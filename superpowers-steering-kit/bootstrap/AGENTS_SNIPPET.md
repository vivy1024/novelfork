# Project Adapter Bootstrap Snippet

Use this alongside upstream Superpowers.

Before planning, editing, validating, committing, or reporting completion in this project:

1. Apply `using-project-steering` to load only the relevant project guidance.
2. Apply `applying-project-profile` when generic Superpowers guidance needs project-specific interpretation.
3. Apply `kiro-spec-adapter` for `.kiro/specs/<feature>/{requirements,design,tasks}.md` workflows.
4. Apply `feature-closure-gate` before claiming any user-visible feature or release-facing behavior is complete.

Generic Superpowers remains the method layer; this project profile/steering layer defines repository-specific truth and evidence.

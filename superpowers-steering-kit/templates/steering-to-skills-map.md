# Steering to Skills Map

Use this when converting project steering into skills.

## Keep as Steering

- Stable project facts
- Architecture overview
- Commands and paths
- Release/version policy
- Domain-specific truth

## Convert to Skill

- Repeated decision workflows
- Completion gates
- Review checklists
- Routing logic
- Failure recovery patterns

## Delete After Conversion

- Duplicated summaries
- Completed one-off task notes
- Historical claims superseded by docs/status
- Rules that are now enforced by tests/scripts
- Anything contradicted by current project profile

## Conversion Template

```text
Source steering file:
Reusable rule:
Trigger condition:
Skill name:
Project-specific facts to keep in profile:
Old content safe to delete:
```

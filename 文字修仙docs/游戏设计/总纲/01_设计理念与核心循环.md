## 一、文档现状判断

你现在的文档体系已经很强：

- 世界观底座成型
- LEVSS 物理模型成型
- 灵材/本草数据库成型
- 丹药/法器/材料工业成型
- 势力、经济、百年推演成型

当前真正薄弱的，不是“规则够不够”，而是：

> **从规则到玩家交互的翻译层还不够厚。**

也就是说，问题不是“世界有没有设计好”，而是：

- 玩家到底看什么
- 玩家怎么理解公式
- 玩家如何从一次修炼中学到东西
- 海量系统怎样以统一范式被玩家消化

这部分不补，规则再强也容易变成作者内部资料库。

## 二、最高设计原则：公式开放，玩家自己算

### 2.1 公式不是后台黑盒

这个项目最独特的地方，不应该是“修仙味”，而应该是：

> **它像 KSP、Factorio、Opus Magnum 那样，让玩家主动学习一套虚构但自洽的科学体系。**

类比：

| 其他游戏 | 本项目 |
|---------|--------|
| KSP 让玩家学轨道力学 | 本项目让玩家学 LEVSS 耦合物理 |
| Factorio 让玩家算产能比 | 本项目让玩家算 `Match_C` 配伍评分 |
| Opus Magnum 让玩家设计分子路径 | 本项目让玩家设计炼丹工艺链和法器阻抗匹配 |

因此：

- 底层公式不能只藏在后台
- 玩家不能只能看到“成功/失败”
- 公式必须作为世界知识的一部分被自然公开

### 2.2 公式开放的三层结构

| 层级 | 玩家看到什么 | 何时解锁 |
|------|-------------|---------|
| **观察层** | 结果 + 白话解释 | 初始即有 |
| **公式层** | 公式骨架 + 参数含义 + 本次代入值 | 胎息期教学后 |
| **原始层** | 完整参数手册、阈值、系数 | 知识库/图书馆/高阶课程可查 |

示例：

- 观察层
  - “本次修炼储能 +0.03”
- 公式层
  - `Risk(t) = O(t) + M(t) + P(t) + Psi(t)`
  - “你的 `O(t)=0.15`，因为灵气吸入速率偏高”
- 原始层
  - `theta_warn = 0.38`
  - `theta_crit = 0.62`
  - `theta_loss = 0.86`

### 2.3 公式公开必须是“世界内合理”

公式不应以“第四面墙 debug 面板”的形式出现，而应当以世界内知识形态出现：

- 学院教材会写
- 开源社区论坛会讨论
- AI 道侣会解释
- 导师会拿案例教学
- 图书馆和数据库可以查原始手册

也就是说，玩家看到公式，不是因为系统在“透底”，而是因为这个世界本来就已经把修炼工程化、学科化、标准化了。

### 2.4 游戏内知识面板设计

```text
+--------------------------------------------------+
|  [book] 修炼知识库                    [搜索...]    |
+--------------------------------------------------+
|                                                   |
|  > 基础物理                                        |
|    +-- LEVSS 耦合能                                |
|    +-- 五行 = 五种耦合模式                          |
|    +-- 灵气浓度 Q 与噪声 N                          |
|                                                   |
|  > 修炼公式                                        |
|    +-- Risk(t) 风险四分量      <- 胎息承明轮解锁    |
|    +-- L_fit 灵根适配评分      <- 胎息青元轮解锁    |
|    +-- HardGate / SoftGate    <- 练气期解锁        |
|                                                   |
|  > 炼丹公式                                        |
|    +-- Match_C 丹药品质评分    <- 首次炼丹后解锁    |
|    +-- Interference_spec 配伍  <- 同上              |
|    +-- ProcessVariance 工艺方差 <- 三次炼制后       |
|                                                   |
|  > 炼器公式                                        |
|    +-- ArtifactScore 法器品质  <- 首次接触炼器      |
|    +-- Z 灵气阻抗匹配          <- 同上              |
|    +-- UserMatch 使用者匹配    <- 同上              |
|                                                   |
|  > 个人实验记录                                     |
|    +-- 每次修炼/炼丹/炼器的完整参数记录             |
+--------------------------------------------------+
```

每条公式都要附带三部分：

- **白话解释**
  - “`Risk` 越高越危险，超过 0.38 AI 道侣会预警”
- **你的数据**
  - “你当前 `L_fit=52`，属于正常可修范围”
- **来源出处**
  - “摘自《QESA 标准修炼安全手册》第三版”

## 三、三条核心循环的统一范式

三条循环不该分裂成三套玩法，而应共享同一个认知模型：

> **选材料 → 调参数 → 看公式反馈 → 记录数据 → 积累经验**

```text
+--- 修炼循环 ---+--- 资源循环 ---+--- 社会循环 ---+
| 选功法/灵材/环境 | 采集/采购原料  | 势力关系/委托  |
| 调参数(呼吸/时长)| 鉴定品质参数   | 节点事件触发   |
| 看公式(Risk/V_d) | 看公式(Match_C)| 世界状态变化   |
| 记录修炼日志     | 记录炼制日志   | 选择与后果     |
+-----------------+---------------+---------------+
         三者通过 时间 和 风险 交织在一起
```

统一之后，玩家会自然理解：

- 修炼是在给自己这台机器调参
- 炼丹是在给药材系统调参
- 炼器是在给材料和模块调参

这样就不会出现“修炼像文字游戏、炼丹像合成表、炼器像另一个世界”的割裂感。

## 四、交互的四个核心原则

### 原则一：公式透明，玩家能看、能算、能决策

#### 修炼准备界面示例

```text
+-- 修炼准备界面 -----------------------------------------+
|                                                         |
|  功法：清源引气诀(中性偏木)                               |
|  灵材：低纯度石英(sigma_q=中, Spec=[0.70,0,0.10,0.05,0.15])|
|  环境：住处静室(Q=2.5, N=0.3)                            |
|                                                         |
|  -- 公式预览 --                                          |
|  灵气吸入速率 InputRate = Q * sigma_q * 呼吸系数          |
|             = 2.5 * 0.45 * 0.8                           |
|             = 0.90                                       |
|                                                         |
|  风险预估 Risk = InputRate / CapacityMargin              |
|              = 0.90 / 3.2 = 0.28                         |
|                                                         |
|  [v] 0.28 < theta_warn(0.38)，安全范围内                 |
|                                                         |
|  [!] 如果换用萤石(sigma_q=中偏高)，InputRate 会升至       |
|     1.15，Risk 升至 0.36——接近预警线                      |
|                                                         |
|  [开始修炼]  [调整参数]  [查看公式详情]                    |
+---------------------------------------------------------+
```

#### 炼丹工作台示例

```text
+-- 炼丹工作台 -------------------------------------------+
|                                                         |
|  配方：静心散                                            |
|  原料已放入：酸枣仁皂苷(m=0.45,P=0.60)                   |
|             茯苓多糖(m=0.35,P=0.55)                      |
|             灵石粉(m=0.20,P=0.30)                        |
|                                                         |
|  -- 实时计算 --                                          |
|  配伍矩阵：木*水 -> Phi=+1.0(相生)                       |
|  Interference_spec = -0.18(配伍增效，主峰*1.3)            |
|                                                         |
|  SideVec 峰值预估：Addiction = 0.10                      |
|  -> [!] 长期服用会削弱自主入静能力                        |
|                                                         |
|  当前工序：水提酸枣仁，目标温度 70C                       |
|  你的温度设定：[  72  ]C                                 |
|  -> 在 60-80C 窗口内 [v]                                 |
|  -> 偏高 2C 会使提取效率 +3% 但杂质 +1%                  |
|                                                         |
|  [执行此步]  [查看 Match_C 公式]                          |
+---------------------------------------------------------+
```

### 原则二：动作从“物体关系”中生长

推荐操作范式：

1. 玩家看到自己在某个区域卡里
2. 手里有功法卡、灵材卡、丹药卡、法器卡
3. 把对象放进工作槽
4. 系统展示公式预算结果与可执行动作
5. 玩家理解数据后再决定

也就是说，动作不是先列一个“按钮菜单”，而是从：

- 地点
- 功法
- 资源
- 当前状态

这四者的关系里自然长出来。

### 原则三：过程必须可见

一次修炼不应该只在最后告诉玩家“成功/失败”，而应让过程本身成为学习对象。

示例：

```text
感气修炼过程(30分钟推演)：
  [校准] "心率 72bpm，体温 36.4C"
         -> Risk 基线 = 0.05
  [导入] "灵气开始进入...... InputRate = 0.90"
         -> Risk 累积至 0.15
  [锁定] "灵气稳定。O(t)=0.08, M(t)=0.04, P(t)=0.02, Psi(t)=0.01"
         -> Risk = 0.15，安全
  [收功] "储能 V_d +0.03 (公式：dV_d = InputRate * 耦合效率 * 时长)"
         -> 完整参数记入修炼日志
```

### 原则四：失败必须有信息量

理想的失败反馈不是：

- “修炼失败”

而是：

```text
修炼失败——过载型走火(I级)
  公式追溯：O(t) = InputRate / CapacityMargin
                 = 1.85 / 3.2 = 0.58 > theta_warn(0.38)
  原因：呼吸节律加快导致 InputRate 从 0.90 飙升至 1.85
  AI 道侣在 Risk=0.42 时触发橙色预警并强制中止
  建议：控制呼吸频率；或换用 sigma_q 更低的灵材
```

失败既是惩罚，也是教学。

## 五、如何让角色进步

### 5.1 进步 = 公式参数逐步达标

角色进步不应抽象成一根神秘经验条，而应表现为一组参数逐步走到门槛线。

```text
+-- 日常修炼 -------------------------------------------+
|  每次修炼 -> 微量数值变化(玩家可见公式)：               |
|    dV_d = InputRate * eta_couple * t_eff               |
|    -> 储能 V_d +0.01~0.05                              |
|    -> 经脉阻抗降，神识升，污染升(各有公式)              |
|  玩家看得到每个变量为什么变、变了多少                    |
+-------------------------------------------------------+
         | 积累到 HardGate 条件达标
+-- 境界突破 -------------------------------------------+
|  知识面板展示跃迁条件族：                               |
|    * V_d >= 阈值                           [v]         |
|    * 经脉贯通度 >= 阈值                    [v]         |
|    * Risk 历史峰值 < 安全线                [v]         |
|    * 连续修炼成功次数 >= N                 [x] (差2次) |
|                                                       |
|  玩家自行决定是否尝试突破——可以算风险、备材料            |
|    -> 成功：境界提升，公式体系解锁新参数                |
|    -> 失败：公式追溯告诉你哪个参数爆了                  |
+-------------------------------------------------------+
```

### 5.2 胎息期 = 新手教学 = 案例驱动学公式

胎息期六轮子阶最适合承担整个游戏的“实验课”功能。

| 轮 | 解锁的公式/概念 | 教学案例 |
|----|----------------|---------|
| 玄景轮 | `Q(灵气浓度)` | 比较住处 `Q=2.5` 和实验楼 `Q=4.0` 对 `InputRate` 的影响 |
| 承明轮 | `Risk(t)` 四分量 | 理解 `O(t)` 为什么会因过量吸入而上升 |
| 周行轮 | 经脉阻抗与导灵损耗 | 理解为什么更合适的辅助材料能降低损耗 |
| 青元轮 | 五行频谱 `Spec` | 验证木/水灵根为何更适合某类灵材 |
| 玉京轮 | `L_fit` 灵根适配评分 | 对比自己与导师的适配向量 |
| 灵初轮 | `HardGate / SoftGate` | 把已学过的公式重新组织为跃迁条件 |

### 5.3 玄景轮教学节奏示例

导师对话：

> “修炼的第一件事，是理解灵气浓度 `Q`。  
> 你的住处 `Q=2.5`，够用，但不算高。  
> 看看这个公式：`InputRate = Q * sigma_q * 呼吸系数`。  
> `Q` 越高，你每次呼吸吸入的灵气越多。  
> 但不要贪——`Q` 太高，你的身体承受不住。  
> 先在住处练三次，每次记录你的 `InputRate` 和 `Risk`。”  

第 1 次修炼后：

- `InputRate = 0.90`
- `Risk 峰值 = 0.15`
- AI 道侣标注：安全，可继续

第 3 次修炼后：

> “你已经积累了三次数据。  
> 注意看——每次 `Risk` 都在 0.12-0.18 之间。  
> 这个范围就是你当前的‘安全窗口’。  
> 想提高效率？有两条路：换更好的灵材，或去更高 `Q` 的地方修炼。  
> 但 `Risk` 也会上升。  
> 这就是修炼的核心权衡。”  

每一轮完成后，系统应自动完成三件事：

- 知识面板永久解锁对应公式页
- 修炼日志回顾本轮数据趋势
- AI 道侣给出基于玩家真实数据的个性化建议

## 六、庞大的物品/系统如何实现

### 6.1 总体策略：分层解锁 + 公式驱动

海量内容不该一上来全摊给玩家，而应随认知和公式能力分层开放。

| 阶段 | 玩家认知范围 |
|------|-------------|
| 胎息期 | 5-10 种基础灵材、1-2 种丹药，学 `Risk / InputRate` |
| 练气期 | 20-30 种灵材、5-8 种丹药，学 `Match_C` |
| 筑基期 | 接触矿物工业与基础炼器，学 `ArtifactScore` |
| 紫府期 | 接触跨势力灵材体系、自研配方与法器设计 |

### 6.2 灵物/灵植：鉴定即学习

理想的灵植鉴定交互：

1. 外观观察
   - “一株叶片带紫边的草本植物”
2. 气味/触感检查
   - “可能属于唇形科植物”
3. 灵气感应
   - “`Spec` 偏木/水”
4. 成分分析
   - “含酸枣仁皂苷类成分，`P=0.58`”
5. 查阅知识库
   - “确认：酸枣仁，C级，完整参数卡解锁”

这个流程的意义在于：

- 获得条目，不等于完全理解条目
- 每次鉴定都在扩展玩家的公式输入信息
- 物品收集本身就变成学习行为

### 6.3 丹药：Rule-C 公式全开放

炼丹是“公式开放原则”最直接的示范。

理想炼丹循环：

1. 选方剂
   - 查看目标原料参数与公式结构
2. 选原料
   - 实时比较手头原料与标称值的偏差
3. 调工艺
   - 温度、顺序、时长窗口可见
4. 执行工序
   - 中间产物参数实时更新
5. 产出结算
   - 完整展示 `Match_C` 的计算过程
6. 自动记录
   - 全参数进入个人知识库

结算示例：

```text
sigmoid 输入 = 0.35*UsefulPeak - 0.25*SidePeak - 0.15*ProcVar
             + 0.20*Conf + 0.10*UserFit
           = 0.3240

Match_C = 100 * sigmoid(0.3240) = 58.0
-> C级偏上
```

玩家看过几次完整算式后，就会开始反向思考：

- “如果把原料纯度从 0.55 提到 0.70，能提升多少？”
- “如果降低朱砂配比，毒性会降多少？”

这时玩家就不再只是消费者，而是配方设计者。

### 6.4 灵器/法器：ArtifactScore + 阻抗匹配

灵器系统建议按阶段开放：

| 阶段 | 玩家能力 | 可做的事 |
|------|---------|---------|
| 胎息期 | 只能使用 | 购买和使用低阶法器 |
| 练气期 | 初步维护 | 法器保养、理解耐久度与匹配度 |
| 筑基期 | 基础炼器 | 制备单模块、学 `ArtifactScore` |
| 紫府期 | 专业炼器 | 多模块设计、阻抗匹配 `Z` |

核心体验不该是“捡到更强装备”，而应是：

- 选材料
- 选模块
- 算阻抗
- 看 `UserMatch`
- 理解为什么这件法器适合你或不适合你

### 6.5 阵法：多节点协同公式

阵法应被理解为：

> **多个阵眼 + 导灵路径 + 阵心放大结构**组成的协同系统。

最早期只需要让玩家学会一种简单思路：

```text
简易聚灵阵：
  3 块石英三角排列
  -> 中心 Q 提升 20%
  -> 修炼效率公式里的 Q 从 2.5 变成 3.0
  -> 玩家自己算出 InputRate 和 Risk 的变化
```

这样阵法就不是“玄学开关”，而是环境参数调制器。

## 七、建议的优先实现顺序

### Phase 1：修炼循环 + 公式教学系统

- 知识面板 UI
- 修炼流程：选材 → 调参 → 看公式反馈 → 结算 → 记录
- 胎息六轮教学案例
- AI 道侣作为安全系统和公式解说员
- 个人修炼日志

### Phase 2：炼丹 + 资源循环

- 灵材采集、购买、鉴定
- 炼丹工作台，实时计算 `Match_C`
- 3-5 个教学型方剂
- 简易阵法布置

### Phase 3：社会循环激活

- 势力关系界面
- 节点事件触发
- 价格与交易反馈
- 月结算与世界时间推进

### Phase 4：炼器 + 深度系统

- 炼器工作台
- `ArtifactScore + 阻抗匹配` 教学
- 境界突破
- 走火入魔完整机制
- 更高层级开放世界探索

## 八、一句话总结

> **修仙 = 生理工程 + 材料科学。公式不是后台秘密，而是世界观本身。玩家不是在“玩升级”，而是在学习一套虚构但自洽的科学体系，然后用这套知识去选材料、调参数、控风险、记数据，并从失败中真正成长。**

## §1 时钟制日循环

### 1.1 设计理念

不用「行动点」，用「时间」。每天从 06:00 开始，每个行动消耗真实分钟数。玩家不是在"花行动点"，而是在"安排一天的时间"。

这个选择的核心原因：修炼是工程，工程需要时间管理。一个修士的日常不是"点三下修炼按钮"，而是"早上花两个时辰打坐，下午去采药，傍晚炼一炉丹"。

### 1.2 一天的结构

```
06:00  晨起阶段 ─── 事件引擎抽取当日事件（daily.yaml）
  │                  检查主线剧情触发（story.yaml）
  │                  检查任务生成（quest_service）
  │
  ├─ 行动阶段 ─── 玩家自由安排行动，每个消耗分钟数
  │   修炼    120min   核心进度推进
  │   炼丹    90min    消耗材料产出丹药
  │   采集    60min    获取灵材
  │   交易    30min    买卖物品
  │   探索    60min    发现新地点/奇遇
  │   休息    60min    恢复疲劳
  │   移动    30min    切换地点
  │   切磋    45min    与NPC战斗
  │   冥想    90min    提升心性三维
  │   炼器    120min   制作法器
  │   净化    60min    降低污染
  │
  ├─ 硬约束检查 ─── 每次行动前检查四道门槛
  │   疲劳 ≥ 0.9     → 强制休息
  │   污染 ≥ 0.8     → 禁止修炼/炼丹
  │   心性 ≤ 0.15    → 禁止高风险行动
  │   深夜 ≥ 22:00   → 只能休息/睡觉
  │
24:00  夜间结算 ─── 睡眠恢复（疲劳×0.7，污染自然衰减）
                     NPC每日推演（npc_simulator）
                     个人事件检查（personal.yaml）
                     Buff/Debuff tick
```

### 1.3 行动后的反馈链

每次行动完成后，系统按顺序执行：

1. **FormulaTrace 生成** — 本次计算的完整过程（受知识解锁过滤）
2. **知识自动解锁** — 首次执行某类行动时解锁对应知识条目
3. **实验记录写入** — experiment_log 表记录本次数据
4. **个人事件检查** — 连续行为可能触发条件事件
5. **主线剧情检查** — story.yaml 节点触发
6. **任务进度检查** — 行动是否推进了当前任务
7. **势力声望检查** — 行动是否影响势力关系
8. **动机选择提示** — 关键行动后弹出动机三选一

### 1.4 玩家体验目标

- **早期（胎息）**：一天只能做 2-3 件事，时间很紧，需要取舍
- **中期（练气）**：效率提升，一天能做 4-5 件事，开始有余裕探索
- **后期（筑基+）**：时间不再是瓶颈，决策质量成为核心

## §2 周报系统

### 2.1 设计理念

每 7 天，AI 道侣生成一份周报。这不是"系统公告"，而是"你的私人助理帮你整理了这周的数据"。

### 2.2 周报六模块

| 模块 | 内容 | 玩家价值 |
|------|------|---------|
| 修炼进度 | 本周 L_fit 变化、熟练度增长、风险趋势 | 知道自己在进步还是停滞 |
| 资源流水 | 收入/支出/库存变化 | 经济决策依据 |
| 灵压趋势 | 当前地点灵压 7 天走势 | 是否需要换地点 |
| 势力动态 | 声望变化、势力间关系变动 | 社会层面的信息 |
| 市场行情 | 物价波动、稀缺物品提示 | 交易时机判断 |
| AI 建议 | 基于数据的下一步建议 | 降低信息过载 |

### 2.3 与玩家节奏的关系

周报是玩家"抬头看路"的时刻。日循环让玩家低头做事，周报让玩家思考方向。

## §3 月度推演

### 3.1 设计理念

每 30 天，世界自己往前走一步。这不是玩家触发的，是世界自己在运转。

### 3.2 月度推演七步

```
Step 1: NPC 突破检查 ─── 满足条件的 NPC 尝试突破（可能失败/走火）
Step 2: NPC 死亡检查 ─── 高龄/重伤 NPC 可能死亡
Step 3: NPC 老化 ─────── 所有 NPC 年龄 +1，属性自然衰减
Step 4: 新 NPC 生成 ──── 补充世界人口
Step 5: 势力结算 ─────── 势力资源/领地/关系变动
Step 6: 灵压波动 ─────── 各地点灵压随机波动
Step 7: 月报生成 ─────── 汇总本月世界大事
```

### 3.3 玩家感知

玩家不直接参与月度推演，但会感受到结果：
- "你的师兄突破了练气期"
- "南瞻学院和古法联盟的关系恶化了"
- "昆仑山的灵压本月上升了 12%"
- "集市上灵石草的价格涨了 30%"

这些变化让世界感觉是活的，不是等着玩家去推动的。

### 3.4 自动存档

月度推演完成后自动存档（WAL checkpoint），防止推演结果丢失。

## §4 三层节奏的体验设计

| 层级 | 周期 | 玩家心态 | 设计目标 |
|------|------|---------|---------|
| 日循环 | 每天 | "今天做什么" | 战术决策，即时反馈 |
| 周报 | 7天 | "这周怎么样" | 战略反思，方向调整 |
| 月推演 | 30天 | "世界变了" | 叙事惊喜，长期目标 |

三层嵌套的关键：**日循环是可控的，月推演是不可控的**。玩家能决定今天做什么，但不能决定世界往哪走。这种"局部可控、全局不可控"的张力是游戏的核心体验。

### 2.1 时间预算

```
日起始    06:00  =  360 min
日上限    24:00  = 1440 min
可用时间  24:00 - 06:00 = 1080 min (18h)
```

一天最多可执行的行动组合受 1080 min 硬约束。典型排布：

```
06:00  修炼(180)  → 09:00
09:00  采集(120)  → 11:00
11:00  炼丹(240)  → 15:00
15:00  修炼(180)  → 18:00
18:00  社交(60)   → 19:00
19:00  休息(60)   → 20:00
20:00  修炼(180)  → 23:00
23:00  睡觉       → 次日 06:00
已用: 180+120+240+180+60+60+180 = 1020 min，剩余 60 min
```

| 途径 | 恢复量 | 代码 |

### 3.3 一天典型场景计算

**场景 A：稳健修炼日**

```
起始疲劳: 0.00（假设前夜充分恢复）

06:00  修炼(标准)  → 疲劳 = 0.00 + 0.08 = 0.08
09:00  采集        → 疲劳 = 0.08 + 0.06 = 0.14
11:00  炼丹        → 疲劳 = 0.14 + 0.10 = 0.24
15:00  修炼(标准)  → 疲劳 = 0.24 + 0.08 = 0.32
18:00  休息        → 疲劳 = 0.32 - 0.25 = 0.07
19:00  修炼(标准)  → 疲劳 = 0.07 + 0.08 = 0.15
23:00  睡觉        → 疲劳 = max(0.15 - 0.40, 0) = 0.00

次日起始疲劳: 0.00 ✅ 完全恢复
```

**场景 B：激进修炼日**

```
起始疲劳: 0.10（前夜残留）

06:00  修炼(加速)  → 疲劳 = 0.10 + 0.16 = 0.26
11:00  修炼(加速)  → 疲劳 = 0.26 + 0.16 = 0.42
16:00  修炼(标准)  → 疲劳 = 0.42 + 0.08 = 0.50
19:00  探索        → 疲劳 = 0.50 + 0.08 = 0.58
22:00  睡觉        → 疲劳 = max(0.58 - 0.40, 0) = 0.18

次日起始疲劳: 0.18 ⚠️ 残留累积
```

**场景 C：走火入魔日**

```
起始疲劳: 0.20

06:00  修炼(加速)  → 疲劳 = 0.20 + 0.16 = 0.36
11:00  修炼(加速)  → 疲劳 = 0.36 + 0.16 = 0.52
16:00  修炼(加速)  → 疲劳 = 0.52 + 0.16 = 0.68
       ↳ 走火入魔！ → 疲劳 = 0.68 + 0.30 = 0.98 ⚠️ 接近强制休息
21:00  睡觉        → 疲劳 = max(0.98 - 0.40, 0) = 0.58

次日起始疲劳: 0.58 🔴 严重残留，需要休息日
```

## §4 风险系统

### 4.1 四维风险模型

代码位置：`rule_engine.py → calc_risk`

```
Risk_total = w_O × Overload + w_M × Dissonance + w_P × Pollution + w_Ψ × Mental
```

| 分量 | 代码键 | 权重 | 含义 | 静态计算公式 |
|------|--------|------|------|-------------|
| 过载 O | `overload` | 0.35 | 灵气输入超载 | `(fatigue×0.8 + injury×0.5) × ticks × 0.3 × ai_factor` |
| 失谐 M | `dissonance` | 0.25 | 灵气频谱错误 | `0.05 × ticks × ai_factor` |
| 污染 P | `pollution` | 0.25 | 杂质累积 | `pollution_load × 0.6 × 0.4 × ai_factor` |
| 心境 Ψ | `mental` | 0.15 | 心理压力 | `(1 - heart_stability) × 0.5 × ai_factor` |

### 4.2 AI 衰减

```python
ai_reduction = 0.35          # AI 道侣降低 35% 风险
ai_factor    = 1.0 - 0.35    # = 0.65
```

所有四个分量的原始值都乘以 `ai_factor = 0.65`。

### 4.3 时间行为版本（§9 扩展）

当 `prev_state` 不为 None 时，四分量具有不同的时间特性：

| 分量 | 时间类型 | 修炼中行为 | 停止后行为 |
|------|---------|-----------|-----------|
| O（过载） | 瞬时型 | 基于当前输入率 | 立即归零 |
| M（失谐） | 会话型 | α=0.95 累积 | α=0.5 快衰 |
| P（污染） | 累积型 | 只升不降 | 需净化清除 |
| Ψ（心境） | 滞后型 | EMA(λ=0.8) | 缓慢衰减 |

### 4.4 四级预警阈值

代码位置：`rule_engine.py` L856–L867

| 等级 | Risk 范围 | 代码标识 | 后果 |
|------|----------|---------|------|
| 🟢 绿色 | < 0.38 | `green` | 安全，正常行动 |
| 🟡 黄色 | 0.38 ≤ R < 0.62 | `yellow` | 注意，AI 道侣记录 |
| 🟠 橙色 | 0.62 ≤ R < 0.86 | `orange` | 警告，AI 建议降低强度 |
| 🔴 红色 | ≥ 0.86 | `red` | 危险，30% 概率走火入魔 |

### 4.5 走火入魔判定

```python
# rule_engine.py → resolve_cultivation
if risk["alert_level"] == "red":
    roll = rng.random()
    if roll < 0.3:                    # 30% 概率
        accident = "走火入魔（轻微）"
        progress *= max(0.1, preserve) # 进度大幅缩水
        fatigue_delta += 0.30          # 疲劳暴增
```

走火入魔分型由 Risk 四分量的主导分量决定：

| 主导分量 | 走火类型 | 主要伤害 | 次要伤害 |
|---------|---------|---------|---------|
| overload | 过载型 | 伤势 injury | 泄漏率 leak_rate |
| dissonance | 失谐型 | 经脉损伤 | 临时 L_fit 下降 |
| pollution | 污染型 | 污染加重 | 依赖性 |
| mental | 心理型 | 心性崩塌 | MindScore 下降 |

后遗症等级（I–V 级）：

| 等级 | 描述 | 恢复天数 | 伤害倍率 |
|------|------|---------|---------|
| I 级 | 可逆短期损伤 | 7 天 | ×0.3 |
| II 级 | 需药物干预 | 30 天 | ×0.6 |
| III 级 | 需法器/高阶干预 | 90 天 | ×1.0 |
| IV 级 | 永久性部分损伤 | 365 天 | ×1.5 |
| V 级 | 社会性毁损 | 不可逆 | ×2.0 |

## §8 数值调参原则

1. **所有阈值在 YAML 中定义**，不硬编码在 Python 里（`_HARD_GATES` / `_SOFT_WEIGHTS` / `_BASE_SUCCESS_RATE` 等均支持 YAML 覆盖）
2. **规则集三层架构**：排序规则（不可改）→ 分级阈值（可调）→ 内部系数（版本迭代）
3. **FormulaTrace 暴露所有中间值**，方便调参时观察每一步计算
4. **测试覆盖**：274+ 个测试确保调参不破坏公式逻辑
5. **单调性保证**：所有 calc_* 函数的输入-输出关系必须满足设计意图的单调性（如 L_fit 越高修炼效率越高）
6. **边界安全**：所有数值 clamp 到合法范围，`power = max(1.0, power)`，`fatigue = max(0, min(1, ...))`

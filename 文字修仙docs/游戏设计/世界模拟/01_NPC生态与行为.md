# NPC 生态与行为设计

> **一句话版**：NPC 不是对话树，而是有目标函数的行动者——他们的行为由属性驱动，不是脚本驱动。
> **关联文档**：[[04_势力声望与任务]] · [[03_叙事节奏控制]] · [[03_心性与动机系统]]
> **代码对照**：`npc_simulator.py` · `npc_interactions.yaml` · `npc_dialogues.yaml` · `npcs_fixed.yaml`

---

## §1 设计哲学

### 1.1 NPC 是玩家世界的"环境压力"

NPC 的存在不是为了给玩家提供服务，而是让世界有**真实的社会质感**：
- 他们有自己的修炼进度、势力归属和性格特征
- 玩家与 NPC 的关系是资产——值得投入但也会衰减
- NPC 的"态度"影响玩家能获得什么、能去哪里

### 1.2 三个不做

- **不做冗长对话树**（所有对话从 `npc_dialogues.yaml` 模板生成）
- **不做纯功能性 NPC**（每个 NPC 都有立场和性格，影响交互结果）
- **不做固定剧本**（NPC 状态随时间和玩家行为动态变化）

## §2 NPC 属性模型

### 2.1 核心属性

```yaml
npc_id: shen_lin
name: 沈霖
realm: 练气5层
faction: 学院
faction_loyalty: 65        # 对势力的忠诚度（影响透露的信息深度）
traits:
  - 严谨
  - 博学
relation_value: 0          # 与玩家的关系值（动态更新）
relation_type: 陌生         # 陌生/友好/信任/师徒/对立
location: academy_lab      # 常驻地点
schedule:
  morning: academy_lab
  afternoon: academy_lab
  evening: nanzhan_outer_home
```

### 2.2 固定 NPC vs 随机 NPC

| 类型 | 数量 | 定义方式 | 作用 |
|------|------|---------|------|
| 固定 NPC | ~10人 | npcs_fixed.yaml 手工定义 | 剧情锚点/关系深度 |
| 随机 NPC | 动态 | npc_generator.py 程序生成 | 世界人口填充 |

随机 NPC 生成参数：名字池（姓+名）、境界加权分布、性格特征随机 1-3 条、势力归属按区域概率分配。

### 2.3 性格特征对交互的影响

| 性格 | 对话倾向 | 交互偏好 |
|------|---------|---------|
| 严谨 | 信息精确，少废话 | 请教效果好，切磋认真 |
| 热心 | 主动分享，情绪外显 | 对话升关系快 |
| 好奇 | 追问，探索欲强 | 对新事物反应积极 |
| 冷漠 | 短句，低频互动 | 关系提升慢但稳 |
| 博学 | 引经据典，知识密度高 | 请教有额外收益 |
| 谨慎 | 试探性表达 | 需先建立足够信任 |
| 贪财 | 交易导向 | 赠礼效果最大化 |

### 2.4 关系值体系

```
relation_value  关系类型    解锁内容
< 0            对立         无特殊交互，交易溢价 12%
0~9            陌生         仅基础对话/交易
10~39          普通         切磋/赠礼
40~69          友好         请教/拜师（条件满足）
≥ 70           信任         teach_secret / faction_ask 深度互动
```

**关系衰减**：每 30 天无互动，relation_value -2（不低于 0）。

## §3 NPC 每日行为推演

### 3.1 目标驱动决策

每天开始时，`npc_simulator.py` 对每个 NPC 执行：

```
1. 更新位置（按 schedule 或随机游走）
2. 检查修炼状态 → 可能触发自主修炼/采集
3. 检查与玩家距离 → 可能触发主动问候（基于 relation_value）
4. 检查 faction_loyalty → 可能触发势力相关行为
```

NPC 有两个时间窗口的目标：
- **短期（3天）**：当前最紧迫的需求（缺灵材/需修炼/约定见面）
- **长期（30天）**：人生方向（冲击下一境界/积累声望/拓展关系）

### 3.2 O(1) 统计等效公式

不逐 tick 模拟每个 NPC，而是在每日推演时统计等效：

```
NPC 本月修炼进度 = base_cultivate_speed × motivation_factor × time_available
```

结果正确（从分布上看），但计算量极小（O(1) 而非 O(N×tick)）。

### 3.3 NPC 主动接触规则

| 触发条件 | 说明 |
|---------|------|
| relation_value ≥ 20 | 关系够好，会主动打招呼 |
| 玩家刚突破（3天内） | 几乎所有认识的 NPC 都会反应 |
| 叙事节奏需要 | storyteller.py 判断需要社交事件时提升概率 |
| 共同 NPC 介绍 | 第三方介绍认识，双方初始 relation_value +5 |

## §4 交互体系

### 4.1 六种基础交互（已实现）

| 交互 | 耗时 | 核心效果 | 冷却 |
|------|------|---------|------|
| 对话（dialogue） | 30分 | 关系+1~3，疲劳-0.02 | 无 |
| 切磋（spar） | 90分 | 关系±2~5，疲劳+，修炼熟练度+ | 1天 |
| 交易（trade_npc） | 30分 | 物品交换 | 无 |
| 拜师（apprentice） | 60分 | 一次性，关系+10~20 | 一次性 |
| 赠礼（gift） | 30分 | 关系+（按物品价值×0.02） | 无 |
| 请教（ask_guidance） | 60分 | L_fit微增0.001，关系+1~3 | 1天 |

### 4.2 深度互动（高关系解锁）

| 交互 | 解锁条件 | 核心收益 |
|------|---------|---------|
| 传授秘法（teach_secret） | relation ≥ 70 + 师徒关系 | 世界观信息/特殊配方/古法残页 |
| 势力话题（faction_ask） | relation ≥ 30 + NPC 有势力归属 | 派系内部情报，影响势力声望 |

### 4.3 对话模板生成流程

```
交互类型 + 关系等级 + NPC性格特征
        ↓
从 npc_dialogues.yaml 匹配模板分支
        ↓
{npc} → NPC名  /  {player} → 玩家名
        ↓
随机选取模板 → 最终展示文本
```

## §6 NPC 生命周期

| 事件 | 触发条件 | 玩家感知 |
|------|---------|---------|
| NPC 突破 | 月度推演+概率 | 主动联系玩家，可能分享心得 |
| NPC 走火入魔 | 高风险修炼失败 | 事件日志+关系变化 |
| NPC 离开区域 | 势力任务/随机 | 位置更新，暂时不可交互 |
| 新 NPC 出现 | 探索/剧情触发 | 探索行动结果 |

# 12_叙事节奏控制设计

> **一句话版**：一个 RimWorld 式的 AI Storyteller 在幕后调控事件密度——玩家永远不会连续遭遇太多坏事，也不会长期无事发生。
> **关联文档**：[[06_核心循环设计]] · [[16_事件系统设计]]
> **代码对照**：`storyteller.py` · `event_engine.py`

---

## §1 设计哲学

玩家不应该感知到叙事控制器的存在。它的工作是让游戏"感觉对"——不会太无聊，也不会太压抑。

借鉴 RimWorld 的 AI Storyteller 思路：不是预编剧本，而是根据当前状态动态调制事件概率。

## §2 四条调制规则

### 2.1 事件密度控制

```
如果最近 3 天内已触发 ≥ 2 个事件 → 降低今日事件概率 50%
```

防止事件过密导致玩家信息过载。

### 2.2 连续负面回弹

```
如果最近 5 天内负面事件 ≥ 3 个 → 强制插入一个正面/中性事件
```

防止"霉运连击"摧毁玩家体验。这不是"天道补偿"（世界观不允许），而是"概率回归均值"的自然表现。

### 2.3 突破后社交效应

```
如果玩家刚完成突破（3天内）→ 提升 NPC 主动接触概率 ×2
```

突破是大事，周围的人会注意到。这创造了"突破后的社交窗口"，让玩家感受到世界的反应。

### 2.4 当天防过密

```
同一天内最多触发 1 个主动事件（不含行动后的被动检查）
```

一天一件大事就够了。

## §3 与事件系统的协作

```
事件引擎（event_engine.py）
    │
    ├── 条件过滤：哪些事件满足触发条件
    │
    └── 叙事控制器（storyteller.py）
            │
            ├── 密度调制：今天该不该触发事件
            ├── 正负平衡：该触发正面还是负面
            └── 权重修正：调整各事件的抽取权重
                    │
                    └── 加权随机抽取 → 最终触发的事件
```

叙事控制器不决定"发生什么"，只决定"该不该发生"和"倾向发生什么类型"。

## §4 参数调优指南

| 参数 | 当前值 | 调整方向 |
|------|--------|---------|
| 密度窗口 | 3天/2事件 | 增大→更稀疏，减小→更密集 |
| 负面回弹阈值 | 5天/3负面 | 降低→更快回弹，提高→更残酷 |
| 突破社交倍率 | ×2 | 提高→更强社交反馈 |
| 日事件上限 | 1 | 提高→更忙碌的一天 |

# 02_事件系统

> **一句话版**：事件不是硬编码的剧本，而是 YAML 驱动的条件引擎——满足条件就触发，不满足就跳过，世界自己讲故事。
> **关联文档**：[[03_叙事节奏控制]] · [[03_心性与动机系统]] · [[01_NPC生态与行为]]
> **代码对照**：`event_engine.py` · `data/events/daily.yaml` · `data/events/personal.yaml` · `data/events/world.yaml` · `data/events/story.yaml`

## §1 四级事件架构

### 1.1 事件层级

| 层级 | 触发时机 | 数据源 | 数量 | 示例 |
|------|---------|--------|------|------|
| daily | 每天晨起抽取 1 个 | daily.yaml | 15 | 灵气潮汐、污染预警、市场促销 |
| world | 月度推演时检查 | world.yaml | 19 | 势力冲突、灵泉喷涌、QESA 巡查 |
| personal | 每次行动后检查 | personal.yaml | 22 | 导师来访、炼丹顿悟、心魔前兆 |
| story | 行动后检查 | story.yaml | 11 | 主线剧情节点（第一卷） |

### 1.2 条件解析器

`_check_conditions(conditions, context)` 支持：

| 操作符 | 语法 | 示例 |
|--------|------|------|
| gte | `{gte: value}` | `player.realm: {gte: "练气"}` |
| lte | `{lte: value}` | `player.fatigue: {lte: 0.5}` |
| eq | `{eq: value}` | `player.location_id: {eq: "academy_lab"}` |
| gt / lt | `{gt: value}` | `world.hot_war_index: {gt: 0.3}` |
| mod | `{mod: [divisor, remainder]}` | `tick: {mod: [90, 0]}` — 每 90 tick |

**特殊条件**：

| 条件 | 语法 | 说明 |
|------|------|------|
| NPC 附近 | `npc_nearby: {realm_gte: "练气", relation_gte: 20}` | 同地点有满足条件的 NPC |
| NPC 性格 | `npc_nearby: {trait: "贪财"}` | 同地点有特定性格 NPC |
| 境界比较 | `player.realm: {gte: "筑基"}` | 自动转为境界序号比较 |

### 1.3 上下文字典

条件检查的上下文由 `_build_context()` 构建：

```python
context = {
    "player": {
        "realm", "realm_minor", "fatigue", "location_id",
        "prof_cultivate", "prof_alchemy", "prof_gather", "prof_explore",
        "pollution_load", "heart_stability", "l_fit_base",
        "money", "spirit_reserve", "mind_score", "mind_insight",
        "mind_integration", "injury_level", "dantian_v_d",
    },
    "world": {
        "era_phase", "hot_war_index", "ai_competition_index",
        "public_order", "resource_stress", "global_pollution",
    },
    "tick": current_tick,
}
```

## §2 事件结构规范

### 2.1 基础字段

```yaml
- id: mentor_visit
  emoji: "👨‍🏫"
  title: 导师来访
  desc: "你的导师注意到你近期的刻苦修炼，特地前来指点。"
  weight: 6                    # 抽取权重（越高越容易触发）
  conditions:                  # 触发条件（全部满足才触发）
    player.prof_cultivate: {gte: 0.30}
    player.realm: {gte: "练气"}
    npc_nearby: {realm_gte: "练气", relation_gte: 30}
  effects:                     # 触发效果
    prof_cultivate_delta: 0.02
    heart_delta: 0.02
    realm_scaled:              # 境界分层奖励
      胎息: 1.0
      练气: 1.2
      筑基: 1.5
      default: 1.0
```

### 2.2 效果字段完整列表

| 字段 | 类型 | 说明 |
|------|------|------|
| `fatigue_delta` | float | 疲劳变化 |
| `heart_delta` | float | 心性变化（注意：仅限非社交来源） |
| `pollution_delta` | float | 污染变化 |
| `money_delta` | float | 金钱变化 |
| `prof_cultivate_delta` | float | 修炼熟练度变化 |
| `spirit_pressure_delta` | float | 当前地点灵压变化 |
| `spirit_reserve_delta` | float | 灵储变化 |
| `mind_insight_delta` | float | 内省维度变化 |
| `mind_integration_delta` | float | 整合维度变化 |
| `free_herb` | int | 赠送灵草数量 |
| `realm_scaled` | dict | 境界分层奖励倍率 |

### 2.3 事件链

事件可以触发后续事件：

```yaml
- id: beast_nest_discovery
  chain_from: wild_beast        # 前置事件 ID
  chain_probability: 0.3        # 链式触发概率
  conditions:
    player.prof_explore: {gte: 0.15}
  effects:
    free_herb: 2
    money_delta: 200
```

`check_chain_events()` 在事件触发后搜索所有 `chain_from` 匹配的事件，满足条件且概率通过则触发。

设计原则：链不超过 3 步，避免"事件瀑布"。

### 2.4 叙事节奏控制

`storyteller.py` 对事件权重进行调制：

- 连续正面事件后压低正面权重（避免"一直好运"）
- 连续负面事件后提升正面权重（避免"一直倒霉"）
- RimWorld 式的戏剧性曲线

```python
adj_weight = apply_modifier_to_weight(raw_weight, event_id, modifiers, "daily")
```

## §3 主线剧情（story.yaml）

### 3.1 特殊字段

主线事件比普通事件多几个字段：

| 字段 | 说明 |
|------|------|
| `unique: true` | 只触发一次，写入 `story_flags` 表 |
| `requires_story: node_id` | 前置节点依赖（必须先触发 X 才能触发 Y） |
| `choice: true` | 有分支选择 |
| `choices[]` | 选项列表，每个含 id/label/effects/next_node |
| `narrative_type` | 分类标签（exposition/conflict/choice/resolution） |

### 3.2 选择与后果

主线选择与动机系统联动：

```yaml
choices:
  - id: help_academy
    label: "帮助学院"
    effects:
      faction_reputation_delta: {faction: "nanzhan_academy", value: 10}
    motivation_prompt: true    # 触发动机选择弹窗
  - id: stay_neutral
    label: "保持中立"
    effects: {}
```

选择 `help_academy` 后，如果 `motivation_prompt: true`，前端弹出动机三选一（真心/权宜/模糊），结果影响 apparent/real Integration。

### 3.3 API

| 端点 | 说明 |
|------|------|
| `GET /story/check` | 检查当前可触发的主线节点 |
| `POST /story/choose` | 处理玩家选择 `{node_id, choice_id}` |
| `GET /story/flags` | 获取所有已触发的剧情标记 |

## §4 如何新增事件

### 4.1 新增 daily 事件

1. 编辑 `data/events/daily.yaml`
2. 添加条目（id/emoji/title/desc/weight/conditions/effects）
3. 条件用 `player.*` 和 `world.*` 路径
4. 重启服务器即生效（模块级缓存在启动时加载）

### 4.2 新增 personal 条件事件

1. 编辑 `data/events/personal.yaml`
2. 条件可以用 `npc_nearby` 检测附近 NPC
3. 支持 `chain_from` 做事件链
4. 支持 `realm_scaled` 做境界分层奖励

### 4.3 新增 story 主线节点

1. 编辑 `data/events/story.yaml`
2. 设置 `unique: true` + `requires_story`（前置依赖）
3. 如果有分支选择，设置 `choice: true` + `choices[]`
4. 选择效果可以包含 `motivation_prompt: true` 触发动机系统
5. 用 `GET /story/check` 验证触发条件

### 4.4 测试验证

```bash
# 跑事件引擎相关测试
python -m pytest tests/test_event_engine.py -v

# 手动验证：创建存档后调用
curl -X GET http://localhost:8000/story/check
```

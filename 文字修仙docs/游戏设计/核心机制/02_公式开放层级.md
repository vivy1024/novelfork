# 公式开放层级方案

> **一句话版**：公式不是后台黑盒，分三层向玩家开放——观察层（人人可见）→ 公式层（胎息期解锁）→ 原始层（知识库/图书馆可查）。每层对应不同的认知深度需求。
>
> **关联文档**：[[01_游戏设计综合分析]] · [[02_玩家交互范式]] · [[05_新手教学设计]]
>
> **实现状态**：FormulaTrace.vue ✅ · 知识面板 KnowledgePanel.vue ✅ · 三层完整过滤 🔲

---

## 一、为什么要分层

**不分层的危险**：

- 全显示 → 新手看到一堆希腊字母，立刻关掉
- 全隐藏 → 老玩家无法自己推导、无法验证AI道侣的建议
- 靠游戏教程弹窗解释 → 没有人看教程弹窗

**分层的逻辑**：公式是这个世界里的**客观知识**，就像现实里的物理学一样。学院教材给初学者写白话，专业文献给研究者写公式，数据库给工程师提供原始参数。游戏内的三层，就是这个结构的映射。

## 二、三层结构定义

### Layer 0 · 观察层（所有玩家，初始可见）

**形态**：白话结果 + 颜色/状态

```
示例：
✓ 本次修炼正常完成
  储能 V_d +0.024
  经脉阻抗 -0.2%
  AI 道侣：状态良好，可继续
```

```
示例（走火）：
⚠️ 过载型走火·I级
  AI 道侣强制中止修炼
  建议：降低灵材品质，或换安静环境
```

**解锁条件**：无，默认显示

---

### Layer 1 · 公式层（胎息期教学后解锁）

**形态**：公式骨架 + 本次代入值 + 结果 + 简短参数说明

```
示例：
InputRate = Q × σ_q × 呼吸系数
          = 0.45 × 0.45 × 0.8
          = 0.162

Risk(t) = MAX( O(t), M(t), P(t), Ψ(t) )
        = MAX( 0.08, 0.04, 0.02, 0.01 )
        = 0.08  ← 绿色安全

V_d 增量 = InputRate × η_couple × t_eff
         = 0.162 × 0.72 × 2.0
         = +0.234
```

**解锁条件**：对应的胎息轮完成

| 公式 | 解锁轮 |
|------|--------|
| Q / InputRate | 玄景轮（第1轮）|
| Risk(t) 四分量 | 承明轮（第2轮）|
| 经脉阻抗/损耗 | 周行轮（第3轮）|
| 五行频谱 Spec | 青元轮（第4轮）|
| L_fit 完整公式 | 玉京轮（第5轮）|
| HardGate/SoftGate | 灵初轮（第6轮）|
| Match_C 炼丹评分 | 首次炼丹后 |
| ArtifactScore | 首次接触法器后 |

---

### Layer 2 · 原始层（知识库/图书馆可查，主动检索）

**形态**：完整参数手册，含阈值、系数、推导过程、历史数据

```
示例：
Risk 系数表（引自《QESA 标准修炼安全手册》v3）:
  θ_warn  = 0.38   → AI 道侣黄色预警
  θ_crit  = 0.62   → AI 道侣橙色，建议停止
  θ_loss  = 0.86   → 走火几率 >80%

过载分量 O(t) 完整公式：
  O(t) = InputRate / CapacityMargin(t)
  CapacityMargin(t) = V_d_capacity - V_d_current - BlockedChannels
  BlockedChannels = Σ(经脉阻抗 × 截面积)

参考文献：rule_engine.py · calc_risk_four_components()
```

**解锁方式**：
- 知识面板对应条目的"原始层"按钮（需该公式已在 Layer 1 解锁）
- 学院图书馆地点可查阅全量参数手册

## 三、FormulaTrace 展示规范

`FormulaTrace.vue` 是公式层的主要载体，每次行动后展示在 ScenePanel 底部。

### 3.1 必须包含的字段

```typescript
interface FormulaTrace {
  action: string          // 行动类型（cultivate/alchemy/gather等）
  steps: TraceStep[]      // 公式推演步骤
  summary: string         // 白话总结（Layer 0）
  knowledge_refs: string[] // 关联知识条目 ID（点击可跳转）
}

interface TraceStep {
  label: string           // 步骤名（如"计算 InputRate"）
  formula: string         // 公式字符串（带实际代入值）
  result: number | string // 计算结果
  unit?: string           // 单位
  highlight?: boolean     // 是否高亮（异常值）
  layer: 0 | 1 | 2       // 所属层级
}
```

### 3.2 分层过滤规则（待实现）

```
玩家未解锁目标公式 → 只展示 Layer 0（summary）
玩家已解锁 → 展示 Layer 0 + Layer 1（formula + result）
玩家在知识库/图书馆 → 可申请查看 Layer 2
```

## 五、公式来源的世界内合理性

**公式的四种合法来源**（避免"系统透底"感）：

| 来源 | 对应游戏内情境 |
|------|-------------|
| 学院教材 | 胎息六轮教学（沈霖讲解） |
| AI 道侣 | 修炼完成后的实时分析 |
| 知识面板 | 图书馆/学院数据库可查询 |
| 开源社区 | 灰巷/古法派等NPC可透露非官方公式 |

**禁止的形式**：
- ❌ 界面直接显示"游戏数据：theta_warn=0.38"（像在看 debug 面板）
- ✅ AI 道侣说："根据 QESA 安全手册，你当前风险值 0.38 已达预警线"

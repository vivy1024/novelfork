## §1 设计哲学

### 1.1 心性 ≠ 心情

传统游戏的"心情系统"：做好事心情好，做坏事心情差。本游戏完全不是这样。

心性衡量的是**内在价值体系的一致性**。一个始终如一的恶人，心性可以很高。一个反复摇摆的好人，心性可能崩溃。

### 1.2 为什么这样设计

世界观约束：修仙者有超忆症（L_fit 越高记忆越清晰）。普通人可以"忘掉"自己的矛盾行为，修仙者不行。每一个言行不一都被完美记住，无法自欺。

所以心魔不是"天道惩罚"，而是"记忆保真度过高导致的自我认知崩溃"。

## §2 三维模型

| 维度 | 含义 | 驱动方式 | 范围 |
|------|------|---------|------|
| Insight（洞察） | 对世界规律的理解深度 | 修炼/冥想/知识解锁 | 0-100 |
| Integration（整合） | 内在价值体系的统一程度 | 动机选择的一致性 | 0-100 |
| Stability（稳定） | 面对冲击时的心理韧性 | 长期积累，突破后可能骤降 | 0-100 |

**MindScore = 0.3 × Insight + 0.4 × Integration + 0.3 × Stability**

代码位置：`rule_engine.py → _MIND_WEIGHTS`

```python
_MIND_WEIGHTS = {"insight": 0.3, "integration": 0.4, "stability": 0.3}
```

MindScore 影响：突破成功率、走火入魔心性减免、战力心性系数（MindScore/100）。

## §3 动机选择系统

### 3.1 触发时机

关键行动后（帮助NPC、势力任务、突破前选择等），弹出动机三选一：

| 选项 | 含义 | Integration 效果 | 矛盾风险 |
|------|------|-----------------|---------|
| **真心（A）** | "我真的相信这样做是对的" | +0.5/次 | 如果和之前的A矛盾 → -3.0 |
| **权宜（B）** | "我不一定认同，但现在需要这样做" | +0 | 无矛盾风险 |
| **模糊（C）** | "我还没想清楚" | +0.1 | 轻微矛盾风险 |

### 3.2 矛盾检测

系统追踪所有"真心"选择的历史。如果新的"真心"选择与之前的"真心"选择在立场上矛盾（比如之前真心帮助学院派，现在真心帮助古法派），触发矛盾惩罚。

矛盾惩罚 = -3.0 × (1 - 合理化缓冲)

合理化缓冲随境界提升而降低（高境界 = 高记忆保真度 = 更难自欺）。

```
合理化缓冲 = max(0.01, 1 - MemFidelity)
```

| 境界 | MemFidelity | 合理化缓冲 | 矛盾实际惩罚 |
|------|------------|-----------|-------------|
| 胎息 | 0.40 | 0.60 | -3.0 × 0.40 = -1.20 |
| 练气 | 0.55 | 0.45 | -3.0 × 0.55 = -1.65 |
| 筑基 | 0.76 | 0.24 | -3.0 × 0.76 = -2.28 |
| 紫府 | 0.88 | 0.12 | -3.0 × 0.88 = -2.64 |
| 金丹 | 0.96 | 0.04 | -3.0 × 0.96 = -2.88 |
| 道胎 | 0.99 | 0.01 | -3.0 × 0.99 = -2.97 |

### 3.3 双值追踪

| 值 | 含义 | 可见性 |
|----|------|--------|
| integration_apparent | 玩家看到的 Integration | 始终可见 |
| integration_real | 真实的 Integration（扣除被合理化掩盖的矛盾） | 突破时才暴露 |

低境界时，合理化能力强，apparent 和 real 差距可以很大。突破到高境界时，MemFidelity 突然提升，apparent 被强制拉到 real——所有被掩盖的矛盾同时涌回。

### 3.4 玩家路线分析

| 路线 | 典型选择 | Integration 轨迹 | 心魔风险 |
|------|---------|-----------------|---------|
| 信念坚定者 | 总选A且方向一致 | 快速增长 | 低 |
| 伪善者 | 选A但前后矛盾 | 崩溃 | 极高 |
| 策略家/卧底 | 关键时选B | 增长慢但稳定 | 低 |
| 随遇而安 | 混合B+C | 中等偏低但稳定 | 极低 |
| 极端善/极端恶 | 总选A，方向一致 | 快速增长 | 低 |

## §5 心魔公式

代码位置：`rule_engine.py → calc_inner_demon`

### 5.1 心魔积累公式

```
InnerDemon(t) = Σ Trauma_i × MemFidelity^(t - t_i) × (1 - IntegrationRate_i)
```

- `Trauma_i`：第 i 次创伤的强度（默认 0.3）
- `MemFidelity`：记忆保真度，境界越高越接近 1.0
- `t - t_i`：距创伤发生的时间（tick 数）
- `IntegrationRate_i`：该创伤已被整合的比例

### 5.2 MemFidelity 境界表

代码位置：`rule_engine.py → _MEM_FIDELITY`

| 境界 | MemFidelity | 含义 | 记忆衰减速度 |
|------|------------|------|-------------|
| 胎息 | 0.40 | 记忆模糊 | 快速遗忘（0.4^t） |
| 练气 | 0.55 | 记忆较清晰 | 中速遗忘 |
| 筑基 | 0.76 | 记忆清晰 | 慢速遗忘 |
| 紫府 | 0.88 | 近乎完美记忆 | 极慢遗忘 |
| 金丹 | 0.96 | 超忆症 | 几乎不遗忘 |
| 道胎 | 0.99 | 完美记忆 | 永不遗忘 |

**关键洞察**：MemFidelity 越高，`MemFidelity^(t-t_i)` 衰减越慢。金丹修士 10 tick 前的创伤仍保留 `0.96^10 ≈ 0.665`，而胎息修士只剩 `0.40^10 ≈ 0.0001`。

### 5.3 MentalCapacity 境界表

代码位置：`rule_engine.py → _MENTAL_CAPACITY`

| 境界 | MentalCapacity | 含义 |
|------|---------------|------|
| 胎息 | 30 | 心理承载力低 |
| 练气 | 50 | — |
| 筑基 | 120 | — |
| 紫府 | 300 | — |
| 金丹 | 800 | — |
| 道胎 | 2000 | 心理承载力极高 |

当 InnerDemon 超过 MentalCapacity 时，触发心魔失控。

### 5.4 合理化缓冲

```
合理化缓冲 = max(0.01, 1 - MemFidelity)
```

低境界时缓冲大（能"骗自己"），高境界时缓冲趋近于零（无法自欺）。

## §6 突破时心魔爆发

代码位置：`rule_engine.py → calc_breakthrough_integration_reveal`

### 6.1 揭露机制

突破成功后，MemFidelity 从旧境界跳到新境界。如果 `apparent > real`（存在隐藏矛盾），合理化失效，矛盾被强制暴露。

```python
mf_jump = new_mf - old_mf                          # MemFidelity 跳跃幅度
reveal_ratio = min(mf_jump × 3.0, 1.0)             # 揭露比例，最多 100%
gap = apparent_integration - real_integration       # 隐藏差距
gap_revealed = gap × reveal_ratio                   # 本次揭露量
new_apparent = apparent_integration - gap_revealed   # 新的表面值
inner_demon_surge = gap_revealed × 0.5              # 心魔涌入量
```

### 6.2 各跃迁的揭露比例

| 跃迁 | old_mf | new_mf | mf_jump | reveal_ratio |
|------|--------|--------|---------|-------------|
| 胎息→练气 | 0.40 | 0.55 | 0.15 | min(0.45, 1.0) = 0.45 |
| 练气→筑基 | 0.55 | 0.76 | 0.21 | min(0.63, 1.0) = 0.63 |
| 筑基→紫府 | 0.76 | 0.88 | 0.12 | min(0.36, 1.0) = 0.36 |
| 紫府→金丹 | 0.88 | 0.96 | 0.08 | min(0.24, 1.0) = 0.24 |
| 金丹→道胎 | 0.96 | 0.99 | 0.03 | min(0.09, 1.0) = 0.09 |

**设计意图**：练气→筑基的揭露比例最高（0.63），这是"第一次真正面对自己"的关键时刻。高境界跃迁虽然揭露比例低，但此时 apparent-real 差距通常已经很小（高境界合理化缓冲极低，矛盾难以隐藏）。

### 6.3 示例计算

假设练气→筑基突破，apparent=60, real=30：

```
gap = 60 - 30 = 30
reveal_ratio = min(0.21 × 3.0, 1.0) = 0.63
gap_revealed = 30 × 0.63 = 18.9
new_apparent = 60 - 18.9 = 41.1
inner_demon_surge = 18.9 × 0.5 = 9.45
```

玩家看到 Integration 从 60 骤降到 41.1，同时心魔涌入 9.45。

## §8 心魔直面术风险

代码位置：`rule_engine.py → calc_mind_training` L2265–L2275

### 8.1 失败概率

```python
fail_chance = max(0.3 - heart_stability × 0.3, 0.05)
```

| heart_stability | 失败概率 | 说明 |
|----------------|---------|------|
| 0.0 | 30% | 心性极低，非常危险 |
| 0.5 | 15% | 中等风险 |
| 0.8 | 6% | 较安全 |
| 1.0 | 5% | 下限，永远有风险 |

### 8.2 失败后果

```python
if rng.random() < fail_chance:
    risk_event = {
        "type": "心魔爆发",
        "desc": "心魔直面术失败，创伤记忆失控",
        "heart_loss": 0.15,          # 心性永久损失
        "stability_loss": 5.0,       # Stability 维度损失
    }
    d_int *= 0.3                     # Integration 提升缩水为 30%
    d_sta = -abs(d_sta) * 2          # Stability 变为负值（大幅下降）
```

### 8.3 成功 vs 失败对比

以 Insight=50, Integration=50, Stability=50 为例（scale=0.25）：

**成功时**：
```
Insight:     +0.3 × 0.25 × rand = [+0.06, +0.09]
Integration: +2.5 × 0.25 × rand = [+0.50, +0.75]  ← 最高整合提升
Stability:   -0.5 × 0.25 × rand = [-0.15, -0.10]  ← 轻微下降
```

**失败时**：
```
Insight:     +0.3 × 0.25 × rand = [+0.06, +0.09]  ← 不变
Integration: +2.5 × 0.25 × rand × 0.3 = [+0.15, +0.225]  ← 缩水 70%
Stability:   -|0.5 × 0.25 × rand| × 2 = [-0.30, -0.20]  ← 双倍下降
额外: heart_loss=0.15, stability_loss=5.0
```

### 8.4 设计意图

心魔直面术是"高风险高回报"的选择：
- 成功时 Integration 提升远超其他方法（月基础 2.5 vs 混合训练 1.5）
- 失败时不仅收益缩水，还会造成永久性心性损伤
- 即使 heart_stability 很高，也有 5% 的最低失败率——修仙没有绝对安全

## §9 设计目标

- **不惩罚任何立场**：善恶都可以走通，惩罚的是矛盾
- **选择有重量**：每个"真心"选择都是对角色核心价值的定义
- **长期后果**：低境界的矛盾会在高境界爆发
- **角色 ≠ 玩家**：角色有自己的心理历史，玩家是引导者不是操控者

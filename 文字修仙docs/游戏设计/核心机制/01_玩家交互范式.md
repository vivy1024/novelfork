# 玩家交互范式 · 修炼/炼丹/炼器统一模型

> **一句话版**：三条核心循环（修炼/炼丹/炼器）共享同一个交互逻辑——「选对象 → 看公式预算 → 执行 → 读结果 → 记录」。玩家不是在点按钮，是在做工程决策。
>
> **关联文档**：[[02_公式开放层级]] · [[子系统/01_炼丹工作台]] · [[总纲/01_设计理念与核心循环]]

## 一、统一交互五步模型

无论是修炼、炼丹还是炼器，玩家经历的认知流程完全一致：

```
Step 1 · 选对象     ─→  Step 2 · 看公式预算  ─→  Step 3 · 执行
    功法/灵材/环境         实时参数代入/结果预估      确认后系统推演

Step 4 · 读结果     ─→  Step 5 · 记录
    FormulaTrace 展开      自动写入知识库/修炼日志
```

这五步对应着不同层次的认知参与：

| 步骤 | 玩家做什么 | 认知收益 |
|------|----------|---------|
| 选对象 | 组合功法/材料/环境 | 理解变量如何影响结果 |
| 看预算 | 读公式代入值，确认风险范围 | 建立直觉——不靠猜 |
| 执行 | 点确认，系统推演 | 验证自己的预期 |
| 读结果 | 看 FormulaTrace，找出与预期的差 | 从误差中学习 |
| 记录 | 完整参数自动进知识库 | 积累可检索的个人档案 |

## 二、修炼循环

### 2.1 准备阶段（选对象 + 看预算）

玩家在"修炼准备"区可见：

```
功法     ：清源引气诀（中性偏木）
辅助灵材 ：低纯度石英  sigma_q=0.45
当前地点 ：学院练气实验楼  Q=0.45
------------------------------------------
实时预算：
  InputRate = Q × σ_q × 呼吸系数
            = 0.45 × 0.45 × 0.8
            = 0.162

  过载风险 O(t) = InputRate / CapacityMargin
               = 0.162 / 0.58
               = 0.279  ← 绿色安全

  ⚠️  换用萤石（σ_q=0.72）→ InputRate 0.259，O(t)=0.447，超过警戒线
```

关键设计点：
- **数字必须是真实代入值**，不是模糊的"高/低"
- **反事实预警**：实时显示"如果换成X，风险变成Y"
- **AI 道侣颜色**：绿→黄→橙→红，对应 Risk 的四个区间

### 2.2 推演阶段（执行 + 读结果）

执行后 ScenePanel 展示：

```
[修炼推演] 120分钟模式

  [校准] 心率 72bpm，体温 36.4°C  → Risk基线 0.05
  [导入] 灵气流入... InputRate = 0.162
         O(t)=0.08  M(t)=0.04  P(t)=0.02  Ψ(t)=0.01
         → Risk = MAX(0.15) ← 安全
  [锁定] 灵气稳定，丹田储能增长
  [收功] V_d +0.024（公式：dV_d = InputRate × η_couple × t_eff）

  ✓ 修炼完成  │  经脉阻抗 -0.2%  │  L_fit 积累 +0.3
```

### 2.3 记录要求

每次修炼自动记录进 `experiment_log`：

| 字段 | 示例 |
|------|------|
| 地点 Q | 0.45 |
| 灵材 σ_q | 0.45（石英） |
| L_fit | 0.37 |
| V_d 增量 | +0.024 |
| Risk 峰值 | 0.15 |
| 走火 | 否 |

## 三、炼丹循环

### 3.1 选材 + 实时预算

炼丹工作台（AlchemyWorkbench.vue）显示：

```
配方   ：静心散（target: 安神益智）
投入   ：酸枣仁皂苷 0.45g  P_target=0.60
         茯苓多糖   0.35g  P_target=0.55
         灵石粉     0.20g  P_target=0.30
------------------------------------------
实时 Match_C 预估：

  配伍矩阵：木×水 → Φ=+1.0（相生加成）
  Interference_spec = -0.18（配伍增效，主峰×1.3）

  有效向量峰值 UsefulPeak = 0.58
  副作用峰值  SidePeak   = 0.10 ← 可接受

  ⚠️  长期服用会削弱自主入静能力（Addiction=0.10）

sigmoid 输入 = 0.35×0.58 - 0.25×0.10 - 0.15×0.12
             + 0.20×0.82 + 0.10×0.65
             = 0.3240
Match_C 预估 = sigmoid(0.3240) × 100 = 58.0 → C级偏上
```

### 3.2 工序执行

每步工序玩家设置参数，系统实时反馈：

```
当前工序：水提酸枣仁
目标温度窗口：60-80°C
你的设定：[ 72 ]°C  ← 在窗口内 ✓

偏高 2°C → 提取效率 +3% / 杂质 +1%
（这是你的选择，需要权衡）
```

### 3.3 产出结算

全参数 FormulaTrace 展开，Match_C 精确值，品质等级，副作用记录。

## 五、三循环的交叉交织

```
              修炼循环
             /        \
        灵材采集      境界突破
           |              |
        资源循环      社会循环
           \              /
            势力声望/任务/月报
```

**时间是三者的共同货币**：

- 修炼消耗时间 → 不能无限堆叠
- 采集消耗时间 → 有机会成本
- 社会事件消耗注意力 → 选择即放弃

玩家每天从06:00到24:00共1080分钟，每个行动消耗真实分钟数（修炼120min/炼丹90min/采集60min等），时间分配本身就是策略决策。

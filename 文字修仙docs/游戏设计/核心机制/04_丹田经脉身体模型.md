# 04_丹田经脉身体模型

> **一句话版**：丹田是灵气的"电池"，经脉是灵气的"电路"——修炼的物理基础不是玄学，是可计算的储能与传导模型。

## §1 丹田储能模型

### 1.1 核心公式

**丹田容量 V_d**

```
V_d = θ0 + θ1·Stage + θ2·Physique + θ3·Material + θ4·Training
```

| 系数 | 含义 | 设计意图 |
|------|------|---------|
| θ0 | 基础容量（1.0） | 所有人都有最低储能能力 |
| θ1·Stage | 境界加成 | 境界越高容量越大（胎息~1→金丹~50） |
| θ2·Physique | 体质加成 | 体质好的人"电池"更大 |
| θ3·Material | 材料加成 | 辅材/法器可以扩展容量 |
| θ4·Training | 有效训练量 | 含 KnowledgeInfra——代际加速的核心机制 |

**储能效率 StoreEff**

```
StoreEff = L_fit × (1 - Noise) × BodyStability × MethodSeal
```

为什么"练得多不一定存得多"：即使修炼时间相同，L_fit 低的人灵气转化效率差，Noise 高的环境灵气损耗大，经脉不稳定的人传导有损耗。StoreEff 是这些因素的乘积，任何一项短板都会拖累整体。

**泄漏率 LeakRate**

```
LeakRate = λ0 + λ1·Poll + λ2·Noise + λ3·Overdrive - λ4·Seal - λ5·Integrity
```

| 增加泄漏 | 原因 |
|---------|------|
| λ1·Poll（污染） | 杂质破坏储能结构的微观完整性 |
| λ2·Noise（噪声） | 环境干扰导致灵气共振失稳 |
| λ3·Overdrive（过载） | 强行灌注超出容量，结构微裂 |

| 降低泄漏 | 原因 |
|---------|------|
| λ4·Seal（功法封存） | 好的功法能在收功时"锁住"灵气 |
| λ5·Integrity（经脉完整度） | 经脉健康则传导无损，不漏气 |

### 1.2 凝丹条件族

突破到金丹需要同时满足四个门槛：

| 门槛 | 条件 | 物理含义 | 设计意图 |
|------|------|---------|---------|
| V_d | ≥ 30.0 | 丹田容量足够大 | 确保有足够灵气完成凝丹 |
| Coherence | ≥ 0.6 | 灵气相干度足够高 | 灵气必须"同频"才能凝聚 |
| LeakRate | ≤ 0.15 | 泄漏率足够低 | 凝丹过程中不能大量漏气 |
| Pollution | ≤ 0.10 | 污染度足够低 | 杂质会破坏丹药结构 |

**为什么凝丹是"工程里程碑"而不是"天赋检查"**：四个条件都是可以通过玩家行动改善的——修炼增加 V_d，净化降低 Pollution，好的功法降低 LeakRate，稳定修炼提高 Coherence。没有任何条件是"天生不行就永远不行"的。

### 1.3 玩家体验

- **丹田容量增长曲线**：前期增长快（θ1·Stage 跳升），中期靠训练积累（θ4），后期靠材料和法器突破上限（θ3）
- **储能效率的反馈**：FormulaTrace 会显示 StoreEff 的每个因子，玩家能看到"换个好环境修炼效率提升 20%"
- **泄漏作为长期管理**：不修炼时灵气会慢慢流失（LeakRate > 0），这不是惩罚而是"维护成本"——就像电池有自放电

## §2 经脉阻抗网络

### 2.1 六节点链路模型

经脉不是"等级"，是一个有物理参数的传导网络：

```
[丹田] ─── [膻中] ─── [百会] ─── [命门] ─── [涌泉] ─── [劳宫]
```

每条经脉有四个参数（R/C/L/S），类比电路：

| 参数 | 电路类比 | 灵气含义 | 范围 |
|------|---------|---------|------|
| R（阻抗） | 电阻 | 灵气流动的阻力，越低越好 | 0.1-5.0 |
| C（容量） | 电容 | 经脉可承载的灵气量 | 0.1-3.0 |
| L（损耗） | 漏电 | 传导过程中的灵气损耗 | 0.0-1.0 |
| S（稳定度） | 绝缘 | 经脉结构的稳定性 | 0.0-1.0 |

**为什么用电路模型而不是"经脉等级"**：等级是一维的（1→2→3），但经脉的状态是多维的——一条经脉可以阻抗低（通畅）但稳定度也低（脆弱），这意味着"好用但容易坏"。这种多维状态让玩家的经脉管理有真正的策略深度。

### 2.2 MeridianFlow 通畅度

```
MeridianFlow = Σ (C_i × S_i) / (R_i × (1 + L_i))
```

通畅度影响三个核心系统：

| 系统 | 影响方式 |
|------|---------|
| 修炼效率 | 直接乘数——通畅度 2.0 = 修炼效率翻倍 |
| 突破条件 | 练气→筑基需 ≥1.0，筑基→紫府需 ≥1.5，紫府→金丹需 ≥2.0 |
| Risk M（失谐） | spectral_mismatch = 0.3 - MeridianFlow×0.1，通畅度越高失谐越低 |

**动态更新**：经脉不是练一次就永久提升的，它会随行动变化：

| 行动 | R（阻抗） | C（容量） | L（损耗） | S（稳定度） |
|------|----------|----------|----------|-----------|
| 修炼成功 | -0.02 | +0.01 | -0.01 | +0.01 |
| 修炼失败 | +0.01 | — | +0.02 | -0.01 |
| 走火入魔 | +0.1 | -0.05 | +0.05 | -0.1 |
| 污染积累 | — | — | +0.02 | -0.02 |
| 净化 | — | — | -0.01 | +0.01 |
| 导灵针法器 | -0.05 | — | -0.02 | +0.02 |

### 2.3 玩家体验

- **经脉是基础设施，不是升级项**：需要持续维护，走火会造成永久损伤（后遗症 I-V 级）
- **走火对经脉的分型损伤**：
  - 失谐型：节点锁死（R 暴增），需要法器干预
  - 过载型：经脉灼伤（S 暴降），需要长期休养
  - 污染型：传导堵塞（L 暴增），需要排毒净化
- **经脉可视化**：前端六节点网络图，每个节点颜色反映 S（绿=健康，红=受损），边的粗细反映 C/R 比

## §3 丹田+经脉联动

### 3.1 修炼时灵气流向

```
环境灵气(Q×Spec_env)
    ↓
经脉传导(MeridianFlow) ──→ Risk M（失谐，频谱不匹配时）
    ↓
丹田储存(StoreEff) ──→ Risk O（过载，超出容量时）
    ↓
实际进度 ΔV = progress × StoreEff × MeridianFlow - V_d × LeakRate
```

### 3.2 突破时的联合检查

突破不是单一条件，而是丹田+经脉+心性的联合检查：

| 检查层 | 条件 | 来源 |
|--------|------|------|
| HardGate | V_d ≥ 门槛 | 丹田 |
| HardGate | MeridianFlow ≥ 门槛 | 经脉 |
| HardGate | Coherence ≥ 门槛 | 丹田 |
| HardGate | LeakRate ≤ 门槛 | 丹田 |
| HardGate | MindScore ≥ 门槛 | 心性 |
| SoftGate | 经脉通畅度加分 | 经脉 |
| SoftGate | 心性储备加分 | 心性 |

突破成功后：
1. 丹田容量 V_d 跳升（境界加成 θ1）
2. 经脉适应新境界（R 微降，C 微升）
3. MentalCapacity 增加（心魔容量扩大）
4. MemFidelity 提升（合理化缓冲下降——隐藏矛盾可能暴露）

突破失败后（走火分型决定）：
1. 经脉受损（R↑ S↓，严重时节点锁死）
2. 丹田可能泄漏（LeakRate 上升）
3. 心性受损（Stability 下降）
4. 后遗症 I-V 级（恢复时间 7天-永久）

## §6 与死亡系统的交互

丹田和经脉是「身体崩溃」死亡线的主要触发源（详见 [[13_死亡机制]]）：

| 死亡触发 | 丹田/经脉参数 | 机制 |
|---------|-------------|------|
| 污染过载 | pollution > 0.8 持续 3 天 | LeakRate 恶性循环→body_hp↓ |
| 经脉全断 | 所有经脉 R > 0.95 | 灵气通路完全堵塞→身体崩溃 |
| 丹田爆裂 | V_d 应力超临界 | 过载灌注→StorageStress→丹田炸裂 |

**濒死缓冲**：body_hp ≤ 0.3 时进入濒死状态，禁止修炼/突破，给玩家自救时间。

## §7 biology 块集成（realms.yaml）

每个境界的 `biology` 块定义该境界对丹田和经脉的参数影响：

```yaml
# realms.yaml 示例
练气:
  biology:
    dantian_capacity_bonus: 5.0    # θ1 跳升的具体值
    meridian_r_reduction: 0.1       # R 初始降低
    leak_rate_base_change: -0.02    # 泄漏率基础改善
    coherence_boost: 0.05           # 相干度基础提升
    lifespan: 144                   # 寿命（tick）
```

`biology` 是境界不仅是数值门槛、更是身体改造的设计载体——突破不是解锁技能，是身体在 LEVSS 场中的进化。

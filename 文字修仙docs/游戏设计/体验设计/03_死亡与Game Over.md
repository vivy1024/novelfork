# 03_死亡与 Game Over 体验

> **一句话版**：死亡是最好的老师——Game Over 不是终点，而是一堂完整的「工程事故分析课」。
> **关联文档**：[[13_死亡机制]] · [[02_公式开放层级]] · [[14_成就系统]]
> **代码对照**：`GameOverScreen.vue` · `death_check.py` · `GameResponse.game_over`

## §1 设计哲学

- **不惩罚失败，奖励理解**：死亡展示完整公式追溯，让玩家理解每一步决策链
- **死得明白**：每条死因都可追溯到具体参数和行为序列
- **情感节奏**：从震惊→理解→接受→重来，每个阶段有对应的 UI 表达

## §2 Game Over 屏幕

### 展示层次

```
1. 震惊层：全屏暗红渐入 + 死因标题
   「身体崩溃 — 污染过载」

2. 信息层：死因详细分析
   存活天数: 47
   临终境界: 练气中期
   ─────────────
   死因链路:
   · Day 38: 污染升至 0.65（警告阈值 0.5）
   · Day 40: 污染持续，body_hp 开始下降
   · Day 44: body_hp 降至 0.15（濒死）
   · Day 47: body_hp = 0.0（死亡）
   ─────────────
   公式追溯:
   pollution = Σ(daily_intake) - Σ(purification)
   = 0.95（累计摄入）- 0.0（未做净化）

3. 教学层：结论提示
   「污染不会自愈。修炼消耗灵材会产生杂质残留，
   需要主动执行「净化」行动来降低污染。
   下次尝试在污染 > 0.3 时及时净化。」

4. 选择层：
   [读取最近存档]  [重新开始]
```

### 情感设计

| 死因 | 情感基调 | 视觉风格 |
|------|---------|---------|
| 污染过载 | 缓慢侵蚀 | 暗绿→黑，毒雾粒子 |
| 走火致命 | 突发爆炸 | 红色闪光→碎裂 |
| 心性崩溃 | 内在瓦解 | 蓝紫渐暗，文字扭曲 |
| 寿元耗尽 | 平静落幕 | 金色渐淡，像黄昏 |
| 灰巷追杀 | 暴力终结 | 暗红急入，裂纹 |

## §3 死亡成就（第 11 章「失败学」）

死亡不白死——特定死法解锁专属成就：

| 成就 | 死因 | 教学 |
|------|------|------|
| 工程事故 | 首次死亡（任何原因） | 死亡是可追溯的 |
| 数据取证 | 死后查看公式追溯 | 每个参数都有意义 |
| 污染陷阱 | pollution > 0.5 时死亡 | 污染不会自愈 |
| 债务螺旋 | 因灰巷追杀死亡 | 利滚利的数学 |

## §4 濒死体验

身体濒死（body_hp ≤ 0.3）或意识濒死（mind_score ≤ 20）时：

- **UI 变化**：屏幕边缘暗红闪烁 + 行动按钮变灰（被禁止的行动）
- **消息提示**：AI 道侣语气警告（「LEVSS Monitor 紧急：生命体征危急，建议立即休息」）
- **允许行动**：只能休息/交易/就寝——给玩家自救窗口

## §5 死后数据保留

死亡后玩家的实验数据、配方发现、知识条目保留在存档中（但需要读档回到最近的save点）。

设计意图：
- 失败产生的知识不应该白白浪费
- 下次从存档重来时，玩家带着「死亡教训」重新决策
- 这不是 roguelike 的全局继承——只是存档系统的自然结果

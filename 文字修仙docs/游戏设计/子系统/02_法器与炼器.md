## §1 法器效能模型

### 1.1 Output 公式

```
Output = SourceCap × PathEff × CoreConv × UserMatch
```

| 因子 | 物理含义 | 数据来源 | 默认值 |
|------|---------|---------|--------|
| SourceCap | 核心灵材的灵气吸收截面 σ_q | 翻译函数 A | 1.0 |
| PathEff | 导灵通路材料的传导效率 η_cond | 翻译函数 A | 0.7 |
| CoreConv | 反应核材料的极化转换率 | 矿物数据库 Polar | 0.5 |
| UserMatch | 使用者 L_fit 与法器五行频谱的一致度 | calc_l_fit | 0.6 |

**为什么是乘法而不是加法**：任何一个因子为零，整个法器就不工作。一把导灵通路断裂的法器（PathEff=0），无论核心多好都没用。这反映了工程系统的"木桶效应"。

### 1.2 品质评分（6因子加权）

```
ArtifactScore = f1·CoreQuality + f2·PathQuality + f3·ShellStability + f4·CraftSkill - f5·DefectPenalty + f6·Synergy
```

| 系数 | 因子 | 默认权重 | 设计意图 |
|------|------|---------|---------|
| f1 | 核心材料品质 | 0.30 | 法器上限由核心灵材决定 |
| f2 | 通路材料品质 | 0.20 | 好材料也需要好通路 |
| f3 | 外壳稳定性 | 0.15 | 封装保护灵气不泄漏 |
| f4 | 工艺水平 | 0.20 | 好材料也需要好工匠 |
| f5 | 缺陷惩罚 | 0.15 | 工艺缺陷直接降品质 |
| f6 | 部件协同 | 0.10 | 五行搭配的额外加成 |

品质等级映射：

| ArtifactScore | 等级 | 适用阶段 |
|--------------|------|---------|
| 0-20 | 凡器 | 几乎无灵气增幅 |
| 20-40 | 下品法器 | 感气-引气 |
| 40-60 | 中品法器 | 筑基-开窍 |
| 60-80 | 上品法器 | 紫府-金丹 |
| 80-100 | 极品法器 | 需 S 级灵材 + 大师工艺 |

品质影响 Buff 强度：`实际Buff = 基础Buff × (ArtifactScore / 50)`。50 分为基准，低于 50 打折，高于 50 加成。

### 1.3 类型判定（6维功能向量）

法器类型不预设，由功能向量的主导维度决定：

```
FuncVec = [Gather, Store, Conduct, Project, Shield, Assist]
```

| 主导维度 | 法器类型 | 功能 | 关键物理参数 |
|---------|---------|------|------------|
| Gather | 聚灵器 | 聚集环境灵气 | Polar + ApertureArea |
| Store | 储灵器 | 存储灵气 | Trap + ShellSeal |
| Conduct | 导灵器 | 引导灵气流动 | PathEff + Anisotropy |
| Project | 投射器 | 灵气转化为外部效果 | CoreConv + FocusGeometry |
| Shield | 防护器 | 灵气屏障/降噪 | ShellStability + NoiseAbsorb |
| Assist | 辅修器 | 辅助修炼降低风险 | UserResonance + RiskDamping |

**同一材料+不同工艺→不同类型**：石英做成球形聚灵器（Gather 主导），做成针形导灵器（Conduct 主导），做成片状防护符（Shield 主导）。材料决定上限，工艺决定方向。

## §2 Buff 映射系统

### 2.1 各类型法器的 Buff

| 法器类型 | Buff | 数值（基准品质） |
|---------|------|----------------|
| 聚灵器 | cultivation_speed +10%, spirit_gather_bonus +15% | 修炼加速 |
| 储灵器 | spirit_reserve_bonus +20, leak_rate_reduce -0.02 | 灵储扩展 |
| 导灵器 | meridian_flow_bonus +0.3, l_fit_bonus +0.02 | 经脉增强 |
| 投射器 | combat_power_bonus +5, damage_bonus +10% | 战斗增强 |
| 防护器 | risk_reduction -0.05, damage_reduction -10% | 风险降低 |
| 辅修器 | fatigue_reduce +0.02, cultivation_speed +5%, risk_reduction -0.03 | 综合辅助 |

### 2.2 装备槽位

| 槽位 | 名称 | 数量 | 说明 |
|------|------|------|------|
| main_artifact | 主法器 | 1 | 修炼时使用的主要法器 |
| accessory | 辅助法器 | 2 | 提供被动加成 |

装备法器时自动添加永久 Buff（`active_effects` 表，`duration=-1`），卸下时自动移除。多法器 Buff 叠加（`get_effect_sum` 求和）。

## §3 炼器工艺

### 3.1 炼器流程

**一键炼器**（`craft_artifact`）：
```
选配方 → 投入材料 → 品质计算 → 类型判定 → 生成法器 → 入背包
```

**多步炼器**（`start_project` + `advance_step`，3步）：

| 步骤 | 名称 | 耗时 | 技能检查 | 决策事件 |
|------|------|------|---------|---------|
| 1 | 材料加工 | 3h | prof_refining ≥ 0.10 | 材料裂纹（重做/填补/废弃） |
| 2 | 灵路刻画 | 4h | prof_refining ≥ 0.20 | 灵感涌现（重新设计/按计划） |
| 3 | 组装封印 | 2h | prof_refining ≥ 0.15 | 共振测试（微调/接受） |

与炼丹的区别：
- 炼器 3 步 vs 炼丹 5 步
- 炼器更依赖 prof_refining，炼丹依赖 prof_alchemy
- 炼器材料是矿物灵材，炼丹材料是本草灵材
- 法器是持久装备，丹药是消耗品

### 3.2 初级配方（5个）

| 配方 | 类型 | 核心材料 | 通路材料 | 技能门槛 |
|------|------|---------|---------|---------|
| 聚灵石 | 聚灵器 | 石英(C级) | 铜线(D级) | 0.10 |
| 储灵坠 | 储灵器 | 萤石(C级) | 银丝(D级) | 0.15 |
| 导灵针 | 导灵器 | 磁铁矿(C级) | 铜线(D级) | 0.20 |
| 护身符 | 防护器 | 玉石(C级) | 朱砂(D级) | 0.15 |
| 辅修环 | 辅修器 | 电气石(C级) | 银丝(D级) | 0.20 |

### 3.3 玩家体验

- **筑基期解锁炼器**：因为需要经脉通畅度 ≥1.0 才能感知材料的灵气属性
- **法器是长期投资**：制作成本高（消耗矿物灵材），但装备后持续提供 Buff
- **公式透明**：FormulaTrace 展示 ArtifactScore 的 6 个因子 + FuncVec 的 6 个维度 + Output 的 4 个乘数

## §4 材料来源

### 4.1 四段循环（Batch 7）

```
采集原矿 → 精炼加工(块体/烧结体/陶胚) → 灵材认证(民用/特许/管制) → 装配入器
```

### 4.2 材料族（Batch 6）

| 材料族 | 来源 | 优势 | 劣势 |
|--------|------|------|------|
| 天然矿物 | 矿脉开采 | 天然灵气印记 | 品质不稳定 |
| 人造陶瓷 | 高温烧结 | 成分可控、批量生产 | 灵气亲和度低 |
| 半导体 | 晶体生长 | 陷阱态储能高 | 成本极高 |
| 聚合物 | 合成/3D打印 | 形状自由、低成本 | 灵气亲和度最低 |

所有材料族共用翻译函数 A 的 7 维特征向量接口。

# 14_成就系统

> **一句话版**：成就不是奖杯，是课程表——90 个成就分 12 章，引导玩家从「观察者」到「设计者」的认知成长。
> **关联文档**：[[01_设计理念与核心循环]] · [[02_公式开放层级]] · [[01_新手教学]]
> **代码对照**：`achievement_service.py` · `data/achievements.yaml` · `GET /achievements`

---

## §1 设计哲学

- **成就 = 教学**：每个成就教会玩家一个概念或技巧
- **12 章 = 12 门课**：从灵气物理到失败学，覆盖全部核心机制
- **自然触发**：不是刻意刷成就，而是正常游玩中自然解锁
- **Steam 集成**：每个成就对应一个 Steam API name

## §3 成就结构

```yaml
- id: safe_window
  name: 安全窗口
  chapter: 1
  category: 灵气物理
  description: 连续 3 次修炼 Risk 均在 θ_warn(0.38) 以下
  hidden: false
  trigger: action_complete
  condition:
    type: consecutive_safe_cultivate
    threshold: 3
  steam_api_name: ACH_SAFE_WINDOW
```

### 数据字段

| 字段 | 说明 |
|------|------|
| trigger | 触发时机（action_complete / node_complete / realm_change 等） |
| condition.type | 检查器类型，映射到代码中的检查函数 |
| hidden | 是否隐藏（第 10 章全部隐藏） |
| steam_api_name | Steam Achievement API 对应名称 |

## §4 检查器架构

```
玩家行动
    ↓
AchievementService.check_achievements()
    ↓
按 trigger 分发到检查器
    ↓
40+ 个检查函数，每个对应一种 condition.type
    ↓
条件满足 → 写入 achievement 表 + 返回通知
```

### 6 个埋点

| 埋点位置 | 触发的 trigger |
|---------|---------------|
| action_service | action_complete |
| node_runner | node_complete |
| event_engine | story_flag |
| knowledge | knowledge_unlock |
| faction_service | faction_change |
| world_engine | world_event |

## §5 前端展示

- **AchievementPanel**：12 章分类浏览，已解锁/未解锁/隐藏分组
- **AchievementToast**：解锁时全屏通知（粒子动画 + 音效）
- **GET /achievements**：全量返回，含进度百分比
- **GET /achievements/recent**：最近 5 个解锁成就

## §6 典型成就举例

### 第一章「灵气物理」—— 教学线

| 成就 | 触发 | 教了什么 |
|------|------|---------|
| 第一缕灵气 | 完成首次修炼 | 基础操作 |
| 环境变量 | 两个地点各修炼 1 次 | 灵压影响 InputRate |
| 风险四象限 | 查看 Risk 完整说明 | Risk 不是单一数字 |
| 深夜代价 | 22:00 后修炼 | 疲劳的隐藏加成 |

### 第十一章「失败学」—— 逆教学线

| 成就 | 触发 | 教了什么 |
|------|------|---------|
| 污染陷阱 | pollution > 0.5 | 污染不会自愈 |
| 债务螺旋 | 总债务 > 2000 | 利滚利的数学 |
| 工程事故 | 首次死亡 | 死亡有原因 |
| 数据取证 | 死后查看公式追溯 | 每个参数都有意义 |

## §1 设计哲学

- **经济压力是核心驱动力**：参考《没钱修什么仙》张羽的 70 万→百亿债务轨迹
- **借贷不是暂停键**：借来的钱要还，利息不会停，催收不会停
- **三个渠道 = 三种风险等级**：学院低息安全、老张中息灵活、灰巷高息无门槛
- **惩罚递进**：不是突然爆炸，而是一步步收紧——关系下降→行动限制→催收→追杀

## §2 三级借贷渠道

| 渠道 | 月利率 | 额度 | 好感度门槛 | 风格 |
|------|--------|------|-----------|------|
| 学院助学贷 | 3% | 500 SQU | 学院好感 ≥ 15 | 安全规范，审批严格 |
| 老张私贷 | 8% | 2000 SQU | 老张好感 ≥ 5 | 灵活人情，利息不低 |
| 灰巷高利贷 | 15% | 5000 SQU | 无门槛 | 来者不拒，利滚利 |

### 设计意图

- **学院贷**：正道之选，利率低但门槛高，让玩家维护学院关系有回报
- **老张贷**：中间路线，代表人情社会——老张帮你但不是白帮
- **灰巷贷**：最后的稻草，任何时候都能借，但利率会吃人

## §3 借还流程

### 借款

```
POST /action/do { action: "borrow", target_id: "gray_loan", quantity: 200 }

检查链：
1. 渠道是否存在
2. 对应势力好感度是否达标
3. 该渠道剩余额度是否足够
4. → 创建贷款记录 + 资金入账 + 更新 debt 总额
```

### 还款

```
POST /action/do { action: "repay", target_id: "loan_xxxx", quantity: 100 }

检查链：
1. 贷款是否存在且为 active 状态
2. 玩家余额是否足够
3. → 扣款 + 更新贷款余额（或结清）+ 更新 debt 总额
```

## §4 月度利息

每月推演时（`_step7_economy`）自动计算：

```
total_owed = principal × (1 + interest_rate)
```

利息直接加到欠款总额上。月报中显示利息增长数字。

## §5 六级债务惩罚阶梯

| 级别 | 触发阈值 | 效果 | 消息 |
|------|---------|------|------|
| 1 | 200 SQU | 所有势力好感 -1/月 | 「欠债的名声开始传开了」 |
| 2 | 500 SQU | 部分地点不欢迎你 | 「某些场所开始回避你」 |
| 3 | 1000 SQU | NPC 拒绝与你交易 | 「没人愿意跟欠债的做生意」 |
| 4 | 2000 SQU | 催收事件（30%/月） | 「有人堵在你家门口讨债」 |
| 5 | 3000 SQU | QESA 信用降级 | 「你的信用评级被下调」 |
| 6 | 5000 SQU | 灰巷追杀（30%/月扣 HP） | 「灰巷的人找上门了...」 |

## §6 玩家体验设计

```
初期（胎息）：月度账单 60 SQU → 修炼消耗灵材 → 钱不够用 → 考虑借贷
中期（练气）：打工/卖丹/月俸 → 收支平衡 → 还清旧债 → 稳定发展
后期（筑基）：多渠道收入 → 借贷变成投资工具 → 借低息投高回报
失败线：灰巷借太多 → 利滚利 → 惩罚累积 → 被追杀 → Game Over
```

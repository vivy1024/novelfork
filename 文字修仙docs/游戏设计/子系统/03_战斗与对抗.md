# 03_战斗与对抗

> **一句话版**：战斗不是核心循环，而是修炼成果的验证场——战力由公式决定，玩家的选择在战前而非战中。
> **关联文档**：[[12_竞技比赛]] · [[02_法器与炼器]] · [[13_死亡机制]]

## §1 设计哲学

本游戏不是动作游戏。战斗是**文字推演**，不是实时操作。

玩家的战斗决策发生在战前：选什么功法修炼、装什么法器、吃什么丹药、心性状态如何。战斗本身是这些准备的自然结果。

这和世界观一致：修仙者之间的对抗，本质是"谁的工程做得更好"。

| 筑基 | 35 | +15 |

| 金丹 | 80 | +25 |
| 道胎 | 120 | +40 |

### 2.2 战力公式

代码位置：`rule_engine.py → calc_combat_power`

```
CombatPower = base_power                    # 境界基准
            + realm_minor × 2.5             # 小境界阶段加成
            + prof_cultivate × 10.0         # 修炼熟练度加成
            + (l_fit - 0.5) × 8.0           # L_fit 修正（0.5 为中性）
            + artifact_output × 10.0        # 法器输出加成
            + buff_bonus                    # 丹药/Buff 加成
            + (env_spirit_pressure - 0.3) × 5.0  # 环境灵压修正
            - fatigue × 8.0                 # 疲劳惩罚
            - injury_level × 15.0           # 伤势惩罚
```

最终值 clamp 到 `max(1.0, power)`。

### 2.3 各项系数解读

| 项目 | 系数 | 范围示例 | 贡献范围 | 设计意图 |
|------|------|---------|---------|---------|
| 阶段加成 | ×2.5 | minor 0–9 | 0–22.5 | 同境界内的成长感 |
| 熟练度 | ×10.0 | prof 0–1 | 0–10.0 | 勤修苦练有回报 |
| L_fit | ×8.0 | l_fit 0–1 | -4.0–+4.0 | 天赋差异 |
| 法器 | ×10.0 | artifact 0–1 | 0–10.0 | 装备投资 |
| 灵压 | ×5.0 | sp 0–1 | -1.5–+3.5 | 地利优势 |
| 疲劳 | ×8.0 | fatigue 0–1 | 0–-8.0 | 疲劳战斗的代价 |
| 伤势 | ×15.0 | injury 0–1 | 0–-15.0 | 带伤战斗极其危险 |

### 2.4 战力计算示例

**练气五阶修士，中等装备，轻微疲劳**：

```
base_power = 20                          # 练气基准
+ 5 × 2.5 = 12.5                        # 五阶加成
+ 0.5 × 10.0 = 5.0                      # 熟练度 50%
+ (0.6 - 0.5) × 8.0 = 0.8              # L_fit 0.6
+ 0.3 × 10.0 = 3.0                      # 法器输出 0.3
+ 0                                      # 无 buff
+ (0.4 - 0.3) × 5.0 = 0.5              # 灵压 0.4
- 0.2 × 8.0 = -1.6                      # 疲劳 0.2
- 0.0 × 15.0 = 0                        # 无伤
= 40.2
```

## §3 五行克制矩阵

代码位置：`rule_engine.py → _ELEMENT_ADVANTAGE`

### 3.1 克制关系

```
金(metal) → 木(wood) → 土(earth) → 水(water) → 火(fire) → 金(metal)
```

### 3.2 完整矩阵

| 攻方 ↓ \ 守方 → | metal | wood | earth | water | fire |
|-----------------|-------|------|-------|-------|------|
| **metal** | 0 | **+3.0** | **-1.5** | 0 | 0 |
| **wood** | **-1.5** | 0 | **+3.0** | 0 | 0 |
| **earth** | 0 | **-1.5** | 0 | **+3.0** | 0 |
| **water** | 0 | 0 | **-1.5** | 0 | **+3.0** |
| **fire** | **+3.0** | 0 | 0 | **-1.5** | 0 |

- **克制（相克）**：+3.0 战力差加成
- **反克（被克）**：-1.5 战力差惩罚
- **无关系**：0

### 3.3 五行加成应用

```python
effective_diff = (atk_power - def_power) + element_advantage
```

五行加成直接叠加到战力差上，影响胜率计算。

## §4 胜率与伤害计算

代码位置：`rule_engine.py → calc_combat_result`

### 4.1 胜率公式

```python
beta = 0.15
raw_rate = 1.0 / (1.0 + exp(-beta × effective_diff))
win_rate = clamp(raw_rate, 0.01, 0.99)
```

sigmoid 函数特性：
- `effective_diff = 0` → `win_rate = 0.50`（势均力敌）
- `effective_diff = +10` → `win_rate ≈ 0.82`（明显优势）
- `effective_diff = +20` → `win_rate ≈ 0.95`（压倒性优势）
- `effective_diff = -10` → `win_rate ≈ 0.18`（明显劣势）
- 永远不会 100% 必胜或必败（夹到 [0.01, 0.99]）

### 4.2 胜率速查表

| 战力差 (effective_diff) | 胜率 |
|------------------------|------|
| -20 | 5% |
| -15 | 10% |
| -10 | 18% |
| -5 | 32% |
| 0 | 50% |
| +5 | 68% |
| +10 | 82% |
| +15 | 90% |
| +20 | 95% |

### 4.3 伤害计算

```python
alpha = 0.04
base_damage = uniform(24, 36)              # 基础伤害随机
power_factor = exp(alpha × |effective_diff|)  # 战力差放大系数

# 赢家
if attacker_wins:
    damage_to_defender = base_damage × power_factor    # 输家受重伤
    damage_to_attacker = base_damage / power_factor    # 赢家受轻伤
else:
    damage_to_attacker = base_damage × power_factor
    damage_to_defender = base_damage / power_factor
```

### 4.4 切磋伤害减半

```python
spar_mult = 0.5 if is_spar else 1.0
damage_to_attacker *= spar_mult
damage_to_defender *= spar_mult
```

### 4.5 HP 归一化

```python
hp_base = 100.0
dmg_atk_norm = min(damage_to_attacker / hp_base, 1.0)
dmg_def_norm = min(damage_to_defender / hp_base, 1.0)
```

所有伤害归一化到 0–1 范围，对应 injury_level 的增量。

### 4.6 伤害示例

**战力差 +10 的正式战斗（攻方胜）**：

```
base_damage = 30（取中值）
power_factor = e^(0.04 × 10) = e^0.4 ≈ 1.492

守方受伤: 30 × 1.492 = 44.8 → 归一化 44.8/100 = 0.448
攻方受伤: 30 / 1.492 = 20.1 → 归一化 20.1/100 = 0.201
```

**同样条件的切磋**：

```
守方受伤: 44.8 × 0.5 = 22.4 → 归一化 0.224
攻方受伤: 20.1 × 0.5 = 10.0 → 归一化 0.100
```

## §5 战斗场景

### 5.1 切磋

- 触发：玩家主动对 NPC 发起
- 前提：NPC 关系 ≥ 中等，且 NPC 愿意
- 后果：伤害减半（`spar_mult = 0.5`），主要获得经验
- 战利品：无物品掉落，仅获得修炼经验和少量声望
- 设计目的：安全的战力测试场

### 5.2 遭遇战

- 触发：探索时随机遭遇敌对 NPC/野兽
- 前提：地点危险等级 + 随机
- 后果：完整战斗后果（`spar_mult = 1.0`）
- 战利品（胜利时）：
  - 灵材掉落（品质与敌方境界相关）
  - 声望 +3~8（视敌方强度）
  - 小概率获得法器碎片
- 失败后果：
  - injury_level 大幅上升
  - 可能丢失背包中的灵材（10% 概率）
  - 声望 -1~3
- 设计目的：探索的风险成本

### 5.3 势力冲突

- 触发：月度推演中势力关系恶化
- 前提：玩家所在势力与敌对势力冲突
- 后果：可能被迫参战或选择回避
- 战利品（胜利时）：
  - 势力声望 +5~15
  - 势力资源奖励
  - 可能解锁新地点或配方
- 回避代价：
  - 势力声望 -5
  - 可能影响 NPC 关系
- 设计目的：世界活性的体现

## §6 玩家体验目标

- **不鼓励刷怪**：战斗有疲劳和受伤成本，频繁战斗会影响修炼
- **鼓励准备**：战前的法器选择、丹药使用、心性调整才是关键
- **公式透明**：战力计算过程通过 FormulaTrace 完全展示，玩家可以看到每一项加成和惩罚
- **结果有意义**：胜利带来声望和资源，失败带来真实代价（伤势恢复需要时间）
- **五行策略**：选择与敌方相克的功法/法器可以获得 +3.0 的战力差加成，相当于半个小境界的优势
- **境界碾压**：跨境界战斗几乎不可能翻盘（练气 vs 筑基 = 20 vs 35，差距 15 → 胜率仅 10%）

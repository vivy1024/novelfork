## §1 设计哲学

- **知识 = 亲历**：没采过的灵材你不认识，没做过的配方你不知道效果
- **三层逐步深入**：物种识别 → 样本鉴定 → 配伍经验
- **与公式追溯联动**：未解锁的知识在 FormulaTrace 中显示为「???」，引导玩家主动探索

## §2 三层知识架构

| 层级 | 类别 | 触发条件 | 解锁内容 |
|------|------|---------|---------|
| **第一层** | 物种识别 | 首次采集/接触灵材 | 名称、基本形态、五行归属 |
| **第二层** | 样本鉴定 | 对样本执行「鉴定」行动（消耗 1 回合） | 纯度、品质、详细参数 |
| **第三层** | 配伍经验 | 炼丹时使用过的材料组合 | 配伍匹配度、最佳工序记录 |

### 解锁流程

```
采集灵石草 → 第一层解锁：「识别：灵石草」
    ↓
鉴定灵石草·粉末 → 第二层解锁：「鉴定：灵石草·粉末」纯度 72%，品质 0.65
    ↓
灵石草+泽兰炼丹 → 第三层解锁：「配伍经验：灵石草+泽兰 → 回灵丹，Match_C 52」
```

## §5 数据存储

- `knowledge_entry` 表：`entry_id / category / title / unlocked / tick / source`
- `experiment_log` 表：鉴定和配伍的详细数据记录
- 知识条目 ID 格式：`herb:灵石草` / `sample:灵石草:粉末` / `recipe:灵石草+泽兰`

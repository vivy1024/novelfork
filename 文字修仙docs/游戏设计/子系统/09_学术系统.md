## §1 设计哲学

学术系统让「修仙 = 工程」从修炼维度扩展到知识维度：

- **实验→发现→发表**：修炼/炼丹产生实验数据，数据积累到阈值自动检测到规律
- **论文 = 知识输出**：玩家不仅是知识消费者，也是知识生产者
- **学术声望 = 另一种资本**：论文被引用、评审通过，声望提升，解锁高级资源

## §2 三步循环

```
实验记录（experiment_log）
    ↓ 自动检测
规律发现（pattern_detection）
    ↓ 玩家撰写
论文发表 → 同行评审 → 学术声望
```

## §3 规律检测引擎

引擎定期扫描 `experiment_log` 表，匹配预定义模板：

| 规律模板 | 触发条件 | 示例 |
|---------|---------|------|
| `l_fit` 收敛 | 连续 N 次修炼 L_fit 趋势上升 | 「你发现长期修炼使灵气适配度稳步提升」 |
| Risk 分量关联 | Risk 四分量出现特定相关性 | 「高灵压下 O(t) 与 P(t) 正相关」 |
| 炼丹配伍规律 | Match_C 在特定材料组合下稳定高分 | 「泽兰+石膏组合的 Match_C 稳定高于 60」 |
| 采气效率 | 特定灵气类型在特定地点效率异常 | 「城郊采样区的水系灵气纯度最高」 |
| 丹田参数 | V_d 或泄漏率出现拐点 | 「V_d 超过 15 后泄漏率增速明显加快」 |

### 检测参数

- **扫描窗口**：最近 30 条实验记录
- **最低样本量**：每种模板至少 5 次相关实验
- **显著性阈值**：相关系数 ≥ 0.7 或趋势 p-value < 0.05

## §4 论文系统

### 4.1 撰写论文

```python
write_paper(pattern_id, title)
# 消耗 3 小时，需要已发现的规律作为素材
# 论文质量 = f(规律检测置信度, 实验样本量, 在相关领域的学术积累)
```

### 4.2 同行评审

```python
submit_and_review_paper(paper_id)
# 3 个随机 NPC 审稿人，各自评分 0-100
# 评审维度：创新性 / 严谨性 / 实用性
# 结果：接受(均分≥60) / 修改后接受(40-59) / 拒稿(<40)
```

### 4.3 学术声望

| 声望事件 | 声望值 |
|---------|--------|
| 论文被接受 | +10-20（取决于质量） |
| 论文被拒稿 | 0（不扣分） |
| NPC 引用你的论文 | +5/次 |
| 论文获奖 | +30 |
| 累计发表 5 篇 | +20（里程碑） |

### 4.4 NPC 学术行为

每月推演时，NPC 有 40% 概率发表自己的论文。月报中展示 NPC 学术动态，让世界感觉「活」的。

## §5 玩家体验

- **不是必做**：学术系统完全可选，不做论文也能推进
- **但有回报**：学术声望解锁高级知识条目、NPC 对话选项、特殊材料折扣
- **与势力联动**：学院势力特别重视学术，高学术声望 = 学院好感度加速

# 13_死亡机制

> **一句话版**：修仙是工程，工程会出事故——死亡不是惩罚，是两条工程事故线（身体崩溃 / 意识崩溃）的自然后果，加上寿命兜底。
> **关联文档**：[[04_丹田经脉身体模型]] · [[03_心性与动机系统]] · [[02_公式开放层级]]
> **代码对照**：`death_check.py` · `day_manager.py → _phase_death_check` · `api/routes/action.py → alive 检查`

---

## §1 设计哲学

- **死亡 = 工程事故**：不是天劫降临，不是随机暴毙——每一次死亡都可追溯到具体参数
- **两条死亡线**：身体崩溃（物理层面）和意识崩溃（精神层面），互相独立
- **濒死缓冲**：不会一瞬间死亡，先进入濒死状态，给玩家自救机会
- **工程日志**：死因展示完整公式追溯，让玩家「死得明白」

## §2 身体崩溃（4 条触发线）

| 触发条件 | 机制 | 可预防 |
|---------|------|--------|
| 污染过载 | `pollution > 0.8` 持续 3 天 → body_hp 每天 -0.1 | 及时净化 |
| 经脉全断 | 所有经脉阻抗 > 0.95 | 避免连续走火 |
| 丹田爆裂 | 丹田应力 > 临界值 | 控制灵力注入 |
| 走火致命 | V 级走火（最严重） | 控制 Risk < θ_crit |

### body_hp 系统

```
body_hp ∈ [0.0, 1.0]
  1.0 = 完全健康
  0.5 = 重伤
  0.3 = 濒死（触发行动限制）
  0.0 = 死亡
```

## §3 意识崩溃（2 条触发线）

| 触发条件 | 机制 | 可预防 |
|---------|------|--------|
| 心性三维归零 | 内省/整合/稳定 ≤ 0 持续 5 天 | 冥想/休息/社交修复 |
| 心魔累积 | inner_demon > 0.9 持续 5 天 → mind_score 每天 -5 | 正念认知训练 |

### mind_score 系统

```
mind_score = f(introspection, integration, stability)
  ≤ 0 → 意识崩溃 → 死亡
```

## §4 寿命系统

自然寿命兜底——即使不出事故，也有时间限制。

| 境界 | 寿命（tick） |
|------|------------|
| 胎息 | 96 |
| 练气 | 144 |
| 筑基 | 280 |
| 紫府 | 640 |
| 金丹 | 1500 |
| 元婴 | 3600 |

**每日检查**：`current_tick > max_lifespan` → 死亡（死因：「寿元耗尽」）

## §5 濒死状态

当 body_hp ≤ 0.3 或 mind_score ≤ 20 时进入濒死状态：

| 限制 | 效果 |
|------|------|
| 禁止修炼 | 修炼会加速死亡 |
| 禁止突破 | 风险太高 |
| 禁止炼丹 | 身体无法承受 |
| 禁止探索 | 无力行动 |
| 可以：休息、交易、就寝 | 自救窗口 |

## §6 Game Over 流程

```
death_check() → {game_over: true, death_info: {...}}
    ↓
GameResponse.game_over = true
    ↓
前端：GameOverScreen
    → 展示死因（完整公式追溯）
    → 选择：读取存档 / 重新开始
```

### 死因展示

| 字段 | 内容 |
|------|------|
| cause | 死因类别（污染过载/走火致命/心性崩溃/寿元耗尽） |
| trigger_detail | 具体触发参数（如「pollution = 0.95, 持续 5 天」） |
| survival_days | 存活天数 |
| realm_at_death | 临终境界 |

## §7 与其他系统的交互

| 系统 | 交互 |
|------|------|
| 修炼 | 走火→body_hp↓，过载→pollution↑ |
| 炼丹 | 副作用→pollution↑ |
| 借贷 | 灰巷追杀→body_hp↓ |
| 心性 | 心魔→mind_score↓ |
| 竞技 | 正式赛受伤→body_hp↓ |

# 02_本草系统精简总览

> 状态日期：2026-03-12  
> 目的：把 `22-29` 这些过程文档压缩成一篇能快速建立全局认识的总览。

## 1. 一句话结论

当前本草系统已经形成 4 层：

1. `上游源`  
   TCMID / TCMBank 等原始数据库
2. `Rule-C library`  
   能进入规则层的本草大库
3. `master`  
   合并来源、分类、关系后的总表
4. `world_enriched`  
   在不覆盖原始源字段的前提下，为玩法层补全 `产地` 和 `ingredient`

如果你现在只想“拿来写世界、写剧情、写玩法”，重点看第 4 层。

## 3. 这四层分别解决什么问题

### 上游源

解决“有没有资料”。

### Rule-C library

解决“能不能把本草条目映射进规则引擎”。

### master

解决“来源、分类、关系能不能放在一张表里查清楚”。

### world_enriched

解决“原始源缺字段时，玩法层还能不能先用”。

## 4. 为什么还要 world_enriched

因为当前原始主库的缺口非常明确：

- `origin missing = 972`
- `rows_with_ingredient_links = 23`
- `rows_with_ingredient_targets = 7`
- `rows_with_ingredient_diseases = 6`

这意味着：

- 如果只依赖原始源层，玩法层会大量卡在“没有产地”或“没有 ingredient link”
- 所以必须加一层“世界化补全”，先让系统能运行

## 5. world_enriched 具体怎么补

### 产地

按：

- 五行频谱
- 资产类别
- 高灵区地理模板

做可复现推断。

### ingredient

按三层优先级补：

1. `tcmbank_linked`
2. `component_bridge`
3. `family_template`

也就是说，先用真实关系；没有真实关系时，用已降噪成分桥接；再不够，就按规则模板补。

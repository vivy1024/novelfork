# 01_本草主库现状速览

> 状态日期：2026-03-12  
> 目的：把当前本草数据链压缩成一页能看懂的说明，避免在 `25-28` 之间反复对数字。

## 1. 一句话结论

当前项目已经不再是“只有一些零散灵植样本”，而是已经有了一套 **`1018` 条可直接进 Rule-C 的本草主库**。  
在此基础上，又新增了一层 **`world_enriched` 世界化补全层**，用于把缺失的 `产地` 和 `ingredient` 先补成可玩的玩法字段。  
但原始源层仍然不是一个完成度很高的“产地资源库”或“关系药理库”，主缺口仍在：

- 产地层过薄
- 物种认证还没全部压实
- ingredient -> target / disease 关系刚打通，覆盖仍低

## 2. 三层对象怎么读

这几层最容易看混，先明确：

- `上游源`
  - 外部抓回来的原始资料，不直接等于可玩资产。
- `Rule-C herb library`
  - 已经过规则映射、成分降噪、批量下沉后的可用本草大库。
- `classified`
  - 在大库基础上补了类别、物种认证、产地信号。
- `master`
  - 把 `library + classified + TCMID + TCMBank` 合并后的总表。
- `world_enriched`
  - 在不覆盖原始源字段的前提下，按世界地理禀赋和 Rule-C 字段为玩法层补全最终可用的 `产地` 与 `ingredient`。

一句话理解：

- `library` 解决“能不能玩”
- `classified` 解决“是什么资产”
- `master` 解决“来源、分类、关系能不能放在一张表里统一查”
- `world_enriched` 解决“原始源不够完整时，玩法层还能不能先用”

## 7. 世界化补全层

当前玩法层不再直接卡死在原始缺口上，而是新增了一层：

- `tcm_materia_medica_world_enriched.csv`
- `tcm_materia_medica_world_enriched.sqlite`
- `tcm_materia_medica_world_enriched_summary.json`

它做两件事：

1. 对缺失产地的条目，按 `五行频谱 + 资产类别 + 高灵区模板` 生成可复现的世界化产地
2. 对缺失 ingredient link 的条目，按 `真实 ingredient -> Comp_i_primary -> 成分族模板` 三层回填

当前补全结果：

- `rows_with_world_origin_provinces = 1018`
- `rows_with_world_ingredient_final = 1018`
- `origin_mode`: `source_recorded 29 / source_plus_geo_inferred 17 / geo_inferred 972`
- `ingredient_mode`: `tcmbank_linked 23 / component_bridge 379 / family_template 616`

## 8. 现在最该怎么理解这套系统

如果只想抓住全局，可以按这个顺序理解：

1. 规则层已经够用了，问题不在 Rule-C 本身。
2. herb 大库已经够大了，问题也不在“样本只有一百条”。
3. 现在真正缺的是：
   - 产地
   - 物种压实
   - ingredient 关系扩深
4. 如果是玩法层使用，先读 `world_enriched`；如果是源数据治理，继续读 `master`。
5. 所以下一步重点不是继续无脑扩总量，而是补“高价值深度”。

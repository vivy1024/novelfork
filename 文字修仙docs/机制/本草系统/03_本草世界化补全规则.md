# 03_本草世界化补全规则

> 状态日期：2026-03-12  
> 目的：说明如何在不覆盖原始源字段的前提下，把缺失的 `产地` 和 `ingredient` 补成玩法层可用版本。

## 1. 原则

这层不是“伪造源数据”，而是：

- 保留原始字段不动
- 新增一组玩法层字段
- 所有推断都可复现，不是每次乱跳

也就是说：

- `master` 负责保存原始世界
- `world_enriched` 负责保存可玩世界

## 2. 产地补全怎么做

### 已有源字段时

- 如果已有 `province_level` 或更强信息，直接保留
- 但仍允许补一个 `高灵区模板`，方便世界层使用

### 缺失时

按以下三项推断：

1. `Spec_i_json` 的主导五行
2. `asset_major_class`
3. 世界内已定义的高灵区模板

当前使用的高灵区模板包括：

- 昆仑山脉
- 长白山
- 峨眉山
- 终南山
- 武当山
- 龙虎山
- 崂山
- 华山
- 衡山
- 罗浮山
- 武夷山
- 黄山
- 神农架
- 九华山

### 为什么是这些地点

因为它们已经在 [13_高灵区地理志](../规则底座/13_高灵区地理志.md) 里有明确地质与五行禀赋描述，可以拿来做稳定模板。

## 3. ingredient 补全怎么做

### 第一层：`tcmbank_linked`

如果主库已经有真实 ingredient link，就直接用。

### 第二层：`component_bridge`

如果没有真实 ingredient link，但已有：

- `Comp_i_primary_json`
- `Comp_i_rulec_source`

就直接把它桥接成玩法层 ingredient。

### 第三层：`family_template`

如果前两层都没有，就按：

- `Comp_i_family_json`
- `recommended_dan_category_rulec`
- `dominant_effect_axis_rulec`
- `Spec_i_json`
- `asset_major_class`

生成一组可解释的模板 ingredient。

## 4. 为什么这样合理

因为当前真实缺口很大：

- 大多数条目没有可靠产地
- 大多数条目没有真实 ingredient link

如果坚持只用“纯原始源”，玩法层就会大面积断掉。  
所以必须先有一层 **“结构正确、来源分级明确、可复现”** 的补全结果。

## 5. 当前补全模式分布

### 产地

- `source_recorded = 29`
- `source_plus_geo_inferred = 17`
- `geo_inferred = 972`

### ingredient

- `tcmbank_linked = 23`
- `component_bridge = 379`
- `family_template = 616`

这能很直观地看出：

- 真实源仍然薄
- 但玩法层已经不再缺字段

## 6. 使用边界

这层适合：

- 世界构建
- 剧情节点
- 地图资源分布
- 价格与黑市逻辑
- 丹药与配伍的玩法层原型

这层不适合：

- 充当真实学术数据库
- 替代原始源审计
- 反向宣称“这些产地和 ingredient 是真实数据”

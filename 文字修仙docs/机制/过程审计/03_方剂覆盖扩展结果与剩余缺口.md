# 03_方剂覆盖扩展结果与剩余缺口

> 状态日期：2026-03-12
> 目的：记录 `expand_rule_c_herb_coverage.py` 落地后的 herb 扩库结果、方剂覆盖提升幅度，以及当前仍未解决的尾部缺口。
> 文档类型：过程 / 审计文档。快速理解当前系统，请先看 [[02_本草系统精简总览]]、[[03_本草世界化补全规则]]、[[04_本草给玩法层的直接接口]]。

## 1. 这轮做了什么

本轮不是继续盲目抓新源，而是改成“按方剂缺口反推 herb 扩库”：

- 读取 `data/processed/tcmid_formula_components.csv`
- 找出当前 `tcm_herb_rulec_final.csv` 里仍未命中的药材
- 从 `TCMID / TCMBank` 回填最小可用 herb 条目
- 直接生成扩展 herb 库，再重跑 formula 聚合

对应脚本：

- `scripts/expand_rule_c_herb_coverage.py`
- `scripts/build_rule_c_formula_db.py`

对应产物：

- `data/processed/tcm_herb_rulec_expanded.csv`
- `data/processed/tcm_herb_rulec_expanded.sqlite`
- `data/processed/tcm_herb_rulec_expanded_summary.json`
- `data/processed/tcm_formula_rulec_expanded.csv`
- `data/processed/tcm_formula_rulec_expanded.sqlite`
- `data/processed/tcm_formula_rulec_expanded_summary.json`

## 2. herb 扩库结果

扩库前：

- 基础 herb final：`40`
- 已知方剂缺口：`111` 个 component row
- 唯一缺口 herb：`78`

扩库后：

- herb 总量：`118`
- 新增补入：`78`
- `Comp_i_rulec_source` 分布：
  - `tcmbank_ranked = 71`
  - `proxy_fallback = 12`
  - `tcmid_proxy = 35`

这说明现在单味药层已经不再只是“小样本 PoC”，而是形成了一个能支撑当前方剂样板集的扩展 herb 库。

## 3. 方剂覆盖提升

旧结果：

- `processed_formulas = 10`
- coverage：`covered = 6`, `empty = 4`
- grade：`D = 8`, `C = 2`
- top formula：`丹益片 51.74`

新结果：

- `processed_formulas = 10`
- coverage：`covered = 10`
- grade：`B = 10`
- top formula：`九香止痛丸 67.99`

典型提升：

- `五黄养阴颗粒` 从缺 `黄连 / 红芪 / 姜黄`，变成 `coverage_ratio = 1.0`
- `二十七味定坤丸` 从大量缺口，变成 `coverage_ratio = 1.0`
- `九香止痛丸` 从 `coverage_ratio = 0.4444` 提升到 `1.0`

## 4. 当前剩余风险

扩库后，当前这批 `10` 个方剂样板已经做到：

- `unmatched_components = 0`
- `coverage_ratio` 全部大于 `0.9`
- 方剂分级全部进入 `B`

但还剩一个质量层面的风险，不是 coverage 风险：

- `枫荷桂`
  - 当前是通过 `formula component` 自身字段占位补入
  - 也就是“coverage 已补齐，但本地 herb 源详情仍未补齐”

所以它现在属于：

- 可匹配
- 可进入 Rule-C
- 但证据强度低于 `TCMBank ranked` 直连条目

## 5. 判断

当前 bottleneck 已经变了。

之前卡的是：

- 常见 herb 覆盖不足
- 方剂层只能做原型

现在卡的是：

- 地方性药材与特殊物种条目的证据强度
- 别名链和物种认证链
- `formula_component` 占位条目的后续实证补全

也就是说，下一步不该再优先做“继续大规模抓普通 herb”，而应该转向：

1. 优先补 `枫荷桂` 的真实 herb 源条目
2. 做 `POWO / GBIF / BOLD / PolyU` 的物种与鉴定层
3. 再进入“配伍冲突、矿植耦合、丹方工程化”阶段

## 6. 结论

这轮扩库已经把 herb 层从“Rule-C 可接入”推进到了“能支撑当前方剂样板集”的状态。

接下来真正的难点，不再是数量，而是：

- 名称统一
- 特殊地方药识别
- 物种认证
- 工艺与配伍细化

## 5. 当前关系层状态

这里最容易误解，所以直接给结论。

主库里的关系层已经接通，但还不厚：

- `rows_with_ingredient_links = 23`
- `rows_with_direct_targets = 4`
- `rows_with_direct_diseases = 5`
- `rows_with_ingredient_targets = 7`
- `rows_with_ingredient_diseases = 6`

这不是主库脚本漏接，而是上游详情深度仍有限。当前本地 `TCMBank` 详情抓取深度是：

- `herb_detail_fetched = 300`
- `ingredient_detail_fetched = 300`

所以现在主库的判断应该是：

- 字段层已经并齐得比较完整
- 关系层已经开始长出来
- 但还没有形成大面积覆盖

## 6. 四张子表现在能干什么

### `plant`

- 灵植生态
- 药园与栽培
- 区域资源扩散

### `animal`

- 灵兽副产物
- 伦理监管
- 濒危资源与黑市

### `mineral`

- 与矿物灵材系统联动
- 矿药、矿丹、导体 / 容器 / 反应核联动

### `processed`

- 炮制
- 胶类、油类、粉类、制剂
- 工业提取与标准化生产

也就是说，四张子表已经能直接接玩法，不需要再从混合总表里手动拆。

## 7. 当前最准确的判断

现在的状态不是“信息不够”，而是：

- 主库已经把现有可抓字段基本并齐
- 但产地层和关系层仍明显弱于规则层与分类层

所以下一步要分两条线并行：

1. 结构化继续推进  
   把四张子表接进经济、地图、黑市、监管规则
2. 源数据继续加深  
   继续补 `TCMBank` 详情与关系  
   接 `PolyU / POWO / GBIF / BOLD`

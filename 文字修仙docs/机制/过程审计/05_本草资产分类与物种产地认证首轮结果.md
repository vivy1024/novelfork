# 05_本草资产分类与物种产地认证首轮结果

> 状态日期：2026-03-12  
> 目的：把当前 `1018` 条 Rule-C herb / materia medica 大库拆成可运营的资产层，并补第一轮物种认证与产地信号。
> 文档类型：过程 / 审计文档。快速理解当前系统，请先看 [[02_本草系统精简总览]]、[[03_本草世界化补全规则]]、[[04_本草给玩法层的直接接口]]。

## 1. 这轮做了什么

在 [25_批量下沉herb大库结果](25_批量下沉herb大库结果.md) 的 `1018` 条大库基础上，新增一层 `classified` 产物：

- 把条目拆成 `plant / animal / mineral / processed`
- 给 `processed` 再细分 `processed_plant / processed_animal / processed_mineral`
- 补 `species_cert_level`
- 补 `origin_level / origin_supply_mode`

对应脚本：

- `scripts/classify_rule_c_materia_medica.py`

对应产物：

- `data/processed/tcm_herb_asset_classified.csv`
- `data/processed/tcm_herb_asset_classified.sqlite`
- `data/processed/tcm_herb_asset_classified_summary.json`

## 2. 当前分类结果

来自 `tcm_herb_asset_classified_summary.json` 的最新统计：

- `total_rows = 1018`
- `plant = 848`
- `animal = 44`
- `mineral = 22`
- `processed = 104`

细分类拆开后：

- `processed_plant = 89`
- `processed_animal = 9`
- `processed_mineral = 6`

这说明当前大库已经不再是一张混合本草表，而是能明确区分：

1. 自然植物药材
2. 动物药材
3. 矿物药材
4. 炮制、提取、胶类、粉类、油类等派生材料

## 3. 物种认证结果

当前 `species_cert_level` 分布：

- `barcode_verified = 487`
- `species_latin_supported = 120`
- `materia_medica_latin_only = 296`
- `name_only = 73`
- `not_applicable_mineral = 28`
- `derived_material_only = 14`

可以先把它们理解成三档：

- **高置信物种层**：`487 + 120 = 607`
- **中间层**：`296`
- **低置信待补层**：`73`

也就是说，当前 `1018` 条里，已经有 `607` 条带较强物种信号；但还有相当一部分仍停留在“本草条目级”，没有真正压到标准物种级。

## 4. 产地结果

当前 `origin_level` 分布：

- `province_level = 29`
- `macro_region_level = 8`
- `habitat_only = 9`
- `missing = 972`

`origin_supply_mode` 分布：

- `cultivated = 4`
- `wild = 23`
- `mixed = 11`
- `unknown = 980`

结论很直接：

- 分类层已经跑通
- 物种认证层已经初步成型
- 产地层仍然极弱，是当前主缺口

这不是分类脚本的问题，而是上游条目普遍缺少稳定产地字段。

## 5. 当前可直接使用的字段

`classified` 层新增了这些关键字段：

- `asset_major_class`
- `asset_detail_class`
- `asset_base_class`
- `asset_processing_flag`
- `asset_class_confidence`
- `asset_class_basis_json`
- `species_candidate_name`
- `species_cert_level`
- `species_cert_score`
- `species_barcode_ids_json`
- `origin_level`
- `origin_supply_mode`
- `origin_provinces_json`
- `origin_macro_regions_json`
- `origin_habitat_tags_json`

这意味着后续已经可以直接做：

1. 按类别拆资源表
2. 按认证等级决定市场准入和科研可信度
3. 按产地缺失度决定黑市、不确定性和地图探索空间

## 6. 判断

这一轮最重要的不是简单加总数，而是**把结构切出来了**。

现在可以明确地说：

- `1018` 条库已经是可运营的本草资产库
- 但它还不是高完成度的物种认证库
- 更不是完整的产地资源分布库

所以后面工作的优先级是：

1. 压 `materia_medica_latin_only`
2. 消化 `name_only`
3. 集中补 `origin_level`

## 7. 下一步

最合理的后续顺序是：

1. 把这层结果继续用于主库与四张子表
2. 先补 `name_only` 和 `materia_medica_latin_only` 的物种认证
3. 接 `PolyU / POWO / GBIF / BOLD`，把产地和物种认证做实
4. 再把分类层接到经济、地图资源分布、黑市与监管规则里

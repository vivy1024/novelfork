# 01_灵植灵药扩展数据源清单

> 状态日期：2026-03-12
> 目的：为灵植、灵药、丹方、药材物种认证的数据扩展提供一份可执行的数据源清单，并标明当前机器的可访问性与接入顺序。
> 文档类型：过程 / 审计文档。快速理解当前系统，请先看 [[02_本草系统精简总览]]、[[03_本草世界化补全规则]]、[[04_本草给玩法层的直接接口]]。

## 0. 当前落地状态

截至 `2026-03-12`，灵植数据层已经不再停留在“只有 40 条 herb 的概念验证版”，而是形成了 4 条可复跑的数据流水线：

1. `build_tcm_herb_db.py`
   - 已完成基础 Rule-C herb 判定 PoC
   - 当前基础 herb 库为 `40` 条去重后条目
2. `build_tcmid_db.py`
   - 已完成 `TCMID` 主骨架抓取
   - 当前结果为 `1206` 条 herb 索引、`7443` 条 formula 索引、`118` 条 formula component 明细
3. `build_tcmbank_db.py`
   - 已完成 `TCMBank` herb / ingredient / relation 主索引抓取
   - 当前结果为 `9191` 条 herb、`61965` 条 ingredient，并已有 `herb_ingredients / ingredient_targets / ingredient_diseases` 等关系表
4. `enrich_herb_screening_with_tcmbank.py`
   - 已完成基础 herb 与 `TCMBank` 的第一轮富化
   - 当前 `40` 条基础 herb 中，`30` 条已匹配到 `TCMBank`，其中 `28` 条已拿到真实成分，`17` 条已有 targets，`14` 条已有 diseases

当前真正的瓶颈已经从“找不到数据源”转为：

- `Comp_i` 原始成分池过长、别名重复、噪声较多，不能直接拿来做游戏层字段
- `PolyU` 的炮制/性状细节还没接入
- 植物学物种归一、条形码认证、道地产区还没进入正式流水线

因此，本文档的角色也从“列候选网站”升级为“说明哪些源已经接入、哪些仍待接入”。

## 1. 当前缺口

当前 `data/processed/tcm_herb_screening.sqlite` 只有 40 条去重后药材，且存在以下结构性缺口：

- `Comp_i` 仍以 `effect_inferred` proxy 为主，不是真实成分库
- 缺少 `Mineral_i`，无法把药材和灵矿耦合写成完整链路
- 缺少丹方/方剂表，无法运行配伍和君臣佐使
- 缺少物种学、条形码、道地产区、种植产量等数据层

因此，扩展顺序不能只追求“更多药材条目”，必须同时补：

1. 药材基础条目
2. 真实成分与结构
3. 方剂/配伍
4. 物种认证与分布
5. 采收、炮制、质量与产地

## 2. 一级数据源：优先直接接入

### 2.1 TCMID

- 官网：https://www.bidd.group/TCMID/about.html
- 统计页：https://www.bidd.group/TCMID/stats.html
- 当前可访问性：可访问，`requests`/`curl` 正常
- 当前工程状态：已接入，主骨架抓取完成
- 当前价值：
  - `7,443` 条 TCM prescriptions
  - `2,751` 个 prescription components，覆盖约 `1,400` 常用 herbs
  - `7,375` 个 chemical ingredients
  - 含 treated diseases、targets、geo-authentic habitats、plant DNA barcode 等信息
- 最适合补的字段：
  - `herbs`：中文名、拉丁名、英文名、归经、毒性、道地产区、条形码
  - `formulae`：方名、组成、功能、症状、疾病映射
  - `ingredients`：真实成分名、来源药材、targets
- 风险与限制：
  - 页面是传统网页结构，不像现代 API 那样直接
  - 更适合先做 HTML 抓取器，再做实体清洗

### 2.2 TCMBank

- 官网：https://tcmbank.cn/
- Help：https://www.tcmbank.cn/Help
- Download：https://www.tcmbank.cn/Download
- 当前可访问性：可访问，主页、Help、Download 均正常
- 当前工程状态：已接入，主索引与第一轮关系抓取完成
- 当前价值：
  - `9,191` herbs
  - `61,965` components
  - `15,179` targets
  - `32,529` diseases
  - Help 页明确支持结构检索；BOLD/分子结构相关字段较丰富
- 最适合补的字段：
  - `Comp_i`：从 proxy 升级为真实成分清单
  - `ingredients`：分子式、分子量、logP、极性表面积、Mol2/SDF 之类结构数据
  - `targets` / `diseases`：给丹药和现代医学解释增加桥梁
- 风险与限制：
  - Download 页是前端渲染，批量下载方式要继续探
  - herb 数量大，但不是所有条目都适合直接进“灵植”主库，需要清洗动物药、矿物药、复方等异质条目

### 2.3 PolyU Chinese Herbal Medicine Database

- 官网：https://herbaltcm.sn.polyu.edu.hk/en/
- 当前可访问性：可访问
- 当前工程状态：尚未写入专用抓取脚本
- 当前价值：
  - herb 单体页稳定，适合抓药材性状、显微鉴别、采收炮制、来源植物、图片
  - 当前首页已经能直接看到 herb 详情链接
- 最适合补的字段：
  - `harvest_process`
  - `material_traits`
  - `property_text`
  - `contraindications`
  - 药材鉴别图像和药用部位说明
- 风险与限制：
  - 暂未发现公开 bulk export
  - 更适合做“高质量详情补完源”，不适合当第一主源

## 3. 二级数据源：做物种、分布和认证

### 3.1 GBIF API

- 入口：https://api.gbif.org/v1/
- 示例：https://api.gbif.org/v1/species/search?q=Panax%20ginseng
- 当前可访问性：可访问
- 当前价值：
  - 学名规范化
  - occurrence / distribution / vernacular names
  - 物种层级结构
- 最适合补的字段：
  - `species_name`
  - `accepted_name`
  - `synonyms`
  - `distribution`
  - `occurrence_count`
- 风险与限制：
  - 它是生物多样性库，不懂中医药炮制和药性
  - 必须和 TCM herb 主库联表，不能单独当药材数据库用

### 3.2 POWO

- 官网：https://powo.science.kew.org/
- 当前可访问性：可访问
- 当前价值：
  - Kew 的植物学名、接受名、同物异名、分布
  - 很适合把中药条目的拉丁名清洗成稳定 taxon
- 最适合补的字段：
  - `accepted_latin_name`
  - `synonyms`
  - `taxonomic_status`
  - `distribution_ref`
- 风险与限制：
  - 植物学很强，但不提供中医药药性逻辑

### 3.3 BOLD Portal

- API Docs：https://portal.boldsystems.org/api/docs
- OpenAPI：https://portal.boldsystems.org/openapi.json
- 当前可访问性：可访问
- 当前价值：
  - 条形码查询
  - image / geo / tax / ids 检索
  - 可用于药材真伪鉴定、物种混淆排查
- 最适合补的字段：
  - `barcode_ref`
  - `species_validation`
  - `sample_images`
- 风险与限制：
  - 适合做认证链，不适合做药性链

### 3.4 NCBI GenBank

- 官网：https://www.ncbi.nlm.nih.gov/genbank/
- Datasets Docs：https://www.ncbi.nlm.nih.gov/datasets/docs/v2/how-tos/genomes/get-genome-metadata/
- 当前可访问性：可访问
- 当前价值：
  - 物种级序列与元数据
  - 条形码与基因组的权威补充来源
- 最适合补的字段：
  - `sequence_ref`
  - `barcode_ref`
  - `species_validation`
- 风险与限制：
  - 使用门槛高，字段清洗成本大
  - 优先级低于 BOLD

## 4. 三级数据源：理论上重要，但当前机器不稳定

### 4.1 SymMap

- 下载页：https://www.symmap.org/download/
- 当前可访问性：当前机器直连失败，报 `ConnectionAbortedError(10053)`
- 理论价值：
  - herb / ingredient / symptom / disease / formula 的桥接非常强
  - 很适合做“中医语义 -> 现代疾病/症状网络”的中间层
- 当前结论：
  - 保留为高价值候选源
  - 等网络路径稳定后再接

### 4.2 HERB

- 官网：https://herb.ac.cn/
- 当前可访问性：当前机器连接被远端重置，报 `10054`
- 理论价值：
  - herb-ingredient-target 关系强
  - 很适合补现代药理映射
- 当前结论：
  - 先列入候选，不作为第一批工程源

### 4.3 ETCM

- 官网：https://www.tcmip.cn/ETCM/
- 当前可访问性：当前机器连接被远端重置，报 `10054`
- 理论价值：
  - 成分、药材、方剂的结构化程度高
- 当前结论：
  - 当前机器上不稳定，先不作为主源

## 5. 推荐接入顺序

### Phase 1：先把 herb 基底做厚

1. `TCMID`：已完成主骨架抓取
2. `TCMBank`：已完成主索引抓取与第一轮富化
3. `PolyU`：待补高质量详情

目标：

- herb 主骨架从 `40` 条基础判定扩展到 `1000+` 的可检索条目
- `Comp_i` 从 proxy 升到“真实成分 + proxy 兜底”
- 把动物药、植物药、矿物药拆类

### Phase 2：把物种和产地做实

1. `POWO`
2. `GBIF`
3. `BOLD`
4. `GenBank`

目标：

- 每个 herb 绑定稳定学名
- 有分布和道地产区证据
- 有条形码/序列认证链
- 为后续“灵植种植基地 / 采收窗口 / 地域价格差”提供可结构化字段

### Phase 3：把丹方和现代解释补全

1. `TCMID prescriptions`
2. `TCMBank targets/diseases`
3. `[[02_灵植成分降噪与Rule-C接入方案]]`
4. `SymMap / HERB / ETCM`（网络恢复后）

目标：

- 建 `formulae` 表
- 运行配伍矩阵和方剂评分
- 把丹药与现代靶点/病症桥接

## 6. 对应当前 schema 的补位建议

| 目标字段/表 | 当前状态 | 最优来源 | 备注 |

## 8. 当前结论

现阶段最优路线已经不再是继续找零散中文 CSV，而是：

- 用 `TCMID` 维持 herb + formula 主骨架
- 用 `TCMBank` 补真实成分和 target 网络
- 用 `PolyU` 补药材性状和炮制细节
- 用 `POWO` / `GBIF` / `BOLD` / `GenBank` 把物种认证做实
- 用 [[02_灵植成分降噪与Rule-C接入方案]] 把“可审计原始数据”变成“可驱动丹药规则的游戏层字段”

只要这一步做完，灵植层才会真正从“概念验证版”升级为“可驱动丹方、经济系统和真伪鉴定的生产级基础库”。

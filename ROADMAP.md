# NovelFork 路线图

**更新日期**: 2026-04-20

---

## 当前阶段

NovelFork 当前不再以“继续做 PWA/Node/monorepo 外壳”为目标，
而是进入 **平台回正阶段**：

> 从旧的源码工程形态，迁回 NarraFork 风格的本地单体应用形态。

---

## Phase 0：口径统一（当前）

- [x] 统一 docs 主口径
- [x] 删除误导性发布文档
- [x] 建立平台纠偏说明
- [x] 形成平台迁移方案文档
- [ ] 清理根目录旧文档与旧引用

---

## Phase 1：运行时回正

- [ ] 建立 Bun 主入口 `main.ts`
- [ ] 支持 `bun run main.ts` 直接启动完整应用
- [ ] 降低 `packages/cli -> packages/studio` 的中心地位
- [ ] 梳理 studio server 与 core 的直接耦合关系

---

## Phase 2：构建与分发回正

- [ ] 前端资源内嵌方案
- [ ] `bun compile` 单可执行文件验证
- [ ] 自动打开浏览器
- [ ] 单入口本地服务形态稳定

---

## Phase 3：数据层与基础设施评估

- [ ] 评估 `node:sqlite` 在 Bun 下的兼容性
- [ ] 评估 `bun:sqlite` 适配层
- [ ] 评估 `better-sqlite3 + drizzle` 替换收益
- [ ] 审视 MCP / 工具执行 / 日志与配置体系

---

## Phase 4：产品收口

- [ ] 选择可直接迁移的 studio UI
- [ ] 清理旧外壳残留（spawn、双进程启动脚本、无效 desktop 叙事）
- [ ] 重写最终安装说明
- [ ] 形成真实可用的本地单体产品体验

---

## 暂不优先

在平台回正完成前，以下事项默认不优先：
- 继续堆旧外壳上的 UI 小功能
- 假发布 / 假 release note
- 继续强化 PWA 叙事
- 围绕旧 CLI/旧 workspace 补大量绑定性测试

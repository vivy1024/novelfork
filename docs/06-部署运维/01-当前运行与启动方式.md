## 当前源码运行方式

当前 NovelFork 的**正统运行方式**已经切到 Bun 主入口：

```text
novelfork studio
  → packages/cli/src/commands/studio.ts
  → 优先定位根 main.ts
  → bun run main.ts --root=<project>
  → 启动本地 Studio 服务
  → 最佳努力打开浏览器 http://localhost:4567
```

直接运行也成立：

```text
bun run main.ts --root=. --port=4567
  → 优先使用 embedded assets
  → 启动本地 Studio 服务
```

## 当前主要端口

- 前端默认：`4567`
- studio server：由 `NOVELFORK_STUDIO_PORT` 控制，默认也是 `4567`
- 旧脚本里曾拆成前端 `4567` / 后端 `4569`，但那不是当前最终方案

## 当前主要问题

- 仍有部分历史测试与类型债未清
- 仍有少量 Node 绑定停留在测试/构建兜底层
- 目前仍是 Bun-first 交付形态，尚未进入完整产品化分发体验

## Package 2 当前正式交付边界

当前可以对外统一表述为“**已正式闭环**”的范围，只到以下三项：

- `pnpm bun:compile`
- embedded/filesystem 双路径
- startup recovery 最小链路（含 runtime-state bootstrap、index rebuild、结构化 `recoveryReport`）

以下内容**仍在 Package 2 边界外**，不能表述为已正式交付闭环：

- installer
- signing
- auto-update
- first-launch UX
- 更完整的 repair / migration 终态

## 当前推荐运行方式

当前推荐直接使用：

```text
pnpm bun:compile
  → bun:build-client
  → bun:embed-assets
  → bun build ./main.ts --compile --outfile dist/novelfork

./dist/novelfork.exe --root=. --port=4567
  → 使用 embedded assets 启动本地 Studio
```

开发体验则优先：

```text
bun run main.ts --root=. --port=4567
```

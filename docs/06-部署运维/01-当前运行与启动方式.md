## 当前源码运行方式

当前 NovelFork 仍是过渡期结构：

```text
novelfork studio
  → packages/cli/src/commands/studio.ts
  → 搜索 packages/studio/src/api/index.ts
  → 用 node + tsx loader 或 node dist/api/index.js 启动 studio server
  → 打开浏览器 http://localhost:4567
```

## 当前主要端口

- 前端默认：`4567`
- studio server：由 `NOVELFORK_STUDIO_PORT` 控制，默认也是 `4567`
- 旧脚本里曾拆成前端 `4567` / 后端 `4569`，但那不是当前最终方案

## 当前主要问题

- 依赖 `pnpm workspace`
- 依赖 `tsc -> dist` 或 `tsx loader`
- CLI 通过 spawn 拉起 studio
- 不符合目标平台（Bun 单入口 + bun compile）

## 目标运行方式

目标应迁移为：

```text
bun run main.ts
  → 直接启动本地服务
  → 打开浏览器
  → 最终 bun compile 生成单可执行文件
```

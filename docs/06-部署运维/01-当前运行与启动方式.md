## 当前主口径

当前 NovelFork 的**源码主入口**是仓库根目录 `main.ts`。

但 `novelfork studio` 这条 CLI 路径还没有消失，它现在的职责是：

- **优先把请求转交给 Bun 主入口**
- 在 Bun 主入口不可用时，优先回落到已安装 / 已构建的 Studio 包入口
- 最后才保留 legacy 源码桥作为兼容兜底

所以当前不是“CLI 已废弃”，而是：

> **CLI 负责过渡期拉起；仓库级 Bun 主入口负责正式运行时。**

---

## 1. 当前真实启动链路

### 1.1 `novelfork studio`

```text
novelfork studio
  → packages/cli/src/commands/studio.ts
  → resolveStudioLaunch(root)
  → 优先寻找仓库级 main.ts
  → 命中后执行 bun run <main.ts> --root=<project>
  → 传入 NOVELFORK_STUDIO_PORT
  → 最佳努力打开浏览器 http://localhost:<port>
```

### 1.2 `resolveStudioLaunch()` 的当前优先级

当前代码里的启动优先级是：

1. **仓库级 Bun 主入口**：`main.ts`
2. **Studio 已构建包入口**：`@vivy1024/novelfork-studio/dist/api/index.js`
3. **Studio 源码兼容桥**：`packages/studio/src/api/index.ts`

其中第 2、3 条仍然存在，但都属于**兼容/过渡职责**，其中源码桥只保留最后兜底能力，不再是优先路径。

---

## 2. Bun 主入口当前实际做什么

入口：`main.ts`

当前行为：

- 解析 `--root=` / `--port=` 参数
- 优先加载 `packages/studio/src/api/embedded-assets.generated.ts`
- 若 embedded 资源不可用，则回退到 `packages/studio/dist`
- 若两者都缺失，则只启动 API，不承诺前端页面可访问
- 最终调用 `startStudioServer(projectRoot, port, ...)`

### 2.1 `startStudioServer()` 当前启动期自动动作

入口：`packages/studio/src/api/server.ts`

当前启动时只会自动做两件事：

1. 若 `novelfork.json` 不存在，则补建最小项目目录
2. 启动 API + 静态资源服务

**不会**自动做的事：

- 不会启动时扫描所有书籍
- 不会统一重建章节索引
- 不会统一恢复搜索索引
- 不会自动跑 state repair / write sync

对应清单见：

- `docs/06-部署运维/03-启动期迁移与修复清单.md`

## 3. 当前推荐运行方式

### 3.1 开发 / 源码验证

这是当前最稳定、最符合现状的运行方式：

```bash
bun run main.ts --root=. --port=4567
```

或：

```bash
novelfork studio --port 4567
```

其中第二条命令本质上仍会**优先转交给 Bun 主入口**。

### 3.2 打包链路验证

仓库当前已经接好 compile 脚本链：

```bash
pnpm bun:compile
```

它会执行：

1. `pnpm bun:build-client`
2. `pnpm bun:embed-assets`
3. `bun build ./main.ts --compile --outfile dist/novelfork`

但这里的当前口径必须写清楚：

- 这说明**仓内已经具备 compile 构建脚本**
- **不等于** 已完成安装器、签名、自动更新、跨平台 smoke、正式分发承诺
- 因此现在只能把它当作**本地构建/验证路径**，不能写成“分发能力已完成”

## 4. 当前端口与环境变量

- 默认端口：`4567`
- CLI 拉起时通过 `NOVELFORK_STUDIO_PORT` 传给子进程
- Bun 主入口也支持：
  - `NOVELFORK_PROJECT_ROOT`
  - `NOVELFORK_STUDIO_PORT`

---

## 5. embedded assets 当前口径

当前 embedded assets 的职责是：

- 给 Bun 主入口和 compile 产物提供前端静态资源来源
- 让运行时不必强依赖源码目录下的 `packages/studio/dist`

当前相关文件：

- 生成脚本：`scripts/generate-embedded-assets.mjs`
- 生成产物：`packages/studio/src/api/embedded-assets.generated.ts`
- 运行时选择：`main.ts`
- 静态资源读取：`packages/studio/src/api/static-provider.ts`

注意：

- embedded 资源链路已接线
- 但“正式分发已完成”仍然不是当前代码事实

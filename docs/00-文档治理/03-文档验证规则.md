# 文档验证规则

**版本**: v1.0.0
**创建日期**: 2026-04-28
**更新日期**: 2026-04-28
**状态**: ✅ 当前有效
**文档类型**: current

---

## 验证入口

固定命令：

```bash
bun run docs:verify
```

脚本位置：`../../scripts/verify-docs.ts`。

## 验证项目

1. Markdown 相对链接必须指向存在的文件或目录。
2. 同一目录内编号前缀不得重复，`README.md` 除外。
3. 每个 `docs/` 下目录必须有 `README.md`。
4. 每篇 Markdown 必须包含版本、创建日期、更新日期、状态和文档类型。
5. 文档类型必须是 `current`、`planning`、`reference`、`archived`、`deprecated` 之一。
6. 根 README 当前事实入口不得指向 `reference`、`archived` 或 `deprecated` 文档。
7. 用户侧 current 文档不得把 legacy `Bible` 作为主称呼。
8. 禁止把 `process-memory`、`prompt-preview`、`unsupported`、`mock`、`fake`、`noop` 等过渡状态写成真实完成。
9. README 的文件列表必须覆盖所在目录的直接子文档和子目录。

## 失败输出

验证失败必须输出文件、问题类型和修复建议；脚本只检查，不自动修改文档。

**版本**: v1.0.1

## 本文定位

本文整理 NovelFork Studio 创作工作台当前已经存在的真实接口、透明过渡语义和仍未补齐的边界。它比 [Studio API总览](./01-Studio%20API总览.md) 更聚焦“作者在工作台里会直接碰到什么接口”，但仍然不是逐字段穷举 schema 文档。

接口口径必须反查 `packages/studio/src/api/routes/` 的真实实现；本文只记录当前事实，不把未来 spec 目标提前写成已完成。新前端访问这些能力时必须先经过 `packages/studio/src/app-next/backend-contract/` 的集中 contract client / adapter；组件内不得散写未登记 API 字符串，也不得把未登记能力渲染成可点击按钮。

## 状态术语

| 术语 | 含义 |
|---|---|
| `current` | route 已存在，且返回真实数据、真实持久化结果或真实错误 |
| `process-memory` | 只保存在当前进程内存，重启后不保证存在 |
| `prompt-preview` | 只返回 prompt / promptPreview，不代表已经生成或写入正文 |
| `chunked-buffer` | 已拿到完整 AI 结果后再切块发送，不是上游原生流式 |
| `unsupported` | 当前未接入，必须返回明确错误或透明占位，不得假成功 |

## 2. 候选稿、草稿与 AI Draft 边界

### 候选稿 CRUD 与非破坏性处理

| 方法 | 路径 | 作用 | 当前响应/边界 | 持久化 |
|---|---|---|---|---|
| `GET` | `/api/books/:id/candidates` | 列出候选稿 | 返回 `{ candidates }`，正文来自 `generated-candidates/*.md`；缺正文时该条返回 `content: null` 与 `contentError` | 真实读取 |
| `POST` | `/api/books/:id/candidates` | 新建候选稿 | 请求体至少需要 `title`、`content`、`source`；缺失返回 400 | 真实写入 |
| `POST` | `/api/books/:id/candidates/:candidateId/accept` | 接受候选稿 | `action` 必须是 `merge` / `replace` / `draft`；返回 `{ candidate }` 或 `{ candidate, draft }` | 真实写入 |
| `POST` | `/api/books/:id/candidates/:candidateId/reject` | 拒绝候选稿 | 返回状态更新结果 | 真实写入 |
| `POST` | `/api/books/:id/candidates/:candidateId/archive` | 归档候选稿 | 返回状态更新结果 | 真实写入 |

`accept` 的关键边界：

- `merge` / `replace` 会把候选正文写入正式章节。
- `draft` 会把候选正文写入 `books/<bookId>/drafts/<draftId>.md`，并返回新草稿对象。
- 任何非法 `action` 都返回 400，而不是静默吞掉。

### 当前草稿接口边界

草稿资源 CRUD 已补齐，当前可用接口：

| 方法 | 路径 | 作用 | 当前响应/边界 | 持久化 |
|---|---|---|---|---|
| `GET` | `/api/books/:bookId/drafts` | 列出草稿资源 | 返回 `{ drafts }`，每条包含 `id`、`bookId`、`title`、`content`、`updatedAt`、`wordCount` | 真实读取 |
| `GET` | `/api/books/:bookId/drafts/:draftId` | 读取单个草稿 | 返回 `{ draft }`；不存在返回 404 | 真实读取 |
| `POST` | `/api/books/:bookId/drafts` | 新建草稿 | 请求体支持 `title`、`content`；返回 `{ draft }` | 真实写入 |
| `PUT` | `/api/books/:bookId/drafts/:draftId` | 更新草稿 | 请求体支持 `title`、`content`；返回 `{ draft }` | 真实写入 |
| `DELETE` | `/api/books/:bookId/drafts/:draftId` | 删除草稿 | 成功返回 `{ success: true }`；不存在返回 404 | 真实删除 |

候选稿接受为 `draft` 时会落盘到 `drafts/` 目录，并返回可由上述接口读取的新草稿。

`POST /api/books/:id/draft` 仍是 AI draft 异步动作启动接口，返回 `{ status: "drafting", bookId }`，不等同于草稿资源 CRUD。

## 4. AI 动作接口

### 当前已接入或已有真实 route 的动作

| 方法 | 路径 | 作用 | 当前边界 |
|---|---|---|---|
| `POST` | `/api/books/:id/write-next` | 生成下一章 | 当前返回 `{ status: "writing", bookId }` 启动异步流程，并通过 `/api/events` 广播 `write:*` 事件；不是同步返回候选稿正文 |
| `POST` | `/api/books/:id/draft` | 触发 AI draft | 返回 `{ status: "drafting", bookId }`；是异步动作启动，不等于草稿资源 CRUD |
| `POST` | `/api/books/:id/audit/:chapter` | 连续性/审校 | 成功直接返回审校结果；找不到章节 404 |
| `POST` | `/api/books/:id/revise/:chapter` | 修订草稿 | 成功直接返回 pipeline revise 结果 |
| `POST` | `/api/books/:id/rewrite/:chapter` | 重写章节 | 当前会先 rollback，再启动异步重写流程 |
| `POST` | `/api/books/:id/resync/:chapter` | 重建章节产物 | 返回 resync 结果 |
| `POST` | `/api/books/:id/detect/:chapter` | 单章 AI 痕迹检测 | 返回分析结果 |
| `POST` | `/api/books/:id/detect-all` | 全书 AI 痕迹检测 | 返回 `{ bookId, results }` |
| `POST` | `/api/ai/complete` | 行内补全 SSE | 返回 `chunked-buffer` 语义，不是上游原生流式 |
| `GET` | `/api/events` | Studio 全局 SSE | 广播 `write:*`、`draft:*`、`audit:*` 等事件 |

### 当前透明边界

- `write-next` 与 `draft` 是**异步动作入口**，当前文档不能把它们写成“同步返回正文”。
- `/api/ai/complete` 会在拿到完整结果后再分块发送，SSE payload 含 `streamSource: "chunked-buffer"`。
- AI 失败时返回真实错误或 SSE `error` event，不能伪造成功结果。

## 5. 写作模式接口

当前写作模式已接入 LLM 真实生成。无 session LLM 时优雅降级为 prompt-preview。

| 方法 | 路径 | 结果 | 当前口径 |
|---|---|---|---|
| `POST` | `/api/books/:bookId/inline-write` | `mode: "generated"` + `content` + `promptPreview` + `usage` | continuation / expansion / bridge 真实生成 |
| `POST` | `/api/books/:bookId/dialogue/generate` | `mode: "generated"` + `content` | 对话生成真实生成 |
| `POST` | `/api/books/:bookId/variants/generate` | `mode: "generated"` + `variants[]` | 多版本真实生成 |
| `POST` | `/api/books/:bookId/outline/branch` | `mode: "generated"` + `content` | 大纲分支真实生成 |
| `POST` | `/api/books/:bookId/writing-modes/apply` | `{ target, requestedTarget, resourceId, status, metadata }` | 安全写入 candidate/draft |
| `POST` | `/api/works/import` | 解析导入文本 | 只返回解析结果，不直接落库 |
| `GET` | `/api/style/personal-profile` | 合并个人文风 profile | `profiles` 缺失或 JSON 非法返回 400 |
| `POST` | `/api/books/:bookId/style/drift-check` | 返回 `drift` | 真实分析结果 |

`POST /api/books/:bookId/writing-modes/apply` 已提供安全应用 route：可以写入候选稿或草稿；正式章节 insert/replace 首版会转为候选稿，避免无确认地改动正式正文。Workspace UI 已接入预览、目标选择、确认和写入结果展示；生成接口在有 session LLM 时返回真实 `mode: "generated"` 结果，无 session LLM 或模型不可用时降级为 `prompt-preview`，因此 prompt-preview 不代表真实写入，只代表透明预览。

## 7. Provider、模型池与 AI gate

Provider routes 由 `createProvidersRouter()` 挂载到 `/api/providers`：

| 方法 | 路径 | 作用 | 当前响应/边界 |
|---|---|---|---|
| `GET` | `/api/providers` | 列出 provider | 返回脱敏后的 `{ providers }` |
| `GET` | `/api/providers/status` | 返回运行时 provider 状态 | 基于 runtime store |
| `GET` | `/api/providers/models` | 统一模型池 | 只返回 enabled provider/model/可用账号组合 |
| `POST` | `/api/providers` | 新建 provider | 缺 `id/name/type` 返回 400 |
| `PUT` | `/api/providers/:id` | 更新 provider | 不存在返回 404 |
| `POST` | `/api/providers/:id/toggle` | 启停 provider | 返回脱敏 provider |
| `POST` | `/api/providers/reorder` | 调整顺序 | 非法 id 返回 400 |
| `POST` | `/api/providers/:id/models/refresh` | 刷新模型列表 | adapter unsupported / auth 缺失返回失败 envelope |
| `PATCH` | `/api/providers/:id/models/:modelId` | patch 单模型 | 不存在返回 404 |
| `POST` | `/api/providers/:id/models/:modelId/test` | 测试单模型 | 失败时返回 envelope + model；成功返回 `{ success: true, latency, model }` |
| `POST` | `/api/providers/:id/test` | 测试 provider 当前启用模型 | 无启用模型返回 400 |
| `DELETE` | `/api/providers/:id` | 删除 provider | 成功返回 `{ success: true }` |

关键边界：

- 当前 runtime model pool 已是唯一运行时模型来源。
- `/api/providers` 不回传明文 `apiKey`，通过 `apiKeyConfigured` 表示密钥状态；平台集成账号的 `credentialJson` 脱敏边界属于 `/api/platform-integrations` 视图。
- adapter 未实现时必须返回真实失败 envelope，而不是伪造模型可用。
- provider adapter 失败统一使用 `{ success: false, code, error, capability? }`：`unsupported` 返回 501，`auth-missing` / `config-missing` 返回 422，`upstream-error` / `network-error` 返回 502，未知失败返回 500。

## 8. 平台集成 unsupported 边界

`/api/platform-integrations/:platformId/accounts/import-json` 对未接入的 JSON 导入能力必须返回统一 unsupported failure：`{ success: false, code: "unsupported", capability, error }` + HTTP 501，不能用空账号、空对象或假成功替代。

## 9. 导出与 legacy 边界

| 方法 | 路径 | 作用 | 当前口径 |
|---|---|---|---|
| `GET` | `/api/books/:id/export` | 读取导出内容 | 当前已存在真实 route，属于 legacy 导出读取路径 |
| `POST` | `/api/books/:id/export` | 创建导出内容 | 当前已存在真实 route，但统一导出范围选择仍未按新前端合同收敛 |
| `POST` | `/api/books/:id/export-save` | 保存导出结果 | 当前已存在真实 route |

当前工作台已有导出相关 route；现阶段文档必须写清楚：**导出可作为 current/legacy 能力接入，但统一导出入口、范围选择和新前端呈现还未按 Backend Contract 收敛。**

## 10. Backend Contract 与前端重建边界

`.kiro/specs/backend-contract-v1/` 是新前端重建前的能力合同入口，`.kiro/specs/frontend-refoundation-v1/` 是后续前端实现主线。前端不得继续按旧三栏 UI 的想象直接接 API，而必须先通过合同矩阵确认能力状态。

首版合同矩阵把创作工作台能力分成以下 UI 可依赖组：

| 组 | 前端用途 | 合同入口 |
|---|---|---|
| 启动与侧栏 | 书籍列表、会话列表、provider/model 状态、进度 | `/api/books`、`/api/sessions`、`/api/providers*`、`/api/progress` |
| Agent Shell | 会话快照、WebSocket、ack/replay、工具确认门 | `/api/sessions/:id/chat/state`、`/api/sessions/:id/chat/history`、`WS /api/sessions/:id/chat`、`/api/sessions/:id/tools*` |
| Writing Workbench | 章节、候选稿、草稿、story/truth、经纬、叙事线 | `/api/books/:id*`、`/api/books/:id/candidates*`、`/api/books/:id/drafts*`、`/api/books/:bookId/jingwei/*`、`/api/books/:bookId/narrative-line*` |
| AI 写作动作 | session-native 写下一章、writing modes、hooks、审校/检测 | `cockpit.*`、`pgi.*`、`guided.*`、`candidate.create_chapter`、`/api/books/:id/write-next`、`/api/books/:bookId/writing-modes/*` |
| Provider 与模型 | provider 管理、模型刷新、模型测试、工具支持降级 | `/api/providers*` |

前端规则：

- 新前端只能通过 `createContractClient`、`fetch-json-contract-client` 及 `session/resource/provider/writing action` 领域 client、`resource-tree-adapter`、`writing-action-adapter`、session WebSocket helper 访问登记能力；若需要新增 route，先补 Backend Contract 矩阵、共享类型和 contract 测试，再接 UI。
- 组件内不得散写 fetch 字符串调用未登记 API；不得用 mock/fake/noop 数据填充 contract client 失败结果。`backend-contract-verification.ts` 的 app-next API guard 是当前收口检查来源。
- `current` 能力可启用，但仍要展示真实错误。
- `process-memory` 必须标注临时状态。
- `prompt-preview` 只允许预览/复制/显式 apply。
- `chunked-buffer` 可显示流式体验，但不能写成上游原生流式。
- `unsupported` 必须禁用或显示失败说明。
- `null` 指标或 `unknown` metric 必须显示未知/待接入，不得伪装为绿色成功。

## 11. 当前缺口与禁止写法

### 当前缺口

1. writing modes 安全 apply route 已落地；生成接口在有 session LLM 时返回真实生成结果，无 session LLM 或模型不可用时降级为 prompt-preview，prompt-preview 只可查看/复制。
2. `write-next` / `draft` 仍是异步启动接口，不是同步正文接口。
3. 工作台主入口已转为右侧叙述者会话；历史 Workspace AI action route map 仍可复用，但驾驶舱 / 经纬 / 写作能力应优先作为 session tools、工具结果卡片或中间画布组件呈现。
4. 导出仍是当前 route 与后续 spec 设计并存阶段。
5. 删除类 destructive 能力当前保持硬删除语义：book/chapter/story file/truth file/candidate/draft 由 service 执行真实文件或索引删除，route adapter 保留前端确认入口、旧响应 envelope 与不存在/非法文件名失败状态。

### 禁止写法

- 不得把 `prompt-preview` 写成“已生成正文”。
- 不得把 `process-memory` 描述成“重启后仍保留”的长期状态。
- 不得把 `unsupported` 写成“当前可用”。
- 不得把模型不支持工具调用时的 `unsupported-tools` 写成工具已执行；正确降级是只读解释、prompt-preview 或引导用户切换支持 tools 的模型。
- 不得把 OpenAI-compatible 上游 function name 约束忽略掉；session tool 内部名可包含点号，但发给上游的工具名必须经过 provider-safe 映射，收到 tool call 后再还原为内部工具名。
- 不得把异步 `POST /api/books/:id/draft` 写成草稿资源 CRUD；草稿资源 CRUD 以 `/api/books/:id/drafts*` 为准。

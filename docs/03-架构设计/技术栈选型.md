## 概述

NovelFork 是一个 AI 辅助的中文网文创作工作台，从 InkOS v1.1.1 fork 而来。技术栈选型遵循以下原则：

### 核心原则

1. **渐进式 Web 应用（PWA）优先**：优先选择 Web 技术，避免 Electron 的资源开销
2. **类型安全**：全栈 TypeScript，编译期捕获错误
3. **开发体验**：快速热重载、完善的工具链、丰富的生态
4. **性能优先**：客户端持久化、离线能力、流式响应
5. **可维护性**：清晰的架构分层、模块化设计、测试覆盖

### 技术栈概览

### Zustand 5

**选型理由**：

1. **极简 API**：无 Provider、无 Context、无样板代码
2. **性能优秀**：细粒度订阅，避免不必要的重渲染
3. **DevTools 支持**：Redux DevTools 集成
4. **TypeScript 友好**：类型推导完善
5. **中间件丰富**：persist、immer、devtools

**状态管理实践**：

```typescript
// stores/windowStore.ts
import { create } from 'zustand';
import { devtools, persist } from 'zustand/middleware';
import { immer } from 'zustand/middleware/immer';

interface ChatWindow {
  id: string;
  bookId: string;
  title: string;
  position: { x: number; y: number };
  size: { width: number; height: number };
  messages: Message[];
}

interface WindowStore {
  windows: ChatWindow[];
  activeWindowId: string | null;
  
  addWindow: (window: ChatWindow) => void;
  removeWindow: (id: string) => void;
  updateWindow: (id: string, updates: Partial<ChatWindow>) => void;
  setActiveWindow: (id: string) => void;
}

export const useWindowStore = create<WindowStore>()(
  devtools(
    persist(
      immer((set) => ({
        windows: [],
        activeWindowId: null,
        
        addWindow: (window) => set((state) => {
          state.windows.push(window);
        }),
        
        removeWindow: (id) => set((state) => {
          state.windows = state.windows.filter(w => w.id !== id);
        }),
        
        updateWindow: (id, updates) => set((state) => {
          const window = state.windows.find(w => w.id === id);
          if (window) Object.assign(window, updates);
        }),
        
        setActiveWindow: (id) => set({ activeWindowId: id }),
      })),
      { name: 'window-store' }
    )
  )
);
```

- Redux：样板代码过多，学习曲线陡峭
- MobX：响应式系统复杂，调试困难
- Jotai：原子化状态适合小型应用，大型应用管理复杂

### IndexedDB (idb 8.0.3)

**选型理由**：

1. **大容量存储**：支持 GB 级数据（LocalStorage 仅 5-10MB）
2. **结构化数据**：支持索引、事务、游标查询
3. **异步 API**：不阻塞主线程
4. **离线能力**：PWA 核心存储方案
5. **idb 库封装**：Promise 化 API，易用性强

**存储架构**：

```typescript
// storage/db.ts
import { openDB, DBSchema, IDBPDatabase } from 'idb';

interface NovelForkDB extends DBSchema {
  books: {
    key: string;
    value: BookMetadata;
    indexes: { 'by-genre': string; 'by-status': string };
  };
  chapters: {
    key: string;
    value: ChapterData;
    indexes: { 'by-book': string; 'by-status': string };
  };
  drafts: {
    key: string;
    value: DraftData;
    indexes: { 'by-chapter': string };
  };
  cache: {
    key: string;
    value: { data: unknown; timestamp: number };
  };
}

let db: IDBPDatabase<NovelForkDB>;

export async function initDB() {
  db = await openDB<NovelForkDB>('novelfork', 1, {
    upgrade(db) {
      // Books store
      const bookStore = db.createObjectStore('books', { keyPath: 'id' });
      bookStore.createIndex('by-genre', 'genre');
      bookStore.createIndex('by-status', 'status');
      
      // Chapters store
      const chapterStore = db.createObjectStore('chapters', { keyPath: 'id' });
      chapterStore.createIndex('by-book', 'bookId');
      chapterStore.createIndex('by-status', 'status');
      
      // Drafts store
      const draftStore = db.createObjectStore('drafts', { keyPath: 'id' });
      draftStore.createIndex('by-chapter', 'chapterId');
      
      // Cache store
      db.createObjectStore('cache');
    },
  });
  return db;
}

// CRUD operations
export async function saveBook(book: BookMetadata) {
  return db.put('books', book);
}

export async function getBooksByGenre(genre: string) {
  return db.getAllFromIndex('books', 'by-genre', genre);
}

export async function getChaptersByBook(bookId: string) {
  return db.getAllFromIndex('chapters', 'by-book', bookId);
}
```

**缓存策略**：

```typescript
// 带过期时间的缓存
export async function getCached<T>(key: string, maxAge: number): Promise<T | null> {
  const cached = await db.get('cache', key);
  if (!cached) return null;
  
  const age = Date.now() - cached.timestamp;
  if (age > maxAge) {
    await db.delete('cache', key);
    return null;
  }
  
  return cached.data as T;
}

export async function setCache(key: string, data: unknown) {
  await db.put('cache', { data, timestamp: Date.now() }, key);
}
```

**为什么不选 LocalStorage/SessionStorage**：

- LocalStorage：容量限制 5-10MB，同步 API 阻塞主线程
- SessionStorage：标签页关闭即清空，不适合持久化

export function InstallPrompt() {
  const [deferredPrompt, setDeferredPrompt] = useState<any>(null);

      setShowPrompt(false);

    setDeferredPrompt(null);

      <p>安装 NovelFork 到桌面，获得更好的体验</p>
      <button onClick={handleInstall}>安装</button>
      <button onClick={() => setShowPrompt(false)}>稍后</button>
    </div>
  );
}

### WebSocket 实时通信

**选型理由**：

1. **双向通信**：服务器主动推送，无需轮询
2. **低延迟**：毫秒级响应，适合实时协作
3. **连接复用**：一个连接多个消息，减少开销

**实现方案**：

```typescript
// api/routes/chat.ts
import { Hono } from 'hono';
import { upgradeWebSocket } from 'hono/ws';

const chatRouter = new Hono();

chatRouter.get(
  '/api/chat/:bookId/stream',
  upgradeWebSocket((c) => {
    const bookId = c.req.param('bookId');
    
    return {
      onOpen: (event, ws) => {
        console.log(`WebSocket opened for book ${bookId}`);
      },
      
      onMessage: async (event, ws) => {
        const message = JSON.parse(event.data);
        
        // 流式生成回复
        for await (const chunk of generateReply(message)) {
          ws.send(JSON.stringify({
            type: 'chunk',
            data: chunk,
          }));
        }
        
        ws.send(JSON.stringify({ type: 'done' }));
      },
      
      onClose: (event, ws) => {
        console.log(`WebSocket closed for book ${bookId}`);
      },
      
      onError: (event, ws) => {
        console.error('WebSocket error:', event);
      },
    };
  })
);
```

**客户端连接管理**：

```typescript
// hooks/useWebSocket.ts
import { useEffect, useRef, useState } from 'react';

export function useWebSocket(url: string) {
  const ws = useRef<WebSocket | null>(null);
  const [isConnected, setIsConnected] = useState(false);
  const [messages, setMessages] = useState<any[]>([]);

  useEffect(() => {
    ws.current = new WebSocket(url);
    
    ws.current.onopen = () => setIsConnected(true);
    ws.current.onclose = () => setIsConnected(false);
    
    ws.current.onmessage = (event) => {
      const message = JSON.parse(event.data);
      setMessages((prev) => [...prev, message]);
    };
    
    return () => {
      ws.current?.close();
    };
  }, [url]);

  const send = (data: any) => {
    if (ws.current?.readyState === WebSocket.OPEN) {
      ws.current.send(JSON.stringify(data));
    }
  };

  return { isConnected, messages, send };
}
```

### Git Worktree 多线并行

**选型理由**：

1. **真正的并行**：多个分支同时工作，互不干扰
2. **零切换成本**：无需 stash/checkout，直接切换目录
3. **安全隔离**：每个 worktree 独立工作区，避免冲突
4. **原生支持**：Git 内置功能，无需额外工具

**实现方案**：

```typescript
// api/lib/git-utils.ts
import { exec } from 'node:child_process';
import { promisify } from 'node:util';

const execAsync = promisify(exec);

const GIT_NO_PROMPT_ENV = {
  GIT_TERMINAL_PROMPT: '0',
  GIT_ASKPASS: '',
};

export async function createWorktree(
  repoPath: string,
  slug: string,
  branch: string
): Promise<string> {
  // 严格验证 slug，防止路径遍历
  const VALID_SLUG = /^[a-zA-Z0-9._-]+$/;
  if (!VALID_SLUG.test(slug) || slug.length > 64) {
    throw new Error(`Invalid worktree slug: ${slug}`);
  }
  
  const worktreePath = join(repoPath, '.worktrees', slug);
  
  // 快速恢复路径（避免重复 git 调用）
  const existingHead = await readWorktreeHeadSha(worktreePath);
  if (existingHead) {
    return worktreePath;
  }
  
  // 创建 worktree
  await execAsync(
    `git worktree add "${worktreePath}" -b "${branch}"`,
    { cwd: repoPath, env: { ...process.env, ...GIT_NO_PROMPT_ENV } }
  );
  
  return worktreePath;
}

export async function removeWorktree(
  repoPath: string,
  slug: string
): Promise<void> {
  const worktreePath = join(repoPath, '.worktrees', slug);
  
  await execAsync(
    `git worktree remove "${worktreePath}" --force`,
    { cwd: repoPath, env: { ...process.env, ...GIT_NO_PROMPT_ENV } }
  );
}

export async function listWorktrees(repoPath: string): Promise<Worktree[]> {
  const { stdout } = await execAsync('git worktree list --porcelain', {
    cwd: repoPath,
  });
  
  return parseWorktreeList(stdout);
}
```

// components/WorktreeManager.tsx
import { useState, useEffect } from 'react';

export function WorktreeManager({ bookId }: { bookId: string }) {
  const [worktrees, setWorktrees] = useState<Worktree[]>([]);

  useEffect(() => {
    fetchWorktrees();
  }, [bookId]);

  const fetchWorktrees = async () => {
    const res = await fetch(`/api/books/${bookId}/worktrees`);
    const data = await res.json();
    setWorktrees(data.worktrees);
  };

  const createWorktree = async (slug: string) => {
    await fetch(`/api/books/${bookId}/worktrees`, {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({ slug, branch: `feature/${slug}` }),
    });
    fetchWorktrees();
  };

  return (
    <div>
      <h2>Worktree 管理</h2>
      <ul>
        {worktrees.map((wt) => (
          <li key={wt.path}>
            {wt.branch} - {wt.path}
          </li>
        ))}
      </ul>
      <button onClick={() => createWorktree('new-feature')}>
        创建新 Worktree
      </button>
    </div>
  );
}

## 技术对比与权衡

### PWA vs Electron

| 维度 | PWA | Electron | 选择 |
|------|-----|----------|------|
| 包体积 | 2MB | 150MB | ✅ PWA |
| 内存占用 | 100MB | 300-500MB | ✅ PWA |
| 启动速度 | <1s | 2-5s | ✅ PWA |
| 离线能力 | Service Worker | 原生支持 | ⚖️ 平手 |
| 系统集成 | 受限 | 完整 | ❌ Electron |
| 更新机制 | 自动 | 手动下载 | ✅ PWA |
| 开发体验 | 热重载快 | 打包慢 | ✅ PWA |

**结论**：PWA 优先，未来可选 Tauri（Rust + WebView，体积仅 3MB）

---

### Hono vs Express vs Fastify

| 维度 | Hono | Express | Fastify | 选择 |
|------|------|---------|---------|------|
| 性能 | 4x | 1x | 2x | ✅ Hono |
| 类型安全 | 端到端 | 无 | 部分 | ✅ Hono |
| 包体积 | 12KB | 200KB | 150KB | ✅ Hono |
| 边缘支持 | ✅ | ❌ | ❌ | ✅ Hono |
| 生态成熟度 | 新兴 | 成熟 | 成熟 | ⚖️ 可接受 |

**结论**：Hono 性能和类型安全优势明显，生态正在快速成长

---

### TipTap vs Slate vs Quill

| 维度 | TipTap | Slate | Quill | 选择 |
|------|--------|-------|-------|------|
| 扩展性 | 插件化 | 高度定制 | 受限 | ✅ TipTap |
| API 稳定性 | 稳定 | 频繁变动 | 稳定 | ✅ TipTap |
| 协作编辑 | Y.js 集成 | 需自行实现 | 不支持 | ✅ TipTap |
| Markdown | 双向转换 | 需自行实现 | 不支持 | ✅ TipTap |
| 学习曲线 | 中等 | 陡峭 | 平缓 | ✅ TipTap |

**结论**：TipTap 平衡了扩展性和易用性，适合小说创作场景

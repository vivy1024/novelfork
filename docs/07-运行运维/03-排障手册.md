## 本文定位

本文提供 NovelFork 当前运行和文档维护的排障顺序。它不是所有异常的完整知识库，而是帮助维护者先按事实源定位问题，避免把透明过渡能力误判成 bug 或把未接入能力写成已完成。

## 总排障顺序

1. 看当前运行方式：[01-当前运行与启动方式.md](./01-当前运行与启动方式.md)。
2. 看当前测试状态：[../08-测试与质量/01-当前测试状态.md](../08-测试与质量/01-当前测试状态.md)。
3. 涉及 mock、临时状态或未接入能力时，对照 [Studio能力矩阵](../01-当前状态/02-Studio能力矩阵.md)。
4. 涉及接口时，对照 [Studio API总览](../06-API与数据契约/01-Studio%20API总览.md)。
5. 涉及文档体系时运行 `bun run docs:verify`。

## 启动失败

| 现象 | 优先检查 | 处理方向 |
|---|---|---|
| 命令不存在 | Bun / pnpm / Node 是否安装 | 补齐运行时工具 |
| 端口无法监听 | `4567` 是否被占用，是否设置了 `NOVELFORK_STUDIO_PORT` | 换端口或关闭占用进程 |
| 项目配置缺失 | `novelfork.json` 是否存在 | 查看启动日志是否触发自动初始化 |
| SQLite 初始化失败 | session store 目录是否可写 | 检查权限和路径 |
| 页面打不开但服务启动 | static provider 是否缺失 | 执行前端构建或 embedded assets 生成 |

## 页面或工作台异常

| 现象 | 判断方式 | 注意事项 |
|---|---|---|
| 页面空白 | 查看浏览器控制台和 server 日志 | 不要直接归因于 API |
| 资源树为空 | 检查 `/books` 和 `/books/:id` 数据 | 没有作品时应显示空态 |
| 章节打开失败 | 检查章节 API 和 book id | 非法 book id 应返回结构化错误 |
| 保存失败 | 看错误 message | 保存失败时正文不应丢失 |
| 候选稿动作异常 | 检查候选稿 API 和状态 | 候选稿不应自动覆盖正文 |

## AI 和模型问题

| 现象 | 优先检查 |
|---|---|
| 模型列表为空 | 设置页 provider runtime store 是否有启用供应商 |
| AI gate 出现 | 凭据是否缺失、模型是否不可用 |
| AI 没有生成正文 | 确认该功能是不是 prompt preview 或未接入 |
| 供应商请求失败 | 看错误 envelope、HTTP 状态和 provider 错误 |
| 平台账号导入失败 | 区分 Codex/Kiro 已接路径和其他透明过渡路径 |

原则：凭据缺失或模型不可用时，系统应该返回真实错误或 AI gate，不生成假正文。

## 文档验证失败

运行：

```bash
bun run docs:verify
```

常见失败类型：

| 类型 | 含义 | 修复方式 |
|---|---|---|
| `broken-link` | Markdown 链接指向不存在文件或目录 | 修正相对路径或补 README |
| `duplicate-number` | 同一目录有重复编号 | 重命名其中一个文件 |
| `missing-readme` | 目录缺 README | 新增或补全 README |
| `missing-header` | 文档缺标准 header | 补版本、日期、状态、文档类型 |
| `invalid-current-entry` | 根 README 当前入口指向 reference/archived | 移出当前入口 |
| `forbidden-user-term` | 用户侧 current 文档用了 legacy 主称呼 | 改成故事经纬 / 经纬 |
| `forbidden-claim` | 把临时或未接入能力写成完成 | 改成透明过渡或 planning |

## 透明边界速查

| 术语 | 正确含义 |
|---|---|
| `process-memory` | 当前进程临时状态，不保证重启后保留 |
| `unsupported` | 当前未接入，不是失败但也不是完成 |
| `prompt-preview` | 只预览提示词，不代表生成或写入 |
| `chunked-buffer` | 已拿到完整结果后切片发送，不是上游原生流式 |
| `unknown` | 指标缺事实源，不应填固定满分 |

## 什么时候不要修代码

以下情况优先改文档或标状态，不要急着改代码：

- 文档把归档内容写成当前事实。
- 文档把 reference 写成当前路线。
- 文档把 planning 能力写成已完成。
- 用户侧文档仍使用 legacy 主称呼。
- README 指向已迁移旧路径。

## 什么时候需要看代码

以下情况必须看代码或测试：

- API 路径文档和实际 route 不一致。
- UI 显示成功但没有真实写入证据。
- 保存、候选稿处理、provider 配置等 current 路径失败。
- docs:verify 不能判断业务真假。

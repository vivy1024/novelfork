## 本文定位

本文记录当前运行运维层面的配置入口。它不替代设置页的模型供应商配置，也不把旧 CLI 配置文档恢复为当前入口。

## 配置分层

| 层级 | 例子 | 用途 |
|---|---|---|
| 进程启动参数 | `--root`、`--port` | 控制当前 Studio 进程 |
| 环境变量 | `NOVELFORK_STUDIO_PORT`、`NOVELFORK_MODE` | 控制运行模式、端口、存储目录等 |
| 项目配置 | `novelfork.json` | 当前项目根目录的基本配置 |
| Runtime store | provider runtime store、平台账号 | 供应商、模型和账号运行时配置 |
| 用户输入 | 设置页表单、工作台动作 | 当前会话中的操作状态 |

文档中必须写清配置属于哪一层，避免把运行参数、项目配置和模型配置混成一个概念。

## 常用启动参数

| 参数 | 说明 | 示例 |
|---|---|---|
| `--root` | 指定项目根目录 | `--root=.` |
| `--port` | 指定 Studio 端口 | `--port=4567` |

开发运行示例：

```bash
bun run main.ts --root=. --port=4567
```

编译运行示例：

```bash
./dist/novelfork.exe --root=. --port=4567
```

## 当前常见环境变量

| 环境变量 | 用途 | 当前口径 |
|---|---|---|
| `NOVELFORK_STUDIO_PORT` | 控制 Studio 服务端口 | 默认 `4567` |
| `NOVELFORK_MODE` | 控制运行模式 | 默认 `standalone`，可设 `relay` |
| `NOVELFORK_WORKBENCH_TOKEN` | 控制 workbench 文件操作 token | 有需要时设置 |
| `NOVELFORK_SESSION_STORE_DIR` | 指定 session store / SQLite 诊断目录 | 默认使用 runtime storage dir |
| `NOVELFORK_DEFAULT_LANGUAGE` | 自动初始化项目时的默认语言 | 默认 `zh` |
| `NOVELFORK_LLM_PROVIDER` | 自动初始化默认 LLM provider | 可选，不能替代设置页完整配置 |
| `NOVELFORK_LLM_BASE_URL` | 自动初始化默认 LLM base URL | 可选 |
| `NOVELFORK_LLM_MODEL` | 自动初始化默认模型 | 默认可落到 `gpt-4o` |

## 模型配置当前口径

模型供应商配置以 Studio 设置页和 runtime provider store 为当前主线：

```text
设置页
  -> Provider runtime store
  -> runtime model pool
  -> AI gate / WritingTools / ConversationSurface
```

旧环境变量或旧 CLI 示例只能作为过渡参考，不能作为唯一 current 入口。

## 安全规则

- API Key、Token、密码不得写入仓库。
- 文档示例只能使用脱敏占位值。
- 排障日志不得输出完整密钥。
- 分享截图时应隐藏供应商凭据。
- `.env`、账号 JSON、真实 token 不应提交。

## 配置排障顺序

1. 先确认启动参数是否正确，例如 `--root` 和 `--port`。
2. 再确认环境变量是否覆盖了默认值。
3. 再检查项目根目录是否存在 `novelfork.json`。
4. 模型问题再进入设置页检查 provider runtime store。
5. 如果是 AI 调用失败，看错误 envelope，不要假设是网络或模型本身。

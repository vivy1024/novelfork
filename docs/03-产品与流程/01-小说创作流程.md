# 小说创作流程

**版本**: v1.0.0  
**创建日期**: 2026-05-10  
**状态**: ✅ 当前有效  
**文档类型**: current

---

## 概述

NovelFork Studio 的创作流程围绕"作品 → 资源 → 写作 → 审校 → 导出"展开。作者可以纯手动写作，也可以借助 AI Agent 生成候选稿后人工审核合入。AI 产出永远不会直接覆盖正式章节，必须经过候选稿审核或确认门。

---

## 1. 创建作品

| 项目 | 说明 |
|------|------|
| API | `POST /api/books/create` |
| 参数 | title（必填）、genre（题材）、targetChapters（目标章数）、chapterWordCount（章节字数）、language |
| 返回 | `{ bookId }` |
| 状态 | ✅ 真实可用 |

创建作品后，系统初始化 worktree 目录结构（chapters/、generated-candidates/、drafts/、story/、truth/）。如果选择了经纬模板，会自动应用模板创建分区。

创建过程为异步，可通过 `GET /api/books/:bookId/create-status` 轮询状态（creating → ready / error）。

---

## 2. 浏览资源

资源树通过 `loadResourceTreeFromContract` 并行加载，展示为左侧资源栏。

| 分组 | 节点类型 | 说明 |
|------|---------|------|
| 章节 | chapter | 正式章节列表 |
| 候选稿 | candidate | AI 生成的待审核稿件 |
| 草稿 | draft | 手动或从候选稿转入的草稿 |
| 大纲与设定 | story | 大纲、设定等 story 文件 |
| 真相文件 | truth | 作为 AI 上下文的真相文件 |
| 经纬资料 | jingwei-section / jingwei-entry | 故事经纬分区与条目 |
| 叙事线 | narrative-line | 叙事线快照 |

状态：✅ 真实可用（7 个分组全部实现）

---

## 3. 手动写作

| 步骤 | 操作 | API | 状态 |
|------|------|-----|------|
| 创建章节 | 在章节分组下新建 | 通过 worktree 文件系统创建 | ✅ 真实可用 |
| 编辑正文 | 在画布中打开章节 viewer，编辑 textarea | — | ✅ 真实可用 |
| 保存 | 触发 `saveResourceAndHydrate` | `PUT /api/books/:bookId/chapters/:number` | ✅ 真实可用 |
| 批准 | 章节状态从 draft 变为 approved | 通过章节 status 字段管理 | ✅ 真实可用 |

保存流程会先校验资源可写性（`assertSaveable`），保存后自动回读详情（hydrate），确保画布显示最新内容。

---

## 4. AI 辅助写作

| 步骤 | 说明 | 状态 |
|------|------|------|
| 选择写作动作 | 从写作动作按钮列表选择（写下一章、AI draft、writing modes 等） | ✅ 真实可用 |
| 创建/复用会话 | Session-native 动作使用 Agent 会话；异步动作直接调用 API | ✅ 真实可用 |
| Agent 交互 | Session-native 链路：cockpit.get_snapshot → pgi.generate_questions → guided.enter/exit → candidate.create_chapter | ✅ 真实可用 |
| 候选稿产出 | AI 结果进入候选稿容器，不直接写入正式章节 | ✅ 真实可用 |

当前可用的写作动作：

| 动作 ID | 标签 | 输出边界 |
|---------|------|---------|
| session-native.write-next | Session-native 写下一章 | candidate-artifact |
| ai.write-next.async | 异步写下一章 | async-start |
| ai.draft.async | AI draft 异步动作 | async-start |
| writing-modes.preview | Writing modes 预览 | prompt-preview |
| writing-modes.apply | Writing modes 安全应用 | candidate-artifact |
| hooks.generate | 生成伏笔建议 | gate |
| hooks.apply | 应用伏笔到 pending_hooks.md | draft-artifact |
| ai.audit | 连续性审校 | analysis |
| ai.detect | AI 味检测 | analysis |

## 5. 处理候选稿

候选稿生命周期：`candidate` → `accepted` / `rejected` / `archived`

| 操作 | 说明 | API | 状态 |
|------|------|-----|------|
| 查看 | 在画布中打开候选稿 viewer，只读展示正文 | — | ✅ 真实可用 |
| 接受 — replace | 用候选稿内容替换目标章节 | `POST /api/books/:id/candidates/:cid/accept` body: `{ action: "replace" }` | ✅ 真实可用 |
| 接受 — merge | 将候选稿追加到目标章节末尾 | 同上，body: `{ action: "merge" }` | ✅ 真实可用 |
| 接受 — draft | 将候选稿另存为草稿 | 同上，body: `{ action: "draft" }` | ✅ 真实可用 |
| 拒绝 | 标记为 rejected | `POST /api/books/:id/candidates/:cid/reject` | ✅ 真实可用 |
| 归档 | 标记为 archived | `POST /api/books/:id/candidates/:cid/archive` | ✅ 真实可用 |
| 删除 | 物理删除候选稿文件 | `DELETE /api/books/:id/candidates/:cid` | ✅ 真实可用 |

前端通过 `CandidateActionsBar` 组件提供操作入口，包含状态 Badge、接受下拉菜单、拒绝/归档/删除按钮。

## 6. 维护经纬

| 操作 | 说明 | API | 状态 |
|------|------|-----|------|
| 查看分区列表 | 列出当前书籍的经纬分区 | `GET /api/books/:bookId/jingwei/sections` | ✅ 真实可用 |
| 创建分区 | 指定 key、name、description 等 | `POST /api/books/:bookId/jingwei/sections` | ✅ 真实可用 |
| 更新分区 | 修改分区属性 | `PUT /api/books/:bookId/jingwei/sections/:sectionId` | ✅ 真实可用 |
| 删除分区 | 软删除 | `DELETE /api/books/:bookId/jingwei/sections/:sectionId` | ✅ 真实可用 |
| 查看条目 | 按分区或全书列出条目 | `GET /api/books/:bookId/jingwei/entries?sectionId=` | ✅ 真实可用 |
| 创建条目 | 指定 title、contentMd、tags 等 | `POST /api/books/:bookId/jingwei/entries` | ✅ 真实可用 |
| 编辑条目 | 在画布中编辑并保存 | `PUT /api/books/:bookId/jingwei/entries/:entryId` | ✅ 真实可用 |
| 删除条目 | 软删除 | `DELETE /api/books/:bookId/jingwei/entries/:entryId` | ✅ 真实可用 |
| 应用模板 | blank/basic/enhanced/genre-recommended | `POST /api/books/:bookId/jingwei/templates/apply` | ✅ 真实可用 |
| 上下文预览 | 预览 AI 生成时会注入哪些经纬条目 | `POST /api/books/:bookId/jingwei/preview-context` | ✅ 真实可用 |

经纬条目中 `participatesInAi=true` 的条目会被注入 AI 写作上下文。

## 7. 审校与检查

| 功能 | 说明 | API | 状态 |
|------|------|-----|------|
| 连续性审计 | 检测剧情矛盾、人物设定冲突 | `POST /api/books/:id/audit/:chapter` | ✅ 真实可用 |
| AI 味检测 | 检测 AI 生成痕迹 | `POST /api/books/:id/detect/:chapter` | ✅ 真实可用 |
| Writing modes 预览 | 选区变换预览（不直接修改） | `POST /api/books/:bookId/inline-write` | ✅ 真实可用 |
| Writing modes 应用 | 将变换结果作为候选稿产出 | `POST /api/books/:bookId/writing-modes/apply` | ✅ 真实可用 |
| 文风漂移检测 | 基于真相文件的风格一致性检查 | — | planned |

审校结果通过 `analysis` 输出边界返回，在前端以分析面板展示，不修改正式章节。

## 8. 导出

| 格式 | 说明 | 状态 |
|------|------|------|

## 流程图

```
创建作品 → 浏览资源树 → 手动写作 / AI 辅助写作
                                    ↓
                              候选稿产出
                                    ↓
                        查看 → 接受/拒绝/归档/删除
                                    ↓ (接受)
                        merge / replace / draft
                                    ↓
                              审校与检查
                                    ↓
                                导出
```

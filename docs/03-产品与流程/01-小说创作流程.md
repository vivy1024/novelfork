# 小说创作流程

**版本**: v1.0.4
**创建日期**: 2026-04-28
**更新日期**: 2026-05-01
**状态**: ✅ 当前有效

## 本文定位

本文从作者视角描述 NovelFork Studio 当前推荐的小说创作流程，并明确每一步现在处于哪种状态。它不是逐端点 API 文档，也不是把规划中能力提前声明为完成。

NovelFork 的创作流程围绕“正式正文”和“可确认的中间产物”展开：作者可以手写、编辑、调用 AI、维护经纬资料和运行检查，但 AI 结果必须先成为预览、候选稿或草稿，不能直接覆盖正式章节。

## 状态口径

| 状态 | 含义 |
|---|---|
| 真实可用 | 有真实 API、持久化或真实 runtime 调用，失败时返回真实错误。 |
| 透明过渡 | 功能仍是临时态、`process-memory`、`prompt-preview`、`chunked-buffer` 或 `unsupported`，但 UI/API 明确标注，不冒充正式完成。 |
| 内部示例 | 只用于开发、测试或 demo，不应作为生产入口。 |
| 待迁移 | 存在旧入口或历史能力，当前不应继续扩展，需迁移到新工作台真实边界。 |
| 规划中 | spec 已定义目标，但当前还不是可用闭环。 |

## 主流程总览

```text
创建或打开作品
  -> 在资源管理器选择章节 / 候选稿 / 草稿 / 大纲 / 经纬 / Story / Truth / 素材 / 发布报告
  -> 新建或导入章节
  -> 编辑并保存正式正文
  -> 可选：补充问卷、生成前追问、经纬资料和写作约束
  -> 调用 AI 动作或写作模式，生成预览 / 候选稿 / 草稿
  -> 阅读候选稿或草稿
  -> 用户确认合并、替换、另存或放弃
  -> 使用写作工具、经纬资料、AI 味和发布检查辅助审校
  -> 导出或进入发布准备
```

## 关键对象

| 对象 | 含义 | 当前口径 |
|---|---|---|
| 作品 | 一本小说的根对象，承载章节和资料 | 真实可用 |
| 卷 | 章节分组，适合长篇网文结构 | 规划中 |
| 正式章节 | 用户确认后的正文，是作品内容事实源 | 真实可用 |
| 候选稿 | AI 生成或改写产生的待确认内容 | 真实可用 |
| 草稿 | 暂存想法、备选片段或不立即合并的文本 | 真实可用 |
| 大纲 | 作品、卷、章节的结构规划 | 规划中 |
| Story / Truth 文件 | hooks、章节摘要、风格、规则等文本事实 | 真实可用 / 规划中混合 |
| 故事经纬 | 人物、地点、势力、伏笔、世界规则等连续性资料 | API 底座真实可用；Workspace 透明过渡 |
| 问卷回答 | 用户在生成前填入的题材、卖点、目标、冲突等意图 | 真实可用 |
| 生成前追问 | 系统基于经纬资料和章节位置提出的本章关键问题 | 真实可用 |
| 核心设定变更 | 对主前提、主线冲突、世界规则、人物弧的可审计变更 | 真实可用 |
| 写作工具结果 | 节奏、对话、健康、AI 味、冲突、语气等辅助判断 | 真实可用 / 透明过渡混合 |

## AI 输出流向

```text
AI 操作或写作模式
  -> 生成前模型 gate
  -> 读取章节、经纬、写作约束和用户输入
  -> 返回 prompt 预览 / 真实结果 / 错误
  -> 写入候选稿或草稿，或保持临时预览
  -> 用户明确确认
  -> 才能合并或替换正式章节
```

关键规则：

1. `prompt-preview` 只代表提示词预览或未写入结果，不代表已经生成正文。
2. `chunked-buffer` 只代表拿到完整 AI 结果后再分块发送，不代表上游原生 streaming。
3. `process-memory` 只代表当前进程临时状态，不持久化，重启后不保证保留。
4. `unsupported` 必须作为明确错误或透明占位展示，不得返回假成功。
5. provider、model、run id 或 request metadata 必须随 AI 结果最小可追踪地保存或展示，便于用户知道内容来源。

## 用户侧最小闭环

当前能作为最小闭合理解的是：

```text
选择作品
  -> 新建章节或打开已有章节
  -> 编辑正文
  -> 保存正文
  -> 可选：问卷、生成前追问或经纬资料补上下文
  -> 生成或打开候选稿
  -> 明确选择合并、替换、另存草稿或放弃
  -> 用写作工具、AI 味报告和经纬资料辅助检查
```

完整长篇闭环仍需继续补齐：Story/Truth viewer 收敛、Workspace 经纬 CRUD、大纲 viewer/editor、发布检查数据来源、统一导出入口、真实状态统计和浏览器 E2E。

## 流程中的禁止事项

- 禁止 AI 结果直接覆盖正式正文。
- 禁止把候选稿显示成正式章节。
- 禁止把 `prompt-preview` 写成真实写入。
- 禁止把 `process-memory` 写成不持久化以外的长期保存。
- 禁止把 `chunked-buffer` 写成上游原生流式。
- 禁止把 `unsupported` 写成当前可用。
- 禁止把问卷、追问或变更历史写成已自动改正文。
- 禁止把未接入能力显示为成功。
- 禁止把参考资料或历史路线当成当前产品承诺。

## 概述

故事经纬（Jingwei）是 NovelFork Studio 的结构化世界观管理系统。它将小说的人物、地点、势力、物品、规则等设定组织为"分区 + 条目"的二级结构，并通过可见性规则和 AI 参与标记控制哪些设定在什么时候注入 AI 写作上下文。

---

## 系统架构

```
经纬系统
├── Section（分区）— 设定分类容器
│   ├── Entry（条目）— 具体设定内容
│   ├── Entry
│   └── ...
├── Section
│   ├── Entry
│   └── ...
└── ...
```

分区是逻辑分类（如"人物"、"地点"、"势力"），条目是具体的设定卡片（如"张三"、"青云山"）。

## 可见性规则（visibilityRule）

可见性规则决定条目在哪些章节范围内对 AI 可见。

| 类型 | 说明 | 适用场景 |
|------|------|---------|
| `global` | 全局可见，所有章节都能看到 | 核心设定（主角、世界规则） |
| `nested` | 嵌套在特定章节范围内可见 | 阶段性设定（某卷的反派） |
| `tracked` | 关键词追踪，出现关键词时才注入 | 次要角色、特定物品 |

### 规则结构

```typescript
interface JingweiVisibilityRule {
  type: "global" | "nested" | "tracked";
  visibleAfterChapter?: number;   // nested: 从第几章开始可见
  visibleUntilChapter?: number;   // nested: 到第几章结束可见
  keywords?: string[];            // tracked: 触发关键词列表
  parentEntryIds?: string[];      // nested: 父条目 ID（层级嵌套）
}
```

### 可见性判定逻辑

- `global`：始终注入 AI 上下文
- `nested`：当前章节在 `[visibleAfterChapter, visibleUntilChapter]` 范围内时注入
- `tracked`：当前场景文本中出现 keywords 或 aliases 中的关键词时注入

## 模板系统

通过模板快速初始化经纬分区结构。

| 模板 ID | 说明 | 创建的分区 |
|---------|------|-----------|
| `blank` | 空白模板 | 无预设分区 |
| `basic` | 基础模板 | 人物、地点、物品等基础分区 |
| `enhanced` | 增强模板 | 基础 + 势力、规则、时间线等 |
| `genre-recommended` | 题材推荐 | 根据 genre 参数推荐分区组合 |

### 模板应用 API

```
POST /api/books/:bookId/jingwei/templates/apply
```

请求体：

```json
{
  "templateId": "genre-recommended",
  "genre": "xianxia",
  "selectedSectionKeys": ["characters", "locations", "cultivation-system"]
}
```

应用规则：
- 已存在相同 key 的分区不会重复创建
- 返回应用后的完整分区列表
- 创建了新分区时返回 201，否则返回 200

## AI 上下文参与

### 参与条件

条目被注入 AI 写作上下文需要同时满足：

1. 所属分区 `participatesInAi = true`
2. 条目自身 `participatesInAi = true`
3. 可见性规则在当前章节/场景下满足

### 上下文预览

```
POST /api/books/:bookId/jingwei/preview-context
```

请求体：

```json
{
  "currentChapter": 5,
  "sceneText": "张三走进了青云山的大殿",
  "tokenBudget": 2000
}
```

返回预览结果，展示哪些条目会被注入、预估 token 消耗。用于作者在写作前确认 AI 能"看到"哪些设定。

### Token 预算

- 条目级 `tokenBudget`：限制单个条目注入的最大 token 数
- 请求级 `tokenBudget`：限制整个经纬上下文的总 token 数
- 超出预算时按优先级裁剪（global > nested > tracked）

## 与叙事线的关系

经纬条目通过 `relatedChapterNumbers` 和 `relatedEntryIds` 与叙事线建立关联：

- `relatedChapterNumbers`：标记条目在哪些章节中出场或产生影响
- `relatedEntryIds`：标记条目与其他条目的关联（如人物关系、物品归属）

叙事线（narrative-line）是章节维度的事件序列快照，经纬是设定维度的知识库。两者互补：
- 叙事线回答"发生了什么"
- 经纬回答"世界是什么样的"

AI 写作时，叙事线提供剧情连续性上下文，经纬提供世界观一致性上下文。

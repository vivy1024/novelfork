**版本**: v1.0.2
**创建日期**: 2026-04-28
**更新日期**: 2026-04-28
**状态**: ✅ 当前有效
**文档类型**: current

## 本文定位

本文定义 NovelFork 中 AI 输出如何进入作品。核心原则是：AI 可以辅助生成、续写、审校和改写，但 AI 结果不能直接覆盖正式正文。

## 固定流向

```text
AI 操作
  -> 可选：问卷 / 生成前追问 / 经纬上下文装配
  -> 生成章节 / 候选稿 / 草稿候选
  -> 用户打开查看
  -> 用户确认处理动作
  -> 合并、替换、另存草稿或放弃
```

这条流向是创作安全边界。任何新 AI 功能都应服从它。

## 为什么必须有候选稿层

1. **保护正文**：正式章节是作品内容事实源，不能被 AI 静默覆盖。
2. **方便比较**：作者可以同时保留多个候选版本。
3. **降低误操作风险**：替换正文必须是显式动作。
4. **支撑后续版本管理**：候选稿天然适合接入草稿库、对照视图和快照。
5. **便于审校**：AI 输出可以先经过节奏、对话、连续性和 AI 味检查。

## AI 输出类型

| 输出类型 | 说明 | 当前处理方式 |
|---|---|---|
| 生成下一章 | 根据上下文生成新章节候选 | 进入候选稿路径 |
| 问卷回答 | 生成前补充题材、卖点、目标、冲突等意图 | 进入经纬资料或生成上下文，不直接写正文 |
| 生成前追问 | 根据冲突和伏笔等规则提出本章问题 | 进入本章作者指示，不直接写正文 |
| 续写 | 在已有章节基础上补写段落 | 需要进入候选或草稿，不能直接写正文 |
| 审校 | 提供问题、建议或修订稿 | 建议类结果不能自动改正文 |
| 改写 | 生成替代文本 | 应作为候选稿或草稿候选 |
| 去 AI 味 | 生成优化建议或改写稿 | 需用户确认后才能影响正文 |
| 连续性检查 | 生成诊断结果 | 应作为报告或提示，不直接改正文 |
| prompt preview | 只预览提示词 | 不代表生成或写入 |

## 生成前引导边界

引导式生成只允许改变“本次 AI 的上下文”和“可审查资料”，不能绕过候选稿层：

| 引导能力 | 可以做 | 不能做 |
|---|---|---|
| 问卷中心 | 保存草稿、提交回答、用 AI 建议补一个问题 | 静默提交答案或覆盖正文 |
| 生成前追问 | 根据本章风险提出问题，把答案注入本章作者指示 | 因用户跳过就伪造答案 |
| 经纬上下文装配 | 按章节范围和可见性裁剪资料 | 把所有资料不加区分塞进 prompt |
| 核心设定变更 | 提出、接受、拒绝、记录影响范围 | 接受后自动重写既有章节 |

## 候选稿状态机

```text
candidate
  -> accepted-merge
  -> accepted-replace
  -> saved-as-draft
  -> rejected
```

当前文档不要求状态名与代码完全一致，但产品行为必须满足：候选稿处理前不影响正式正文，处理后结果可追踪。

## 用户动作定义

| 动作 | 说明 | 风险等级 |
|---|---|---|
| 合并到正式章节 | 把候选稿作为补充内容加入目标章节 | 中 |
| 替换正式章节 | 用候选稿替换原正文 | 高 |
| 另存为草稿 | 保存为备选材料，不改正文 | 低 |
| 放弃候选稿 | 删除或标记候选稿不可用 | 低 |

高风险动作必须有足够清楚的文案，避免用户误点后覆盖正文。

## UI 必须表达清楚的内容

候选稿打开时，界面应明确展示：

- 这是候选稿，不是正式正文。
- 目标章节是哪一章。
- 来源是什么，例如生成下一章、续写、改写、审校。
- 当前状态是什么。
- 可以执行哪些处理动作。
- 哪些动作会修改正式正文。

## 禁止事项

- 禁止 AI 结果直接覆盖正式章节正文。
- 禁止把候选稿显示为正式章节。
- 禁止在无安全写入目标时把按钮显示为成功。
- 禁止把 `prompt-preview` 写成真实写入。
- 禁止把问卷、生成前追问、核心设定变更写成自动改正文。
- 禁止把 mock、fake、noop 路径包装成真实生成。

## 当前状态

生成下一章已走候选稿路径；问卷中心、生成前追问、经纬上下文装配、核心设定变更和 AI 味报告已有已实现底座。续写、审校、改写、连续性检查在具体页面上仍需根据真实 route 或未接入状态表达，不能假成功。

## 后续补齐方向

后续完整闭环应补齐：

1. 草稿资源树。
2. 候选稿对照视图。
3. 候选稿另存草稿后的刷新和追踪。
4. 候选稿状态持久化和审计记录。
5. 与故事经纬、hooks、发布检查的联动。

# 资源管理器模型

**版本**: v1.0.0  
**创建日期**: 2026-05-10  
**状态**: ✅ 当前有效  
**文档类型**: current

---

## 概述

资源管理器是 NovelFork Studio 工作台的核心数据层。它将书籍的所有内容（章节、候选稿、草稿、大纲、真相文件、经纬、叙事线）统一抽象为资源树，提供一致的加载、展示、编辑、保存流程。

---

## 资源树数据模型

### ContractResourceNode

后端契约层的资源节点定义，是资源树的基础数据结构。

```typescript
interface ContractResourceNode {
  id: string;                              // 唯一标识，格式如 "chapter:bookId:1"
  kind: ContractResourceKind;              // 节点类型
  title: string;                           // 显示标题
  path?: string;                           // 文件路径（story/truth 类型使用）
  content?: string | null;                 // 正文内容（列表加载时可能为 preview）
  capabilities: ContractResourceCapabilities; // 能力标记
  metadata?: Record<string, unknown>;      // 扩展元数据
  children?: ContractResourceNode[];       // 子节点（group 类型使用）
}
```

### ContractResourceKind — 节点类型

| 类型 | 说明 | 来源 |
|------|------|------|
| `book` | 书籍根节点 | `GET /api/books/:bookId` |
| `group` | 分组容器（章节、候选稿、草稿等） | 前端构造 |
| `chapter` | 正式章节 | BookDetailResponse.chapters |
| `candidate` | AI 生成的候选稿 | `GET /api/books/:bookId/candidates` |
| `draft` | 草稿 | `GET /api/books/:bookId/drafts` |
| `story` | 大纲与设定文件 | `GET /api/books/:bookId/story-files` |
| `truth` | 真相文件（AI 上下文参考） | `GET /api/books/:bookId/truth-files` |
| `jingwei-section` | 经纬分区 | `GET /api/books/:bookId/jingwei/sections` |
| `jingwei-entry` | 经纬条目 | `GET /api/books/:bookId/jingwei/entries` |
| `narrative-line` | 叙事线快照 | `GET /api/books/:bookId/narrative-line` |
| `unsupported` | 加载失败的占位节点 | 错误回退 |

### ContractResourceCapabilities — 能力标记

```typescript
interface ContractResourceCapabilities {
  read?: BackendCapability;      // 读取能力
  edit?: BackendCapability;      // 编辑能力
  delete?: BackendCapability;    // 删除能力
  apply?: BackendCapability;     // 应用能力（候选稿接受等）
  unsupported?: BackendCapability; // 不支持标记
}
```

每个 `BackendCapability` 包含 `status` 字段，取值为：
- `current` — 当前可用
- `unsupported` — 当前不支持
- `process-memory` — 仅进程内存态可用
- `prompt-preview` — 仅预览模式

## 资源树加载流程

### loadResourceTreeFromContract

入口函数，并行加载所有资源域并组装为树结构。

```
loadResourceTreeFromContract(resource, bookId)
  ├── resource.getBook(bookId)              // 必须成功，否则返回错误树
  ├── resource.listCandidates(bookId)       // optional，失败记录到 errors
  ├── resource.listDrafts(bookId)           // optional
  ├── resource.listStoryFiles(bookId)       // optional
  ├── resource.listTruthFiles(bookId)       // optional
  ├── resource.listJingweiSections(bookId)  // optional
  ├── resource.listJingweiEntries(bookId)   // optional
  └── resource.getNarrativeLine(bookId)     // optional
```

返回结构：

```typescript
interface ContractResourceTreeLoadResult {
  ok: true;
  tree: ContractResourceNode[];   // 完整资源树
  errors: ContractResourceNode[]; // 加载失败的节点（kind: "unsupported"）
}
```

加载失败的域不会阻塞整棵树，而是以 `unsupported` 节点出现在对应分组中。

### 树结构

```
book:${bookId}
├── group:chapters        → chapter 节点列表
├── group:candidates      → candidate 节点列表
├── group:drafts          → draft 节点列表
├── group:story-files     → story 节点列表
├── group:truth-files     → truth 节点列表
├── group:jingwei         → jingwei-section + jingwei-entry 节点
└── group:narrative-line  → narrative-line 节点
```

## 详情加载

### resourceNeedsDetailHydration

判断节点是否需要从后端加载完整详情（列表接口只返回 preview）。

```typescript
function resourceNeedsDetailHydration(node: WorkbenchResourceNode): boolean
```

需要 hydration 的条件：
- 节点可打开（`capabilities.open`）且非 unsupported
- 节点 `detailSource` 不是 `"detail"`
- chapter 类型：来源为 preview 或无 content
- story / truth 类型：来源为 preview

### loadResourceDetailState

根据节点类型调用对应的详情 API：

| 节点类型 | 详情 API |
|---------|---------|
| chapter | `GET /api/books/:bookId/chapters/:number` |
| story | `GET /api/books/:bookId/story-files/:fileName` |
| truth | `GET /api/books/:bookId/truth-files/:fileName` |
| draft | `GET /api/books/:bookId/drafts/:draftId` |
| candidate | 使用列表返回的 content（无独立详情接口） |
| jingwei-entry / jingwei-section / narrative-line | 使用 metadata 中的 snapshot |

返回状态：

```typescript
type ResourceDetailState =
  | { status: "idle" }
  | { status: "loading"; resourceId: string }
  | { status: "ready"; resourceId: string; content: string; source: "detail" | "preview"; loadedAt: string; metadata?: Record<string, unknown> }
  | { status: "error"; resourceId: string; error: unknown; message: string }
```

## 保存流程

### saveResourceAndHydrate

统一保存入口，保存后自动回读详情确保一致性。

```typescript
async function saveResourceAndHydrate(
  resource: ResourceDomainClient,
  fallbackBookId: string,
  node: WorkbenchResourceNode,
  content: string,
): Promise<WorkbenchResourceNode>
```

保存前校验（`assertSaveable`）：
- 节点必须有 edit 能力且非 readonly / unsupported
- 节点必须已完成 detail hydration（禁止保存 preview 内容）
- candidate 类型禁止直接保存（必须通过接受流程）

各类型保存路径：

| 节点类型 | 保存 API | 回读方式 |
|---------|---------|---------|
| chapter | `PUT /api/books/:bookId/chapters/:number` | loadResourceDetailState |
| draft | `PUT /api/books/:bookId/drafts/:draftId` | getDraft 回读 |
| truth | `PUT /api/books/:bookId/truth/:fileName` | loadResourceDetailState |
| jingwei-entry | `PUT /api/books/:bookId/jingwei/entries/:entryId` | 使用返回的 entry 数据 |

## WorkbenchResourceNode 与 ContractResourceNode 的映射

前端工作台使用 `WorkbenchResourceNode`，它是 `ContractResourceNode` 的 UI 适配层。

```typescript
interface WorkbenchResourceNode {
  id: string;
  kind: WorkbenchResourceKind;  // 扩展了 bible-entry / storyline / tool-result
  title: string;
  content?: string;
  path?: string;
  metadata?: Record<string, unknown>;
  capabilities: WorkbenchResourceCapabilities;
  children?: WorkbenchResourceNode[];
}

interface WorkbenchResourceCapabilities {
  open: boolean;        // 是否可在画布中打开
  readonly: boolean;    // 是否只读
  unsupported: boolean; // 是否为不支持的类型
  edit: boolean;        // 是否可编辑
  delete: boolean;      // 是否可删除
  apply: boolean;       // 是否可应用（候选稿接受）
}
```

映射规则：
- `open` = kind 非 group 且 read 能力为 current
- `readonly` = edit 能力不为 current
- `unsupported` = kind 为 unsupported 或 unsupported 能力存在
- `edit` / `delete` / `apply` = 对应能力为 current

## 资源 Viewer 注册表

画布根据资源 kind 选择对应的 viewer 组件渲染。

| Viewer Kind | 对应资源 | 说明 |
|-------------|---------|------|
| chapter | chapter | 可编辑 textarea，支持保存 |
| candidate | candidate | 只读展示 + CandidateActionsBar |
| draft | draft | 可编辑 textarea，支持保存 |
| story | story | 只读展示大纲/设定文件 |
| truth | truth | 可编辑 textarea，支持保存 |
| bible-entry | bible-entry | 圣经条目展示 |
| storyline | storyline | 故事线展示 |
| jingwei-section | jingwei-section | 经纬分区信息展示 |
| jingwei-entry | jingwei-entry | 经纬条目编辑器 |
| narrative-line | narrative-line | 叙事线快照展示 |
| tool-result | tool-result | 工具调用结果展示 |
| generic | 未知类型 | 通用回退 viewer |

未知 kind 自动回退到 `generic` viewer。

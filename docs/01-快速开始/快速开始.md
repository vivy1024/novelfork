**创建日期**: 2026-04-18  
**版本**: v0.0.1

## 📋 概述

这份快速开始描述的是当前仓库的源码运行方式，用于开发和过渡期验证。它不是最终产品交付形态；最终目标仍是 NarraFork 风格的本地单体应用。

`★ 核心价值 ─────────────────────────────────────`
NovelFork 通过多 Agent 协作管线，实现了零人工干预的章节创作。内置 33 维度连续性审计、文风仿写、去 AI 味等功能，支持玄幻、仙侠、都市、科幻等题材。
`─────────────────────────────────────────────────`

## 🚀 5分钟快速体验

### 前置条件

```bash
# ✅ 1. Node.js 环境（必需）
node --version          # ≥20.0.0
pnpm --version          # ≥9.0.0

# ✅ 2. Git 环境（必需，用于 Worktree 多线并行）
git --version           # ≥2.30

# ✅ 3. 基础硬件要求
#    - 内存：≥8GB（推荐16GB）
#    - 磁盘：≥10GB可用空间
#    - CPU：≥4核心
```

### 步骤1：安装 NovelFork

```bash
# 从源码安装
git clone https://github.com/vivy1024/novelfork.git novelfork
cd novelfork
pnpm install
pnpm build
npm link packages/cli
```

**预期输出**：
```
✓ Built packages/core
✓ Built packages/cli
✓ Built packages/studio
✓ Linked novelfork command
```

### 步骤2：配置 LLM 提供商

> 如果你当前只想先体验本地工作流，这一步可以暂时跳过。未配置模型时，仍可创建本地书籍、整理故事经纬、编辑章节；只有在触发 AI 续写、评点、深度 AI 味检测或工作台 Agent 时，系统才会提示配置模型。

```bash
# 全局配置（推荐，只需一次）
novelfork config set-global \
  --provider openai \
  --base-url https://api.openai.com/v1 \
  --api-key sk-xxx \
  --model gpt-4o
```

**预期输出**：
```json
{
  "success": true,
  "message": "全局配置已保存到 ~/.novelfork/.env"
}
```

### 步骤3：创建第一本书

```bash
# 创建新书
novelfork book create \
  --title "吞天魔帝" \
  --genre xuanhuan \
  --chapter-words 3000 \
  --target-chapters 100
```

**预期输出**：
```

  题材: 玄幻
```

### 步骤4：写第一章

```bash

novelfork write next 吞天魔帝
```

**预期输出**：

## 🧭 Studio 首次打开建议

1. 推荐优先配置 AI 模型，但可以跳过。
2. 未配置模型时，可先完成：创建本地书籍、选择故事经纬结构、手工维护经纬栏目、编辑章节正文。
3. 如果你已经有现成栏目目录，现在也可以在故事经纬的栏目管理里使用 **Markdown 目录导入** 快速建立栏目。
4. 《没钱修什么仙》只作为高级范本导入样例，不是默认标准；栏目结构应以当前作品需求为准。

### 步骤5：查看创作结果

```bash
# 查看书籍状态
novelfork status 吞天魔帝

# 导出全书
novelfork export 吞天魔帝 --format txt
```

## 🎯 三种使用方式

### 方式1：完整管线（推荐新手）

```bash
# 一键式写作（草稿 → 审计 → 修订）
novelfork write next 吞天魔帝

# 连续写5章
novelfork write next 吞天魔帝 --count 5

# 指定字数
novelfork write next 吞天魔帝 --words 5000
```

**适用场景**：
- ✅ 零人工干预的自动化创作
- ✅ 保证质量的批量生产
- ✅ 守护进程后台运行

### 方式2：原子命令（可组合）

```bash
# 规划章节意图
novelfork plan chapter 吞天魔帝 --context "本章重点写师徒矛盾"

# 编排上下文
novelfork compose chapter 吞天魔帝

# 只写草稿
novelfork draft 吞天魔帝

# 审计指定章节
novelfork audit 吞天魔帝 1

# 修订指定章节
novelfork revise 吞天魔帝 1
```

**适用场景**：
- ✅ 精细控制创作流程
- ✅ 调试和优化
- ✅ 自定义工作流

### 方式3：自然语言 Agent 模式

```bash
# 自然语言指令
novelfork agent "帮我写一本都市修仙，主角是个程序员"

# 复杂任务
novelfork agent "先扫描市场趋势，然后根据结果创建一本新书"

# 创作指导
novelfork agent "写下一章，重点写师徒矛盾，字数3000字"
```

**适用场景**：
- ✅ 快速原型验证
- ✅ 复杂任务编排
- ✅ 自然交互体验

## 🏗️ 项目结构概览

```
novelfork/
├── packages/
│   ├── core/                    # 核心逻辑
│   │   ├── agents/             # 8个Agent实现
│   │   ├── memory/             # 长期记忆系统
│   │   ├── audit/              # 33维度审计
│   │   └── llm/                # LLM提供商适配
│   │
│   ├── cli/                     # 命令行工具
│   │   ├── commands/           # CLI命令
│   │   └── utils/              # 工具函数
│   │
│   └── studio/                  # Web工作台
│       ├── src/api/            # Hono后端
│       ├── src/components/     # React组件
│       └── src/stores/         # 状态管理
│
└── story/                       # 书籍数据（自动生成）
    ├── state/                  # 7个真相文件（JSON）
    ├── runtime/                # 运行时产物
    ├── chapters/               # 章节正文
    └── memory.db               # SQLite时序记忆
```

### 核心特性

- **多 Agent 管线**: 8个专业 Agent 接力完成章节创作
- **连续性审计**: 33维度检查，确保逻辑自洽
- **长期记忆**: 7个真相文件 + SQLite 时序数据库
- **去 AI 味**: 内置词汇疲劳词表、禁用句式、文风指纹
- **文风仿写**: 提取统计指纹，注入创作流程
- **多模型路由**: 不同 Agent 使用不同模型
- **Web 工作台**: 可视化管理、实时监控、数据分析

## ⚠️ 常见问题

### Q: 安装失败怎么办？

```bash
# 检查 Node.js 版本
node --version  # 需要 ≥20.0.0

# 检查 pnpm 版本
pnpm --version  # 需要 ≥9.0.0

# 清理缓存重新安装
rm -rf node_modules
pnpm install
```

### Q: API 调用失败？

```bash
# 诊断配置问题
novelfork doctor

# 检查全局配置
novelfork config show-global

# 测试 API 连通性
curl -X POST https://api.openai.com/v1/chat/completions \
  -H "Authorization: Bearer $NOVELFORK_LLM_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"model":"gpt-4o","messages":[{"role":"user","content":"test"}]}'
```

### Q: 审计不通过怎么办？

```bash
# 查看审计报告
novelfork audit 吞天魔帝 1 --json

# 手动修订
novelfork revise 吞天魔帝 1

# 重写章节
novelfork write rewrite 吞天魔帝 1
```

### Q: 如何提高创作质量？

1. **使用更好的模型**：Claude Opus、GPT-4o
2. **提供创作简报**：`--brief` 传入详细设定
3. **编辑控制文件**：`story/author_intent.md`、`story/current_focus.md`
4. **导入文风指纹**：`novelfork style import`
5. **多模型路由**：写手用 Claude，审计用 GPT-4o

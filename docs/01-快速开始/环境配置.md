## 🔧 配置层级

NovelFork 支持三层配置，优先级从高到低：

```
1. 项目级配置 (.env)          # 最高优先级
   ↓
2. 全局配置 (~/.novelfork/.env) # 中等优先级
   ↓
3. 默认配置                    # 最低优先级
```

---

## 🌍 全局配置（推荐）

### 配置全局 LLM 提供商

```bash
# OpenAI
novelfork config set-global \
  --provider openai \
  --base-url https://api.openai.com/v1 \
  --api-key sk-xxx \
  --model gpt-4o

# Anthropic Claude
novelfork config set-global \
  --provider anthropic \
  --base-url https://api.anthropic.com \
  --api-key sk-ant-xxx \
  --model claude-opus-4-7

# 自定义提供商（兼容 OpenAI 格式）
novelfork config set-global \
  --provider custom \
  --base-url https://your-api.com/v1 \
  --api-key your-key \
  --model your-model
```

### 查看全局配置

```bash
# 查看当前配置
novelfork config show-global

# 预期输出
{
  "provider": "openai",
  "baseUrl": "https://api.openai.com/v1",
  "model": "gpt-4o",
  "apiKey": "sk-***" // 脱敏显示
}
```

### 全局配置文件位置

```bash
# Linux/macOS
~/.novelfork/.env

# Windows
C:\Users\<用户名>\.novelfork\.env
```

## 📁 项目级配置

### 初始化项目

```bash
# 在当前目录初始化
novelfork init

# 在指定目录初始化
novelfork init my-novel
cd my-novel
```

### 编辑项目配置

```bash
# 编辑 .env 文件
vim .env
```

**完整配置示例**：

```bash
# ========== LLM 提供商配置 ==========
# 必填
NOVELFORK_LLM_PROVIDER=openai              # openai / anthropic / custom
NOVELFORK_LLM_BASE_URL=https://api.openai.com/v1
NOVELFORK_LLM_API_KEY=sk-xxx
NOVELFORK_LLM_MODEL=gpt-4o

# 可选
NOVELFORK_LLM_TEMPERATURE=0.7              # 温度 (0.0-2.0)
NOVELFORK_LLM_MAX_TOKENS=8192              # 最大输出 token
NOVELFORK_LLM_THINKING_BUDGET=0            # Anthropic 扩展思考预算

# ========== 项目配置 ==========
NOVELFORK_DEFAULT_GENRE=xuanhuan           # 默认题材
NOVELFORK_DEFAULT_CHAPTER_WORDS=3000       # 默认每章字数
NOVELFORK_DEFAULT_TARGET_CHAPTERS=100      # 默认目标章节数

# ========== 守护进程配置 ==========
NOVELFORK_DAEMON_INTERVAL=300              # 守护进程间隔（秒）
NOVELFORK_DAEMON_MAX_CHAPTERS=10           # 守护进程最大章节数

# ========== 通知配置 ==========
NOVELFORK_NOTIFY_TELEGRAM_BOT_TOKEN=       # Telegram Bot Token
NOVELFORK_NOTIFY_TELEGRAM_CHAT_ID=         # Telegram Chat ID
NOVELFORK_NOTIFY_WEBHOOK_URL=              # Webhook URL
NOVELFORK_NOTIFY_WEBHOOK_SECRET=           # Webhook 签名密钥

# ========== 调试配置 ==========
NOVELFORK_DEBUG=false                      # 调试模式
NOVELFORK_LOG_LEVEL=info                   # 日志级别 (debug/info/warn/error)
```

### 使用 CLI 配置项目

```bash
# 设置项目配置
novelfork config set \
  --provider openai \
  --base-url https://api.openai.com/v1 \
  --api-key sk-xxx \
  --model gpt-4o

# 查看项目配置
novelfork config show
```

## 🎯 多模型路由

### 为不同 Agent 配置不同模型

```bash
# 写手使用 Claude Opus（创意强）
novelfork config set-model writer claude-opus-4-7 \
  --provider anthropic \
  --base-url https://api.anthropic.com \
  --api-key-env ANTHROPIC_API_KEY

# 审计员使用 GPT-4o（便宜快速）
novelfork config set-model auditor gpt-4o \
  --provider openai \
  --base-url https://api.openai.com/v1 \
  --api-key-env OPENAI_API_KEY

# 雷达使用本地模型（零成本）
novelfork config set-model radar qwen-7b \
  --provider custom \
  --base-url http://localhost:11434/v1 \
  --api-key-env LOCAL_API_KEY
```

### 查看模型路由

```bash
# 查看当前路由
novelfork config show-models

# 预期输出
{
  "writer": {
    "model": "claude-opus-4-7",
    "provider": "anthropic",
    "baseUrl": "https://api.anthropic.com"
  },
  "auditor": {
    "model": "gpt-4o",
    "provider": "openai",
    "baseUrl": "https://api.openai.com/v1"
  },
  "radar": {
    "model": "qwen-7b",
    "provider": "custom",
    "baseUrl": "http://localhost:11434/v1"
  },
  "default": {
    "model": "gpt-4o",
    "provider": "openai",
    "baseUrl": "https://api.openai.com/v1"
  }
}
```

### 移除模型路由

```bash
# 移除 writer 的模型覆盖（回退到默认）
novelfork config remove-model writer

# 验证
novelfork config show-models
```

## 🔐 API Key 管理

### 环境变量方式（推荐）

```bash
# 在 .env 中设置
NOVELFORK_LLM_API_KEY=sk-xxx

# 或在 shell 中设置
export NOVELFORK_LLM_API_KEY=sk-xxx
```

### 多个 API Key

```bash
# 在 .env 中设置多个 Key
OPENAI_API_KEY=sk-xxx
ANTHROPIC_API_KEY=sk-ant-xxx
LOCAL_API_KEY=local-key

# 在模型路由中引用
novelfork config set-model writer claude-opus-4-7 \
  --provider anthropic \
  --api-key-env ANTHROPIC_API_KEY
```

### API Key 安全

```bash
# ❌ 不要将 API Key 提交到 Git
echo ".env" >> .gitignore

# ✅ 使用环境变量
export NOVELFORK_LLM_API_KEY=sk-xxx

# ✅ 使用密钥管理工具
# 1Password, LastPass, AWS Secrets Manager 等
```

## 🛠️ 故障排除

### 问题1：API Key 无效

**症状**：
```
Error: Invalid API key
```

**解决方案**：
```bash
# 检查 API Key
novelfork config show-global

# 重新设置
novelfork config set-global --api-key sk-xxx

# 测试连通性
novelfork doctor
```

### 问题2：模型不可用

**症状**：
```
Error: Model not found
```

**解决方案**：
```bash
# 检查模型名称
novelfork config show-global

# 查看提供商支持的模型
# OpenAI: https://platform.openai.com/docs/models
# Anthropic: https://docs.anthropic.com/claude/docs/models-overview

# 更新模型
novelfork config set-global --model gpt-4o
```

### 问题3：网络连接失败

**症状**：
```
Error: Network request failed
```

**解决方案**：
```bash
# 检查网络连接
ping api.openai.com

# 检查代理设置
echo $HTTP_PROXY
echo $HTTPS_PROXY

# 设置代理（如果需要）
export HTTP_PROXY=http://proxy.com:8080
export HTTPS_PROXY=http://proxy.com:8080
```

### 问题4：配置文件损坏

**症状**：
```
Error: Invalid configuration file
```

**解决方案**：
```bash
# 备份配置
cp ~/.novelfork/.env ~/.novelfork/.env.backup

# 删除配置
rm ~/.novelfork/.env

# 重新配置
novelfork config set-global \
  --provider openai \
  --base-url https://api.openai.com/v1 \
  --api-key sk-xxx \
  --model gpt-4o
```

```bash
NOVELFORK_LLM_PROVIDER=openai
```

```bash

NOVELFORK_LLM_PROVIDER=custom
```

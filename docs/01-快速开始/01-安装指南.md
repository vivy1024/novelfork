# 安装指南

本指南将帮助你在本地环境中安装和配置 NovelFork。

## 前置要求

在开始之前，请确保你的系统满足以下要求：

### 必需软件

- **Node.js** ≥ 20.0.0（推荐 22.5.0+）
  - 下载地址：https://nodejs.org/
  - 验证安装：`node --version`

- **pnpm** ≥ 9.0.0
  - 安装命令：`npm install -g pnpm`
  - 验证安装：`pnpm --version`

- **Git** ≥ 2.30.0
  - 下载地址：https://git-scm.com/
  - 验证安装：`git --version`

### 系统要求

- 操作系统：Windows 10+、macOS 10.15+、Linux（主流发行版）
- 内存：建议 4GB 以上
- 磁盘空间：至少 500MB 可用空间

## 安装步骤

### 1. 克隆仓库

```bash
git clone https://github.com/vivy1024/inkos.git novelfork
cd novelfork
```

### 2. 安装依赖

```bash
pnpm install
```

这将安装所有必需的依赖包，包括：
- 核心包（`@vivy1024/novelfork-core`）
- CLI 工具（`@vivy1024/novelfork`）
- Studio Web 工作台（`@vivy1024/novelfork-studio`）

### 3. 构建项目

```bash
pnpm build
```

构建过程会编译 TypeScript 代码并生成可执行文件。

### 4. 链接 CLI 命令

```bash
npm link packages/cli
```

这将使 `novelfork` 命令在全局可用。

### 5. 验证安装

```bash
novelfork --version
```

如果显示版本号，说明安装成功。

## 配置 API

NovelFork 需要连接 LLM API 才能工作。支持三种配置方式：

### 方式一：全局配置（推荐）

适合只使用一个 API 提供商的场景，配置一次后所有项目共享。

```bash
novelfork config set-global \
  --provider <openai|anthropic|custom> \
  --base-url <API 地址> \
  --api-key <你的 API Key> \
  --model <模型名>
```

**参数说明：**

- `--provider`：API 提供商类型
  - `openai`：OpenAI 官方 API
  - `anthropic`：Anthropic Claude API
  - `custom`：兼容 OpenAI 格式的第三方 API（中转站、智谱、Gemini 等）

- `--base-url`：API 端点地址
  - OpenAI：`https://api.openai.com/v1`
  - Anthropic：`https://api.anthropic.com/v1`
  - 自定义：你的中转站地址

- `--api-key`：你的 API 密钥

- `--model`：模型名称
  - OpenAI：`gpt-4o`、`gpt-4-turbo`、`gpt-3.5-turbo`
  - Anthropic：`claude-3-5-sonnet-20241022`、`claude-3-opus-20240229`
  - 其他：根据提供商文档填写

**示例：**

```bash
# OpenAI 官方
novelfork config set-global \
  --provider openai \
  --base-url https://api.openai.com/v1 \
  --api-key sk-xxx \
  --model gpt-4o

# Anthropic Claude
novelfork config set-global \
  --provider anthropic \
  --base-url https://api.anthropic.com/v1 \
  --api-key sk-ant-xxx \
  --model claude-3-5-sonnet-20241022

# 自定义中转站
novelfork config set-global \
  --provider custom \
  --base-url https://your-proxy.com/v1 \
  --api-key your-key \
  --model gpt-4o
```

配置保存在 `~/.novelfork/.env`，所有项目共享。

### 方式二：项目级配置

适合不同项目使用不同 API 的场景。

1. 初始化项目：

```bash
novelfork init my-novel
cd my-novel
```

2. 复制环境变量模板：

```bash
cp ../.env.example .env
```

3. 编辑 `.env` 文件：

```bash
# 必填
NOVELFORK_LLM_PROVIDER=custom
NOVELFORK_LLM_BASE_URL=https://your-api.com/v1
NOVELFORK_LLM_API_KEY=your-key-here
NOVELFORK_LLM_MODEL=gpt-4o

# 可选
NOVELFORK_LLM_TEMPERATURE=0.7
NOVELFORK_LLM_MAX_TOKENS=8192
NOVELFORK_LLM_THINKING_BUDGET=0
```

项目级 `.env` 会覆盖全局配置。

### 方式三：多模型路由（高级）

为不同 Agent 分配不同模型，平衡质量与成本：

```bash
# 写手用 Claude（创意强）
novelfork config set-model writer claude-3-5-sonnet-20241022 \
  --provider anthropic \
  --base-url https://api.anthropic.com/v1 \
  --api-key-env ANTHROPIC_API_KEY

# 审计用 GPT-4o（便宜快速）
novelfork config set-model auditor gpt-4o \
  --provider openai \
  --base-url https://api.openai.com/v1 \
  --api-key-env OPENAI_API_KEY

# 查看当前路由
novelfork config show-models
```

未单独配置的 Agent 自动使用全局模型。

## 验证配置

运行诊断命令检查配置是否正确：

```bash
novelfork doctor
```

该命令会检查：
- 环境变量是否正确设置
- API 连通性
- 提供商兼容性
- 依赖版本

## 启动开发服务器

安装完成后，可以启动 Studio Web 工作台：

```bash
novelfork studio
```

默认在 `http://localhost:4567` 打开。使用 `-p` 参数指定端口：

```bash
novelfork studio -p 8080
```

## 常见问题

### 安装失败

**问题：** `pnpm install` 报错

**解决方案：**
1. 确认 Node.js 版本 ≥ 20：`node --version`
2. 确认 pnpm 版本 ≥ 9：`pnpm --version`
3. 清理缓存后重试：
   ```bash
   pnpm store prune
   rm -rf node_modules
   pnpm install
   ```

### 命令未找到

**问题：** `novelfork: command not found`

**解决方案：**
1. 确认已执行 `npm link packages/cli`
2. 检查 npm 全局 bin 目录是否在 PATH 中：
   ```bash
   npm config get prefix
   ```
3. 手动添加到 PATH（临时）：
   ```bash
   export PATH="$(npm config get prefix)/bin:$PATH"
   ```

### API 连接失败

**问题：** `novelfork doctor` 显示 API 连接失败

**解决方案：**
1. 检查网络连接
2. 确认 API Key 正确
3. 确认 base-url 地址正确
4. 如果使用代理，检查代理设置

### 权限错误

**问题：** `npm link` 报权限错误

**解决方案：**
- macOS/Linux：使用 `sudo npm link packages/cli`
- Windows：以管理员身份运行命令提示符

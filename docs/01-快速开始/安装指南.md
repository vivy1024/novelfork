**创建日期**: 2026-04-18

**状态**: 过渡文档（描述当前源码运行方式，不代表目标平台形态）

## 📋 概述

本指南描述的是当前仓库的源码运行方式，用于开发和过渡期验证。它不是最终产品的安装说明；最终目标仍是 Bun 单可执行文件分发。

当前 Package 2 的正式交付边界只覆盖：`pnpm bun:compile`、embedded/filesystem 双路径、startup recovery 最小链路。它**不**覆盖 installer、signing、auto-update、first-launch UX，也不代表更完整 repair / migration 已收口。

## 📦 安装 Node.js

### Windows

**方式1：官方安装包（推荐）**

```bash
# 1. 访问 Node.js 官网
https://nodejs.org/

# 2. 下载 LTS 版本（≥20.0.0）
# 3. 运行安装程序
# 4. 验证安装
node --version
npm --version
```

**方式2：使用 nvm-windows**

```bash
# 1. 下载 nvm-windows
https://github.com/coreybutler/nvm-windows/releases

# 2. 安装 Node.js
nvm install 22.5.0
nvm use 22.5.0

# 3. 验证安装
node --version
```

### macOS

**方式1：使用 Homebrew（推荐）**

```bash
# 1. 安装 Homebrew（如果未安装）
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

# 2. 安装 Node.js
brew install node@22

# 3. 验证安装
node --version
npm --version
```

**方式2：使用 nvm**

```bash
# 1. 安装 nvm
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash

# 2. 重启终端，安装 Node.js
nvm install 22.5.0
nvm use 22.5.0

# 3. 验证安装
node --version
```

### Linux

**方式1：使用包管理器**

```bash
# Ubuntu/Debian
curl -fsSL https://deb.nodesource.com/setup_22.x | sudo -E bash -
sudo apt-get install -y nodejs

# CentOS/RHEL
curl -fsSL https://rpm.nodesource.com/setup_22.x | sudo bash -
sudo yum install -y nodejs

# Arch Linux
sudo pacman -S nodejs npm

# 验证安装
node --version
npm --version
```

**方式2：使用 nvm**

```bash
# 1. 安装 nvm
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash

# 2. 重启终端，安装 Node.js
nvm install 22.5.0
nvm use 22.5.0

# 3. 验证安装
node --version
```

**其他安装方式**：

```bash

# 使用独立脚本（Linux/macOS）
curl -fsSL https://get.pnpm.io/install.sh | sh -

# 使用独立脚本（Windows PowerShell）
iwr https://get.pnpm.io/install.ps1 -useb | iex
```

## 🚀 安装 NovelFork

> 当前仍以源码安装/编译体验为准，尚未提供 installer；signing、auto-update、first-launch UX 也不在本阶段正式交付边界内。

### 方式1：从源码安装（推荐）

```bash
# 1. 克隆仓库
git clone https://github.com/vivy1024/novelfork.git novelfork
cd novelfork

# 2. 安装依赖
pnpm install

# 3. 构建项目
pnpm build

# 4. 链接 CLI 命令
npm link packages/cli

# 5. 验证安装
novelfork --version
```

**预期输出**：
```
novelfork v0.0.1
```

### 方式2：从 npm 安装（未来支持）

```bash
# 全局安装（未来版本）
npm install -g @vivy1024/novelfork-cli

# 验证安装
novelfork --version
```

## 🛠️ 故障排除

### 问题1：Node.js 版本过低

**症状**：
```
Error: NovelFork requires Node.js ≥20.0.0
```

**解决方案**：
```bash
# 升级 Node.js
nvm install 22.5.0
nvm use 22.5.0
```

### 问题2：pnpm 未找到

**症状**：
```
command not found: pnpm
```

**解决方案**：
```bash
# 安装 pnpm
npm install -g pnpm

# 或使用 Corepack
corepack enable
```

### 问题3：权限错误（Linux/macOS）

**症状**：
```
Error: EACCES: permission denied
```

**解决方案**：
```bash
# 方式1：使用 sudo（不推荐）
sudo npm install -g pnpm

# 方式2：修改 npm 全局目录（推荐）
mkdir ~/.npm-global
npm config set prefix '~/.npm-global'
echo 'export PATH=~/.npm-global/bin:$PATH' >> ~/.bashrc
source ~/.bashrc
```

### 问题4：构建失败

**症状**：
```
Error: Build failed
```

**解决方案**：
```bash
# 清理缓存
pnpm store prune
rm -rf node_modules
rm -rf packages/*/node_modules

# 重新安装
pnpm install
pnpm build
```

### 问题5：Git Worktree 不可用

**症状**：
```
Error: Git worktree command not found
```

**解决方案**：
```bash
# 升级 Git
# Windows: 重新下载安装包
# macOS: brew upgrade git
# Linux: sudo apt-get upgrade git

# 验证 Worktree 功能
git worktree --help
```

## 🔄 更新 NovelFork

### 从源码更新

```bash
# 1. 进入项目目录
cd novelfork

# 2. 拉取最新代码
git pull origin master

# 3. 更新依赖
pnpm install

# 4. 重新构建
pnpm build

# 5. 验证版本
novelfork --version
```

### 当前边界说明

当前文档只覆盖源码更新路径。Package 2 目前**不包含**内置 auto-update，也不包含 installer / signing / first-launch UX。

## 🗑️ 卸载 NovelFork

### 完全卸载

```bash
# 1. 取消链接 CLI
npm unlink -g @vivy1024/novelfork-cli

# 2. 删除项目目录
rm -rf novelfork

# 3. 删除全局配置（可选）
rm -rf ~/.novelfork

# 4. 验证卸载
novelfork --version  # 应该显示 command not found
```

# NovelFork 版本演进历史

## 说明

本文件记录的是 **NovelFork** 的项目演进历史。

需要特别说明：
- 当前项目已经是 **NovelFork**
- 不是 InkOS
- InkOS 只是代码来源和历史上游
- 过去一些版本记录和阶段描述混用了 InkOS 口径，这里统一用 NovelFork 视角重新整理

---

## 阶段 0：上游继承期

### 背景
NovelFork 起步时继承了上游 InkOS 的一整套工程基础：
- monorepo 结构
- `packages/core / cli / studio / desktop`
- TypeScript + Node
- CLI + Web 工作台双入口
- 多 Agent 写作管线

### 保留下来的核心资产
- 写作引擎
- 状态与真相文件系统
- 连续性审计
- 题材与规则系统
- Studio 工作台基础设施

---

## 阶段 1：功能扩展期

### 主要方向
这一阶段主要围绕“把写作工作台做强”推进：
- 工具系统
- Git Worktree 能力
- 会话、多窗口、搜索、监控
- Lorebook / NER / RAG 等网文相关能力

### 特征
- 产品能力快速增长
- 交互层越来越接近 NarraFork
- 但平台外壳仍然沿用旧的 Node / pnpm / tsc / 多入口结构

---

## 阶段 2：平台偏航期

### 发生了什么
在 Tauri、SEA 等路线尝试后，项目逐渐把目标写成了：
- PWA 优先
- Node + pnpm + tsc
- CLI 启动 studio
- 继续按源码工程形态运行

### 问题本质
偏航不在核心写作能力，而在于：
- 把 NarraFork 当成 UI/功能参考
- 没把 NarraFork 当成平台原型

结果就是：
- 壳越来越像 Web IDE
- 而不像本地单体产品

---

## 阶段 3：平台回正期（当前）

### 当前判断
当前 NovelFork 的核心结论是：

> 不是推翻项目重来，而是保留写作引擎，替换错误外壳。

### 当前主目标
- 回归 NarraFork 风格的本地单体应用
- Bun 作为首选运行时
- 未来以 `bun compile` 作为主要分发路线
- 继续保留并迁移已有核心写作能力

### 当前关键产出
- docs 体系重构为 `00-07`
- 平台纠偏说明
- 平台迁移方案
- NarraFork 依赖与更新日志参考文档

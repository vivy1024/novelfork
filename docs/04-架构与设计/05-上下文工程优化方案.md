## P0 Critical — 直接影响 token 效率和用户体验

### 1. Micro-compact 白名单扩充

**现状**: 只折叠 `cockpit/pgi/questionnaire/narrative/guided/candidate` 前缀的工具结果。  
**问题**: Read/Bash/Grep/Glob/WebFetch 等通用工具结果占 context 大头，从不折叠。连续写 20 章后 context 膨胀严重。

**方案**:
```ts
// compact/micro-compact.ts
const COMPACTABLE_TOOL_PREFIXES = [
  // 现有
  "cockpit.", "pgi.", "questionnaire.", "narrative.", "guided.", "candidate.",
  // 新增 — 通用 I/O 工具
  "Read", "Bash", "Grep", "Glob", "WebFetch", "WebSearch",
  "Edit", "Write", "Terminal",
  // 新增 — 小说工具
  "jingwei.", "chapter.", "scene.", "pipeline.", "resource.",
  "health.", "hooks.", "beat.", "presets.", "style.",
];
```

**保留策略**: 最近 3 条不折叠 → 改为最近 **5** 条（写作场景上下文依赖更重）。

**折叠格式**（增加更多信息）:
```
[工具结果已折叠: {toolName}({关键参数摘要}) — {前80字符}... ({原始字符数}字符)]
```

**实施文件**: `packages/studio/src/api/lib/compact/micro-compact.ts`

### 2. File Unchanged Dedup（文件去重）

**现状**: 无。相同文件反复读取，每次完整塞入 context。经纬数据 `jingwei.read` 每章调用都重复。  
**Claude Code 做法**: 跟踪 `readFileState`（文件路径 → 最后内容 hash），连续读取同一文件且内容未变时返回 stub。

**方案**:

```ts
// agent-turn-runtime.ts 新增
interface FileReadCache {
  path: string;
  contentHash: string;  // SHA-256 前 16 位
  lastReadStep: number;
}

const fileReadCache = new Map<string, FileReadCache>();

function deduplicateFileResult(toolName: string, input: Record<string, unknown>, result: string): string {
  // 只对文件读取类工具生效
  if (!["Read", "jingwei.read", "chapter.read"].includes(toolName)) return result;
  
  const path = String(input.file_path ?? input.scope ?? input.bookId ?? "");
  if (!path) return result;
  
  const hash = quickHash(result);
  const cached = fileReadCache.get(path);
  
  if (cached && cached.contentHash === hash) {
    return `[文件未变: ${path} — 与第 ${cached.lastReadStep} 步相同，内容省略]`;
  }
  
  fileReadCache.set(path, { path, contentHash: hash, lastReadStep: currentStep });
  return result;
}
```

**注意事项**:
- Cache 生命周期 = 单个 turn（每轮对话重置），不跨 turn 保留
- 对 `jingwei.read(scope=brief)` 特别有效——经纬核心包在写作 7 步流程中被读 2 次
- 对 `chapter.read` 有效——审计时可能反复读同一章

**实施文件**: `packages/studio/src/api/lib/agent-turn-runtime.ts`

### 3. Token 估算修正

**现状**: 简单的 `chars / 2`（CJK）或 `chars / 4`（ASCII），不区分内容类型。  
**问题**: JSON 内容 token 密度约 `chars / 2`，纯中文约 `chars / 1.5`，混合内容约 `chars / 2`。当前估算可能低估 30-50%。

**方案**:

```ts
// context-compaction.ts 改进 estimateTokens
function estimateTokensImproved(text: string): number {
  if (!text) return 0;
  
  // 检测 JSON 内容（密集 token）
  const jsonLikelihood = (text.startsWith("{") || text.startsWith("[")) ? 0.8 : 0;
  
  // CJK 字符占比
  const cjkChars = (text.match(/[\u4e00-\u9fff\u3000-\u303f]/g) || []).length;
  const cjkRatio = cjkChars / text.length;
  
  // 混合估算
  let estimate: number;
  if (jsonLikelihood > 0.5) {
    estimate = Math.ceil(text.length / 2);  // JSON 密集
  } else if (cjkRatio > 0.5) {
    estimate = Math.ceil(text.length / 1.5);  // 中文主导
  } else {
    estimate = Math.ceil(text.length / 3);  // 英文/混合
  }
  
  // 保守系数 ×4/3（Claude Code 做法）
  return Math.ceil(estimate * 4 / 3);
}
```

**应用位置**:
- `context-compaction.ts` 的 `estimateTokenCount`
- `context-breakdown.ts` 的消息历史估算
- `context-budget-manager.ts` 的 `estimateTokensFromText`

**实施文件**: 统一到一个 `token-estimation.ts` 模块，其他文件引用

## P1 Important — 显著改善稳定性

### 4. Prompt-too-long 自动重试

**现状**: API 返回 413/400 context_length_exceeded 时直接报错。  
**Claude Code 做法**: 按 API-round 分组丢弃最旧的轮次，最多重试 2 次。

**方案**:

```ts
// agent-turn-runtime.ts generate 调用处
if (result.code === "upstream-error" && isContextOverflowError(result.error)) {
  // 按轮次分组（assistant tool_call + tool_result 为一组）
  const groups = groupMessagesByRound(messages);
  // 丢弃最旧的 1/4 组
  const dropCount = Math.max(1, Math.floor(groups.length / 4));
  const remaining = groups.slice(dropCount).flat();
  // 插入通知
  remaining.unshift({ type: "message", role: "system", content: "[上下文溢出，已丢弃早期对话。]" });
  // 重试（最多 2 次）
  messages.length = 0;
  messages.push(...remaining);
  continue; // 回到 generate 循环
}
```

**实施文件**: `packages/studio/src/api/lib/agent-turn-runtime.ts`

### 7. Post-compact 状态恢复

**现状**: 压缩后只恢复最近文件。Plan/Goals/Todos/Agent 状态丢失。  
**Claude Code 做法**: `runPostCompactCleanup` 恢复 Plan attachments、Skills、Agent definitions、MCP instructions、运行 SessionStart hooks。

**方案**:

```ts
// context-compaction.ts autoCompact 后
async function postCompactRestore(sessionId: string, compactedItems: AgentTurnItem[]): AgentTurnItem[] {
  const restored = [...compactedItems];
  
  // 1. 恢复 Todos（如果有未完成的）
  const session = await getSessionById(sessionId);
  if (session?.todos?.some(t => t.status !== "completed")) {
    const todosText = session.todos
      .filter(t => t.status !== "completed")
      .map(t => `- [${t.status === "in_progress" ? "→" : "○"}] ${t.content}`)
      .join("\n");
    restored.push({
      type: "message", role: "system",
      content: `[上下文已压缩。以下是未完成的待办事项：]\n${todosText}`,

      type: "message", role: "system",
      content: `[当前活跃目标：]\n${goalsText}`,
    });
  }
  
  // 3. 恢复最近文件（已有）

  return restored;
}
```

### 8. 工具输入 Zod Schema 验证

**现状**: 模型传错参数时直接抛异常，错误信息不友好。  
**方案**: 在 session-tool-executor.ts 入口加 Zod 验证层。

```ts
// session-tool-executor.ts
import { z } from "zod";

function validateToolInput(toolName: string, input: unknown, schema: z.ZodSchema): 
  { valid: true; data: Record<string, unknown> } | { valid: false; error: string } {
  const result = schema.safeParse(input);
  if (result.success) return { valid: true, data: result.data };
  
  const issues = result.error.issues.map(i => `${i.path.join(".")}: ${i.message}`).join("; ");
  return { valid: false, error: `参数错误: ${issues}` };
}

// 执行工具前：
const validation = validateToolInput(toolName, input, getToolSchema(toolName));
if (!validation.valid) {
  return { ok: false, error: "validation-error", summary: validation.error };
}
```

**实施文件**: `packages/studio/src/api/lib/session-tool-executor.ts` + `packages/novel-plugin/src/tool-schemas.ts`（添加 Zod schema）

### 10. Emergency Truncation 改为保留摘要

**现状**: `emergencyTruncateMessages` 直接丢弃 2/3 消息，只保留最近 1/3。  
**方案**: 丢弃的部分生成简短摘要（如果有最近的 compact summary 就用它）。

```ts
function emergencyTruncateMessages(messages: AgentTurnItem[], keepRecent = 10): AgentTurnItem[] {
  if (messages.length <= keepRecent + 2) return messages;
  
  const firstSystem = messages.find(m => m.type === "message" && m.role === "system");
  const actualKeep = Math.min(keepRecent, Math.floor(messages.length / 3));
  const recentMessages = messages.slice(-actualKeep);
  const droppedCount = messages.length - actualKeep - (firstSystem ? 1 : 0);
  
  // 生成丢弃部分的快速摘要（不调 LLM，纯文本提取）
  const dropped = messages.slice(firstSystem ? 1 : 0, -actualKeep);
  const quickSummary = dropped
    .filter(m => m.type === "message" && m.role === "user")
    .map(m => m.content.slice(0, 50))
    .slice(-5)
    .join(" | ");
  
  const notice: AgentTurnItem = {
    type: "message", role: "system",
    content: `[上下文溢出紧急压缩。已丢弃 ${droppedCount} 条较早消息。用户近期请求摘要: ${quickSummary || "无"}]`,
  };
  
  return firstSystem ? [firstSystem, notice, ...recentMessages] : [notice, ...recentMessages];
}
```

**实施文件**: `packages/studio/src/api/lib/agent-turn-runtime.ts`

### 11. Append System Prompt（末尾追加系统指令）

**现状**: 系统指令只在 messages[0] 注入。  
**Claude Code 做法**: 支持在消息列表末尾追加系统级指令（更高优先级，因为 recency bias）。  
**用途**: compact 后恢复行为约束、每轮 turn 动态注入当前目标等。

**方案**:

```ts
// agent-turn-runtime.ts generate 调用前
function injectAppendSystemPrompt(messages: AgentTurnItem[], appendPrompt?: string): AgentTurnItem[] {
  if (!appendPrompt?.trim()) return messages;
  return [
    ...messages,
    { type: "message", role: "system", content: appendPrompt },
  ];
}

// 使用：
const appendPrompt = buildAppendPrompt(session);  // goals + todos + 当前状态
const finalMessages = injectAppendSystemPrompt(messages, appendPrompt);
```

**实施文件**: `packages/studio/src/api/lib/agent-turn-runtime.ts` + `packages/studio/src/api/lib/system-prompt-builder.ts`

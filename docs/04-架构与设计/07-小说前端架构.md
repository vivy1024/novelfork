# 小说前端架构

NovelFork Studio 的写作工作台（Writing Workbench）是一个 Agent-native 的本地 Web 工作台，采用左资源栏 / 中间画布 / 右侧叙述者会话的三栏布局。前端基于 React 19 + TypeScript + Tailwind CSS + shadcn/ui 构建，通过 REST API 和 WebSocket 与后端通信。

## 数据流

### 资源加载

```
后端 /api/books/:bookId/resources
  → loadResourceTreeFromContract()
    → buildWorkbenchResourceTree()
      → useWorkbenchResources() hook
        → { tree, resourceMap, openableNodes, errors }
```

### 资源详情按需加载

```
用户点击资源节点
  → ResourceDetailLoader.resourceNeedsDetailHydration(node)
  → 若需要: fetch /api/books/:bookId/resources/:id/detail
  → 更新 node.content + node.metadata
```

### 保存流程

```
用户编辑 → setContent() → setDirty(true)
  → 点击保存 → onSave(node, content)
    → ResourceSaveController
      → PATCH /api/books/:bookId/resources/:id
      → setDirty(false)
```

### Tab 补全 (续写)

```
用户按 Tab → onTabComplete(currentContent, cursorPosition)
  → POST /api/books/:bookId/inline-write
    body: { mode: "continuation", context: last500chars, maxTokens: 80 }
  → 返回 { text } → 插入光标位置
```

## 状态管理

NovelFork 前端采用 **局部状态 + props 下钻** 模式，不使用全局状态管理库：

| 层级 | 方式 | 说明 |
|------|------|------|
| 路由级 | `useState` in `WritingWorkbenchRoute` | viewMode, showToolsPanel, showCheckpoints, showJingwei |
| 画布级 | `useState` in `WorkbenchCanvas` | content, dirty, saving, saveError |
| 面板级 | `useState` in 各 Panel | 各面板独立的 loading/result/error |
| 跨组件 | props + callbacks | onSave, onOpen, onCanvasContextChange |
| 后端同步 | `useEffect` + fetch | 按需加载，无 SWR/React Query |

### 画布上下文广播

`WorkbenchCanvas` 通过 `onCanvasContextChange` 回调向上层广播当前画布状态：

```typescript
interface WorkbenchCanvasContext {
  activeResourceId: string | null;
  activeKind: WorkbenchResourceKind | null;
  activeTabId?: string;
  activeResource?: WorkspaceResourceRef;
  openTabs: OpenResourceTab[];
  dirty: boolean;
  contentPreview: string; // 前 500 字符
}
```

此上下文被注入到叙述者会话的 system prompt 中，使 Agent 感知用户当前正在编辑的资源。

## 关键 Hooks

| Hook | 文件 | 职责 |
|------|------|------|
| `useWorkbenchResources` | `useWorkbenchResources.ts` | 将后端 ContractResourceNode 转换为前端 WorkbenchResourceNode 树，计算 capabilities |
| `useJingweiEntries` | `jingwei/hooks/useJingweiEntries.ts` | 加载经纬条目列表，支持按分类过滤 |
| `useJingweiProgressions` | `jingwei/hooks/useJingweiProgressions.ts` | 加载经纬进度数据 |

### useWorkbenchResources 详解

```typescript
// 输入: 后端返回的 ContractResourceNode[]
// 输出: { tree, resourceMap, openableNodes, errors }
export function useWorkbenchResources(nodes: readonly ContractResourceNode[]) {
  return useMemo(() => {
    const tree = buildWorkbenchResourceTree(nodes);
    const resourceMap = flattenWorkbenchResourceTree(tree);
    const openableNodes = Array.from(resourceMap.values())
      .filter(node => node.capabilities.open);
    return { tree, resourceMap, openableNodes, errors: [] };
  }, [nodes]);
}
```

核心转换逻辑：
1. `ContractResourceCapabilities` → `WorkbenchResourceCapabilities`（open/readonly/edit/delete/apply）
2. 递归映射子节点
3. 扁平化为 Map 供快速查找

## 经纬子系统前端

`jingwei/` 子目录包含经纬资料管理的完整前端：

```
jingwei/
├── category-schemas.ts    — 分类字段定义（角色/地点/物品/势力/事件等）
├── genre-templates.ts     — 题材模板（玄幻/都市/科幻等预设分类）
├── JingweiCategorySidebar.tsx — 左侧分类导航
├── JingweiEntryForm.tsx   — 条目编辑表单（动态字段）
├── JingweiEntryList.tsx   — 条目列表视图
├── JingweiEntryTree.tsx   — 条目树视图（支持嵌套）
├── JingweiGraphView.tsx   — 关系图可视化
├── JingweiPanel.tsx       — 顶层面板容器
├── JingweiProgressions.tsx — 进度追踪视图
└── hooks/
    ├── useJingweiEntries.ts      — 条目数据加载
    └── useJingweiProgressions.ts — 进度数据加载
```

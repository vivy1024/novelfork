**版本**: v1.0.0

## 概述

驾驶舱（Cockpit）是写作工作台的默认视图——当用户选中一本书但未选中具体资源时显示。它回答一个核心问题：**下一章该怎么写？**

通过聚合作品进度、未关闭伏笔、候选稿、风险提示等信息，帮助作者快速了解当前状态并决定下一步。

## CockpitOverview 组件

在写作工作台中，当 WorkbenchCanvas 没有选中具体资源时，渲染 CockpitOverview：

```
WorkbenchCanvas
├── [无选中资源] → CockpitOverview
│   ├── 作品信息卡片（标题/状态/题材）
│   ├── 章节进度条
│   ├── 字数统计（总字数/均字数/今日字数）
│   ├── 审校状态（通过数/失败数）
│   ├── 风险卡片列表
│   ├── 未关闭伏笔摘要
│   ├── 最近候选稿
│   └── 下一步建议
└── [有选中资源] → ResourceViewer / JingweiEntryEditor / ...
```

## 数据模型

驾驶舱快照的完整类型定义：

```typescript
export interface CockpitSnapshot {
  readonly status: CockpitDataStatus;
  readonly book: CockpitBookSummary | null;
  readonly generatedAt: string;
  readonly progress: CockpitProgressSummary;
  readonly currentFocus: CockpitCurrentFocusSummary;
  readonly recentChapterSummaries: CockpitListResult<CockpitChapterSummaryItem>;
  readonly openHooks: CockpitListResult<CockpitHookItem>;
  readonly recentCandidates: CockpitListResult<CockpitCandidateItem>;
  readonly riskCards: CockpitListResult<CockpitRiskCard>;
  readonly modelStatus?: CockpitModelStatus;
}

export type CockpitDataStatus = "available" | "empty" | "missing" | "unsupported";
```

### 进度摘要

```typescript
export interface CockpitProgressSummary {
  readonly status: CockpitDataStatus;
  readonly chapterCount: number;
  readonly targetChapters: number | null;
  readonly totalWords: number;
  readonly approvedChapters: number;
  readonly failedChapters: number;
  readonly todayWords: number;
  readonly dailyTarget: number;
  readonly streak: number;
  readonly weeklyWords: number;
}
```

### 伏笔项

```typescript
export interface CockpitHookItem {
  readonly id: string;
  readonly text: string;
  readonly sourceChapter: number;
  readonly status: "open" | "payoff-due" | "expired-risk" | "resolved" | "frozen";
  readonly sourceFile: string;
  readonly sourceKind: "pending-hooks";
}
```

伏笔风险计算逻辑：
- `open`：正常未回收
- `payoff-due`：距离埋设超过阈值 70%（默认 10.5 章），提醒即将到期
- `expired-risk`：距离埋设超过阈值（默认 15 章），高风险

### 风险卡片

```typescript
export interface CockpitRiskCard {
  readonly id: string;
  readonly kind: "audit-failure" | "expired-hook" | "tone-drift";
  readonly title: string;
  readonly detail: string;
  readonly chapterNumber?: number;
  readonly navigateTo: string;
  readonly level: "warning" | "danger";
}
```

风险来源：
- 审计失败的章节 → `audit-failure`（danger）
- 长期未回收的伏笔 → `expired-hook`（warning）
- 文风漂移检测 → `tone-drift`（warning）

### 候选稿项

```typescript
export interface CockpitCandidateItem {
  readonly id: string;
  readonly bookId: string;
  readonly title: string;
  readonly source: string;
  readonly status: string;
  readonly createdAt: string;
  readonly updatedAt: string;
  readonly artifact: {
    readonly id: string;
    readonly kind: "candidate";
    readonly title: string;
    readonly resourceRef: { kind: "candidate"; id: string; bookId: string; title: string };
    readonly renderer: "candidate.created";

}
```

候选稿项包含 `artifact` 字段，前端可直接用它在画布中打开候选稿。

## 驾驶舱 API

### GET /api/books/:id/cockpit/snapshot

返回完整驾驶舱快照。

请求参数：
- `includeModelStatus`（可选）：是否包含模型可用性状态

响应：`CockpitSnapshot`

### GET /api/books/:id/cockpit/open-hooks

返回未关闭伏笔列表。

请求参数：
- `limit`（可选）：返回数量上限，默认 10

响应：`CockpitListResult<CockpitHookItem>`

### GET /api/books/:id/cockpit/recent-candidates

返回最近候选稿列表。

请求参数：
- `limit`（可选）：返回数量上限，默认 10

响应：`CockpitListResult<CockpitCandidateItem>`

## CockpitService 实现

位于 `packages/studio/src/api/lib/cockpit-service.ts`：

```typescript
export class CockpitService {
  constructor(options: CockpitServiceOptions);

  async getSnapshot(input: { bookId: string; includeModelStatus?: boolean }): Promise<CockpitSnapshot>;
  async listOpenHooks(input: { bookId: string; limit?: number }): Promise<CockpitListResult<CockpitHookItem>>;
  async listRecentCandidates(input: { bookId: string; limit?: number }): Promise<CockpitListResult<CockpitCandidateItem>>;
}
```

数据来源：
- 作品信息 → `StateManager.loadBookConfig()`
- 章节列表 → `StateManager.loadChapterIndex()`
- 当前焦点 → 读取 `story/current_focus.md`
- 章节摘要 → 读取 `story/chapter_summaries.md`
- 伏笔 → 解析 `story/pending_hooks.md`（`- [ ]` 格式）
- 候选稿 → 读取 `generated-candidates/index.json`
- 模型状态 → `ProviderRuntimeStore` + `buildRuntimeModelPool()`

## 设计原则

1. **默认视图**：驾驶舱是写作工作台的"首页"，用户打开一本书时第一眼看到的就是驾驶舱，无需额外操作。

2. **回答"下一步"**：不只是展示数据，而是通过风险卡片和建议告诉作者应该做什么。

3. **零配置**：从已有的真相文件和章节数据自动聚合，不需要作者额外维护驾驶舱数据。

4. **渐进式**：每个数据块都有 `CockpitDataStatus`，缺失数据显示原因而非报错。新建的空作品也能正常展示驾驶舱（只是各区域显示"暂无"）。

5. **可嵌入对话**：通过工具结果卡片，驾驶舱数据可以在叙述者对话中自然呈现，AI 据此给出写作建议。

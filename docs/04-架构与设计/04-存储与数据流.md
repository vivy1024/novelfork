## 当前定位

存储层基于 SQLite（`~/.novelfork/novelfork.db`），通过 `bun:sqlite` + `drizzle-orm` 驱动，包含 15 个 migration。数据流通过 Studio API 暴露的数据契约对外提供服务。经纬系统使用 `story_jingwei_entry` / `story_jingwei_section` 表持久化（不再使用 md 文件）。请求日志通过 `request_log` 表持久化。

## 数据流原则

- UI 不直接绕过 API 写业务数据。
- 当前进程临时状态必须明确标注为 `process-memory`，不保证重启后保留。
- 迁移、repair、schema 兼容规则必须写入开发者指南和 API 契约。

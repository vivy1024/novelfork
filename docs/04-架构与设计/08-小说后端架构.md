# 小说后端架构

**版本**: v1.0.0
**状态**: ✅ 当前有效

NovelFork 小说后端由三层构成：`packages/core/` 提供通用基础设施（存储/LLM/状态/hooks/MCP/runtime），`packages/novel-plugin/` 提供小说领域引擎（Agent 管线、经纬系统、过滤器、合规检查、预设库、路由和服务），`packages/studio/src/api/` 提供 Hono HTTP 框架层和 Agent 运行时。数据持久化使用 SQLite（bun:sqlite，~/.novelfork/novelfork.db，15 个 migration），AI 调用通过 LlmRuntimeService 统一路由到多供应商。

## 数据流：用户动作 → 存储 → AI

### 写下一章完整链路

```
用户点击"写下一章"
  │
  ▼
[1] GET /api/books/:id/cockpit/snapshot
    → 获取当前书籍状态 + 进度
  │
  ▼
[2] POST /api/books/:id/pgi/generate-questions
    → PGI 引擎分析经纬数据，生成追问
    → 用户回答追问（或跳过）
  │
  ▼
[3] POST /api/books/:id/guided/enter
    → 形成 GuidedGenerationPlan
    → 用户批准计划
  │
  ▼
[4] POST /api/books/:id/candidates/create-chapter
    → PipelineRunner 执行:
      a. buildJingweiLegacyContext() → 注入经纬上下文
      b. PlannerAgent.plan() → 章节意图
      c. ComposerAgent.compose() → 完整上下文包
      d. WriterAgent.write() → 初稿
      e. ContinuityAuditor.audit() → 连续性检查
      f. ReviserAgent.revise() → 修订（如有问题）
      g. StateValidatorAgent.validate() → 状态验证
    → 结果写入 candidates 表
  │
  ▼
[5] 用户在画布中查看候选稿
    → 接受 → 写入 chapters 表
    → 拒绝 → 标记 rejected
```

### AI 味检测链路

```
用户选择章节 → 点击"AI 味检测"
  │
  ▼
POST /api/filter/scan { text, bookId, chapterNumber, persist: true }
  │
  ▼
filter/engine/rules.ts — 12 条本地规则扫描
  ├── 句式重复检测
  ├── 过度修饰检测
  ├── 情感标签堆砌
  ├── 转折词滥用
  └── ...
  │
  ▼
filter/zhuque/client.ts — 朱雀 API 远程检测（可选）
  │
  ▼
filter/suggestions/seven-tactics.ts — 生成修复建议
  │
  ▼
filter/repositories/filter-report-repo.ts — 持久化报告
  │
  ▼
返回 FilterReport { score, issues[], suggestions[] }
```

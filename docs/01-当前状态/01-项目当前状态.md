## 当前一句话结论

NovelFork 当前主线是中文网文小说 AI 辅助创作工作台：以本地 Studio、右侧固定叙述者会话、中间多资源画布、左侧资源栏、AI 候选稿和故事经纬为核心，补齐 Agent-native 创作闭环。

更具体地说，NovelFork 现在不是一个单纯的 CLI 章节生成器，也不是 NarraFork 的叙述者中心复刻。当前产品重心已经转向“作者在一个本地工作台里管理作品、编辑章节、通过右侧叙述者调用工具链、沉淀故事资料、检查发布风险”的写作软件形态。

## 当前已真实可用的能力

### 运行与交付

- `bun run main.ts --root=. --port=4567` 可以作为开发运行入口。
- `pnpm bun:compile` 是当前编译交付入口。
- embedded assets / filesystem assets 双路径已经进入当前交付口径。
- startup recovery 最小链路已经具备，包括 runtime-state bootstrap、index rebuild 和结构化 `recoveryReport`。

### Studio 工作台

- `/next` 已使用 Agent Shell 路由壳，侧栏来自 Backend Contract domain clients，默认入口不再回退旧三栏 WorkspacePage。
- `/next/narrators/:sessionId` 已接入 live Conversation runtime、模型池/权限/推理状态栏、工具结果 renderer 与 session tool 确认刷新。
- `/next/books/:bookId` 可以通过 resource contract 读取章节、候选稿、草稿、经纬、故事文件、真相文件和叙事线资源，并在中间画布打开、编辑、保存或按 capability 显示只读/unsupported。
- `/next/search`、`/next/routines`、`/next/settings` 已挂载真实页面或真实错误/unsupported 状态，不再使用“稍后接线”作为当前事实。
- 候选稿进入非破坏性处理路径，用户需要选择合并、替换、另存或放弃。
- 写作工具和部分健康指标已经接入真实计算或真实文件读写；驾驶舱快照、叙事线、PGI 和 GuidedGenerationPlan 通过工具结果卡片展示。

### 引导式生成与故事经纬

- 首启引导、开始使用清单和模型未配置 gate 已有 API / 组件 / 测试事实；未配置模型不阻断本地建书、整理故事经纬和章节编辑。
- 作品创建路径支持经纬模板选择，基础模板生成人物、事件、设定、章节摘要；增强模板追加伏笔、名场面、核心记忆。
- 故事经纬结构化栏目、条目、模板应用和上下文预览 API 已落地；旧兼容数据仍可作为内部技术兼容层读取。
- 问卷中心、生成前追问、核心设定变更历史和 AI 味报告属于已实现 spec 功能，但需要区分“API/组件已落地”和“当前工作台入口已完整串起”。

### 模型运行时

- Provider runtime store 是当前供应商和模型池的主线。
- Codex / Kiro 平台账号导入已有真实持久化路径。
- Runtime model pool 从持久化 runtime store 派生。
- 凭据缺失或模型不可用时，相关路径应返回错误或 AI gate，不生成假正文。

## 当前透明过渡和规划中能力

| 能力 | 当前口径 | 不得写成 |
|---|---|---|
| 轻量 Book Chat 历史 | 当前进程临时历史，重启不保证保留 | 持久化聊天系统 |
| Pipeline runs | 当前进程临时运行态 | 完整持久化运行中心 |
| Monitor status | 无事实源时返回 unsupported | 已接真实监控 daemon |
| 写作模式生成 | 生成端点仍可返回 prompt-preview；prompt-preview 只可复制/查看，不会进入正文 | 已产出正式正文 |
| 写作模式 apply | 模型结果可在 Workspace 选择候选稿/草稿/章节目标并确认落库，正式章节 insert/replace 转候选稿 | 已能无确认改正式正文 |
| 轻量 Book Chat / Pipeline runs | `process-memory` 会话级边界明确，重启后不保证保留 | 长期持久化运行中心 |
| Monitor / Admin users | 无事实源或本地单用户阶段返回 transparent unsupported | 已接完整 daemon / 多用户系统 |
| AI complete streaming | `chunked-buffer`，不代表上游原生流式 | 原生 LLM streaming |
| 故事经纬深度工作台联动 | 结构化 API、模板、上下文、资源树分类和画布详情已接入主工作台；缺上下文时仍显示 UnsupportedCapability | 无缺口的全量资料库 |
| 完整资源管理器 | 章节、候选稿、草稿、大纲、素材、发布报告、story/truth 文件均已有落点；不是通用 IDE 文件浏览器 | 通用文件系统 IDE |

## 已实现 spec 的 current 文档补齐清单

本轮对照 `.kiro/specs/*/tasks.md`、Studio route、核心测试和 current 文档后，发现不是所有已实现能力都进入了 `01-当前状态/` 事实入口。当前需要重点补齐或已在本文重新点名的能力分为 6 类：

| 类别 | 已实现事实 | 原 current 覆盖问题 | 当前写法 |
|---|---|---|---|
| 引导式生成 | 问卷中心、生成前追问、变更历史、AI 味报告 | 用户/产品 current 文档几乎未解释 | 写成“API/组件/测试已落地，工作台入口按矩阵判断” |
| 首启与模型 gate | 首启弹窗、开始使用清单、AI 动作 gate | 设置文档只提模型，未形成工作台流程 | 写成本地优先、AI 可选增强 |
| 故事经纬结构化能力 | 栏目/条目 API、模板应用、上下文装配 | 之前只写“完整编辑仍待补齐”，没有区分 API 与工作台联动 | 写成结构化底座 current、深度联动 transparent |
| AI 味过滤 | 过滤 API、朱雀配置、报告、改写建议 | 用户流程只轻描淡写“去 AI 味” | 写成生成后审校能力，不自动覆盖正文 |
| 预设与套路资产 | presets / bundles / beats API 和书籍预设绑定 | 只在参考资料里出现 | 写成可用资产，避免冒充模板市场终态 |
| 发布合规 | 敏感词、AI 比例、格式、发布就绪、声明生成 API | 当前只写成待迁移入口 | 写成核心 API/组件可复用，新工作台入口待整合 |

## 当前不再作为主线的内容

- 不再以“回归 NarraFork 路线”作为产品主线。
- NarraFork 只作为设置页、套路页、管理型布局等参考资料，不作为小说创作主流程事实来源。
- 旧 PWA / v0.0.1 发布叙事已归档。
- 旧快速开始中的大量 CLI 自动写作示例已归档，不作为当前用户首选入口。
- 旧测试报告、旧迁移方案、旧路线图进入 `99-历史归档/`，只供追溯。

## 当前产品命名

用户侧统一使用“故事经纬 / 经纬”。技术兼容层、数据库表、旧代码符号和历史引用中仍可能出现 legacy 命名，但它们不代表新的用户文案。

## 当前完成标准

后续任何文档或功能如果要写成 current，至少需要满足其中一种证据：

1. 有代码路径和 API 路由能支撑该能力。
2. 有测试覆盖或浏览器审计证明该路径可走通。
3. 有明确运行时状态和失败反馈，不靠 mock、fake、noop 假成功。
4. 如果能力未接入，文档必须写成 planning、透明过渡或 unsupported。

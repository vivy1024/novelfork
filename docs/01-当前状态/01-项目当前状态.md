## 当前已真实可用的能力

### 运行与交付

- `bun run main.ts --root=. --port=4567` 开发运行入口。
- `bun run bun:compile` 编译交付入口；产物为 `dist/novelfork-v1.0.0-windows-x64.exe`。
- v1.0.0 为当前最新版本。
- embedded assets / filesystem assets 双路径已进入交付口径。
- startup recovery 最小链路已具备，包括 runtime-state bootstrap、index rebuild 和结构化 `recoveryReport`。

### 插件化架构

- **PluginRegistry 机制**：`plugin-loader.ts` 实现插件注册/查询/按 projectType 过滤。
- **novel-plugin 独立包**：小说写作功能从通用 Agent 工作台中彻底拆出（engine 241 文件 + routes 9 个 + handlers 10 个 + pages 53 个组件），studio 成为纯通用平台。
- **fitness-plugin 骨架**：证明架构可扩展到第二领域。
- **core 瘦身**：不再包含任何小说领域代码，仅保留 storage/llm/state/hooks/mcp/runtime。

### 经纬系统（SQLite 持久化）

- **存储迁移**：已从 md 文件迁移到 SQLite（story_jingwei_entry 表）。
- **16 分类结构化 UI**：面板 + 条目列表 + 编辑表单 CRUD。
- **react-flow 关系图谱**：5 种视图模式（关系图谱/角色弧线/矛盾地图/列表/时间线），节点 lifecycle 状态色。
- **上下文可见性系统**：三种可见性（tracked/global/nested）+ visibleAfterChapter 时间线纪律 + 自动链接引擎 + 上下文组装服务（`buildJingweiContext`）。
- **图谱可见性标识**：节点显示可见性图标，`participatesInAi=false` 半透明。
- **批量管理**：多选 checkbox + 统一设置 visibility 和 visibleAfterChapter。

### 驾驶舱（从 SQLite jingwei 读取）

- **经纬图谱工作区**：画布默认显示 react-flow 关系图谱，左侧 CategorySidebar + 右侧 GraphCanvas + 底部 ViewSwitcher。
- **底部状态条（StatusBar）**：一行显示章数/节拍/质量/AI味/警告数，点击展开对应面板。
- **可展开面板系统**：预设配置/节拍进度/质量监控/警告。
- **3 个 Agent 工具**：`presets.get_rules`、`presets.check_compliance`、`beat.get_current`。
- **对话工具配置栏（ToolConfigBar）**：对话顶部一行显示工具启用状态，toggle 同步到 session toolPolicy.deny。
- **Agent 快捷按钮组（AgentQuickActions）**：按角色显示对应操作按钮。
- **质量趋势图**：AI味趋势折线图 + 文风漂移曲线 + 质量评分趋势。

### 引导系统

- **FirstRunDialog**：首次启动弹窗引导。
- **GettingStartedChecklist**：开始使用清单，引导用户完成初始配置。
- 模型未配置 gate 不阻断本地建书、整理故事经纬和章节编辑。

### 5 Agent 写作管线

- Explorer → Planner → Writer → Auditor → Reviser 完整串行管线。
- 每种 Agent 专属 system prompt（200+ 行领域知识）。
- WorkflowProgressCard 可见执行链。
- 大纲生成流程已验证通过。

### 请求日志

- SQLite 持久化（request_log 表），记录所有 LLM 请求。

### 学习中心

- 20 篇文档，覆盖写作技巧、工具使用、最佳实践。

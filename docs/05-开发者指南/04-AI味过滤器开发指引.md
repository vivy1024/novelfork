## 范围

AI 味过滤器 v1 是 NovelFork 的强制核心层：本地 12 特征规则检测、可选朱雀 API、消 AI 味 7 招建议、写作管线检测历史与 Studio 报告 UI。

非目标：自训练模型、自动改写执行、非中文检测、图像/音频检测、晋江/起点直投接口。

## 数据模型

`packages/core/src/storage/migrations/0005_filter_v1.sql` 创建 `filter_report`：

- `book_id` / `chapter_number`：报告归属章节
- `ai_taste_score`：0-100，越高 AI 味越重
- `level`：`clean | mild | moderate | severe`
- `hit_counts_json`：各规则命中次数
- `zhuque_score` / `zhuque_status`：朱雀结果或降级状态
- `details`：完整 `FilterReport` JSON，包含 `hits`、`pgiUsed`、`crossSpecHints`
- `engine_version`：规则版本，当前 `filter-v1`

Repository：`packages/core/src/filter/repositories/filter-report-repo.ts`。

## 规则扩展

核心入口：`packages/core/src/filter/engine/index.ts`。

规则统一接口在 `packages/core/src/filter/engine/rules.ts`：

```ts
interface FilterRule {
  id: string;
  name: string;
  weight: number;
  run(text: string, ctx: RuleContext): RuleHit | null;
}
```

当前规则：

| ID | 规则 | 默认权重 |
|---|---|---:|
| r01 | 过度正式 / 官腔 | 1.0 |
| r02 | 固定句式模板 | 1.2 |
| r03 | 典型 AI 词汇 | 1.5 |
| r04 | 缺乏情感动词具体化 | 0.8 |
| r05 | 缺乏口头禅 / 人称癖好 | 0.6 |
| r06 | 句长方差过低 | 0.7 |
| r07 | 段落长度过于均匀 | 0.6 |
| r08 | 行话 / 术语密度过高 | 0.5 |
| r09 | 形容词堆叠 | 0.9 |
| r10 | 空话密度 | 0.8 |
| r11 | 对话书面语 | 1.1 |
| r12 | 伪人感综合 | 派生 |

新增规则时要求：

1. 使用稳定 `ruleId`，不要复用旧 ID。
2. 返回 `{ ruleId, severity, spans, suggestion, weightContribution }`。
3. 在 `filter-rules-engine.test.ts` 加正反例，至少覆盖命中与误报边界。
4. 若规则依赖词典，放入 `engine/dictionaries.ts` 并使用 `loadFilterDictionary()` 缓存。

## 朱雀 API

客户端：`packages/core/src/filter/zhuque/client.ts`。

配置存储键：`kv_store.settings:zhuque`，JSON 示例：

```json
{
  "apiKey": "<runtime-only>",
  "endpoint": "https://example.com/zhuque/scan",
  "timeoutMs": 10000,
  "retries": 2
}
```

注意：API Key 不入仓库。朱雀失败时返回 `{ status: "failed", error }`，本地规则检测继续可用。

## 写作管线与 Bible/PGI 接口

`scanChapterAndStoreFilterReport()` 会：

1. 读取 `bible_chapter_summary.metadataJson.pgi_answers` 判断 `pgiUsed`。
2. 运行 `runFilter()`。
3. 写入 `filter_report`。
4. 回写 `bible_chapter_summary.metadataJson.filterReportId`。

这形成 Filter 与 Novel Bible 的 1:1 跨 spec 关联。

## Studio API

- `POST /api/filter/scan`：临时扫描；传 `persist + bookId + chapterNumber` 时入库
- `GET /api/books/:bookId/filter/report?groupByPgi=true`：全书趋势 + PGI 分组
- `GET /api/books/:bookId/filter/report/:chapter`：单章详情
- `POST /api/filter/suggest-rewrite`：按 ruleIds 返回 7 招建议
- `PUT /api/settings/zhuque`：保存朱雀运行时配置
- `POST /api/books/:bookId/filter/batch-rescan`：按章节 summary 批量重扫

## UI

组件在 `packages/studio/src/components/filter/`：

- `AiTasteBadge.tsx`：clean 绿色、mild 黄色、moderate 橙色、severe 红色
- `FilterReportTab.tsx`：全书平均分、PGI 分组、章节列表
- `ChapterDetailPanel.tsx`：单章规则命中与建议
- `SevenTacticsDrawer.tsx`：7 招建议入口，不自动覆盖原文

Book Detail / Bible 页面提供「AI 味报告」Tab。

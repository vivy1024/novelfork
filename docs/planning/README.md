# 规划文档

本目录只保留 **当前有效的迁移规划**。

旧的“v0.0.1 发布准备 / PWA IDE 成熟化”路线已不再作为执行依据。
当前规划只服务于一件事：

> **把 NovelFork 从偏航的 Web/monorepo 外壳，拉回 NarraFork 风格的本地单体应用路线。**

---

## 当前有效文档

### 1. [version-plan.md](./version-plan.md)
定义平台回正的阶段计划：
- Phase 0：口径统一与 docs 纠偏
- Phase 1：运行时回正
- Phase 2：构建与分发回正
- Phase 3：数据层与基础设施评估
- Phase 4：UI 与写作能力融合

### 2. [feature-backlog.md](./feature-backlog.md)
记录平台迁移 backlog，只保留真正影响回正的任务。

### 3. [migration-plan.md](./migration-plan.md)
具体的代码迁移方案：
- 当前启动链路分析
- 目标启动链路设计
- 代码资产盘点（可复用 / 可迁移 / 需替换）
- 5 个 Phase 的具体操作步骤
- 风险与缓解
- 代码量估算（总计 ~580 行改动，7-13h）

### 4. [未实现功能路线图.md](./未实现功能路线图.md)
说明旧路线图为什么失效，以及现在该参考哪里。

---

## 当前阶段目标

当前优先级不是继续堆 UI 功能，也不是准备假发布，而是：

1. 统一文档口径
2. 明确目标平台
3. 把运行时 / 构建 / 分发从旧路径上拉回来
4. 之后再决定具体代码迁移顺序

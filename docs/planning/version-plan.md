## 发布流程

### 1. 开发阶段

1. 功能开发（按优先级）
2. 代码审查（code-reviewer agent）
3. 单元测试（tdd-guide agent）
4. 集成测试

### 2. 测试阶段

1. E2E 测试（e2e-runner agent）
2. 性能测试
3. 安全测试（security-reviewer agent）
4. 用户验收测试（UAT）

### 3. 发布准备

1. 更新 CHANGELOG.md
2. 更新 VERSION_HISTORY.md
3. 创建 Release Notes
4. 更新文档

### 4. 发布

1. 创建 Git 标签
2. 推送到远程
3. 创建 GitHub Release
4. 发布公告
5. 社区推广

### 5. 发布后

1. 监控错误日志
2. 收集用户反馈
3. 修复紧急 bug
4. 规划下一版本

## 版本号规范

遵循 [Semantic Versioning 2.0.0](https://semver.org/)：

- **MAJOR**: 不兼容的 API 变更
- **MINOR**: 向后兼容的功能新增
- **PATCH**: 向后兼容的 bug 修复

示例：
- `v0.0.1` → `v0.0.2`: Bug 修复
- `v0.0.2` → `v0.1.0`: 新增功能
- `v0.9.0` → `v1.0.0`: 重大变更

# AI 代理配置

> 控制 Agent 权限、重试、上下文裁剪与安全防护的运行时参数

## 核心概念

**权限模式**决定 Agent 自主程度：

| 模式 | 行为 |
|------|------|
| 全部允许 | Agent 自主执行所有操作 |
| 需要审批 | 每次工具调用前暂停等待确认 |
| 只读 | 只能读取，不能写入或执行 |

**每条消息最大轮次**（默认 200）限制单条消息的工具调用循环次数，防止死循环。

**计划与审批**控制 Agent 是否先提方案再执行：
- 默认计划模式：新会话自动启用，先方案后执行
- 宽松规划：计划模式下工具仍可用（需确认）
- 自动批准：AI 内部反思确认后直接执行，无需人工审批

**安全防护**：
- 危险反思：v0.4.0 增强，调用 LLM 对高风险操作生成安全评估（做什么、可能出什么问题、是否可逆），超时 8 秒自动跳过
- 跳过只读确认：已确认只读的操作不再触发安全暂停

**重试与超时**：
- 最大重试次数：临时性错误（429/500/502）自动重试，默认 5 次
- 退避上限：指数退避最大等待秒数，默认 30 秒
- 首 token 超时：API 无响应超过此秒数则中断重试，0 为禁用
- 沉默工具调用阈值：连续 N 次工具调用无可见输出时要求 Agent 汇报进度，默认 25

**上下文窗口阈值**：
- 裁剪百分比：上下文占用达此比例时渐进删除旧消息
- 压缩百分比：达此比例时触发 AI 摘要压缩
- 保留轮数：压缩后保留最近 N 轮对话，默认 4

**全局白/黑名单**（v0.4.0 实际执行）：
- 目录白名单：Agent 可自由读写的目录
- 目录黑名单：禁止访问的目录，优先级高于白名单
- 命令白名单：自动放行的命令，支持通配符
- 命令黑名单：自动拒绝的命令，可附拒绝提示词

**自定义可重试错误规则**：按 HTTP 状态码或内容关键词匹配，命中则自动重试而非报错。

## 推荐使用流程

1. 首次使用设为「需要审批」，观察 Agent 行为模式
2. 熟悉后切换「全部允许」+ 开启「危险反思」，兼顾效率与安全
3. 重要创作决策（大纲定稿、世界观修改）临时切换计划模式
4. 根据使用历史中的错误率调整重试次数和超时参数

## 最佳实践

- 白名单只添加项目目录，不要添加系统目录
- 上下文阈值保持默认，除非频繁遇到"上下文过长"错误
- 开启「显示 Token 用量」配合使用历史监控成本
- 命令黑名单加入 `rm -rf`、`format` 等危险命令

## 常见坑

- 设为「全部允许」但不开「危险反思」→ Agent 可能误删文件，无二次确认
- 最大轮次设太低（<50）→ 复杂写作管线执行到一半被截断
- 首 token 超时设太短 → 大模型思考时间长被误判为卡死
- 目录黑名单漏配 `.env` 目录 → Agent 可能读取密钥

## Agent 查阅提示

- 权限模式决定你能做什么：全部允许=自主执行，需要审批=每步等确认，只读=只能读
- 沉默阈值触发时输出一句进度说明即可
- 遇到可重试错误自动重试，无需通知用户
- 上下文压缩触发时保留最近对话，旧内容变为摘要

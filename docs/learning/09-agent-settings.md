# AI 代理配置

> 控制 AI Agent 的运行时行为、安全策略和资源限制。所有设置在「设置 → AI 代理」中管理。

---

## 基础设置

### 默认权限模式

决定 AI 执行工具调用时的自主程度：

| 模式 | 行为 | 适用场景 |
|------|------|----------|
| 全部允许 | AI 可自主执行所有操作 | 信任 AI、追求效率 |
| 需要审批 | 每次工具调用前暂停等待确认 | 谨慎模式、学习阶段 |
| 只读 | AI 只能读取，不能写入或执行 | 纯咨询、代码审查 |

### 每条消息最大轮次

限制 AI 对单条用户消息的最大工具调用循环次数（默认 200）。防止 Agent 写作管线陷入死循环。

- 日常写作：100-200 足够
- 复杂管线（规划→编排→写作→审计）：可设 300+
- 设为 -1 表示无限制（不推荐）

### 翻译思考内容

开启后，AI 的推理/思考块会通过摘要模型翻译为中文。适合使用英文模型但希望看到中文推理过程的场景。

### 旧编码支持

自动检测并保留非 UTF-8 文件编码（GBK、Shift_JIS 等）。如果你的小说 TXT 文件是 GBK 编码，开启此选项避免乱码。

### Dump API 请求

开启后，每次 AI 调用的完整请求/响应数据会保存到请求历史中，可在「使用历史」页面查看完整 payload。用于调试 AI 行为异常。

## 计划与审批

### 默认进入计划模式

新建会话时自动启用计划模式。计划模式下 AI 必须先提出方案、获得批准后才能执行。适合重要创作决策。

### 默认宽松规划

计划模式下工具仍然可用（只是需要确认），而非完全禁用。推荐开启，让 AI 在规划时也能查阅资料。

### 全局默认自动批准计划

AI 的计划经过内部反思确认后自动执行，无需人工审批。适合信任 AI 判断力的场景。

## 安全防护

### 危险反思

在「全部允许」模式下，AI 对高风险操作（删除文件、修改配置等）会进行二次反思确认。即使你给了全部权限，AI 也会自我审查危险操作。

**v0.4.0 增强**：开启后，系统会调用摘要模型对高风险操作进行安全评估，生成 1-3 句评估文本（做什么、可能出什么问题、是否可逆），附在确认卡片中供你参考。评估超时 8 秒自动跳过。

### 跳过只读确认

已确认为只读的操作（如读取文件、搜索）不再触发安全暂停。减少不必要的确认弹窗。

## 重试与超时

### 可恢复错误最大重试次数

遇到临时性 API 错误（限流 429、服务器 500/502）时的自动重试次数。默认 5 次。设为 -1 表示无限重试。

### 沉默工具调用阈值

AI 连续执行多少次工具调用但没有输出可见文本时，系统会要求它简短说明当前进度。默认 25 次。防止 AI "闷头干活"不汇报。

### 重试退避时间上限

指数退避的最大等待时间（秒）。重试间隔不会超过此值。默认 30 秒。

### 首 token 超时

API 请求发起后，若在此秒数内没有收到任何实质响应（文本、工具调用等），则中断并重试。0 表示禁用。适合检测 API 卡死。

## 上下文窗口阈值

控制长对话时的自动裁剪和压缩行为。分两档配置：

### 标准窗口 (≤600k tokens)

- **开始裁剪**：上下文占用达到此百分比时，开始渐进式删除旧消息
- **开始压缩**：达到此百分比时，触发 AI 摘要压缩（将旧对话压缩为摘要）

### 大窗口 (>600k tokens)

同上，但阈值可以设得更低（因为窗口大，有更多缓冲空间）。

### 自动压缩保留轮数

压缩后保留最近 N 轮 user/assistant 对话（不被压缩）。默认 4 轮。

### 最大裁剪比例

已裁剪消息占总消息的比例达到此值时，强制启动后台压缩。默认 80%。

## 会话行为

### 发送方式

- **Enter 发送**：按 Enter 直接发送消息（Shift+Enter 换行）
- **Ctrl+Enter 发送**：按 Ctrl+Enter 发送（Enter 换行）

### 会话滚动自动加载

滚动到对话顶部时自动加载更早的历史消息。

### 要求使用用户语言回复

强制 AI 使用与界面相同的语言回复。确保中文界面下 AI 用中文写作。

## 调试

### 显示 Token 用量

每轮对话结束后在消息底部显示输入/输出 Token 数量。用于监控写作成本。

### 显示实时输出速率

在标题栏显示实时字符/秒指示器。用于判断 AI 响应速度是否正常。

---

## 全局白/黑名单

### 目录白名单

对所有会话自动放行的目录。AI 可以自由读写这些目录下的文件。

### 目录黑名单

对所有会话禁止访问的目录。优先级高于白名单。用于保护敏感目录（如 `.env`、密钥目录）。

### 命令白名单

所有会话自动放行的命令。支持通配符（如 `npm*`、`git*`）。

### 命令黑名单

所有会话自动拒绝的命令。支持可选的拒绝提示词，告诉 AI 为什么不能用这个命令。

---

## 自定义可重试错误规则

定义自定义规则，将特定 API 错误标记为可重试。匹配方式：

- **HTTP 状态码**：如 502、429
- **内容关键词**：如 "upstream_error"、"ThrottlingException"

匹配的错误会自动重试，而非直接报错给用户。

---

## 最佳实践

- 日常写作用「全部允许」+ 开启「危险反思」，兼顾效率和安全
- 重要创作决策（如大纲定稿）用「计划模式」
- 上下文窗口阈值保持默认即可，除非遇到"上下文过长"错误
- 白名单只添加你的项目目录，不要添加系统目录
- 开启「显示 Token 用量」监控成本，发现异常及时调整

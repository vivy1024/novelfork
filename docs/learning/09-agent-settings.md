# Agent 设置

> 模型默认值、子代理模型池、推理强度、运行时控制参数。

## 核心概念

**默认模型**：所有叙述者会话使用的主模型。建议选能力最强的模型。

**摘要模型**：对话压缩时使用的模型。可选较便宜的快速模型。

**子代理模型池**：子代理（explore/plan/general/fork）可使用的模型列表。系统按任务类型自动选择。

**推理强度**：控制 AI 思考深度。高推理强度适合复杂规划，低推理强度适合简单续写。

**最大步数**：单次请求中 Agent 最多执行的工具调用步数。防止 Agent 无限循环。

**首 token 超时**：API 请求超时后自动中断（0=禁用）。

## 推荐使用流程

1. 设置 → AI 代理 → 配置默认会话模型
2. 配置摘要模型（推荐用快速便宜的模型）
3. 配置子代理模型池（至少包含一个可用模型）
4. 根据写作需求调整推理强度
5. 最大步数保持默认（通常 25-50），除非有特殊需求

## 最佳实践

- 写书 Agent 用最强模型，审校/伏笔 Agent 可用较快模型
- 推理强度：规划大纲用高，日常续写用中，简单修改用低
- 首 token 超时建议设 30-60 秒，避免 API 卡死时无限等待
- 子代理模型池至少配 2 个模型，一个主力一个备用

## 常见坑

- **子代理无响应** → 子代理模型池为空或模型不可用
- **Agent 执行太慢** → 推理强度过高或模型响应慢，降低强度或换模型
- **Agent 无限循环** → 最大步数设太高，降低到 25-30

## Agent 查阅提示

- 运行时控制通过 `/api/settings/runtime` 读写
- 推理强度映射到模型的 temperature / top_p 参数
- 最大步数限制 Agent 单次请求的工具调用次数
- 子代理继承父代理的供应商配置，但可独立选择模型
- 自定义重试规则：按 HTTP 状态码或内容关键词匹配可重试错误

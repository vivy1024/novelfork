# 写作分析工具

> AI 味检测、章节健康度、文风漂移、角色弧线、合规导出、快照回滚。

## 核心概念

工具分两类：
- **章节级**：AI 味检测、段落节奏、对话比例——作用于单章
- **全书级**：健康度仪表盘、文风漂移、角色弧线、连续性审计——作用于整本书

| 工具 | 作用 |
|------|------|
| AI 味检测 | 12 规则本地扫描，标注 AI 痕迹段落 |
| 章节健康度 | 节奏/对话/句长直方图 |
| 文风漂移检测 | 对比各章与目标文风的偏离 |
| 角色弧线追踪 | 检查角色成长是否连贯 |
| 连续性审计 | 检测剧情矛盾、人设冲突、时间线错误 |
| 平台合规检查 | 敏感词扫描 |
| 导出 | TXT / Word / ePub |
| Checkpoint | 资源快照与回滚 |

## 推荐使用流程

1. 写完一章 → 跑 AI 味检测，修掉高分段落
2. 用段落节奏分析检查句长分布，修正节奏单调区域
3. 每 10 章跑一次全书健康度和文风漂移检测
4. 中后期用角色弧线检查角色成长连贯性
5. 定稿后跑平台合规 → 通过后导出

## 最佳实践

- AI 味检测在初稿完成后立即跑，越早修正越省力
- 文风检测前先在经纬中声明目标文风，否则无基准可比
- 合规检查在每次导出前必跑
- 大改前打 Checkpoint 快照，改坏了一键回滚

## 常见坑

- **AI 味检测结果为空** → 章节内容不足 500 字，本地规则需要最低文本量
- **文风检测不准** → 未在经纬中设置目标文风描述
- **导出格式异常** → 章节标题格式不统一，先跑合规检查

## Agent 查阅提示

- AI 味检测组件：`AiTasteReport`，12 条本地规则优先执行
- 健康度组件：`ChapterHealthCard`（章节级）+ `BookHealthSummary`（全书级）
- 文风漂移：`StyleDriftPanel`，对比当前章与目标文风的向量距离
- 角色弧线：`CharacterArcsPanel`，从经纬角色设定 + 章节内容提取 arc beats
- 合规导出：`CompliancePanel` 扫描 → `ExportPanel` 输出
- Word 导出：零依赖 .docx 生成，直接构建 OpenXML 包
- Checkpoint：`CheckpointPanel`，快照存储在 SQLite，回滚恢复全部资源状态

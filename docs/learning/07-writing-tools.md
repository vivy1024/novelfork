# 写作工具

> 写作工具面板提供一系列分析和生成工具，帮助你从数据角度审视作品质量。

---

## 核心概念

写作工具（WritingToolsPanel）是画布区域的辅助面板，提供 7 个独立工具。每个工具：

- 有明确的输入（作品 ID + 可选的章节号）
- 调用后端 API 获取分析结果
- 以可视化方式展示结果
- 部分工具需要指定章节（章节级分析），部分工具作用于全书

---

## 工具列表

### 1. 章末钩子生成

| 属性 | 值 |
|------|-----|
| ID | hooks |
| 端点 | `/api/books/:bookId/hooks/generate` |
| 需要章节 | 是 |
| 输出 | 3-5 个章末钩子方案 |

**功能**：为指定章节生成章末悬念/钩子方案。

**使用场景**：
- 章节写完后不知道如何收尾
- 想要增强读者继续阅读的动力
- 需要多个方案对比选择

**输出示例**：
- 方案 A：以主角发现关键线索但被打断收尾
- 方案 B：以反派视角的一句独白制造悬念
- 方案 C：以时间跳跃暗示即将发生的变故

---

### 2. 段落节奏分析

| 属性 | 值 |
|------|-----|
| ID | rhythm |
| 端点 | `/api/books/:bookId/chapters/:ch/rhythm` |
| 需要章节 | 是 |
| 输出 | 句长分布图 + 节奏多样性评分 |

**功能**：分析章节的句长分布和节奏变化。

**分析维度**：
- 句长分布（短句/中句/长句比例）
- 节奏多样性指数
- 连续同长度句子检测
- 段落长度变化

**RhythmChart 组件**：以柱状图展示句长分布，高亮节奏单调区域。

---

### 3. 对话比例分析

| 属性 | 值 |
|------|-----|
| ID | dialogue |
| 端点 | `/api/books/:bookId/chapters/:ch/dialogue` |
| 需要章节 | 是 |
| 输出 | 对话占比 + 角色对话分布 |

**功能**：计算章节中对话与叙述的比例。

**分析维度**：
- 对话总占比（百分比）
- 各角色对话量分布
- 对话密度变化（章节内的分布）
- 独白 vs 对话比例

**DialogueAnalysis 组件**：饼图展示对话/叙述比例，柱状图展示角色对话量。

---

### 4. 全书健康仪表盘

| 属性 | 值 |
|------|-----|
| ID | health |
| 端点 | `/api/books/:bookId/health` |
| 需要章节 | 否（全书级） |
| 输出 | 综合健康评分 |

**功能**：从多个维度评估全书质量。

**评估维度**：
- 人设一致性（角色行为是否符合设定）
- 伏笔回收率（已埋伏笔的回收比例）
- AI 味均值（全书 AI 痕迹的平均程度）
- 节奏健康度
- 字数达标率

**BookHealthDashboard 组件**：雷达图 + 各维度详细评分。

---

### 5. 矛盾地图

| 属性 | 值 |
|------|-----|
| ID | conflicts |
| 端点 | `/api/books/:bookId/conflicts/map` |
| 需要章节 | 否（全书级） |
| 输出 | 矛盾关系图 |

**功能**：追踪全书主要/次要矛盾的辩证转化。

**分析维度**：
- 主要矛盾识别
- 次要矛盾列表
- 矛盾转化节点（哪一章矛盾升级/解决）
- 矛盾关联关系

**ConflictMap 组件**：网络图展示矛盾之间的关系和转化路径。

---

### 6. 角色弧线

| 属性 | 值 |
|------|-----|
| ID | arcs |
| 端点 | `/api/books/:bookId/arcs` |
| 需要章节 | 否（全书级） |
| 输出 | 角色成长曲线 |

**功能**：追踪角色的成长弧线和关键转折点。

**分析维度**：
- 角色成长进度（百分比）
- Arc Beat 标记（关键转折章节）
- 角色状态变化轨迹
- 多角色弧线对比

**CharacterArcDashboard 组件**：折线图展示角色成长曲线，标记关键节点。

---

### 7. 文风检测

| 属性 | 值 |
|------|-----|
| ID | tone-check |
| 端点 | `/api/books/:bookId/chapters/:ch/tone` |
| 需要章节 | 是 |
| 输出 | 文风偏离度报告 |

**功能**：检测章节文风与声明基调的偏离程度。

**分析维度**：
- 与目标文风的匹配度
- 偏离区域标记
- 文风特征词频
- 与前几章的一致性

**ToneDriftAlert 组件**：偏离度仪表盘 + 问题段落高亮。

## 使用方式

### 从写作工具面板触发

1. 在工作台中打开 WritingToolsPanel
2. 选择要运行的工具
3. 如需章节号，选择目标章节
4. 点击运行 → 等待分析结果
5. 查看可视化报告

### 从叙述者对话触发

在对话中用自然语言请求：
- "分析第 5 章的节奏"
- "看看全书健康度"
- "给第 10 章生成章末钩子"

---

## 最佳实践

- **定期跑全书健康**：每写 10 章跑一次，及时发现问题
- **节奏分析配合修改**：发现节奏单调的段落，针对性修改
- **对话比例参考**：网文对话占比通常 40-60%，过高或过低都需注意
- **矛盾地图辅助规划**：写到中期时用矛盾地图检查是否有悬而未决的线索
- **文风检测在修订时用**：初稿不必纠结文风，修订时再检测调整

## 常见坑

| 问题 | 原因 | 解决 |
|------|------|------|
| 分析结果为空 | 章节内容太短 | 至少需要 500 字以上 |
| 健康度评分偏低 | 经纬设定不完整 | 补充角色设定后重新分析 |
| 文风检测不准 | 未设置目标文风 | 在经纬中添加文风描述 |
| 工具运行超时 | 全书章节过多 | 分批分析，或等待后端处理 |

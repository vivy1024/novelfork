# 安全与沙箱

NovelFork 提供多层安全机制，确保 AI Agent 在受控环境中运行，不会意外修改或访问你不希望被触及的文件和系统资源。

## 沙箱模式

系统提供三级沙箱隔离：

- **none**：无限制模式。Agent 可自由访问文件系统和执行命令，适合完全信任的本地开发场景。
- **basic**：环境变量隔离。Agent 运行时不会继承你的系统环境变量（如 API Key、PATH 等），防止敏感信息泄露。
- **strict**：容器隔离。尝试在独立容器中运行 Agent，提供最强隔离。如果系统不支持容器，会自动降级到 basic 模式。

## 目录白名单与黑名单

你可以精确控制 Agent 能访问哪些文件路径：

- **白名单**：只允许 Agent 读写指定目录（如项目目录）
- **黑名单**：禁止 Agent 访问特定目录（如 `.env` 所在目录、系统配置目录）

## 命令白名单与黑名单

限制 Agent 可执行的 shell 命令：

- **白名单**：只允许执行特定命令（如 `git`、`node`、`bun`）
- **黑名单**：禁止执行危险命令（如 `rm -rf`、`format`）

## 权限模式

每个操作可设置不同的权限级别：

- **read**：只读模式，Agent 只能查看文件内容
- **edit**：可编辑模式，Agent 修改文件前会请求你确认
- **allow**：自动批准模式，Agent 可直接执行操作无需确认

## 进程管理

Agent 执行的所有命令都有超时保护。如果某个命令运行时间超过设定阈值，系统会自动终止该进程，防止无限循环或资源耗尽。

# 一页理解 NovelFork

> 网文小说 AI 辅助创作工作台，5 个 Agent 协作、引导式生成、本地运行数据不出机。

## 核心概念

NovelFork 是本地运行的网文创作工作台。你掌控方向和设定，5 个 Agent（规划→编排→写作→审计→修订）负责执行。所有数据存在本机 SQLite，不上传任何内容。

三栏布局是主要工作界面：左栏资源树浏览章节/经纬/候选稿，中间画布编辑和阅读，右侧叙述者对话随时可用。叙述者是你与 Agent 管线的唯一交互入口。

引导式生成（PGI + Guided Plan）是核心写作模式。AI 不会直接输出结果——先追问你的意图，再生成计划让你批准，最后才动笔。这保证每一章都在你的掌控下。

经纬系统是 AI 的记忆。你写入的角色/世界观/规则会自动注入写作上下文，AI 写作时能"看到"这些设定。经纬越完善，生成质量越高。

## 推荐使用流程

1. 配置模型：设置页添加 API Key（OpenAI/Anthropic/自定义端点）
2. 创建作品：新书引导向导 11 题，选预设或自定义，自动生成初始经纬
3. 写第一章：点"生成下一章"→ 回答 PGI 追问 → 批准计划 → AI 生成候选稿
4. 审阅采纳：画布中查看候选稿 → 接受/拒绝/修改 → 正式章节入库

## 最佳实践

- 先填经纬再写作，经纬是控制 AI 输出质量的核心杠杆
- 不要跳过 PGI 追问，它帮 AI 精准理解你这一章要什么
- 候选稿是建议不是终稿，接受后仍可手动编辑

## 常见坑

- AI 生成与设定矛盾：经纬条目未开启"参与 AI"开关，AI 看不到你的设定
- 叙述者无响应：模型未配置或 API Key 失效，去设置页检查供应商状态
- 生成速度慢：经纬条目过多撑爆上下文，用可见性规则精准控制哪些章节能看到哪些设定

## Agent 查阅提示

- 写作请求必须经过完整管线（规划→编排→写作→审计→修订），不可跳过任何环节
- 上下文组装顺序：经纬条目 → 前文摘要 → 驾驶舱快照 → 用户 PGI 回答 → Guided Plan
- 候选稿生成后必须等待用户确认，禁止自动合并到正式章节
- 经纬条目的 participatesInAi 开关决定是否注入上下文，默认关闭

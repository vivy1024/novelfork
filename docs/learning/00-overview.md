# 一页理解 NovelFork

> 你说方向，AI 执行。所有数据存本机，不上传任何内容。

## 它是什么

NovelFork 是本地运行的网文创作工作台。你通过对话告诉 AI 想写什么，AI 帮你规划、写作、审校。你掌控方向和设定，AI 负责产出文字。

## 界面布局

三栏结构：

- **左栏**：资源树 — 书籍、章节、经纬资料、伏笔
- **中间**：画布 — 阅读和编辑章节内容
- **右侧**：叙述者对话 — 你和 AI 交流的主入口

## 5 分钟上手

1. **配置模型**：设置 → 供应商 → 添加 API Key 和地址
2. **创建作品**：左栏「新建作品」→ 填书名、选题材
3. **开始写作**：右侧对话框告诉叙述者你想写什么
4. **审阅结果**：AI 生成的候选稿出现在画布中，确认后入库

## 核心概念

| 概念 | 一句话解释 |
|------|-----------|
| 叙述者 | 右侧的 AI 对话助手，你的写作搭档 |
| 经纬 | 世界观/角色/设定的结构化存储，AI 写作时自动注入 |
| 候选稿 | AI 生成的草稿，确认后才正式入库 |
| 预设 | 写作规则（如"禁止 AI 味表达"），注入 AI 写作指令 |
| 节拍 | 故事结构模板（三幕式、网文开篇钩子等），指导情节节奏 |
| 套路 | Agent 行为规则集，定义命令/工具/权限/提示词 |

## 最佳实践

- **先聊再写**：先和叙述者讨论想法、人物、冲突，再动笔
- **经纬越全越好**：AI 能看到的设定越多，生成质量越高
- **预设让 AI 守规矩**：不想要 AI 味启用 anti-ai 预设，想要硬核逻辑启用 logic-risk 预设
- **候选稿不是终稿**：接受后仍可手动编辑

## 常见坑

| 问题 | 解决 |
|------|------|
| 叙述者无响应 | 设置页检查供应商状态，确认 API Key 有效 |
| AI 写的和设定矛盾 | 检查经纬条目是否已创建并开启 AI 参与 |
| 生成太慢 | 经纬条目过多撑爆上下文，精简不相关条目 |
| 预设没生效 | 让叙述者帮你设置："帮我选择适合这本书的预设" |

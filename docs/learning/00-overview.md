# 一页理解 NovelFork

> NovelFork 是一个网文小说 AI 辅助创作工作台，帮助作者用 Agent 驱动的方式完成从大纲到成稿的全流程。

---

## 核心概念

### NovelFork 是什么

NovelFork 是一个本地运行的 Web 工作台（TypeScript + Bun + React 19 + Hono + SQLite），专注中文网文创作场景。它不是一个"一键生成小说"的工具，而是一个**人机协作**的创作环境：

- 作者掌控方向、设定、风格
- AI 负责草稿生成、连续性审校、文风检测
- 叙述者（Narrator）作为对话界面，让你用自然语言驱动创作

### 心智模型

```
作品 (Book)
├── 章节 (Chapter)
│   ├── 候选稿 (Candidate) ← AI 生成的草稿
│   └── 正式正文 (Approved) ← 你接受后的最终版
├── 经纬 (Jingwei) ← 世界观/人物/设定的结构化存储
│   ├── 分区 (Section) ← 如"角色"、"地点"、"势力"
│   └── 条目 (Entry) ← 具体的设定卡片
└── 叙述者 (Narrator) ← 对话驱动的 AI 助手
    ├── 会话 (Session) ← 一次对话上下文
    └── 确认门 (Confirmation Gate) ← 写入正式资源前的审批
```

**关键流转**：

1. 你发起写作请求 → 叙述者生成候选稿
2. 候选稿出现在画布中 → 你审阅
3. 接受（merge/replace/draft）或拒绝 → 正式正文更新

### 经纬 = AI 上下文

经纬系统不只是笔记本。每个条目都有 `participatesInAi` 开关——开启后，AI 写作时会自动将该条目纳入上下文。这意味着：

- 你在经纬里写的角色设定，AI 写作时能"看到"
- 可见性规则控制哪些章节能看到哪些设定（避免剧透）
- 经纬是你控制 AI 输出质量的核心杠杆

## 三栏布局

```
┌─────────────┬──────────────────────┬─────────────────┐
│  左：资源树  │    中：画布区域       │  右：叙述者会话  │
│             │                      │                 │
│ · 作品信息   │ · 驾驶舱总览         │ · 对话消息流     │
│ · 章节列表   │ · 章节编辑器         │ · 确认门        │
│ · 经纬分区   │ · 候选稿对比         │ · 工具调用卡片   │
│ · 候选稿     │ · 经纬条目编辑       │ · 状态栏        │
│ · 叙事线     │ · 写作工具面板       │ · Slash 命令    │
│ · 草稿      │ · 工作流进度         │                 │
│             │                      │                 │
└─────────────┴──────────────────────┴─────────────────┘
```

- **左栏**：资源树（WorkbenchResourceTree），展示作品下所有资源的树形结构
- **中栏**：画布（WorkbenchCanvas），根据选中资源动态渲染对应的查看/编辑器
- **右栏**：叙述者对话（ConversationSurface），始终可见的 AI 交互界面

## 推荐使用流程

### 第一次使用

```
1. 配置模型
   设置 → 供应商配置 → 添加 API Key（支持 OpenAI/Anthropic/自定义兼容端点）

2. 创建作品
   仪表盘 → 新建作品 → 填写题材/平台/目标字数/章节数

3. 回答问卷（Tier 1）
   建书后自动弹出引导式设定问卷，帮助 AI 理解你的创作意图

4. 写第一章
   驾驶舱 → "生成下一章" → 叙述者发起 PGI 追问 → 你回答 → AI 生成候选稿

5. 审阅与接受
   画布中查看候选稿 → 接受/拒绝/修改 → 正式正文入库
```

### 日常写作循环

```
打开工作台 → 查看驾驶舱进度 → 选择写作动作 → 回答追问 → 审阅候选稿 → 接受/修订
```

---

## 最佳实践

- **先填经纬再写作**：经纬越丰富，AI 生成质量越高
- **善用 PGI 追问**：不要跳过追问，它帮助 AI 精准理解你的意图
- **候选稿不是终稿**：把候选稿当作"初稿建议"，接受后仍可编辑
- **定期审校**：用"连续性审校"工具检查前后章节的设定一致性
- **控制上下文**：通过经纬的可见性规则，避免 AI 看到不该看的信息

## 常见坑

| 问题 | 原因 | 解决 |
|------|------|------|
| AI 生成内容与设定矛盾 | 经纬条目未开启 `participatesInAi` | 编辑条目，开启 AI 参与 |
| 生成速度慢 | 上下文过大 | 精简经纬条目，使用可见性规则 |
| 候选稿风格不对 | 缺少风格参考 | 在经纬中添加"文风"分区 |
| 叙述者无响应 | 模型未配置或 API Key 失效 | 检查设置 → 供应商配置 |

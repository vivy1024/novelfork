# 子代理系统

> 将复杂任务拆分给专门的子代理执行，主对话保持清晰。

## 核心概念

**四种内置类型**：

| 类型 | 用途 | 特点 |
|------|------|------|
| explore | 搜索和阅读代码/文档 | 只读，不修改文件 |
| plan | 规划方案和设计架构 | 输出计划，不执行实现 |
| general | 通用任务执行 | 可读写文件、运行命令 |
| fork | 继承当前上下文的分支 | 携带父对话的完整记忆 |

**后台任务**：设置 `run_in_background=true` 后，子代理在后台静默执行，不阻塞主对话。

**Await 等待**：等待后台任务完成并获取结果。支持超时设置。

**自定义子代理**：在套路页面定义专属类型，配置 systemPrompt 和工具权限。

## 推荐使用流程

1. 复杂任务 → 派 plan 代理先规划
2. 耗时操作（全书扫描）→ 派 general 代理后台执行
3. 探索不同写法 → 派 fork 代理尝试，对比后选最优
4. 信息收集 → 派 explore 代理搜索，不影响主对话

## 最佳实践

- explore 代理适合"帮我查一下前面哪章提到过这个角色"
- 后台任务适合批量检查，完成后用 Await 获取结果
- fork 代理适合"试试另一种写法"，不满意直接丢弃

## 常见坑

- **子代理无响应** → 检查子代理模型池配置是否有可用模型
- **后台任务结果丢失** → 未使用 Await 获取结果，任务完成后结果仍可查询
- **fork 代理上下文过大** → 继承了父对话全部历史，考虑先 `/compact` 再 fork

## Agent 查阅提示

- 子代理通过 `Agent` 工具派发，`Await` 工具等待结果
- `Send` 工具可向后台运行的子代理发送消息
- 子代理继承父代理的供应商配置和套路设置
- 后台任务生命周期：Detach 分离到后台 → Attach 重新接入
- MCP 工具自动继承父代理配置
- 自定义子代理在套路的 SubAgents tab 中定义

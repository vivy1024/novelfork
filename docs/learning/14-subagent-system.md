# 子代理系统

> 将复杂任务拆分给专门的子代理执行，主对话保持清晰。

## 四种内置类型

| 类型 | 用途 | 特点 |
|------|------|------|
| **explore** | 搜索和阅读代码/文档 | 只读，不修改任何文件 |
| **plan** | 规划方案和设计架构 | 输出计划文档，不执行实现 |
| **general** | 通用任务执行 | 可读写文件、运行命令 |
| **fork** | 继承当前上下文的分支 | 携带父对话的完整记忆 |

## 自定义子代理

在套路（Routine）页面可定义专属子代理类型，配置：
- **systemPrompt**：注入该代理的角色指令
- **工具权限**：限定可使用的工具集合

例如定义一个"角色扮演审核员"，只允许读取文件和搜索，专门检查角色一致性。

## 后台任务

设置 `run_in_background=true` 后，子代理在后台静默执行，不阻塞你的主对话。适合耗时操作如全书扫描、批量检查。

## Await 等待

使用 Await 工具可等待后台任务完成并获取结果。支持设置超时时间，超时后返回已有的部分结果。

## 使用场景

- 写新章节时，后台派 explore 代理检索前文伏笔
- 用 plan 代理生成大纲，审阅后再派 general 代理执行
- fork 代理尝试不同写法，对比后选择最优版本

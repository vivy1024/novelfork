# 外观与界面

> 主题、字体、编辑器、终端和动画的视觉配置

## 核心概念

**主题模式**：亮色 / 暗色 / 跟随系统三选一。暗色模式下可额外开启 OLED 纯黑（#000000 背景），适合 AMOLED 屏幕省电和沉浸写作。

**字体设置**：
- 字体大小：12-20px 滑块调节，影响全局界面文字
- 字体族：系统默认 / Segoe UI / 微软雅黑等，按偏好选择

**编辑器设置**：
- 行高：控制文本行间距
- Tab 大小：缩进宽度

**终端设置**：
- 终端字体大小：8-32px，影响 Agent 执行日志的可读性
- 终端主题：自动（跟随系统）/ 深色 / 浅色

**高级动画**：消息和卡片的 blur-in 动画效果。关闭可提升低配设备流畅度。

**自动换行**：分别控制 Markdown、代码块、Diff 的换行行为。关闭则水平滚动。

## 推荐使用流程

1. 打开「设置 → 外观与界面」tab
2. 选择主题模式，夜间写作建议暗色 + OLED 纯黑
3. 调整字体大小到舒适值（推荐 14-16px）
4. 终端字体设大一点（16-18px）方便查看 Agent 日志
5. 按需开关自动换行

## 最佳实践

- 长时间写作用暗色 + OLED 纯黑，减少眼睛疲劳
- Markdown 自动换行开启（长段落易读），代码自动换行关闭（保持格式）
- 低配设备关闭高级动画，提升滚动流畅度
- 终端字体不要太小，Agent 执行日志需要快速扫读

## 常见坑

- OLED 纯黑只在暗色模式下生效 → 设为亮色或跟随系统白天时无效果
- 字体族选了系统没安装的字体 → 回退到默认字体，显示不一致
- 关闭代码自动换行后忘记 → 长行代码需要水平滚动才能看全

## Agent 查阅提示

- 用户说"看不清"或"字太小"→ 引导到外观设置调整字体大小
- 用户说"卡顿"→ 建议关闭高级动画
- 外观设置不影响 AI 行为，纯视觉配置

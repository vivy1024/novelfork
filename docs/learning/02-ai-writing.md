# AI 写作功能

> 选段写作、多版本变体、候选稿流程、AI 味检测、写作预设、上下文组装。

## 核心概念

选段写作是核心交互模式。选中文本后可执行续写（从光标处继续）、扩写（展开细节描写）、补写（在两段之间插入过渡）。所有结果进入候选区，不会直接覆盖正文。

多版本变体让你不必押注单一结果。同一段落可生成 2-4 个不同风格的版本，并排对比后选择最满意的。变体面板（VariantsPanel）支持逐句对比和一键采纳。

候选稿是 AI 与你之间的缓冲层。AI 生成 → 进入候选区 → 你审阅 → 接受（merge/replace/draft）或拒绝。这个流程保证你对正文有绝对控制权，AI 永远不会直接改动你的正式章节。

AI 味检测双引擎工作：12 条本地规则实时扫描（重复句式、过度修饰、机械过渡等），朱雀 API 提供深度检测。检测结果标注在文本上，你可以逐条处理或忽略。

写作预设覆盖 26 种流派/文风/基底/逻辑规则组合。选择预设后，AI 的语言风格、叙事节奏、用词偏好都会相应调整。预设可叠加使用。

上下文组装决定 AI "看到"什么。系统自动按优先级注入：经纬条目 → 前文摘要 → 驾驶舱快照 → PGI 回答 → Guided Plan。你通过经纬的可见性规则控制信息边界。

## 推荐使用流程

1. 选择写作动作：续写/扩写/补写，或点"生成下一章"走完整管线
2. 回答 PGI 追问：告诉 AI 这一段/章你想要什么效果、情绪、节奏
3. 审阅候选稿：在画布中查看生成结果，可生成变体对比
4. 采纳或修订：接受满意的版本，或拒绝后重新生成

## 最佳实践

- 用"生成下一章"而非"续写草稿"，前者走完整 PGI + Guided Plan 流程，质量更高
- 不确定方向时先生成变体对比，再决定采纳哪个
- 每写 5-10 章跑一次 AI 味检测，早发现早处理

## 常见坑

- 生成内容重复前文：上下文中前文摘要太长或太粗，手动精简摘要
- 候选稿字数不达标：PGI 追问时没说清字数要求，明确告诉 AI "这章要 3000 字"
- 去 AI 味后文字生硬：过度修改导致，只处理明显 AI 痕迹即可，不必全文润色
- 风格漂移：写作预设中途切换导致前后不一致，一部作品尽量固定一套预设

## Agent 查阅提示

- 所有生成结果必须进入候选区，禁止直接写入正式章节
- 上下文组装顺序固定：经纬 → 前文摘要 → 驾驶舱快照 → PGI → Guided Plan，不可调换
- AI 味检测的 12 条本地规则优先执行，朱雀 API 作为补充（需网络可用）
- 写作预设通过 preset_id 关联，切换预设不影响已生成的候选稿
- 变体生成时每个变体独立走完整管线，共享相同上下文但 temperature 不同

# 设置与套路

> 设置管理 AI 模型连接，套路系统定义 Agent 的行为规则——两者共同决定 NovelFork 的运行方式。

---

## 核心概念

- **设置**：全局配置，包括模型供应商、外观、数据、通知等
- **套路（Routines）**：Agent 的行为规则集，定义命令、工具、权限、技能等

---

## 供应商配置

### API 供应商

配置 AI 模型的 API 连接：

| 字段 | 说明 |
|------|------|
| 供应商名称 | 如 OpenAI、Anthropic、自定义 |
| API Key | 认证密钥 |
| Base URL | API 端点地址 |
| 可用模型 | 该供应商下可用的模型列表 |
| 默认模型 | 新会话的默认模型 |

### 支持的供应商类型

- OpenAI 兼容端点（包括 Sub2API 网关）
- Anthropic Claude
- 自定义 OpenAI-compatible 端点

### 平台集成

除 API 供应商外，还支持平台集成配置：

| 平台 | 用途 |
|------|------|
| 起点中文网 | 发布同步 |
| 番茄小说 | 发布同步 |
| 其他平台 | 自定义集成 |

平台集成包含账号绑定、发布设置等。

## 套路系统

### 什么是套路

套路是一组规则，定义 Agent 在会话中的行为边界。类似于"角色设定"但更结构化。

### 全局 vs 项目套路

| 层级 | 作用域 | 说明 |
|------|--------|------|
| 全局套路 | 所有作品 | 基础行为规则，所有会话继承 |
| 项目套路 | 单个作品 | 作品特定的规则，覆盖全局 |

### 合并策略

```
最终套路 = 全局套路 + 项目套路（项目优先）
```

- 命令：合并，同名命令项目覆盖全局
- 工具：合并，同名工具项目覆盖全局
- 权限：项目权限优先
- 提示词：拼接（全局在前，项目在后）

---

## 套路 Tab

RoutinesNextPage 提供以下 Tab 管理套路：

### 命令 Tab（CommandsTab）

定义可用的 Slash 命令：
- 命令名称和别名
- 命令描述
- 启用/禁用状态
- 自定义命令实现

### 工具 Tab（ToolsTab）

管理 Agent 可调用的工具：
- 内置工具列表
- 工具启用/禁用
- 工具参数默认值

### MCP Tab（MCPToolsTab）

MCP（Model Context Protocol）服务器配置：
- 添加 MCP 服务器
- 配置服务器地址和认证
- 查看可用工具列表
- 启用/禁用特定 MCP 工具

### 权限 Tab（PermissionsTab）

定义 Agent 的权限边界：
- 文件读写权限
- 网络访问权限
- 工具调用权限
- 资源修改权限

### 技能 Tab（SkillsTab）

管理 Agent 的技能模块：
- 写作技能（续写、扩写、审校等）
- 分析技能（节奏、对话、健康度等）
- 自定义技能

### 子代理 Tab（SubAgentsTab）

配置子代理：
- Writer Agent 配置
- Planner Agent 配置
- Auditor Agent 配置
- 自定义子代理

### 提示词 Tab（PromptsTab）

管理系统提示词：
- 全局系统提示词
- 写作风格提示词
- 角色扮演提示词
- 自定义提示词模板

## 套路继承到 sessionConfig

会话创建时，套路配置注入 sessionConfig：

```
套路（全局 + 项目合并）
        ↓
sessionConfig
├── availableCommands  ← 命令 Tab
├── availableTools     ← 工具 Tab + MCP Tab
├── permissionMode     ← 权限 Tab
├── skills             ← 技能 Tab
├── subAgents          ← 子代理 Tab
└── systemPrompt       ← 提示词 Tab
```

会话运行时可通过 Slash 命令动态调整部分配置（如 /model、/permission）。

## 最佳实践

- **先配置供应商**：没有可用模型，其他功能都无法使用
- **全局套路设基础**：把通用规则放全局，避免每个项目重复配置
- **项目套路做定制**：特定题材的写作规则放项目套路
- **权限从严开始**：先用 ask-always，熟悉后再放宽
- **MCP 按需启用**：只启用需要的 MCP 工具，减少干扰

## 常见坑

| 问题 | 原因 | 解决 |
|------|------|------|
| 模型不可用 | API Key 过期或余额不足 | 检查供应商配置 |
| 命令不生效 | 命令被禁用 | 检查套路命令 Tab |
| 权限冲突 | 全局和项目套路矛盾 | 项目套路优先，检查合并结果 |
| MCP 工具超时 | 服务器连接问题 | 检查 MCP 服务器状态 |

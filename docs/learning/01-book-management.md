# 作品与章节管理

> 从创建作品到管理章节的完整流程，包括资源树结构和驾驶舱总览。

---

## 创建作品

### 必填字段

| 字段 | 说明 | 示例 |
|------|------|------|
| 标题 | 作品名称 | 《星河万道》 |
| 题材 | 玄幻/仙侠/都市/科幻/其他 | xuanhuan |
| 平台 | 目标发布平台 | 起点中文网 |
| 章节字数 | 每章目标字数 | 3000 |
| 目标章节数 | 全书计划章节数 | 300 |
| 语言 | 创作语言 | zh-CN |

### 可选配置

- **仓库来源**：新建 / 已有目录 / 克隆远程仓库
- **工作流模式**：大纲优先（outline-first）/ 草稿优先（draft-first）/ 串行操作（serial-ops）
- **模板预设**：题材默认 / 空白 / 网文连载
- **经纬模板**：blank / basic / enhanced / genre-recommended
- **AI 初始化**：自动生成经纬 / 生成 pitch / 生成首章方向

### 创建流程

```
仪表盘 → "新建作品" → 填写基本信息 → 选择模板 → 确认创建
                                                    ↓
                                          自动创建默认叙述者会话
                                          自动弹出 Tier 1 问卷
```

创建完成后，系统返回 `bookId` 和默认会话，自动跳转到写作工作台。

## 资源树结构

左栏资源树按以下分组展示作品资源：

| 图标 | 分组 | 资源类型 | 说明 |
|------|------|---------|------|
| 📖 | 作品 | book | 作品元信息（标题、状态、进度） |
| 📄 | 章节 | chapter | 已批准的正式章节 |
| 📜 | 真相文件 | truth | 经纬/设定的只读快照 |
| 🌍 | 故事 | story | 故事线/叙事线 |
| ✏️ | 候选稿 | candidate | AI 生成的待审阅草稿 |
| 📑 | 草稿 | draft | 工作中的草稿 |
| 📗 | 经纬 | jingwei-section / jingwei-entry | 世界观结构化数据 |
| 🗺️ | 叙事线 | narrative-line | 故事线索追踪 |

### 资源能力标记

每个资源节点有能力标记：

- **编辑**：可修改内容
- **只读**：仅可查看
- **删除**：可删除

## 章节 CRUD

### 创建章节

章节通常通过 AI 写作流程自动创建（候选稿 → 接受 → 正式章节）。也可手动创建：

- 在资源树中右键 → 新建章节
- 指定章节号（自动递增）
- 输入标题和正文

### 编辑章节

- 在画布中打开章节 → 使用 InkEditor 编辑
- 支持自动保存（use-autosave hook）
- 编辑器支持 Markdown 格式

### 章节状态流转

```
candidate（候选） → approved（已批准） → 正式章节
                 → rejected（已拒绝） → 归档/删除
                 → archived（已归档）
```

### 批准与拒绝

通过 CandidateActionsBar 操作：

| 操作 | 说明 |
|------|------|
| 接受 - merge | 合并到现有章节 |
| 接受 - replace | 替换现有章节内容 |
| 接受 - draft | 保存为草稿，不立即入正文 |
| 拒绝 | 标记为已拒绝 |
| 归档 | 保留但不使用 |
| 删除 | 永久删除 |

### 删除章节

- 资源树右键 → 删除
- 确认对话框 → 确认删除
- 删除后不可恢复（Git 历史中仍有记录）

## 驾驶舱总览

驾驶舱（CockpitOverview）是打开工作台时的默认视图，提供作品全局状态：

### 统计卡片

| 指标 | 说明 |
|------|------|
| 总字数 | 全书已写字数（万字格式） |
| 章节进度 | 已完成 / 目标章节数 |
| 已批准 | 通过审阅的章节数 |
| 待审阅 | 等待处理的候选稿数 |
| 审阅失败 | 被拒绝的章节数 |
| 下一章 | 即将写作的章节号 |

### 状态标签

| 状态 | 含义 |
|------|------|
| 大纲阶段 | 正在规划大纲 |
| 写作中 | 正在创作 |
| 修订中 | 正在修改已有章节 |
| 已完成 | 全书完成 |
| 已暂停 | 暂停创作 |

### 进度条

显示当前章节数占目标章节数的百分比，直观展示完成度。

## 导入章节文本

通过 WorkImporter 组件导入已有文本：

- 支持格式：纯文本（.txt）、Markdown（.md）
- 自动按分隔符拆分章节
- 导入后作为草稿存在，需手动批准

---

## 导出

- **Markdown 导出**：全书或单章导出为 .md 文件
- **TXT 导出**：纯文本格式，适合投稿平台
- 导出范围：全书 / 选定章节 / 单章

---

## 最佳实践

- **先规划再写作**：利用大纲优先模式，先生成全书大纲再逐章写作
- **及时审阅候选稿**：不要积压太多候选稿，及时接受或拒绝
- **善用驾驶舱**：每次开始写作前先看驾驶舱，了解当前进度
- **章节命名规范**：使用有意义的章节标题，方便后续检索

---

## 常见坑

| 问题 | 原因 | 解决 |
|------|------|------|
| 章节顺序错乱 | 手动创建时章节号冲突 | 使用自动递增，避免手动指定 |
| 候选稿消失 | 被误操作删除 | 检查归档列表，Git 历史可恢复 |
| 字数统计不准 | 包含了 Markdown 标记 | 系统按纯文本计算，标记不计入 |
| 导入后格式错乱 | 原文编码问题 | 确保文件为 UTF-8 编码 |

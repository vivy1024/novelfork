# 作品与章节管理

> 从创建到组织，管理你的书籍和章节。

## 核心概念

**资源树**（左栏）展示当前作品的全部资源：

```
📖 我的作品
├── 📁 章节
│   ├── 第一章 初入江湖
│   └── 第二章 暗流涌动
├── 📁 经纬资料
│   ├── 故事经纬
│   ├── 卷大纲
│   └── 书籍规则
└── 📁 伏笔
    └── 待处理伏笔
```

**新书引导向导**（NewBookGuide）：创建作品时 11 题三模式——预设选择 / 自定义填写 / 跳过随机。完成后自动生成初始经纬条目。

**章节状态**：草稿 → 候选稿确认 → 正式章节。正式章节可继续编辑。

## 推荐使用流程

1. 点击「新建作品」→ 填写书名和题材
2. 完成新书向导（推荐至少填写主角设定和核心冲突）
3. 向导完成后，经纬自动生成初始条目
4. 在叙述者对话中开始写作，或手动在画布中编辑

## 最佳实践

- 新书向导不要全部跳过，至少填核心前提和主角，AI 才有方向
- 章节命名保持统一格式（如"第X章 标题"），方便导出和排序
- 一本书对应一个主叙述者会话，避免上下文分散

## 常见坑

- **新书向导生成的经纬为空** → 向导全部跳过时生成随机内容，质量不高，建议至少填 3 题
- **章节顺序错乱** → 章节按创建时间排序，可在资源树中拖拽调整
- **删除章节不可恢复** → 删除前先打 Checkpoint 快照

## Agent 查阅提示

- 作品 CRUD 通过 `/api/books` REST 接口
- 章节通过 `candidate.create_chapter` 工具创建候选稿，用户确认后入库
- 资源树数据通过 `/api/books/:bookId/resources` 获取
- 新书向导完成后触发经纬初始化流程，调用 AI 填充 story_bible

## 可跳转功能入口

- 作品列表: 查看和管理所有作品。 (/next/books)
- 作品工作台: 单本书的编辑和写作界面。 (/next/books/:bookId)

# 作品与章节管理

> 创建作品、管理章节、资源树浏览、驾驶舱总览、导入导出全流程。

## 核心概念

### 新书引导向导

创建作品的入口。11 道题目覆盖建书所需的全部核心设定，支持三种模式：

| 模式 | 说明 |
|------|------|
| 预设选择 | 从预设选项中点选，快速开始 |
| 自定义 | 输入自由文本，完全控制 |
| 跳过随机 | 让系统随机决定，适合练手 |

**11 题内容：**

| # | 问题 | 预设示例 |
|---|------|---------|
| 1 | 故事类型 | 玄幻/仙侠/都市/科幻/末日/穿越/系统流/无限流/赛博朋克… (26 种) |
| 2 | 核心前提 | 废柴逆袭成神/穿越异世重生/都市隐藏高手/星际文明崛起… |
| 3 | 主角特质 | 隐忍型天才/热血莽夫/腹黑谋士/佛系咸鱼/冷面杀手… |
| 4 | 金手指 | 系统面板/前世记忆/特殊血脉/神秘空间/时间回溯/无金手指… |
| 5 | 世界观 | 九州修真界/现代都市暗面/星际联邦/架空古代/末日废墟… |
| 6 | 力量体系 | 练气→筑基→金丹→元婴/武者九重/序列途径/异能等级… |
| 7 | 基调文风 | 热血爽文/轻松日常/沉重黑暗/悬疑烧脑/温馨治愈/沙雕轻快… |
| 8 | 创作方式 | 建筑师派（先大纲）/花园派（边写边发现）/混合 |
| 9 | 发布平台 | 起点/番茄/飞卢/晋江/七猫/暂不确定 |
| 10 | 每章字数 | 2000字/3000字/4000字/5000字+ |
| 11 | AI 味容忍度 | 零容忍/低容忍/中等/不在意 |

完成后系统自动生成初始经纬文件（角色卡、世界观、力量体系等），并创建首个叙述者会话。

### 资源树

左栏的树形浏览器，按类别分组：

```
📖 作品名
├── 📁 章节          ← 正式章节（全文可读可编辑）
├── 📁 候选稿        ← AI 生成的待审阅稿件
├── 📁 草稿          ← 用户手写的未完成内容
├── 📁 大纲与设定
├── 📁 经纬资料
│   ├── 故事经纬     ← story_bible.md（核心设定文件）
│   ├── 卷大纲       ← 分卷结构规划
│   ├── 当前状态     ← 进度追踪
│   └── 书籍规则     ← 写作规则约束
├── 📁 伏笔
│   └── 待处理伏笔
└── 📁 叙事线
```

点击任何条目 → 在中间画布打开 Markdown 编辑器，支持全文阅读和编辑。支持多标签页同时打开多个文件。

### 草稿 vs 候选稿

| | 草稿 | 候选稿 |
|---|------|--------|
| 来源 | 用户手写 | AI 生成 |
| 状态 | 无审阅流程 | 需要审阅（接受/拒绝） |
| 用途 | 随手记录、片段积累 | 正式章节的前置步骤 |
| 流转 | 手动移入章节 | 接受后自动合并到章节 |

### 驾驶舱（Cockpit）

打开工作台时的默认视图（"总览"标签页）。从上到下展示：

- **章节进度卡片** — 已写/总计章节数 + 进度条
- **日更进度** — 今日字数 / 日更目标完成度
- **经纬完成度** — 各分区（角色/势力/设定/伏笔/大纲）填充状态
- **候选稿待处理** — 未审阅的 AI 生成稿件数量
- **伏笔追踪** — 开放/已回收/过期伏笔统计
- **角色弧线状态** — 主要角色当前发展阶段
- **健康度评分** — 节奏/对话/句长综合评分

Agent 可通过 `cockpit.get_snapshot` 工具读取驾驶舱全部数据作为写作上下文。

### 顶部工具栏

工作台顶部提供快捷操作入口：

```
总览 | 资源树 | 章节图 | 生成下一章 | 续写草稿 | 扩写/改写 | 连续性审校 | 去AI味检测 | 伏笔建议 | 写作工具 | 快照
```

- **总览** — 驾驶舱视图
- **资源树** — 文件浏览器
- **章节图** — 章节关系可视化
- **生成下一章** — 触发写下一章流程（PGI → Plan → 候选稿）
- **续写草稿** — 从草稿继续写作
- **扩写/改写** — 选段变换
- **连续性审校** — 检测矛盾和设定冲突
- **去AI味检测** — 12 规则本地检测
- **伏笔建议** — AI 建议伏笔回收时机
- **写作工具** — 健康度/节奏/对话比例等分析面板
- **快照** — Checkpoint/Rewind 资源快照

### 章节内容展示

章节在画布中以 Markdown 编辑器形式展示：
- 全文可阅读、可编辑
- 顶部标签页显示文件名 + 路径 + 保存状态（已保存/未保存）
- 支持多标签页同时打开多个章节对比阅读
- 经纬条目初始内容为 `*(待 AI 生成)*`，向导完成后由 AI 填充实际设定

### 章节生命周期

```
AI 生成 → 候选稿 → 用户审阅 → 接受(merge/replace/draft) 或 拒绝
```

手动创建章节也支持，但推荐走 AI 生成流程以保持连续性追踪。

## 推荐使用流程

1. 新建作品：作品列表页点"新建"→ 完成 11 题向导 → 系统自动生成经纬和首个会话
2. 查看驾驶舱：每次开始写作前先看驾驶舱，了解当前进度和待处理候选稿
3. 管理章节：通过资源树浏览章节，点击打开编辑，拖拽调整顺序
4. 导出成品：完成后通过导出面板输出 TXT/Word/ePub 格式

## 最佳实践

- 及时处理候选稿，不要积压超过 3 篇未审阅的候选稿
- 章节标题用有意义的名称，方便资源树中快速定位
- 每次写作前看驾驶舱，它是你和 Agent 共享的状态感知入口

## 常见坑

- 章节顺序错乱：手动创建时章节号冲突，用自动递增避免
- 候选稿"消失"：被误操作归档了，检查归档列表可恢复
- 导入后格式乱：原文件不是 UTF-8 编码，转码后重新导入
- 字数统计偏差：系统按纯文本计算，Markdown 标记不计入字数

## Agent 查阅提示

- 创建作品后必须等待向导完成才能开始写作，向导生成的经纬是后续写作的基础
- 驾驶舱快照通过 `cockpit.get_snapshot` 获取，包含章节进度/字数/健康度等结构化数据
- 候选稿状态流转：candidate → approved/rejected/archived，只有 approved 才进入正式章节
- 资源树分组顺序：章节 > 经纬 > 候选稿 > 草稿 > 叙事线

# 使用历史

> Token 用量追踪、费用统计、按供应商和模型分组查看。

## 核心概念

系统自动记录每次 API 调用的 token 用量和费用：

- **输入 token**：发送给模型的上下文（经纬 + 历史 + 系统提示）
- **输出 token**：模型生成的回复内容
- **费用估算**：按模型定价自动计算（需在供应商配置中设置单价）

支持按维度分组查看：
- 按供应商：哪个供应商花费最多
- 按模型：哪个模型消耗最大
- 按时间：日/周/月趋势

## 推荐使用流程

1. 设置 → 使用历史 → 查看总览
2. 按时间范围筛选（今日/本周/本月/自定义）
3. 按供应商或模型分组，定位高消耗来源
4. 根据数据调整模型选择策略

## 最佳实践

- 定期查看费用趋势，避免意外超支
- 高消耗场景（全书审校）用便宜模型
- 经纬条目过多会显著增加输入 token，精简不必要的条目

## 常见坑

- **费用显示为 0** → 未在供应商配置中设置模型单价
- **用量突然飙升** → 检查是否有子代理在后台持续运行

## Agent 查阅提示

- 用量数据通过 `/api/settings/usage` 接口获取
- 每次 LLM 调用完成后自动记录到 SQLite usage 表
- 费用计算公式：input_tokens × input_price + output_tokens × output_price
- Prompt Cache 命中时 input_tokens 会显著降低

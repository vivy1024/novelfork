# 使用历史

> 查看 Token 用量统计、API 请求明细和成本估算

## 核心概念

**Token 用量统计**：按会话、按模型、按时间维度汇总输入/输出 Token 消耗。页面顶部显示请求数、总 Tokens、总轮次、会话数四个概览卡片。

**API 请求历史**：逐条记录每次 AI 调用，包含时间、供应商、模型、Token 数、TTFT（首 Token 响应时间）、总耗时、状态。错误请求红色高亮，可查看完整 payload。

**使用趋势**：折线图展示 Token 随时间变化，支持小时/天/月粒度切换。

**筛选器**：按供应商、模型、请求状态过滤，定位特定调用。

**成本估算**：按模型分组统计累计 Token，结合模型定价估算花费。

## 推荐使用流程

1. 打开「设置 → 使用历史」tab 查看概览
2. 用趋势图观察日常消耗节奏，发现异常飙升
3. 用筛选器定位具体供应商/模型的请求
4. 点击错误请求查看 payload 排查问题

## 最佳实践

- 每周看一次趋势图，了解写作节奏和成本走向
- 对比不同模型的 TTFT 和耗时，选择性价比最优的模型
- 发现错误率上升时立即排查（API Key 过期、限流、代理故障）
- 开启「AI 代理 → 显示 Token 用量」实时监控每轮消耗

## 常见坑

- Token 突然飙升 → 可能是上下文未压缩导致每轮都发送完整历史
- 错误率高但没注意 → API Key 余额耗尽或被限流，写作中断
- 只看总量不看模型分布 → 不知道贵模型占了大头，成本失控

## Agent 查阅提示

- 用户问"花了多少钱"时引导到使用历史页面
- 用户报告 AI 响应慢时，查看 TTFT 指标判断是 API 慢还是网络慢
- 建议用户用便宜模型做初稿、贵模型做精修来控制成本

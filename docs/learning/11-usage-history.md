# 使用历史与成本监控

> 在「设置 → 使用历史」中查看 AI 请求统计、趋势和逐条明细。

---

## 统计概览

页面顶部显示 4 个统计卡片：

| 指标 | 含义 |
|------|------|
| 请求数 | 累计 AI API 调用次数 |
| 总 Tokens | 输入+输出 Token 总量 |
| 总轮次 | 所有会话的对话轮次总和 |
| 会话数 | 创建过的会话总数 |

每个卡片显示格式化的大数字（如 67.5K）和精确值。

---

## 使用趋势

折线图展示 Token 使用量随时间的变化趋势。

### 时间粒度

- **小时**：查看最近几小时的使用波动
- **天**：查看每日使用量（最常用）
- **月**：查看月度趋势

### 如何解读

- 突然飙升：可能是长篇写作或 Agent 管线密集执行
- 持续平稳：正常日常写作节奏
- 突然归零：可能是 API 配置问题或服务中断

---

## 筛选器

在趋势图下方，可以按条件筛选请求：

| 筛选项 | 用途 |
|--------|------|
| 提供商 | 只看特定供应商的请求（如只看 DeepSeek） |
| 模型 | 只看特定模型的请求 |
| 请求状态 | 只看成功/错误请求 |

点击「应用筛选」生效，「重置」清除所有筛选条件。

---

## 请求明细表

逐条展示每次 AI API 调用的详细信息：

| 列 | 含义 |
|----|------|
| 时间 | 请求发起时间（精确到秒） |
| 提供商 | 使用的 AI 供应商 |
| 模型 | 使用的模型 ID |
| Tokens | 本次请求的总 Token 数 |
| TTFT | 首 Token 响应时间（Time To First Token） |
| 耗时 | 请求总耗时 |
| 状态 | 成功（绿色）或错误（红色） |

### 错误行

错误请求会以红色背景高亮，方便快速定位问题。常见错误：
- 429：限流（请求太频繁）
- 502：上游服务器错误
- 超时：API 响应过慢

---

## 按模型分组统计

页面底部展示按供应商+模型分组的累计统计表：

- 每个模型的输入/输出/总 Token 数
- 使用轮次
- 帮助判断哪个模型消耗最多

---

## 成本控制建议

### 监控成本

1. 开启「AI 代理 → 调试 → 显示 Token 用量」实时查看每轮消耗
2. 定期查看使用历史趋势，发现异常飙升
3. 按模型分组统计，找出高消耗模型

### 降低成本

- 精简经纬条目，减少上下文长度
- 使用更便宜的模型做初稿，贵模型做精修
- 调整上下文窗口阈值，更早触发压缩
- 避免不必要的重试（检查自定义重试规则）

### 排查问题

1. 筛选「错误」状态的请求
2. 查看错误请求的提供商和模型
3. 检查是否是代理问题（切换代理试试）
4. 查看 TTFT 是否异常高（可能是 API 过载）

---

## 最佳实践

- 每周查看一次使用趋势，了解写作节奏
- 发现错误率上升时立即排查（可能是 API Key 过期或限流）
- 用筛选器对比不同模型的性能（TTFT、耗时）
- 长篇写作前检查 Token 余额

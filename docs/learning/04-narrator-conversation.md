# 叙述者对话

> 有状态的 Agent 会话界面，用自然语言驱动一切创作操作。

## 核心概念

叙述者不是聊天机器人。它是有状态的 Agent：能调用工具、修改资源、记住上下文。

**状态栏** — 顶部实时显示：空闲/工作中/等待输入 + 上下文使用率（Context Ring）+ 当前模型 + 权限模式。

**确认门** — AI 写入正式资源前暂停等待用户批准。风险分三级：read（绿）/ write（黄）/ delete（红）。拒绝后 AI 不得执行写入。第三个按钮"始终允许此类操作"可永久放行同类工具调用，免去重复确认。

**Slash 命令** — 输入框中 `/` 触发：`/help` `/model` `/permission` `/compact` `/fork`。不区分大小写，支持别名。

**Bash 实时输出流** — 工具执行 shell 命令时，stdout 实时推送到前端，无需等命令结束即可看到输出。

**重试按钮** — 工具调用失败后，消息气泡显示"重试"按钮，一键重发上次请求。

**长按中断** — 按住中断按钮 1 秒确认后才会中断 Agent 执行，防止误触。

**工具长时间运行通知** — 工具执行超过 10 秒后自动显示"仍在执行中..."提示，避免用户误以为卡死。

## v0.9.x 新增工具

| 工具 | 说明 |
|------|------|
| EnterPlanMode / ExitPlanMode | 计划模式：AI 先调查和规划，不执行写入，计划完成后提交审批 |
| TaskCreate | 待办列表：AI 创建/更新任务清单，追踪多步骤工作进度 |
| AskUserQuestion | 向用户提问：AI 需要澄清需求时弹出选择题，用户选择后继续 |
| Recall | 搜索历史对话：AI 可搜索当前会话的历史消息，找到之前的决策和上下文 |
| Send | 向子代理发消息：主叙述者可向后台运行的子代理发送指令 |
| Agent / Await | 子代理：派发独立任务给子代理执行，Await 等待结果返回 |
| ShareFile | 分享文件：生成临时下载链接，方便分享文件给他人 |
| ForkNarrator | 分叉叙述者：创建独立的新叙述者，继承或不继承当前上下文 |
| StartPipeline / EndPipeline | 管道模式：批量工具调用时压缩中间输出，只保留最终结果 |
| GetGoals / AddGoal / UpdateGoal | 目标管理：追踪长期目标的完成进度 |

## 推荐使用流程

1. 一个作品维护一个主会话，避免上下文分散
2. 用自然语言下达指令（写章节、审校、修改设定）
3. AI 通过确认门征求同意后执行写入
4. 上下文超 70% 时执行 `/compact` 压缩历史
5. 探索不同方向时用 `/fork` 分叉，不满意回主线

## 最佳实践

- Context Ring 接近满时生成质量下降，主动压缩。
- 不要开 auto-approve 除非完全信任当前流程。
- 善用消息多选（Ctrl+Click / Shift 范围选）批量导出或清理对话。

## 常见坑

- **AI 回复越来越短** — 上下文快满了。执行 `/compact`。
- **确认门频繁弹出** — 权限模式为 ask-always。切换到 auto-approve 需谨慎评估。
- **会话恢复失败** — 本地存储被清理。检查浏览器存储设置。
- **模型切换后风格变化** — 不同模型特点不同。在经纬中明确文风要求可缓解。

## Agent 查阅提示

- 消息操作：右键菜单支持回退/分叉/压缩/编辑重生成/删除
- v0.4.0 多选：Ctrl+Click 切换选中 / Shift 范围选 / Escape 清除 → 浮动操作栏（复制/删除/分叉）
- v0.4.0 文件修改面板：追踪 Write/Edit 操作 + diff 预览 + 单文件恢复
- v0.4.0 WebSocket 自动重连 + 断点续传，浏览器刷新后自动恢复会话
- 权限模式：ask-always（每次确认）/ auto-approve（自动批准低风险）

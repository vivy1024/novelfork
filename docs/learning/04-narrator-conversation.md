# 叙述者对话

> 有状态的 Agent 会话界面，用自然语言驱动一切创作操作。

## 核心概念

叙述者不是聊天机器人。它是有状态的 Agent：能调用工具、修改资源、记住上下文。

**每本书有多个叙述者**：写书、审校、伏笔、章末钩子、大纲与经纬——各司其职。

**确认门**：AI 写入正式资源前暂停等待你批准。风险分级：read（绿）/ write（黄）/ delete（红）。"始终允许此类操作"可永久放行同类调用。

**状态栏**：顶部实时显示空闲/工作中/等待输入 + 上下文使用率 + 当前模型 + 权限模式。

**Slash 命令**：输入框中 `/` 触发——`/help` `/model` `/permission` `/compact` `/fork`。

## 推荐使用流程

1. 一个作品维护一个主会话，避免上下文分散
2. 用自然语言下达指令（写章节、审校、修改设定）
3. AI 通过确认门征求同意后执行写入
4. 上下文超 70% 时执行 `/compact` 压缩历史
5. 探索不同方向时用 `/fork` 分叉，不满意回主线

## 最佳实践

- Context Ring 接近满时生成质量下降，主动 `/compact`
- 不要开 auto-approve 除非完全信任当前流程
- 善用消息多选（Ctrl+Click / Shift 范围选）批量操作

## 常见坑

- **AI 回复越来越短** → 上下文快满了，执行 `/compact`
- **确认门频繁弹出** → 权限模式为 ask-always，可切换到 auto-approve（需谨慎）
- **模型切换后风格变化** → 在经纬中明确文风要求可缓解

## Agent 查阅提示

- 叙述者工具集：Read/Write/Edit/Bash/Grep/Glob + 小说专属工具（jingwei/cockpit/candidate/pgi/guided 等）
- 确认门触发条件：写入正式章节、修改经纬、删除资源、执行修订
- 权限模式：ask-always（每次确认）/ auto-approve（自动批准低风险）
- 消息操作：右键菜单支持回退/分叉/压缩/编辑重生成/删除
- WebSocket 自动重连 + 断点续传，刷新后自动恢复会话
- Bash 工具实时输出流：stdout 实时推送到前端

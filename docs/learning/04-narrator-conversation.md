# 叙述者对话

> 叙述者是 NovelFork 的核心交互界面——你用自然语言驱动创作，AI 通过对话完成写作、审校、修订等一切操作。

---

## 核心概念

叙述者（Narrator）不是简单的聊天机器人。它是一个有状态的 Agent，能够：

- 调用写作工具（生成章节、审校、检测）
- 修改作品资源（经纬、章节、大纲）
- 通过确认门征求你的同意
- 记住对话上下文并压缩历史

## NarratorStatusBar

状态栏位于对话区顶部，提供实时状态信息：

### 左侧

| 元素 | 说明 |
|------|------|
| 状态圆点 | 灰=空闲 / 蓝=工作中 / 黄=等待输入 / 灰浅=已归档 |
| 子状态 | error(红) / interrupted(橙) / reasoning(紫) / compacting(橙) / planning(绿) |
| 耗时 | 当前操作已用时间 |

### 右侧

| 元素 | 说明 |
|------|------|
| Context Ring | 上下文使用百分比（环形进度） |
| 模型按钮 | 当前使用的模型，点击切换 |
| 推理强度 | 调整 AI 推理深度 |
| Fast Mode | 快速模式开关（跳过部分确认） |
| 权限按钮 | 当前权限模式 |

### 模型切换

点击模型按钮弹出下拉菜单：
- 显示所有可用模型
- 按供应商分组
- 支持搜索过滤
- 切换后立即生效

### 权限模式

| 模式 | 图标 | 说明 |
|------|------|------|
| ask-always | 🛡️ | 每次写入都需确认 |
| auto-approve | 🛡️⚡ | 自动批准低风险操作 |

## Slash 命令

在输入框中输入 `/` 开头的命令：

| 命令 | 说明 |
|------|------|
| /help | 显示可用命令列表 |
| /status | 显示当前会话状态（模型、权限、会话 ID） |
| /model | 切换模型（等同点击模型按钮） |
| /permission | 切换权限模式 |
| /compact | 压缩对话历史，释放上下文空间 |
| /fork | 从当前对话分叉出新会话 |

### 命令解析

```
/command args → parseSlashCommandInput → { name, args }
```

命令支持别名匹配，不区分大小写。

### 执行结果类型

| 类型 | 说明 |
|------|------|
| status | 显示状态信息 |
| update-session-config | 更新会话配置（模型/权限） |
| fork-session | 创建分叉会话 |
| focus-model | 聚焦模型选择 |
| compact-session | 压缩会话历史 |
| compact-pending | 压缩等待中 |

## 确认门（ConfirmationGate）

当 AI 要执行写入操作时，确认门会弹出：

### 确认门内容

```
┌─────────────────────────────────┐
│ ⚠️ 确认操作                      │
│                                 │
│ 标题：写入第 15 章正文            │
│ 摘要：将候选稿合并到章节 15       │
│ 目标资源：chapter-15             │
│ 风险等级：write                  │
│                                 │
│ [批准] [拒绝]                    │
└─────────────────────────────────┘
```

### 风险等级颜色

| 等级 | 颜色 | 说明 |
|------|------|------|
| read | 绿色 | 只读操作 |
| write | 黄色 | 写入操作 |
| delete | 红色 | 删除操作 |

### 确认门字段

- `title`：操作标题
- `summary`：操作摘要
- `targetResources`：受影响的资源列表
- `risk`：风险等级
- `diff`：变更预览（如有）
- `checkpoint`：是否需要创建检查点
- `questions`：PGI 问题（如有）

## 消息操作

### 右键菜单

对消息右键可执行：

| 操作 | 说明 |
|------|------|
| 回退 | 回退到该消息之前的状态 |
| 分叉 | 从该消息创建新的对话分支 |
| 压缩 | 将该消息之前的历史压缩为摘要 |
| 编辑重生成 | 编辑用户消息并重新生成 AI 回复 |
| 删除 | 删除该消息 |

### 消息编辑

通过 MessageEditor 组件：
1. 点击编辑 → 消息变为可编辑状态
2. 修改内容 → 点击"重新生成"
3. AI 基于修改后的消息重新回复

## 会话恢复与历史分页

### 会话恢复

如果浏览器意外关闭或刷新：
- 系统自动保存会话状态
- 重新打开时显示 RecoveryBadge
- 点击恢复 → 回到上次对话位置

### 历史分页

对话历史过长时：
- 自动分页加载
- 滚动到顶部加载更早的消息
- 压缩后的历史以摘要形式显示

### 会话中心

SessionCenterPage 管理所有会话：
- 查看所有活跃/归档会话
- 恢复已归档会话
- 删除不需要的会话
- 按作品/时间筛选

## 最佳实践

- **一个作品一个主会话**：避免创建过多会话导致上下文分散
- **定期 /compact**：上下文超过 70% 时主动压缩
- **善用分叉**：探索不同方向时用 /fork，不满意可以回到主线
- **关注状态栏**：Context Ring 接近满时生成质量会下降
- **用确认门把关**：不要开启 auto-approve 除非你完全信任当前流程

## 常见坑

| 问题 | 原因 | 解决 |
|------|------|------|
| AI 回复越来越短 | 上下文快满了 | 执行 /compact |
| 确认门频繁弹出 | 权限模式为 ask-always | 切换到 auto-approve（谨慎） |
| 会话恢复失败 | 本地存储被清理 | 检查浏览器存储设置 |
| 模型切换后风格变化 | 不同模型有不同特点 | 在经纬中明确文风要求 |

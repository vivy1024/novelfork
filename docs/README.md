## 文档维护规范

### 目录命名

- 使用数字前缀（01-、02-）表示阅读顺序
- 使用中文目录名，便于理解
- 特殊目录（archive、planning）不加数字前缀

### 文件命名

- 使用小写字母和连字符：`feature-name.md`
- 版本号使用 `v` 前缀：`release-v0.0.1.md`
- 日期格式：`YYYY-MM-DD`

### 文档更新

- 每次更新文档时，更新文档头部的「更新日期」
- 重大变更时，在 CHANGELOG.md 中记录
- 过时文档移动到 `archive/` 目录

## 创建作品

### 新建书籍流程

新建书籍流程已简化为仓库绑定优先 + 可选书名：

1. 点击侧边栏"叙事线（书籍）"分组的新建按钮
2. 选择或创建仓库目录（必填）
3. 输入书名（可选，留空则使用目录名）
4. 选择题材（可选）
5. 确认创建

### 新书引导向导（NewBookGuide）

新书进入工作台后，自动显示引导式创作向导：

- **11 道问题**：覆盖题材、文风、目标读者、核心卖点、世界观、主角设定等
- **每题三种模式**：
  - 预设选择：从推荐选项中快速选择
  - 自定义：手动输入个性化内容
  - 跳过随机：跳过该题，系统随机填充合理默认值
- 引导完成后自动生成初始真相文件，进入正常驾驶舱视图

### API 创建

通过 API 创建新作品，同时可初始化经纬模板：

```
POST /api/books/create
```

请求体示例：

```json
{
  "title": "我的新书",
  "genre": "xianxia",
  "targetChapters": 200,
  "jingweiTemplate": "genre-recommended"
}
```

经纬模板选项：

| 模板 ID | 说明 |
|---------|------|
| `blank` | 空白，不创建任何分区 |
| `basic` | 基础分区（人物、地点、物品） |
| `enhanced` | 增强分区（含势力、功法、时间线等） |
| `genre-recommended` | 根据题材推荐分区，可通过 `selectedSectionKeys` 选择子集 |

支持的题材（genre）：`xuanhuan`（玄幻）、`xianxia`（仙侠）、`dushi`（都市）、`scifi`（科幻）、`other`（其他）。

## 工作台布局

Studio 采用三栏布局：

```
┌──────────────┬────────────────────────────┬──────────────────┐
│  左：资源树   │     中：画布（Canvas）      │  右：叙述者会话   │
│              │                            │                  │
│  章节列表     │  章节编辑器 / 驾驶舱       │  消息流           │
│  经纬资料     │  候选稿查看器              │  Composer         │
│  大纲与设定   │  经纬条目编辑器            │  NarratorStatusBar│
│  真相文件     │  叙事线快照                │                  │
└──────────────┴────────────────────────────┴──────────────────┘
```

- **左栏**（WorkbenchResourceTree）：资源树导航，按类型分组展示所有写作资源
- **中栏**（WorkbenchCanvas）：根据选中资源渲染对应的查看器/编辑器
- **右栏**（ConversationSurface）：固定的叙述者 AI 对话面板

---

## 资源树结构

资源树按以下类型组织：

| 节点类型 | 说明 |
|---------|------|
| `book` | 作品根节点，含驾驶舱总览入口 |
| `chapter` | 章节，按章节号排序 |
| `candidate` | 候选稿，AI 生成的待审内容 |
| `draft` | 草稿，从候选稿接受或手动创建 |
| `outline` | 大纲文件 |
| `setting` | 设定文件（世界观、规则等） |
| `truth` | 真相文件（ground truth） |
| `jingwei-section` | 经纬分区（人物、地点、物品等） |
| `jingwei-entry` | 经纬条目（具体的人物/地点/物品） |
| `narrative-snapshot` | 叙事线快照 |

## 章节 CRUD

### 创建章节

通过工作台或 API 创建新章节，章节文件存储在 `books/{bookId}/chapters/` 目录下，文件名格式为 `{4位章节号}-{标题}.md`。

### 编辑与保存

中栏画布提供基于 TipTap 的富文本编辑器：

- **Markdown 双向转换**：内部以 Markdown 存储，编辑时渲染为富文本，保存时自动转回 Markdown
- **自动保存**：编辑内容自动保存，无需手动操作
- **字数统计**：编辑器底部实时显示当前章节字数

> 注意：编辑器本身不包含选段写作功能（如续写/扩写/改写）。所有 AI 写作操作通过右侧 Agent 对话面板的快捷按钮触发。

### 删除

删除章节会同时清理关联的候选稿引用。

### 批准/拒绝

章节经过连续性审校后，可标记为：
- **批准**（approved）：审校通过，计入已完成章节
- **拒绝**（failed）：审校发现问题，需要修订

## 经纬资料编辑

经纬系统是 NovelFork 的核心设定管理机制，用于维护人物、地点、物品、势力等结构化设定数据。

### 分区管理

```
GET    /api/books/:bookId/jingwei/sections          # 列出所有分区
POST   /api/books/:bookId/jingwei/sections          # 创建分区
PUT    /api/books/:bookId/jingwei/sections/:id      # 更新分区
DELETE /api/books/:bookId/jingwei/sections/:id      # 软删除分区
```

分区属性：

| 字段 | 说明 |
|------|------|
| `key` | 唯一标识键（如 `characters`、`locations`） |
| `name` | 显示名称 |
| `description` | 分区描述 |
| `enabled` | 是否启用 |
| `showInSidebar` | 是否在侧栏显示 |
| `participatesInAi` | 是否参与 AI 上下文组装 |
| `defaultVisibility` | 默认可见性规则：`global`/`nested`/`tracked` |
| `fieldsJson` | 自定义字段定义数组 |

### 条目 CRUD（JingweiEntryEditor 组件）

```
GET    /api/books/:bookId/jingwei/entries           # 列出条目（可按 sectionId 过滤）
POST   /api/books/:bookId/jingwei/entries           # 创建条目
PUT    /api/books/:bookId/jingwei/entries/:id       # 更新条目
DELETE /api/books/:bookId/jingwei/entries/:id       # 软删除条目
```

条目属性：

| 字段 | 说明 |
|------|------|
| `title` | 条目标题（如角色名） |
| `contentMd` | Markdown 格式的详细描述 |
| `tags` | 标签数组 |
| `aliases` | 别名数组（用于 AI 识别） |
| `customFields` | 自定义字段值 |
| `relatedChapterNumbers` | 关联章节号 |
| `relatedEntryIds` | 关联条目 ID |
| `visibilityRule` | 可见性规则（控制 AI 何时能看到此条目） |
| `participatesInAi` | 是否参与 AI 上下文 |
| `tokenBudget` | Token 预算限制 |

### 模板应用

```
POST /api/books/:bookId/jingwei/templates/apply
```

根据选择的模板批量创建分区，已存在的分区（按 key 判断）不会重复创建。

### 上下文预览

```
POST /api/books/:bookId/jingwei/preview-context
```

预览当前经纬数据在指定章节号下组装出的 AI 上下文内容。

## 真相文件与大纲设定

- **真相文件**（truth）：存储在 `books/{bookId}/story/` 目录下，是 AI 写作的权威参考源。包括 `character_matrix.md`、`chapter_summaries.md`、`pending_hooks.md`、`style_profile.json` 等。
- **大纲与设定**：Markdown 格式的世界观、规则、大纲文件，在资源树中以 `outline`/`setting` 类型展示。

在画布中选中这些资源后，可直接查看和编辑其内容。

## 驾驶舱总览（CockpitOverview）

点击资源树中的作品根节点，画布区域显示驾驶舱总览，包含：

| 模块 | 内容 |
|------|------|
| 进度条 | 章节进度百分比（已写/目标章节数） |
| 总字数 | 全书字数统计，含每章均字数 |
| 下一章 | 提示下一章章节号 |
| 审校状态 | 已通过审校章节数、待审章节数 |
| 风险提示 | 审校未通过章节数，红色高亮 |
| 下一步建议 | 根据当前状态给出优先级建议（修复未通过 > 处理待审 > 继续写作） |

作品状态标签：大纲阶段、写作中、修订中、已完成、已暂停。

### 驾驶舱底部面板

驾驶舱底部提供四个可折叠面板：

| 面板 | 功能 |
|------|------|
| 预设配置 | Toggle 启用/禁用各写作预设，修改后自动保存到后端 |
| 节拍进度 | 选择节拍模板，追踪当前写作进度与节拍完成情况 |
| 质量监控 | AI 味趋势图、文风漂移检测、审校通过率统计 |
| 警告汇总 | 汇总当前作品的所有警告信息（审校失败、风格偏移、合规问题等） |

## 叙事线快照查看

叙事线快照（narrative-snapshot）记录特定时间点的故事状态，包括角色位置、情节进展、伏笔状态等。在资源树中选中快照节点后，画布以 CockpitSnapshotCard 组件渲染快照内容。

---

## 经纬图谱（JingweiGraph）

经纬图谱基于 react-flow 提供 5 种可视化视图：

| 视图 | 说明 |
|------|------|
| 关系图谱 | 角色/势力之间的关系网络，节点可拖拽布局 |
| 角色弧线 | 角色在全书中的成长轨迹与情感变化曲线 |
| 矛盾地图 | 当前活跃的矛盾冲突及其关联角色/事件 |
| 列表视图 | 所有经纬条目的表格化展示，支持排序和筛选 |
| 时间线 | 按时间轴展示事件发生顺序与因果关系 |

交互功能：
- **可见性图标**：每个节点旁显示可见性状态图标（global/nested/tracked），点击可快速切换
- **筛选下拉**：顶部下拉菜单按分区、标签、可见性规则筛选显示的节点

## 本文定位

本文描述 NovelFork v1.0.0 当前真实可用的启动方式。推荐把 NovelFork 当成本地 Studio 工作台使用：运行 exe 或开发模式启动后，浏览器访问 `http://localhost:4567`。

## 前置条件

### 使用编译产物（推荐）

| 工具 | 用途 |
|---|---|
| Windows x64 系统 | 运行 exe |
| 浏览器 | 访问 Studio 界面 |

下载 `novelfork-v1.0.0-windows-x64.exe` 即可使用，无需安装其他依赖。

### 从源码运行（开发者）

| 工具 | 用途 |
|---|---|
| Git | 克隆仓库 |
| Node.js | 运行 pnpm |
| pnpm | 安装依赖 |
| Bun | 主运行时 |

> 当前是原生 Windows 环境可用的项目，不需要切换到 WSL。

## 快速开始（编译产物）

1. 下载 `novelfork-v1.0.0-windows-x64.exe`
2. 双击运行
3. 浏览器打开 `http://localhost:4567`

## 从源码运行

```bash
git clone https://github.com/vivy1024/novelfork.git novelfork
cd novelfork
pnpm install
```

开发模式启动：

```bash
bun run main.ts --root=. --port=4567
```

启动后访问：

```text
http://localhost:4567
```

## 编译产物

编译命令：

```bash
pnpm bun:compile
```

编译后运行：

```bash
./dist/novelfork-v1.0.0-windows-x64.exe --root=. --port=4567
```

## 数据存储

NovelFork 的数据存储在用户目录下：

| 文件 | 位置 | 说明 |
|---|---|---|
| 主数据库 | `~/.novelfork/novelfork.db` | SQLite，存储作品、章节、经纬、会话等 |
| 搜索数据库 | `~/.novelfork/novelfork-search.db` | SQLite，全文搜索索引 |
| 请求日志 | `~/.novelfork/novelfork.db` | SQLite 持久化，重启不丢失 |

数据随用户账户走，重装软件不丢失创作数据。

## 首次运行

首次启动 NovelFork 时，会弹出 **FirstRunDialog**（首次运行引导对话框）：

1. 欢迎页面介绍核心功能
2. 配置 AI 供应商（名称、Base URL、API Key）
3. 测试连接
4. 完成配置，进入工作台

未配置 AI 供应商时仍可进行本地创作管理（创建作品、编辑章节、维护经纬），只有触发 AI 写作动作时才需要可用模型。

## 首次打开后应该做什么

1. 完成 FirstRunDialog 配置 AI 供应商
2. 进入创作工作台
3. 创建新作品或导入已有作品
4. 通过新书引导向导设定作品基础信息
5. 开始写作

## 常见启动问题

| 现象 | 优先检查 |
|---|---|
| 端口无法访问 | 是否启动到了 `4567`；是否被其他进程占用 |
| 页面空白或资源缺失 | 是否使用了编译产物而非源码直接运行 |
| 模型调用失败 | 先检查设置页供应商配置，再看凭据和模型是否可用 |

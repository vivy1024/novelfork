## 供应商配置

NovelFork Studio 的 AI 能力依赖模型供应商配置。未配置模型时仍可进行本地创作管理（创建作品、编辑章节、维护经纬），只有触发 AI 动作时才需要可用模型。

### 首次运行引导

首次启动时，FirstRunDialog 会引导你完成 AI 供应商配置：

1. 输入供应商名称
2. 填写 Base URL 和 API Key
3. 测试连接
4. 完成配置

### API 供应商管理

通过设置页维护自定义供应商凭据：

- 添加供应商（名称、Base URL、API Key）
- 配置可用模型列表
- 测试连接可用性
- 启用/禁用供应商

## 套路系统概述

套路（Routines）是 NovelFork 的配置管理机制，控制叙述者会话中可用的命令、工具、权限和行为。

### 全局配置 vs 项目配置

| 层级 | 作用域 | 说明 |
|------|--------|------|
| 全局 | 所有项目 | 用户级默认配置 |
| 项目 | 单个作品 | 项目级覆盖配置 |

### 合并策略

项目配置覆盖全局配置。具体规则：
- 命令/工具/技能：项目级 `enabled` 状态覆盖全局
- 权限：项目级规则优先，全局规则作为兜底
- 提示词：项目级追加到全局之后

## 套路 Tab

套路系统包含 7 个配置标签页：

### 命令（Commands）

管理可用的 Slash 命令：

| 字段 | 说明 |
|------|------|
| `id` | 命令标识（如 `/compact`） |
| `name` | 显示名称 |
| `description` | 命令描述 |
| `prompt` | 命令执行时的提示词模板（支持 `{{input}}` 占位符） |
| `enabled` | 启用/禁用开关 |

内置命令：`/help`、`/model`、`/permission`、`/compact`、`/fork`。

你也可以创建自定义命令，创建后会自动出现在 Composer 的 `/` 补全列表中。

### 工具（Tools）

管理叙述者可使用的工具：

| 字段 | 说明 |
|------|------|
| `name` | 工具名称 |
| `enabled` | 启用/禁用 |
| `description` | 工具描述 |

### 权限（Permissions）

配置工具级别的权限规则：

| 字段 | 说明 |
|------|------|
| `tool` | 工具名称 |
| `permission` | `allow` / `deny` / `ask` |
| `pattern` | 匹配模式（可选，如文件路径 glob） |

### 技能（Skills）

管理全局技能和项目技能：

| 字段 | 说明 |
|------|------|
| `id` | 技能标识 |
| `name` | 显示名称 |
| `description` | 技能描述 |
| `instructions` | 技能指令（注入到系统提示词） |
| `enabled` | 启用/禁用 |

### 子代理（SubAgents）

配置可派发的子代理：

| 字段 | 说明 |
|------|------|
| `id` | 子代理标识 |
| `name` | 名称 |
| `type` | `general-purpose` / `specialized` |
| `systemPrompt` | 系统提示词 |
| `enabled` | 启用/禁用 |

内置类型（explore/plan/general/fork）始终可用，你可以在此定义自定义子代理。

### 提示词（Prompts）

管理全局提示词和系统提示词：

| 类型 | 说明 |
|------|------|
| 全局提示词 | 追加到每次对话的系统消息中 |
| 系统提示词 | 作为系统级指令注入 |

### MCP 工具

管理 MCP（Model Context Protocol）服务器提供的工具：

| 字段 | 说明 |
|------|------|
| `serverName` | MCP 服务器名称 |
| `toolName` | 工具名称 |
| `enabled` | 启用/禁用 |
| `approved` | 是否已审批 |

## 套路继承

创建新叙述者会话时，系统从当前套路配置继承到 `sessionConfig`：

```
Routines（全局 + 项目合并后）
  → 新建叙述者会话
    → sessionConfig.permissionMode（从推荐模式或默认 "edit"）
    → sessionConfig.toolPolicy（从 permissions 派生 allow/deny/ask 列表）
    → 可用命令列表（排除 disabledCommands）
    → 可用工具列表（排除 disabled tools）
```

会话创建后，可在 NarratorStatusBar 中实时调整权限模式和模型，这些调整只影响当前会话，不修改全局套路。

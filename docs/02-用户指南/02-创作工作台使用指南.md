## 本文定位

本文从用户视角说明当前 Studio 创作工作台怎么用，以及哪些能力已经能走通、哪些仍是透明过渡。本文不把规划中能力写成已完成；如果要看逐项能力状态，请读 [Studio能力矩阵](../01-当前状态/02-Studio能力矩阵.md)。

## 工作台三栏心智模型

当前创作工作台按小说写作软件的心智组织，而不是通用 IDE 或 NarraFork 叙述者中心。

```text
左侧：小说资源管理器
  -> 作品、章节、候选稿、大纲、经纬等资源入口

中间：正文编辑区
  -> 打开章节或候选稿，编辑、保存、处理正文

右侧：AI / 经纬 / 写作工具面板
  -> 生成、续写、审校、改写、去 AI 味、连续性资料和工具结果
```

## 当前可走通的基础路径

### 1. 打开作品

1. 启动 Studio。
2. 进入创作工作台。
3. 使用“作品选择”选择当前作品。
4. 左侧资源管理器会根据真实书籍/章节/候选稿数据派生资源树。

当前已确认的资源包括：

- 作品。
- 已有章节。
- 生成章节候选稿。
- 大纲、经纬等分类入口。

完整的草稿、素材、发布报告等资源树还在后续闭环任务中。

### 2. 打开已有章节

在左侧“已有章节”分组里点击章节后，中间正文编辑区会显示：

- 章节标题。
- 章节状态。
- 字数或正文内容。
- 保存状态。
- 正文编辑控件。

已有章节的读取和保存通过迁移后的章节 API 路径完成。保存失败时，当前编辑内容应该保留，并展示错误原因，而不是丢失未保存正文。

### 3. 编辑并保存正文

当前推荐流程：

```text
打开已有章节
  -> 修改正文
  -> 保存状态变为未保存
  -> 点击保存
  -> 成功后显示已保存
  -> 失败时显示失败原因并保留正文
```

这条路径适合手工修订、AI 候选稿合并后的二次修改和章节日常维护。

### 4. 查看 AI 候选稿

生成候选稿会出现在资源管理器的生成章节或候选稿入口中。打开候选稿时，正文编辑区必须明确显示它不是正式正文。

候选稿当前有明确的非破坏性动作：

| 动作 | 含义 |
|---|---|
| 合并到正式章节 | 把候选稿追加或合并到目标章节 |
| 替换正式章节 | 用候选稿替换目标章节，需要用户明确确认 |
| 另存为草稿 | 不影响正式正文，保留为草稿材料 |
| 放弃候选稿 | 丢弃该候选稿 |

核心原则：AI 候选稿不会自动覆盖正式章节。

## AI 输出流向

所有 AI 输出都必须走安全流向：

```text
AI 操作
  -> 可选：引导式资料/问卷/生成前追问
  -> 生成章节 / 候选稿 / 草稿候选
  -> 用户查看
  -> 用户确认处理方式
  -> 合并、替换、另存或放弃
```

这意味着：

- AI 结果不是正式正文。
- 正式正文的变化必须有用户确认动作。
- 没有安全写入目标时，应用按钮应该禁用或显示未接入。
- 只有 prompt preview 的能力不能写成已写入正文。

## 引导式生成怎么用

引导式生成不是一个单独按钮，而是一组把“先问清楚、再生成、再审校”串起来的能力。当前已经有问卷中心、生成前追问、核心设定变更历史、AI 味报告等底座；实际页面是否已经完整挂入主工作台，以 [Studio能力矩阵](../01-当前状态/02-Studio能力矩阵.md) 为准。

### 1. 首次使用：先模型，后本地创作

首次进入 Studio 时，推荐先完成模型配置。模型未配置时：

- 仍然可以创建本地书籍。
- 仍然可以整理故事经纬和编辑章节。
- 需要调用 AI 的动作应该被模型 gate 拦住，并引导去设置模型。

### 2. 新建作品：选择经纬模板

新建作品时可以选择经纬模板。当前底座支持：

| 模板 | 用途 |
|---|---|
| 空白模板 | 只创建最小结构，适合完全自定义 |
| 基础模板 | 生成人物、事件、设定、章节摘要等基础栏目 |
| 增强模板 | 在基础栏目上追加伏笔、名场面、核心记忆等栏目 |
| 题材推荐 | 根据题材给出推荐栏目组合，用户仍可调整 |

模板会写入故事经纬结构，不会自动生成正式正文。

### 3. 问卷中心：把创作意图变成经纬资料

问卷中心用于在生成前收集用户意图，例如一句话卖点、题材气质、主角目标、关键冲突等。当前能力边界：

- 可以保存草稿、提交或稍后再填。
- AI 建议需要已配置模型；未配置时应提示设置模型，不应清空用户已经填写的答案。
- 提交后的答案会映射到对应的故事经纬对象或初始化资料，而不是直接改正文。

### 4. 生成前追问：在写章节前补关键决策

生成前追问会根据已有经纬资料和章节位置提出少量问题，常见触发包括：

- 冲突已经多章没有升级，需要确认本章是否推进。
- 伏笔接近回收窗口，需要确认本章是否回收或延后。
- 本章写作需要更明确的作者指示。

回答会作为“本章作者指示”进入生成上下文。跳过追问时，AI 仍可生成，但 AI 味报告和质量判断应能区分本章是否使用了追问答案。

### 5. 核心设定变更：先提议，再接受

当需要改动主前提、主线冲突、世界规则或人物弧时，当前支持记录核心设定变更：

```text
提出变更
  -> 记录影响对象和受影响章节
  -> 用户接受或拒绝
  -> 接受后写入设定，并保留变更历史
```

这类变更不会自动重写既有正文；它只标记影响范围，后续仍需要用户决定如何修订。

## 右侧写作工具当前口径

当前右侧工具中，有些能力已经接入真实计算或真实文件读写，有些仍是透明过渡。

| 类型 | 当前口径 |
|---|---|
| hooks | 可写入 story truth 相关文件，模型不可用时应返回 AI gate |
| 节奏分析 | 基于当前章节文本即时计算 |
| 对话分析 | 基于当前章节文本即时计算 |
| 书籍健康 | 从章节数、字数、敏感词、矛盾登记等真实数据计算，缺失指标显示 unknown |
| 写作模式 | 当前主要是提示词预览，不代表已写入正文 |
| 问卷与生成前追问 | 已有 route / 组件 / 引擎事实；如果当前页面没有入口，必须按未接入展示 |
| AI 味报告 | 已有本地规则、可选朱雀配置、报告和改写建议；建议不自动覆盖正文 |
| 续写、审校、改写、连续性检查 | 需要真实 route 或 unsupported 透明状态，不能假成功 |

## 故事经纬入口

工作台右侧经纬面板用于辅助查看人物、地点、伏笔、世界规则等资料。当前用户文案统一使用“故事经纬 / 经纬”。

当前已经存在的底座能力：

- 栏目和条目的结构化 API。
- 经纬模板应用。
- 用于 AI 上下文的经纬预览与裁剪。
- 问卷答案映射到经纬资料。

当前边界：

- 结构化 API 可用，不等于工作台内所有分类都已有完整 CRUD。
- 部分分类详情仍可能是透明过渡。
- 如果某路径未接入，应该显示错误、空态或未接入状态，而不是显示假数据。

## 当前不能当作已完成的能力

| 能力 | 当前状态 |
|---|---|
| 新建章节 | 按钮或路径仍待接完整写入闭环 |
| 全书导出 | 当前仍待接 Markdown/TXT 导出路线 |
| 完整草稿资源树 | 仍待做实 |
| 完整大纲编辑 | 仍待做实 |
| 完整故事经纬工作台编辑 | 结构化 API 已可用；资源树与详情 CRUD 仍待完整接入 |
| 素材和发布报告 viewer | 仍待补齐 |

## 推荐使用方式

1. 先完成模型配置；如果暂时不配置，也可以先创建本地书籍。
2. 新建作品时选择合适的经纬模板，先搭资料底座。
3. 使用问卷或生成前追问补齐关键创作意图，再让 AI 生成候选稿。
4. 对候选稿先阅读，再选择合并、替换或另存草稿。
5. 用写作工具检查节奏、对话、健康指标、AI 味和连续性线索。
6. 用故事经纬维护长期资料，但遇到未接入入口时按透明过渡处理。

## 快速开始

1. 安装依赖：`bun install`
2. 启动工作台：`bun run dev`（在 `packages/studio` 目录）
3. 打开浏览器：`http://localhost:4567`
4. 配置 AI 模型：设置 → AI 供应商 → 添加供应商
5. 创建或打开一本书：仪表盘 → 创建新书 / 打开已有书
6. 在右侧叙述者会话输入你的创作目标，或从左侧资源树打开章节与资料

## 工作台布局

打开一本书后，工作台采用 Agent-native 的三栏 session-first 布局：

```text
┌──────────────┬──────────────────────────┬────────────────┐
│ 左侧资源栏     │ 中间画布                  │ 右侧叙述者会话    │
│ 全局导航       │ 章节 / 候选稿 / 经纬 / 工具产物 │ 模型 / 权限 / 工具链 │
└──────────────┴──────────────────────────┴────────────────┘
```

### 左侧：资源栏

左侧同时保留全局入口和当前书籍资源管理器：

- **全局入口** — 仪表盘、工作台、工作流、设置、套路等。
- **已有章节** — 正式定稿章节，可在中间画布打开编辑。
- **生成章节** — AI 候选稿，可查看、编辑、合并、替换、另存草稿或放弃。
- **草稿** — 暂存写作片段。
- **大纲** — 卷大纲和规划资料。
- **经纬 / 资料库** — 人物、地点、势力、物品、伏笔、世界规则、章节摘要。
- **故事文件 / 真相文件** — `pending_hooks.md`、`chapter_summaries.md`、`book_rules.md`、`current_focus.md` 等。
- **素材 / 发布报告** — 外部素材与发布就绪检查结果。

点击左侧资源只会在中间画布打开或切换资源，不会替换右侧叙述者会话。

### 中间：画布

中间画布是主要工作面：

- 支持打开多个资源 Tab，避免切换时覆盖当前编辑内容。
- 章节编辑器使用 TipTap 富文本编辑器。
- 候选稿、草稿、经纬、故事文件、真相文件、发布报告和 Agent 产物使用对应 renderer 展示。
- Agent 创建的候选稿、计划、叙事线快照或工具结果可自动打开到画布。
- 资源存在未保存编辑（dirty）时，写入类工具会被阻断，要求先保存、放弃或另存为候选。

### 右侧：叙述者会话

右侧固定显示当前书籍的默认 Writer / Narrator 会话，是主操作入口：

- 输入自然语言请求，例如「写下一章」「检查伏笔」「读取当前状态」。
- 保留模型选择、权限模式、推理强度、连接状态和恢复信息。
- 展示工具调用链：pending / running / success / error / confirmation。
- 展示专用工具结果卡片，例如驾驶舱快照、PGI 问题、GuidedGenerationPlan、候选稿产物、叙事线快照。
- 写入正式资源、覆盖正文或修改核心资料前必须进入确认门。

历史的驾驶舱 / 经纬 / 写作面板不再是右侧主 Tab；其中仍有价值的能力会作为会话工具结果卡片或中间画布组件复用。

## 写作流程

### 手动写作

1. 从左侧「已有章节」或空态按钮创建 / 打开章节。
2. 在中间画布编辑正文。
3. 保存章节。
4. 如资源 dirty，切换或 Agent 写入前会提示处理未保存内容。

### 对话驱动写下一章

1. 确认右侧叙述者会话处于合适权限模式（通常是 edit 或 allow；正式覆盖仍需确认）。
2. 输入：「写下一章，回收玉佩伏笔，保持林月冷峻性格」。
3. 系统按工具链推进：
   - `cockpit.get_snapshot` 读取当前书状态。
   - `pgi.generate_questions` 生成必要的生成前追问；没有问题时显示 `no-questions` 原因。
   - `guided.enter` / `guided.exit` 形成 GuidedGenerationPlan。
   - 等待用户批准计划。
   - 批准后调用 `candidate.create_chapter` 生成下一章候选稿。
4. 候选稿会进入候选区，并可在中间画布打开。
5. 用户确认后才合并、替换或插入正式章节。

任一步失败时，系统停止后续写入，显示失败原因，并保留已完成的只读调查结果。

### 使用已有工具能力

- 想查看当前项目状态：在右侧输入「看看当前状态」，会调用 cockpit 工具并返回快照卡片。
- 想整理经纬：在右侧要求叙述者读取或提出经纬变更；正式写入前会展示草案和确认门。
- 想查看叙事线：请求「检查叙事线」，会读取 Narrative Line 快照并可在画布打开。

如果当前模型或 provider 不支持工具调用，系统会返回 `unsupported-tools`，降级为只读解释或 prompt-preview，不会假装已经执行工具。

## 管理故事经纬

1. 从左侧资源树打开「经纬 / 资料库」分类或具体条目。
2. 在中间画布查看或编辑人物、事件、设定、章节摘要等资料。
3. 通过右侧叙述者要求整理、检查或提出变更时，Agent 会先返回可审查草案。
4. 影响正式资料库的写入必须经用户确认。

---

## 发布前检查与导出

### 发布前检查

1. 从资源栏或顶部入口打开发布报告。
2. 选择目标平台（起点 / 晋江 / 番茄等）。
3. 运行敏感词、AI 比例、格式规范、连续性等检查。
4. 在中间画布查看报告，并按建议修改。

### 导出

1. 使用导出入口。
2. 选择格式（Markdown / TXT）和范围。
3. 下载文件。

## Agent 会话角色

右侧叙述者会话可使用 5 种 Agent 角色：

| 角色 | 适合场景 |
|------|---------|
| **Writer**（写作） | 日常续写、章节生成 |
| **Planner**（规划） | 制定大纲、情节点设计 |
| **Auditor**（审计） | 连续性检查、质量审校 |
| **Architect**（架构） | 世界观构建、规则体系 |
| **Explorer**（探索） | 只读分析状态、给出建议 |

会话是主对象，窗口或右侧 docked 视图只是展示方式；刷新或断线后会使用 session recovery 恢复消息、ack 边界、未完成工具调用和最近执行链。

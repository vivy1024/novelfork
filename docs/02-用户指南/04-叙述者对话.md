## 概述

叙述者对话是 NovelFork Studio 右侧固定面板，提供与 AI Agent 的实时交互界面。每个叙述者会话（Narrator Session）是一个独立的对话上下文，支持多会话并行。

---

## 顶部工具栏

| 按钮 | 图标 | 功能 |
|------|------|------|
| 在新标签打开 | ExternalLink | 在新浏览器标签中打开当前会话 |

- 工作中时：显示"思考中"动画和计时器
- 空闲时：显示上轮耗时

## NarratorStatusBar 控件

位于 Composer 上方的状态栏，从左到右包含：

### 左侧：状态指示

- **状态圆点**：颜色反映当前状态
  - 灰色：空闲（idle）
  - 蓝色：工作中（working）
  - 黄色：等待中（waiting）
  - 紫色：推理中（reasoning）
  - 橙色：压缩中（compacting）/ 已中断（interrupted）
  - 红色：错误（error）
  - 绿色：计划中（planning）
- **状态文案**：如"空闲"、"工作中"、"推理中"等
- **上轮耗时**：空闲状态下显示上一轮对话的耗时

### 右侧：控件

#### Context Ring（上下文圆环）

SVG 圆环显示上下文使用百分比：
- 蓝色：< 70%
- 黄色：70%-90%
- 红色：> 90%

悬停显示具体 token 数：`已用 / 最大`。

#### 模型切换

DropdownMenu 下拉菜单，按 provider 分组显示可用模型：
- 顶部搜索框可筛选模型
- 当前选中模型高亮显示
- 点击切换下次发言使用的模型
- 触发按钮显示当前模型首字母

#### 推理强度

| 值 | 显示 | 说明 |
|----|------|------|
| low | L | 低推理强度，响应更快 |
| medium | M | 中等推理强度（默认） |
| high | H | 高推理强度，更深入思考 |

#### Fast Mode（快速模式）

闪电图标切换按钮：
- 开启时：黄色高亮，使用更快的模型配置
- 关闭时：灰色

#### 权限模式

| 模式 | 说明 | 适用场景 |
|------|------|---------|
| ask | 逐项询问 | 新书磨合、陌生 Agent |
| edit | 允许编辑 | 写作会话、章节续写 |
| allow | 全部允许 | 已隔离的临时实验 |
| read | 只读 | 审稿会话、连续性排查 |
| plan | 规划模式 | 大纲规划、卷纲拆解 |

图标：`Shield`（受限模式）/ `ShieldOff`（全部允许）。

## Slash 命令

在 Composer 中输入 `/` 开头的命令触发 slash 命令系统：

| 命令 | 说明 |
|------|------|
| `/help` | 显示可用命令列表 |
| `/model <name>` | 切换模型 |
| `/permission <mode>` | 切换权限模式 |
| `/compact` | 压缩上下文，释放 token 空间 |
| `/fork [title]` | 从当前会话分叉出新会话 |

输入 `/` 后会显示命令建议列表，支持模糊匹配。命令通过 RuntimeCommandRegistry 注册，可通过套路系统启用/禁用。

## 确认门（ConfirmationGate）

当叙述者要执行写入正式资源的操作时，会弹出确认门：

| 字段 | 说明 |
|------|------|
| 标题 | 操作描述 |
| 风险等级 | `read`（绿）/ `draft-write`（蓝）/ `confirmed-write`（黄）/ `destructive`（红） |
| 摘要 | 操作详情 |
| 目标资源 | 受影响的资源列表（kind + id + title） |
| Checkpoint | 是否需要创建检查点 |

操作按钮：
- **批准**（绿色）：允许执行
- **拒绝**（灰色）：阻止执行

确认门确保 AI 不会在未经用户同意的情况下修改正式章节或设定。

## 消息右键菜单

对消息可执行的上下文操作：

| 操作 | 说明 |
|------|------|
| 压缩到此消息前 | 将此消息之前的所有消息压缩为摘要，释放上下文空间 |
| 回退 | 回退到此消息之前的状态（planned） |
| 分叉 | 从此消息创建新的对话分支（planned） |
| 编辑重生成 | 编辑用户消息并重新生成回复（planned） |
| 删除 | 删除此消息（planned） |

## 文件修改追踪（FileChangesPanel）

通过顶部工具栏的 FileCode 按钮打开侧边抽屉，显示本次会话中所有文件修改记录：

- 从助手消息的 toolCalls 中提取 Write/Edit 操作
- 每条记录显示：操作类型（新建/编辑）+ 文件路径
- 只追踪写入操作，读取操作不显示

## 会话恢复（Recovery）

当连接中断或会话状态异常时，系统自动进入恢复流程：

| 恢复状态 | 说明 |
|---------|------|
| recovering | 正在恢复 |
| reconnecting | 正在重连 |
| replaying | 正在重放消息 |
| resetting | 正在重置 |
| failed | 恢复失败 |

恢复原因包括：`initial-hydration`、`reconnect`、`replay`、`history-gap`、`server-reset` 等。

恢复期间界面顶部显示黄色通知栏，提示当前恢复状态和原因。

## 历史消息分页加载

消息流支持向上滚动加载历史消息：

- `hasPreviousMessages`：是否还有更早的消息
- 点击"加载更多"或滚动到顶部时触发 `onLoadPreviousMessages`
- 新加载的消息插入到消息流顶部，保持当前滚动位置

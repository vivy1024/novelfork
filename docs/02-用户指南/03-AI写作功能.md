## 写作模式

写作模式通过 inline-write API 触发，生成内容后进入候选稿流程。

```
POST /api/books/:bookId/inline-write
```

| 模式 | mode 值 | 说明 |
|------|---------|------|
| 续写 | `continuation` | 根据前文自然延续，支持 `direction` 指定方向 |
| 扩写 | `expansion` | 对选中文本进行扩展，支持 `expansionDirection`：`sensory`（感官）等 |
| 过渡 | `bridge` | 生成场景过渡段落，支持 `purpose`：`scene-transition`（场景转换）等 |
| 对话生成 | `dialogue` | 根据角色性格生成对话 |
| 变体生成 | `variant` | 对同一段落生成多个风格变体 |
| 大纲分支 | `outline/branch` | 基于当前大纲生成分支走向 |

### 对话生成

```
POST /api/books/:bookId/dialogue/generate
```

参数：
- `characters`：角色数组，每个含 `name`、`personality`、`speechStyle`
- `scene`：场景描述
- `purpose`：对话目的
- `turns`：对话轮数（默认 5）

### 变体生成

```
POST /api/books/:bookId/variants/generate
```

对选中文本生成 2-5 个风格变体（默认 3 个），每个变体独立调用模型。

### 大纲分支

```
POST /api/books/:bookId/outline/branch
POST /api/books/:bookId/outline/branch/:branchId/expand
```

- `branch`：基于当前大纲、伏笔、故事状态生成分支建议
- `expand`：将选定分支扩展为详细章节大纲

## AI 动作

AI 动作是更高层的写作操作，通常由叙述者会话或工具栏触发：

| 动作 | 说明 |
|------|------|
| write-next | 写下一章，综合大纲、经纬、前文上下文 |
| draft | 生成草稿，不直接写入正式章节 |
| audit | 连续性审校，检测剧情矛盾和设定冲突 |
| revise | 修订，支持 5 种模式（见下文） |
| rewrite | 重写整章 |
| detect / detect-all | 去 AI 味检测（单章/全书） |

### 修订模式

修订（revise）支持以下子模式：

| 子模式 | 说明 |
|--------|------|
| style | 文风修订，对齐真相文件中的风格画像 |
| continuity | 连续性修订，修复审校发现的矛盾 |
| rhythm | 节奏修订，调整叙事节奏 |
| dialogue | 对话修订，优化角色语言风格 |
| general | 综合修订 |

## 选区变换（Transform）

在编辑器中选中文本后，可触发选区变换操作：

| 操作 | 说明 |
|------|------|
| polish | 润色：优化文笔，保持原意 |
| condense | 精简：压缩冗余表达 |
| expand | 扩写：丰富细节和描写 |
| audit | 审查：检查选中段落的连续性问题 |

## 行内续写（Complete）

```
POST /api/books/:bookId/inline-write (mode: "continuation")
```

行内续写以 SSE 流式返回，生成 30-80 字的短续写片段，适合在编辑器中光标位置即时补全。

| 操作 | API | 说明 |
|------|-----|------|
| generate | `POST /api/books/:bookId/outline/branch` | 生成大纲分支 |

## 候选稿流程

AI 生成的内容不直接写入正式章节，而是先进入候选稿（candidate）流程：

```
创建 → 查看 → 接受/拒绝/归档/删除
```

### 候选稿 API

```
GET    /api/books/:id/candidates                    # 列出所有候选稿
POST   /api/books/:id/candidates                    # 创建候选稿
POST   /api/books/:id/candidates/:id/accept         # 接受
POST   /api/books/:id/candidates/:id/reject         # 拒绝
POST   /api/books/:id/candidates/:id/archive        # 归档
DELETE /api/books/:id/candidates/:id                 # 删除
```

### 接受动作（accept action）

| 动作 | 说明 |
|------|------|
| `replace` | 用候选稿内容覆盖目标章节 |
| `merge` | 将候选稿追加到目标章节末尾 |
| `draft` | 将候选稿另存为草稿，不修改正式章节 |

### 草稿管理

```
GET    /api/books/:id/drafts                        # 列出草稿
GET    /api/books/:id/drafts/:draftId               # 获取草稿详情
POST   /api/books/:id/drafts                        # 创建草稿
PUT    /api/books/:id/drafts/:draftId               # 更新草稿
DELETE /api/books/:id/drafts/:draftId               # 删除草稿
```

## CandidateActionsBar 组件

候选稿查看器顶部的操作栏，提供以下功能：

| 按钮 | 说明 |
|------|------|
| 接受（下拉） | 展开选择：替换章节 / 合并到章节 / 另存为草稿 |
| 拒绝 | 标记为已拒绝 |
| 归档 | 移入归档 |
| 删除 | 二次确认后永久删除 |

操作栏同时显示候选稿元信息：状态标签、来源、目标章节、创建时间。

## 文风分析与导入

### 风格画像合并

```
GET /api/style/personal-profile?profiles=[...]
```

将多个风格画像（ImportStyleProfile）合并为统一的个人风格画像。

### 风格漂移检测

```
POST /api/books/:bookId/style/drift-check
```

对比当前章节风格与基准风格画像，检测是否发生风格漂移。

### 作品导入与解析

```
POST /api/works/import
```

上传已有作品文本，自动解析章节结构。

## 上下文组装（Context Assembly）

AI 写作时，系统自动组装上下文，包括：

1. **经纬数据**：根据当前章节号和可见性规则，筛选参与 AI 的经纬条目
2. **前文摘要**：已写章节的摘要
3. **真相文件**：角色矩阵、伏笔状态等
4. **风格指南**：风格画像和写作规则
5. **大纲**：当前章节的大纲要求

上下文预览：

```
POST /api/books/:bookId/jingwei/preview-context
```

参数：`currentChapter`（当前章节号）、`sceneText`（当前场景文本）、`tokenBudget`（Token 预算）。

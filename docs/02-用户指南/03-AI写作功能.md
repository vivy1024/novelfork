## Agent 角色

NovelFork 提供 5 个专业 Agent 角色，各司其职：

| Agent | 角色 ID | 独特能力 |
|-------|---------|---------|
| 写手 | `writer` | 根据大纲、经纬、风格画像生成正文；支持续写/扩写/改写/补写 |
| 大纲师 | `outline` | 生成/调整大纲分支；分析故事结构；建议情节走向 |
| 审计官 | `auditor` | 连续性审校；检测剧情矛盾、人物设定冲突；输出审校报告 |
| 钩子师 | `hooks` | 管理伏笔（pending hooks）；建议埋设/回收时机；追踪伏笔状态 |
| 章节钩子 | `chapter-hooks` | 章节级钩子管理；章末悬念设计；章间衔接优化 |

## 对话面板交互

右侧 Agent 对话面板是所有 AI 写作操作的统一入口。

### ToolConfigBar（工具配置栏）

对话面板顶部的工具配置栏，用于快速切换当前 Agent 角色和配置写作参数（温度、模型、上下文范围等）。

### AgentQuickActions（快捷按钮）

对话面板顶部的快捷按钮组，提供常用写作操作的一键触发：

- **续写**：从当前章节末尾或光标位置向后延续
- **扩写**：对编辑器中选中的文本进行扩展
- **改写**：对选中文本进行风格/内容改写
- **补写**：在指定位置插入过渡内容
- **多版本变体**：对同一段落生成多个风格变体供选择

> 所有选段写作操作（续写/扩写/改写/补写/多版本变体）均通过 Agent 对话面板的快捷按钮触发。

### Artifact Surfacing

Agent 生成内容时，对话面板中以 ToolCallCard 组件实时逐行渲染生成结果：

- 流式输出，逐行显示生成的文本
- 生成完成后自动进入候选稿流程
- 支持中途取消生成

---

写作模式通过 inline-write API 触发，生成内容后进入候选稿流程。

```
POST /api/books/:bookId/inline-write
```

| 模式 | mode 值 | 说明 |
|------|---------|------|
| 续写 | `continuation` | 根据前文自然延续，支持 `direction` 指定方向 |
| 扩写 | `expansion` | 对选中文本进行扩展，支持 `expansionDirection`：`sensory`（感官）等 |
| 过渡 | `bridge` | 生成场景过渡段落，支持 `purpose`：`scene-transition`（场景转换）等 |
| 对话生成 | `dialogue` | 根据角色性格生成对话 |
| 变体生成 | `variant` | 对同一段落生成多个风格变体 |
| 大纲分支 | `outline/branch` | 基于当前大纲生成分支走向 |

### 对话生成

```
POST /api/books/:bookId/dialogue/generate
```

参数：
- `characters`：角色数组，每个含 `name`、`personality`、`speechStyle`
- `scene`：场景描述
- `purpose`：对话目的
- `turns`：对话轮数（默认 5）

### 变体生成

```
POST /api/books/:bookId/variants/generate
```

对选中文本生成 2-5 个风格变体（默认 3 个），每个变体独立调用模型。

### 大纲分支

```
POST /api/books/:bookId/outline/branch
POST /api/books/:bookId/outline/branch/:branchId/expand
```

- `branch`：基于当前大纲、伏笔、故事状态生成分支建议
- `expand`：将选定分支扩展为详细章节大纲

## AI 动作

AI 动作是更高层的写作操作，通常由叙述者会话或工具栏触发：

| 动作 | 说明 |
|------|------|
| write-next | 写下一章，综合大纲、经纬、前文上下文 |
| draft | 生成草稿，不直接写入正式章节 |
| audit | 连续性审校，检测剧情矛盾和设定冲突 |
| revise | 修订，支持 5 种模式（见下文） |
| rewrite | 重写整章 |
| detect / detect-all | 去 AI 味检测（单章/全书） |

### 修订模式

修订（revise）支持以下子模式：

| 子模式 | 说明 |
|--------|------|
| style | 文风修订，对齐真相文件中的风格画像 |
| continuity | 连续性修订，修复审校发现的矛盾 |
| rhythm | 节奏修订，调整叙事节奏 |
| dialogue | 对话修订，优化角色语言风格 |
| general | 综合修订 |

## 候选稿流程

AI 生成的内容不直接写入正式章节，而是先进入候选稿（candidate）流程：

```
创建 → 查看 → 接受/拒绝/归档/删除
```

### 候选稿 API

```
GET    /api/books/:id/candidates                    # 列出所有候选稿
POST   /api/books/:id/candidates                    # 创建候选稿
POST   /api/books/:id/candidates/:id/accept         # 接受
POST   /api/books/:id/candidates/:id/reject         # 拒绝
POST   /api/books/:id/candidates/:id/archive        # 归档
DELETE /api/books/:id/candidates/:id                 # 删除
```

### 接受动作（accept action）

| 动作 | 说明 |
|------|------|
| `replace` | 用候选稿内容覆盖目标章节 |
| `merge` | 将候选稿追加到目标章节末尾 |
| `draft` | 将候选稿另存为草稿，不修改正式章节 |

### 草稿管理

```
GET    /api/books/:id/drafts                        # 列出草稿
GET    /api/books/:id/drafts/:draftId               # 获取草稿详情
POST   /api/books/:id/drafts                        # 创建草稿
PUT    /api/books/:id/drafts/:draftId               # 更新草稿
DELETE /api/books/:id/drafts/:draftId               # 删除草稿
```

| 按钮 | 说明 |

## 文风分析与导入

### 风格画像合并

```
GET /api/style/personal-profile?profiles=[...]
```

将多个风格画像（ImportStyleProfile）合并为统一的个人风格画像。

### 风格漂移检测

```
POST /api/books/:bookId/style/drift-check
```

### 作品导入与解析

```
POST /api/works/import
```

上传已有作品文本，自动解析章节结构。

## 上下文组装（Context Assembly）

AI 写作时，系统自动组装上下文，包括：

1. **经纬数据**：根据当前章节号和可见性规则，筛选参与 AI 的经纬条目
2. **前文摘要**：已写章节的摘要
3. **真相文件**：角色矩阵、伏笔状态等
4. **风格指南**：风格画像和写作规则
5. **大纲**：当前章节的大纲要求

上下文预览：

```
POST /api/books/:bookId/jingwei/preview-context
```

参数：`currentChapter`（当前章节号）、`sceneText`（当前场景文本）、`tokenBudget`（Token 预算）。

## AI 味检测与去 AI 味闭环

### AI 味检测（NovelAuditExpanded）

检测章节中的 AI 生成痕迹，问题文本标红高亮显示：

- **12 条本地规则**：检测重复句式、过度修饰、机械过渡等
- **朱雀 API**（可选）：外部 AI 检测服务
- **标红高亮**：NovelAuditExpanded 组件将检测到的问题文本直接标红，定位精确

### 去 AI 味闭环流程

```
检测 → 改写 → 自动重新检测 → 前后对比
```

1. 检测：对章节运行 AI 味检测，标记问题段落
2. 改写：Agent 自动对标记段落进行去 AI 味改写
3. 重新检测：改写完成后自动再次运行检测
4. 前后对比：展示改写前后的 AI 味分数变化

### 文风仿写

基于真相文件的风格迁移：

1. **上传范文**：上传目标风格的参考文本
2. **提取风格特征**：系统自动分析范文的句式、用词、节奏等特征，生成风格画像
3. **写作时自动注入**：后续 AI 写作自动参考风格画像，模仿目标文风

## 写作预设（PresetsPanel）

管理写作预设组合，通过驾驶舱底部面板或 Agent 对话面板配置：

| 类别 | 说明 |
|------|------|
| 流派配置 | 6 个流派（玄幻/仙侠/都市/科幻/言情/悬疑） |
| 文风 tone | 5 类（热血/冷峻/轻松/沉稳/华丽） |
| 时代基底 | 6 类（古代/现代/未来/架空/末世/异界） |
| 逻辑规则 | 8 类自检规则 |
| 预设组合 | 6 个快速应用 Bundle |

### 章节健康度

| 组件 | 功能 |
|------|------|
| ChapterHealthCard | 单章健康指标：节奏分析、对话占比、句长直方图 |
| BookHealthSummary | 全书健康概览：6 项指标汇总 |

### 日更进度追踪

| 组件 | 功能 |
|------|------|
| DailyProgressCard | 日更字数、连续天数、趋势图 |
| BeatProgressBar | 节拍表进度可视化 |

### 平台合规检查（CompliancePanel）

- 敏感词扫描（起点/晋江/番茄/七猫/通用 5 平台规则集）
- AI 比例估算
- 格式规范检查
- 发布就绪检查 + AI 使用声明生成

### 导出（ExportPanel）

支持导出格式：TXT、Word、ePub。

### 角色弧线（CharacterArcsPanel）

追踪角色在全书中的成长弧线和情感变化。

### 文风漂移检测（StyleDriftPanel）

对比当前章节风格与基准风格画像，检测是否发生风格漂移。

### 模板市场（TemplateMarketPanel）

浏览和应用写作模板，包括叙事节拍模板、AI 味过滤预设、文学技法预设等。

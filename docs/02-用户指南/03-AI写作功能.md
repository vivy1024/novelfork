## Agent 角色

NovelFork 提供 5 个专业 Agent 角色，各司其职：

| Agent | 职责 |
|-------|------|
| 写书 | 根据大纲、经纬、风格画像生成正文；支持续写/扩写/改写/补写 |
| 伏笔 | 管理伏笔（pending hooks）；建议埋设/回收时机；追踪伏笔状态 |
| 章末钩子 | 章末悬念设计；章间衔接优化；章节级钩子管理 |
| 审校 | 连续性审校；检测剧情矛盾、人物设定冲突；输出审校报告 |
| 大纲与经纬 | 生成/调整大纲；分析故事结构；维护经纬数据 |

## 引导式生成

NovelFork 采用引导式生成流程，确保 AI 输出符合你的创作意图：

### 流程

```
PGI 追问 → Guided Plan（计划批准）→ 候选稿生成
```

### PGI 追问

生成前，AI 会通过 AskUserQuestion 向你追问关键信息：

- 支持多选题和单选题
- 每题自动提供"其他"选项，可输入自定义内容
- 追问内容根据当前写作上下文动态生成

### Guided Plan（计划批准）

追问完成后，AI 生成写作计划供你审批：

- 展示即将生成的内容大纲
- 你可以批准、修改或拒绝计划
- 批准后才开始正式生成

### 候选稿

AI 生成的内容不直接写入正式章节，而是先进入候选稿流程：

```
生成 → 查看 → 接受/拒绝/归档/删除
```

接受时可选择：
- **替换**：用候选稿内容覆盖目标章节
- **合并**：将候选稿追加到目标章节末尾
- **另存为草稿**：不修改正式章节

## 写作模式

| 模式 | 说明 |
|------|------|
| 续写 | 从当前章节末尾或光标位置向后延续 |
| 扩写 | 对选中文本进行扩展，丰富细节 |
| 改写 | 对选中文本进行风格/内容改写 |
| 补写 | 在指定位置插入过渡内容 |
| 多版本变体 | 对同一段落生成多个风格变体供选择 |

所有写作操作通过右侧 Agent 对话面板的快捷按钮触发。

## AI 味检测与去 AI 味

### 检测

检测章节中的 AI 生成痕迹：

- **12 条本地规则**：检测重复句式、过度修饰、机械过渡等
- **朱雀 API**（可选）：外部 AI 检测服务
- **标红高亮**：问题文本直接标红，定位精确

### 去 AI 味闭环

```
检测 → 改写 → 自动重新检测 → 前后对比
```

1. 检测：对章节运行 AI 味检测，标记问题段落
2. 改写：Agent 自动对标记段落进行去 AI 味改写
3. 重新检测：改写完成后自动再次运行检测
4. 前后对比：展示改写前后的 AI 味分数变化

## 文风仿写

基于真相文件的风格迁移：

1. **上传范文**：上传目标风格的参考文本
2. **提取风格特征**：系统自动分析范文的句式、用词、节奏等特征，生成风格画像
3. **写作时自动注入**：后续 AI 写作自动参考风格画像，模仿目标文风

---

## 上下文组装

AI 写作时，系统自动组装上下文：

1. **经纬数据**：根据当前章节号和可见性规则，筛选参与 AI 的经纬条目
2. **前文摘要**：已写章节的摘要
3. **真相文件**：角色矩阵、伏笔状态等
4. **风格指南**：风格画像和写作规则
5. **大纲**：当前章节的大纲要求

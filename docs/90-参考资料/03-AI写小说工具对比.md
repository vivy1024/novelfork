## 八、NovelFork 的独特定位

### 没人做的事（我们做了）

1. **Agent 驱动的创作流**：不是"用户点按钮→AI 生成"，而是"用户说意图→Agent 自主探索→规划→写作→审校"
2. **网文专用工具链**：伏笔追踪（埋/长/收）、资源账本（修仙/系统流）、敏感词平台规则
3. **完全本地 + 模型自由**：数据不出本机，想接哪个模型接哪个
4. **非破坏性写入原则**：AI 结果永远先进候选区，用户确认才写入正文

### 别人做得更好（我们需要追的）

1. **Sudowrite**：UX 更流畅，Story Engine 一键生成整章体验更好，导出格式多
2. **Scrivener**：20 年积累的小说管理能力，拖拽排序、分屏对照、快照版本
3. **NovelAI**：Lorebook 的世界观自动注入更成熟，图像生成是加分项

### 最值得优先补齐的

| 优先级 | 功能 | 对标 |
|--------|------|------|
| 🔴 | 新手引导全流程烟测 | Sudowrite 的 onboarding |
| 🔴 | EPUB 导出 | Scrivener |
| 🟡 | 章节拖拽排序 | Scrivener |
| 🟡 | 专注写作模式 | 所有竞品都有 |
| 🟡 | ChatWindow 嵌入工作台 | Sudowrite 的 inline AI |
| 🟢 | 设置/配置（Kiro 等平台的初始接入） | 运营需求 |

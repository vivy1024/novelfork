# chapter-complete — 章节完成流程

章节写作完成后，执行以下收尾流程确保经纬状态同步。

## 步骤

1. **获取路径**：调用 `cockpit.get_snapshot` 获取 `storyDir` 和当前章节信息。

2. **保存章节**：使用 `candidate.create_chapter` 工具保存候选稿（不直接覆盖正式章节）。

3. **更新章节摘要**：用 `Read` 读取 `{storyDir}/chapter_summaries.md`，追加本章摘要，用 `Write` 写回。

   格式：
   ```
   ## 第N章：标题
   
   一段话摘要（50-100字），包含关键事件和角色行动。
   ```

4. **更新当前状态**：用 `Edit` 更新 `{storyDir}/current_state.md`：
   - 更新"当前章节"编号
   - 更新"世界状态"（如有变化）
   - 更新"角色状态"（如有变化）
   - 更新"活跃伏笔"（新埋设的加入，已回收的标记）

5. **更新伏笔**：如果本章埋设了新伏笔或回收了旧伏笔，用 `Edit` 更新 `{storyDir}/pending_hooks.md`：
   - 新伏笔：`- [ ] 第N章：描述`
   - 已回收：将对应条目改为 `- [x] 第N章：描述（第M章回收）`

6. **更新情绪弧线**（可选）：如果本章有显著情绪转折，更新 `{storyDir}/emotional_arcs.md`。

## 规则

- 每一步都用 Read 先读取现有内容，合并后再写入，不要覆盖已有数据
- 摘要要简洁（50-100字），不要复述全文
- 伏笔描述要具体到可以验证是否回收
- 所有路径基于 storyDir，不要写到其他位置

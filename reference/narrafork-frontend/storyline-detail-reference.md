# NarraFork 叙事线详情页参考

从 /projects/:id 页面提取。

## 页面结构

叙事线详情页 = 项目详情页。显示项目的章节列表，每个章节是一个对话会话。

```
┌──────────────────────────────────────────────────────────┐
│ 项目名: novelfork                                        │
│ 状态: 活跃                                               │
│ 操作: [删除叙事线] [清理] [批量合并] [命令] [技能] [套路] [设置] [新建章节] │
├──────────────────────────────────────────────────────────┤
│                                                          │
│ 章节列表:                                                │
│                                                          │
│ ┌──────────────────────────────────────────────────────┐ │
│ │ spec任务4                                            │ │
│ │ 活跃                                                 │ │
│ │                                                      │ │
│ │ spec任务4                                            │ │
│ │ 推理                                                 │ │
│ │ — 考虑 Git 命令...                                   │ │
│ │                                                      │ │
│ │ Bash                                                 │ │
│ │ 查看工作树状态                                        │ │
│ │ 0.9s / 2m                                            │ │
│ │ $ git status --short                                 │ │
│ │ 输出: M packages/studio/src/api/routes/admin.test.ts │ │
│ │ ...                                                  │ │
│ └──────────────────────────────────────────────────────┘ │
│                                                          │
└──────────────────────────────────────────────────────────┘
```

## 关键特征

1. **章节 = 对话会话** — 每个章节不是文件，是一个完整的对话历史
2. **章节内容直接展示** — 不需要点击展开，直接在列表中显示最近的对话内容
3. **工具调用内联** — Bash、Read、Write 等工具调用直接在章节内容中展示
4. **操作按钮丰富** — 删除/清理/批量合并/命令/技能/套路/设置/新建章节

## 与 NovelFork 的差异

NarraFork 的"章节"是 git worktree 中的对话会话。
NovelFork 的"章节"是小说的实际章节（正文内容）。

NovelFork 的叙事线详情页应该：
- 显示书的章节列表（正文章节，不是对话）
- 有资源管理器（章节/候选稿/草稿/经纬/大纲树）
- 有编辑器（查看/编辑章节正文）
- 有写作工具（续写/扩写/审校等）
- 选择写作方式时创建绑定该书的叙述者会话

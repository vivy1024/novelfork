# 书籍管理与写作流程 v2

## 概述

当前 NovelFork 的书籍管理和写作流程存在多个断裂点：书籍无法正常删除、引导式创建不写入文件、写作动作按钮不触发 Agent、侧边栏交互不符合预期。本 spec 统一修复这些问题，使"创建书籍 → 引导设定 → 生成大纲 → 写第一章"的端到端流程真正可用。

## 来源

- 用户实际操作反馈（2026-05-10 截图）
- NarraFork 叙事线管理页面参考（卡片列表 + 红色删除按钮）
- `docs/90-参考资料/小说写作与AI调研/01-小说写作与AI调研.md` 产品定位
- `docs/99-历史归档/旧文档体系归档/小说写作与AI调研-v0.1-v0.4-archive.md` 决策映射
- `docs/03-产品与流程/01-小说创作流程.md` 已定义的流程

## 用户故事

1. 作为作者，我希望有一个书籍管理页面，能看到所有书籍、创建新书、删除旧书
2. 作为作者，我希望新书引导完成后，设定内容写入经纬文件，引导消失，进入正常工作台
3. 作为作者，我希望点击"生成下一章"后，Agent 自动开始引导式生成流程（追问 → 规划 → 写作），不需要我手动输入指令
4. 作为作者，我希望写作动作产生的候选稿出现在资源树中，我可以预览、接受或拒绝

## 功能需求

### FR-1: 书籍管理页面

**当前问题**：侧边栏"叙事线（书籍）"只是一个列表，没有管理功能。右键菜单在 embedded Edge 应用中不可用。

**目标**：参照 NarraFork 的叙事线页面，实现一个独立的书籍管理页面。

- 路由：`/next/books`（新增）
- 侧边栏"叙事线（书籍）"标题可点击，进入书籍管理页面
- 页面内容：
  - 标题"我的作品" + 右上角"新建作品"按钮
  - 书籍卡片网格（3 列），每个卡片显示：书名、题材 Badge、章节数/目标章数、字数、状态（活跃/已完成）
  - 点击卡片进入工作台
  - 每个卡片有红色"删除"按钮（带确认弹窗）
- 删除调用 `DELETE /api/books/:id`，成功后刷新列表

### FR-2: 引导式创建完整流程

**当前问题**：
1. "完成设定"按钮点击无反应（可能是 `encodeURIComponent` 问题或 `onComplete` 链断裂）
2. 引导完成后没有触发资源树刷新
3. 引导写入的 `story_bible.md` 内容不在资源树中显示（因为 size 过滤）

**目标**：引导完成后，经纬文件有实际内容，工作台进入正常驾驶舱视图。

- 修复 `handleSubmit`：确保 `postApi` 调用成功后 `onComplete` 被调用
- `onComplete` 回调链：`NewBookGuide.onComplete` → `CockpitOverview.onGuideComplete` → `WorkbenchCanvas.onGuideComplete` → `WritingWorkbenchRouteLive` 刷新资源树
- `guided-setup` API 写入的 `story_bible.md` 内容应该 > 100 字节（通过引导回答生成有意义的内容）
- 引导完成后 localStorage 标记 `guideCompleted`，下次不再显示
- 工作台提供"重新引导"按钮（在写作工具面板或顶部栏）

### FR-3: 写作动作自动触发 Agent

**当前问题**：点击"生成下一章"只是导航到对话页面，Agent 不会自动开始工作。用户需要手动输入指令。

**目标**：写作动作按钮点击后，自动发送对应的 slash command 或初始消息，触发 Agent 的引导式生成流程。

- `session-native.write-next`：导航到会话后自动发送 `/novel:write-next` 命令
- `ai.draft.async`：导航后自动发送 `/novel:draft` 命令
- `ai.audit`：导航后自动发送 `/novel:audit` 命令
- `ai.detect`：导航后自动发送 `/novel:detect` 命令
- `hooks.generate`：导航后自动发送 `/novel:hooks` 命令

实现方式：`onNavigateToConversation(sessionId, action)` 导航时携带 action 信息，对话页面加载后检测到 pending action，自动发送对应消息。

### FR-4: 引导式生成产生的文件

**当前问题**：`guided-setup` API 只写入 `story_bible.md`，没有写入其他经纬文件（大纲、角色、设定等）。

**目标**：引导完成后，根据用户回答生成初始经纬文件集。

引导完成后应写入：
- `story_bible.md` — 故事经纬（题材/前提/主角/金手指/世界观/力量体系/基调/平台/AI味容忍度）
- `volume_outline.md` — 卷大纲骨架（如果用户选择了"建筑师派"创作方式）
- `book_rules.md` — 书籍规则（从题材和文风推导的基本约束）
- `current_state.md` — 初始状态（第 0 章，世界初始状态）

如果用户选择了"花园派"（即大部分跳过），只写入 `story_bible.md` 和 `book_rules.md`。

### FR-5: 侧边栏交互修正

**当前问题**：
- 右键菜单在 embedded Edge 应用中触发的是浏览器原生右键菜单
- 侧边栏书籍列表没有管理入口

**目标**：
- 移除右键菜单方案，改为点击进入书籍管理页面
- 侧边栏"叙事线（书籍）"标题变为可点击链接，进入 `/next/books`
- 每本书旁边不需要删除按钮（删除在管理页面做）

## 非功能需求

- 引导完成到工作台刷新 < 2s
- 写作动作自动发送消息 < 1s（导航完成后立即发送）
- 书籍管理页面加载 < 500ms

## 验收标准

1. 打开 `/next/books` 看到书籍卡片网格，可以删除书籍
2. 新建书籍 → 走完引导 → 点"完成设定" → 引导消失 → 资源树显示有内容的经纬文件
3. 点击"生成下一章" → 导航到对话 → Agent 自动开始引导式生成（PGI 追问）
4. Agent 生成的候选稿出现在资源树"候选稿"分组中
5. 侧边栏"叙事线（书籍）"标题可点击进入管理页面

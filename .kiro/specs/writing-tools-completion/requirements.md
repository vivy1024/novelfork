# 写作工具补全 — Requirements

## 背景

cockpit-redesign spec 将 WritingToolsPanel 的功能拆散到驾驶舱面板和 Agent 对话按钮。但拆散后部分功能变成了"发消息给 Agent"的间接方式，丢失了旧版直接调 API 的即时性。同时调研文档中标为🔴/🟡优先的功能尚未实现。

本 spec 目标：补全所有声称已有但实际不完整的写作功能，并实现调研中标为高优先的缺失功能。

## 用户故事

### US-1: 直接执行写作工具（不依赖 Agent）
作为作者，我希望在驾驶舱中一键执行 AI 味检测、段落节奏分析、对话比例分析，不需要等 Agent 思考。

### US-2: 选段写作即时执行
作为作者，我选中一段文字后，希望直接触发续写/扩写/去AI味/风格改写，立即看到结果，不需要切换到对话面板。

### US-3: EPUB 导出
作为作者，我希望将完成的小说导出为 EPUB 格式，可以在 Kindle/微信读书中阅读校对。

### US-4: 专注写作模式
作为作者，我希望有一个全屏无干扰的写作模式，只显示编辑器和字数统计，隐藏所有面板和工具栏。

### US-5: 章节拖拽排序
作为作者，我希望在资源树中拖拽章节调整顺序，而不是手动改编号。

### US-6: 去 AI 味工作流
作为作者，我希望有一个完整的"检测→高亮→一键改写→二次检测"循环，而不是分散的独立工具。

### US-7: 文风仿写
作为作者，我希望上传一段范文，让 AI 学习其风格后用该风格续写我的章节。

## 功能需求

### FR-1: 驾驶舱工具面板（直接执行）
- 在 ExpandablePanel 中新增第 5 个面板类型：`"tools"`
- StatusBar 添加"🔧"按钮打开工具面板
- 工具面板内容：
  - AI 味检测（直接调 `/api/filter/scan`，显示 AiTasteReport）
  - 段落节奏（直接调 `/api/books/:id/chapters/:ch/rhythm`）
  - 对话比例（直接调 `/api/books/:id/chapters/:ch/dialogue`）
  - 全书健康（直接调 `/api/books/:id/health`）
  - 矛盾地图（直接调 `/api/books/:id/conflicts/map`）
- 需要选中章节的工具，在未选中时显示"请先选择章节"
- 结果内联展示在面板中，不弹 Dialog

### FR-2: 选段写作面板（编辑器内嵌）
- 用户在章节编辑器中选中文字后，浮现工具条：续写 | 扩写 | 去AI味 | 风格改写
- 点击后直接调用 `/api/writing-modes/inline` API
- 结果以 diff 形式展示在编辑器下方
- 支持"采纳"/"放弃"/"生成新版本"
- 不依赖 Agent，直接 API 调用

### FR-3: EPUB 导出
- 导出面板（已有 ExportPanel）新增 EPUB 格式选项
- 使用 `epub-gen` 或类似库生成 .epub 文件
- 包含：封面（可选）、目录、章节正文、作者信息
- 后端 API：`POST /api/books/:bookId/export/epub`
- 返回文件下载链接

### FR-4: 专注写作模式
- 顶部工具栏添加"专注模式"按钮（全屏图标）
- 进入后：隐藏左侧资源树、底部状态条、驾驶舱面板
- 只显示：章节编辑器（居中、最大宽度 800px）+ 底部字数统计
- 按 Esc 或点击退出按钮恢复正常布局
- 支持暗色/亮色主题

### FR-5: 章节拖拽排序
- 资源树中章节节点支持拖拽
- 拖拽完成后自动更新章节编号
- 调用 `PUT /api/books/:bookId/chapters/reorder` 持久化新顺序
- 拖拽时显示插入位置指示线

### FR-6: 去 AI 味工作流
- 在 AI 味检测结果中，每个高亮段落旁显示"一键改写"按钮
- 点击后调用 rewrite.segment 工具（mode: "reduce_ai"）
- 改写完成后自动重新检测该段落
- 显示改写前后的 AI 味分数对比
- 支持批量改写（选中多个高亮段落）

### FR-7: 文风仿写
- 预设面板中支持上传"范文"文件（.txt/.md）
- 系统提取范文的风格特征（句长分布、用词偏好、节奏模式）
- 写作时将风格特征注入 prompt
- API：`POST /api/books/:bookId/style-profile` 上传范文
- API：`GET /api/books/:bookId/style-profile` 获取已提取的风格特征

## 非功能需求

### NFR-1: 直接执行工具响应时间
- AI 味检测 < 3s（本地规则）
- 段落节奏/对话比例 < 1s（纯计算）
- 选段改写 < 10s（依赖 LLM）

### NFR-2: EPUB 质量
- 通过 epubcheck 验证
- 在 Kindle Previewer 中正确渲染
- 支持中文竖排（可选）

### NFR-3: 专注模式
- 进入/退出动画 < 300ms
- 不丢失编辑器状态（光标位置、选中文字）

## 优先级排序

| 优先级 | 功能 | 理由 |
|--------|------|------|
| P0 | FR-1 驾驶舱工具面板 | 旧功能入口丢失，用户无法使用已有能力 |
| P0 | FR-2 选段写作面板 | 核心写作功能，竞品对比表中声称已有 |
| P1 | FR-3 EPUB 导出 | 调研标为🔴，用户强需求 |
| P1 | FR-6 去 AI 味工作流 | 产品核心差异化，当前只有检测没有闭环 |
| P2 | FR-4 专注写作模式 | 调研标为🟡，所有竞品都有 |
| P2 | FR-5 章节拖拽排序 | 调研标为🟡，Scrivener 基础能力 |
| P2 | FR-7 文风仿写 | 竞品对比表声称已有，实际只有 tone-check |

## 验收标准

1. 驾驶舱工具面板中，点击"AI味检测"立即返回结果（不经过 Agent）
2. 编辑器中选中文字后浮现工具条，点击"扩写"3 秒内开始流式输出
3. 导出 EPUB 文件可在 Kindle Previewer 中正确打开
4. 专注模式下只看到编辑器和字数，无其他 UI 干扰
5. 拖拽章节后编号自动更新，刷新页面顺序保持
6. AI 味检测→一键改写→二次检测的完整循环可在 30 秒内完成
7. 上传范文后，续写的文字风格明显接近范文

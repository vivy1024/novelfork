## 组件复用

| 需要的功能 | 复用来源 | 改动 |
|---|---|---|
| Sidebar | 新建的 `sidebar/Sidebar.tsx` | 已有，接入真实数据 |
| 对话框 | 现有 `ChatWindow.tsx` + `NarratorPanel` | 全宽渲染，不嵌套在三栏中 |
| 资源管理器 | 现有 `WorkspacePage` 的 `ResourceTree` | 提取为独立组件 |
| 编辑器 | 现有 `WorkspacePage` 的编辑器组件 | 提取为独立组件 |
| 写作工具 | 现有 `writing-modes/` 组件 | 直接复用 |
| 设置/套路 | 现有页面 | 直接复用 |

## 关键设计决策

### 1. 主界面对话框 = NarratorPanel 全宽

不嵌套在 ResourceWorkspaceLayout 中。直接在内容区全宽渲染 NarratorPanel。ChatWindow 内部已有完整的 WebSocket、消息、工具调用、确认门、模型选择器。

### 2. 叙事线详情页 = 独立页面

点击叙事线的书 → URL 变为 `/next/book/:id` → 渲染叙事线详情页。这个页面有自己的两栏布局（资源管理器 + 编辑器），不影响主界面。

### 3. 写作方式触发叙述者

在叙事线详情页选择写作方式（续写/整章生成/审校）时，自动创建绑定该书的叙述者会话，可以在页面内弹出对话面板，也可以跳转到主界面对话框。

### 4. StudioNextApp 改造

当前 StudioNextApp 用 NextShell（sidebar 导航 + 内容区）。改造为：
- 去掉 NextShell，用新的 Sidebar 组件
- 内容区根据路由渲染不同页面
- 对话框路由下全宽渲染 NarratorPanel
- 叙事线详情页路由下渲染写作工作台

## 不做的事

- 不做 SplitView 可拖拽分割
- 不做三栏布局
- 不做 VS Code 风格的 grid/sash
- 不重写 ChatWindow 内部逻辑

# 写作工作台交互重构 v1

## 概述

当前写作工作台的交互设计有严重问题：所有写作按钮行为相同（跳转新对话），写作工具面板塞在底部看不见，功能分类混乱。本 spec 重新设计工作台交互架构。

## 核心设计：每本书固定 5 个 Agent 对话窗口

创建叙事线（书籍）时，自动创建 5 个绑定的 Agent 会话：

| Agent | 职责 | 使用的工具 |
|-------|------|-----------|
| **写书 Agent** | 引导式生成下一章（PGI 追问 → 计划确认 → 写作） | pgi.* + guided.* + candidate.create_chapter + cockpit.* + jingwei.read_context |
| **伏笔 Agent** | 伏笔管理、建议回收时机、埋设新伏笔 | cockpit.list_open_hooks + narrative.* + chapter.read |
| **章末钩子 Agent** | 生成章末钩子方案（3-5 个选项） | chapter.read + cockpit.get_snapshot |
| **审校 Agent** | 连续性审计、矛盾检测、人设一致性检查 | chapter.read + jingwei.read_context + health.read_summary |
| **大纲/经纬 Agent** | 生成大纲、维护经纬、添加角色/设定/势力 | write_truth_file + read_truth_files + questionnaire.* |

### 用户体验

侧边栏叙事线下方显示 5 个固定 Agent 入口：
```
▼ 这个世界修仙讲科学
  📝 写书（引导式生成）
  🎣 伏笔管理
  🪝 章末钩子
  🔍 审校
  📋 大纲与经纬
```

点击任一 Agent → 进入对应的对话窗口（右侧），对话上下文自动绑定当前书籍。

### 与当前设计的区别

| 当前 | 新设计 |
|------|--------|
| 顶部 6 个按钮全部跳转新对话 | 5 个固定 Agent 窗口，点击直接进入 |
| 每次点击创建新会话 | 复用已有会话（持续对话） |
| 用户看不到 Agent 做了什么 | 对话里展示完整过程 |
| 写作工具面板在底部 | 去掉面板，即时工具用弹窗 |

## 选段操作（画布内）

选中文本后出现浮动工具条：

| 操作 | 行为 |
|------|------|
| 续写 | 从选区末尾续写 30-80 字 |
| 扩写 | 将选区扩展为更详细的描写 |
| 改写 | 用不同风格重写选区 |
| 多版本 | 生成 3 个改写版本供对比 |

这些走 InlineWritePanel 的 API（`/api/books/:id/inline-write`），结果直接插入/替换画布内容。

## 去掉的东西

- ❌ 底部写作工具面板（WritingToolsPanel 作为独立面板）
- ❌ 顶部 6 个写作动作按钮（替换为 5 个 Agent 入口）
- ❌ "写作工具"按钮（工具分散到各自位置）

## 保留的东西

- ✅ 资源树（左侧）
- ✅ 画布编辑器（中间）
- ✅ 驾驶舱（无选中资源时显示）
- ✅ 快照/回滚按钮
- ✅ 视图切换（资源树/章节图）
- ✅ 预设管理（移到设置页）
- ✅ 模板市场（移到设置页或独立入口）
- ✅ 节拍表（移到驾驶舱）

```
[书名] [状态]                    [AI味检测] [节奏] [对话] [文风] [发布] [导出] [快照] [视图切换]
```

即时工具按钮在顶部栏右侧，点击弹窗显示结果。

## 实现步骤

1. 创建书籍时自动创建 5 个 Agent 会话（绑定 projectId + agentId）
2. 侧边栏显示 5 个 Agent 入口
3. 去掉顶部 6 个写作动作按钮
4. 去掉底部写作工具面板
5. 即时工具按钮移到顶部栏，点击弹窗
6. 选段操作改为浮动工具条
7. 预设/模板/节拍表移到设置页或驾驶舱

# 会话运行时状态 — 需求文档

## 背景

NarraFork 的叙述者状态栏精确反映 AI 当前行为：
- 发出消息后立即显示"思考中"
- 遇到反思模型切换为"反思中"
- 进入计划模式切换为"计划中"
- 工具调用时显示工具名
- 完成后显示"空闲 · 上轮耗时 Xs"

NovelFork 当前问题：
- 对话进行中状态栏只显示"空闲"，没有实时状态
- 用户不知道 AI 当前在做什么（思考？调工具？等待 API？）
- 中断后点继续无反应，后台无日志
- 后台日志只有 `[anthropic.generate]`，没有状态变化信息

## 目标

1. 状态栏实时反映 AI 当前行为（思考中/工具调用中/反思中/计划中）
2. 中断/继续功能正常工作
3. 后台日志显示完整的状态变化链

## 用户故事

- 作为用户，我发送消息后能立即看到"思考中"状态，知道 AI 在工作
- 作为用户，AI 调用工具时我能看到"调用 Glob..."等具体状态
- 作为用户，我点击中断后 AI 停止，点击继续后 AI 恢复工作
- 作为用户，我能从状态栏看到上轮对话的精确耗时

## 参考

NarraFork 状态机：
```
idle → working（收到用户消息）
working → thinking（等待 AI 响应）
thinking → tool_calling（AI 发起工具调用）
tool_calling → thinking（工具返回结果，继续思考）
thinking → reflecting（进入反思）
thinking → planning（进入计划模式）
working → waiting（需要用户确认）
working → idle（完成）
```

## 约束

- 不改变 WebSocket 协议格式（在现有 envelope 基础上扩展）
- 向后兼容现有会话
- 状态更新必须低延迟（< 100ms）

# 小说写作特有功能 — 需求文档

**状态**: 🟡 Active
**优先级**: P0-P2
**前置**: 新书引导向导已实现（NewBookGuide）、Agent 管线已就绪、Truth Files 已就绪

---

## 背景

NovelFork 的底层 Agent 工作台能力（对话/工具/上下文管理/Coding Agent）已从 NarraFork 继承并完善。写作管线（Architect → Planner → Composer → Writer → Auditor → Reviser）和 Truth Files 体系也已就绪。

**但缺少面向作者的"预设/检测/数据/节拍"层** — 引擎有了，方向盘和仪表盘还没装。

本 spec 覆盖调研文档（`docs/90-参考资料/小说写作与AI调研/01-小说写作与AI调研.md`）中确定的小说特有功能。

---

## 用户故事

### US-1: 写作预设库
**作为**网文作者，**我想要**一组内置的 AI 写作预设（续写、改写、评点、检测等），**以便**不用自己写提示词就能让 AI 帮我完成常见写作任务。

### US-2: 消 AI 味后处理
**作为**要在起点/番茄发文的作者，**我想要**写完后自动检测 AI 味并一键降 AI，**以便**我的文章不被平台检测系统标记。

### US-3: 章节健康度
**作为**追求数据驱动的作者，**我想要**每章看到完读率估算、节奏评分、情绪曲线、AI 味分数，**以便**知道哪里需要改进。

### US-4: 脂砚斋评点
**作为**想提升写作水平的作者，**我想要** AI 像脂砚斋一样给我夹批式反馈（技法/伏笔/情绪/节奏），**以便**知道具体哪里写得好哪里写得差。

### US-5: 节拍表驱动
**作为**情节型作者，**我想要**按救猫咪15节拍/英雄之旅/三幕结构来约束 AI 生成，**以便**故事结构不散。

### US-6: 同章多变体
**作为**想探索不同走向的作者，**我想要**同一章在不同风格/情节走向下各生成一版并排对比，**以便**择优选用。

### US-7: 日更进度追踪
**作为**日更作者，**我想要**看到今日字数/目标/连续天数/周均/月均/日历热力图，**以便**保持写作节奏。

### US-8: 黄金三章特检
**作为**新人作者，**我想要**写完前3章后 AI 按5要素（主角/金手指/钩子/期待感/世界观）打分，**以便**知道开篇是否合格。

### US-9: 平台合规检查
**作为**要发文的作者，**我想要**发布前自动扫描敏感词 + 估算 AI 含量 + 生成 AI 使用标注，**以便**合规发布。

### US-10: 文风漂移检测
**作为**长篇作者，**我想要**每隔 N 章自动检测文风是否偏离初始设定，**以便**及时校准。

### US-11: 模板市场
**作为**想快速开书的作者，**我想要**从模板市场导入流派模板（含预设设定+提示词+推荐节拍），**以便**不用从零搭建世界观。

### US-12: 导出
**作为**要投稿/排版的作者，**我想要**把作品导出为 Word/ePub/TXT 格式，**以便**提交给编辑或上传平台。

## 功能需求

### FR-1: 写作预设库（P0）

| ID | 预设 | 触发场景 | 核心能力 |
|---|------|---------|---------|
| FR-1.1 | 黄金三章特检 | 写完前3章 | 按5要素打分 + 指出薄弱点 |
| FR-1.2 | 续写下一节 | 章节中断处 | 注入经纬 + 前文 + 节奏规则 |
| FR-1.3 | 风格改写 | 选中段落 | 保持内容不变，切换文风（热血/冷酷/搞笑/悲情/平淡） |
| FR-1.4 | 消 AI 味润色 | 任意段落 | 按12特征检测 + 9手段改写 |
| FR-1.5 | 脂砚斋评点 | 任意段落 | 夹批式：技法/内容/伏笔/情绪密度 |
| FR-1.6 | 爽点设计 | 卡文时 | 基于节拍表给3个备选方案 |
| FR-1.7 | 金手指设计 | 开书前 | 8问自检 + 限制条件建议 |
| FR-1.8 | 反套路思考 | 老套路想创新 | 给出反转/颠覆方案 |
| FR-1.9 | 人设辩论 | 角色言行不一致 | 多agent模拟角色对话 |
| FR-1.10 | 大纲展开 | 有短纲无长纲 | 按McKee五层结构展开 |
| FR-1.11 | 同章多变体 | 想尝试不同走向 | 同章3-5种不同处理并排对比 |
| FR-1.12 | 短剧分镜化 | 想改编短剧 | 按15-20集约60秒/集生成分镜 |

### FR-2: 消 AI 味后处理管线（P0）

| ID | 能力 | 说明 |
|---|------|------|
| FR-2.1 | 12 特征检测 | 扫描段落，标记命中的 AI 味特征 |
| FR-2.2 | 高亮展示 | 在编辑器中高亮 AI 味段落，悬浮显示具体特征 |
| FR-2.3 | 一键降 AI | 按9招自动改写（长短句交错/感官细节/口语化等） |
| FR-2.4 | 循环检测 | 降 AI 后二次检测，直到分数达标 |
| FR-2.5 | AI 味分数 | 0-100 分，越低越好，显示在章节卡片上 |
| FR-2.6 | 朱雀集成（可选） | 调用朱雀 API 做外部验证（需用户配置） |

### FR-3: 章节健康度（P1）

| ID | 指标 | 说明 |
|---|------|------|
| FR-3.1 | 字数 | 本章字数 + 与目标对比 |
| FR-3.2 | 估算完读率 | 基于番茄公式（节奏/钩子/情绪密度）估算 |
| FR-3.3 | 节奏评分 | 爽点分布是否合理（每N字一个小爽点） |
| FR-3.4 | 情绪曲线 | 按段落标注情绪强度，画曲线 |
| FR-3.5 | AI 味分数 | FR-2.5 的结果 |
| FR-3.6 | 开篇吸引力 | 前500字的钩子强度 |
| FR-3.7 | 结尾卡点 | 最后200字的悬念/期待感 |

### FR-4: 节拍表系统（P1）

| ID | 能力 | 说明 |
|---|------|------|
| FR-4.1 | 内置模板 | 救猫咪15节拍 / 英雄之旅17阶段 / 三幕结构 / 网文开篇12式 |
| FR-4.2 | 章节标记 | 每章/每场景可标记当前节拍 |
| FR-4.3 | 约束生成 | AI 生成时注入当前节拍要求（如"Midpoint必须有镜像时刻"） |
| FR-4.4 | 进度可视化 | 节拍表进度条，显示当前在哪个阶段 |

### FR-5: 日更进度追踪（P1）

| ID | 能力 | 说明 |
|---|------|------|
| FR-5.1 | 今日字数 | 实时统计当日新增字数 |
| FR-5.2 | 目标设定 | 可设日更目标（如6000字） |
| FR-5.3 | 连续天数 | 连续达标天数 |
| FR-5.4 | 日历热力图 | 按日显示写作量（类似 GitHub contribution） |
| FR-5.5 | 周/月统计 | 周均/月均/总字数趋势 |

### FR-6: 平台合规检查（P1）

| ID | 能力 | 说明 |
|---|------|------|
| FR-6.1 | 敏感词扫描 | 按平台规则扫描（起点/番茄/晋江各有不同） |
| FR-6.2 | AI 含量估算 | 估算全书 AI 生成比例 |
| FR-6.3 | 发布就绪检查 | 一键检查：字数/格式/敏感词/AI味/标注 |
| FR-6.4 | AI 使用标注生成 | 自动生成合规的 AI 辅助使用声明 |

### FR-7: 文风漂移检测（P2）

| ID | 能力 | 说明 |
|---|------|------|
| FR-7.1 | 基线建立 | 前5章建立文风基线（句长分布/词频/修辞密度） |
| FR-7.2 | 定期检测 | 每10章自动对比当前文风与基线 |
| FR-7.3 | 漂移报告 | 指出哪些维度偏离 + 建议校准方向 |

### FR-8: 模板市场（P2）

| ID | 能力 | 说明 |
|---|------|------|
| FR-8.1 | 流派模板 | 26种流派各含：默认设定条目 + 提示词预设 + 推荐节拍 |
| FR-8.2 | 文风预设 | 悲苦/冷峻/古典/黑色幽默/沙雕 5种 |
| FR-8.3 | 导入机制 | 一键导入模板到当前书籍的经纬 |
| FR-8.4 | 社区共享（远期） | 作者可上传/下载模板 |

### FR-9: 导出（P2）

| ID | 格式 | 说明 |
|---|------|------|
| FR-9.1 | TXT | 纯文本，按章节分割 |
| FR-9.2 | Word (.docx) | 带格式（标题/正文/分页） |
| FR-9.3 | ePub | 电子书格式 |
| FR-9.4 | 平台格式 | 按起点/番茄投稿格式要求输出 |

## 非功能需求

| ID | 要求 |
|---|------|
| NFR-1 | 所有预设的提示词必须中文优先，支持中文网文术语 |
| NFR-2 | AI 味检测必须在本地完成（不依赖外部 API），朱雀为可选增强 |
| NFR-3 | 章节健康度计算必须在 3 秒内完成（不阻塞写作） |
| NFR-4 | 预设库必须可扩展（作者可自定义预设） |
| NFR-5 | 所有检测结果必须可视化展示，不能只有文字报告 |

---

## 验收标准

1. 每个预设在真实 LLM 调用下产出合理结果
2. 消 AI 味管线能将 AI 味分数从 70+ 降到 30 以下
3. 章节健康度卡片在工作台中可见且数据来自真实分析
4. 节拍表能约束 AI 生成内容符合当前阶段要求
5. 日更追踪数据持久化到 SQLite，跨会话保留
6. 导出的 Word/ePub 在对应阅读器中格式正确

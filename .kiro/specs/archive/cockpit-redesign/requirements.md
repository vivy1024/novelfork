# 驾驶舱重构 — Requirements

## 背景

当前驾驶舱（CockpitOverview）是一个信息堆砌的长列表，把用户当成"每天手动写 6000 字的网文作者"来设计。实际上 NovelFork 是 AI 写小说软件——用户通过 5 个对话 Agent（大纲与经纬、写书、伏笔、章末钩子、审校）下达指令，AI 执行写作。用户的角色是导演/制片人，需要的是：

1. 可视化编辑和查看经纬（世界观/角色/关系图谱）
2. 配置和监控预设/节拍对 AI 写作的约束效果
3. 查看 AI 产出的质量指标

## 用户角色

- **身份**：AI 写作的导演/制片人
- **工作模式**：在画布区域管理经纬/配置/质量 ↔ 切到左侧 Agent 对话下达写作指令 ↔ 回来看结果
- **核心诉求**：让 AI 写得好，而不是管理自己的写作进度

## 功能需求

### FR-1：经纬图谱为画布默认视图

- 引导向导完成后，画布区域默认显示经纬关系图谱（react-flow）
- 左侧分类侧栏：按分类（角色/势力/设定/伏笔/力量体系等）筛选
- 点击节点：弹出编辑面板（就地编辑，不跳转）
- 底部视图切换：图谱视图 / 列表视图 / 时间线视图
- 支持缩放、拖拽、框选
- 节点显示：名称 + 分类图标 + 生命周期状态（active/dormant/retired）
- 连线显示：关系类型标签

### FR-2：底部状态条

- 画布底部固定一行状态条，始终可见
- 显示内容：当前章数/目标章数 | 当前节拍位置 | 质量均分 | AI味均值 | 警告数
- 每个指标可点击，展开对应的详情面板
- 状态条高度固定（约 36px），不可隐藏

### FR-3：可展开面板系统

- 从底部状态条点击某项，向上展开对应面板
- 每个面板支持：
  - [×] 关闭（收回状态条）
  - [□] 独占画布（面板全屏，经纬图谱隐藏）
  - [↕] 拖拽调整高度（面板和图谱共享画布空间）
- 面板之间互斥：同时只展开一个（避免空间不够）
- 用户的面板偏好（上次打开的面板、高度）持久化到 localStorage

### FR-4：预设配置面板

- 展示当前书籍启用的所有预设（按分类分组：流派/文风/基底/逻辑风险/AI过滤/文学技法）
- 每个预设卡片显示：名称 + 分类标签 + promptInjection 摘要（可展开全文）
- 支持启用/禁用单个预设（toggle）
- 支持从模板市场添加预设套装
- 最近 5 章的预设命中报告：哪些预设的约束被触发了

### FR-5：节拍进度面板

- 显示当前选用的节拍模板名称（如"救猫咪 15 节拍"）
- 可视化进度条：已完成节拍 / 总节拍数
- 当前节拍详情：名称、情绪基调、字数分配比例、网文特化建议
- 节拍列表：每个节拍的完成状态 + 对应章节范围
- 支持切换节拍模板

### FR-6：质量监控面板

- AI味趋势图（最近 20 章折线图）
- 文风漂移曲线（与基线的偏离度）
- 审校通过率（通过/未通过/待审）
- 逻辑风险触发统计（按风险类型分组）
- 章节质量列表：每章的质量评分 + AI味 + 审校状态
- 点击某章可跳转到该章节编辑器

### FR-7：警告面板

- 汇总所有需要用户注意的问题：
  - 审校未通过的章节
  - 逾期伏笔（埋设后超过 N 章未推进）
  - 经纬矛盾（连续性审计发现的问题）
  - 文风漂移超阈值
- 每个警告提供操作入口：[跳转到对应章节] [跳转到对应 Agent 对话]

### FR-8：经纬图谱扩展视图模式

- 图谱视图支持多种模式切换：
  - **关系图谱**（默认）：所有条目 + 关系连线
  - **角色弧线**：角色节点按章节展开，显示成长/变化轨迹
  - **矛盾地图**：冲突关系高亮，显示 protagonist/antagonist/stakes/resolution
- 视图模式切换在底部 ViewSwitcher 中

### FR-11：对话工具配置机制

每个绑定书籍的对话 session，通过工具启用/禁用来控制 Agent 能访问哪些书籍数据。工具全局注册在 tool-schemas.ts（套路管理可见），每个对话 session 选择启用哪些。

#### 新增全局工具

| 工具 | 功能 |
|------|------|
| `presets.get_rules` | 获取当前书籍启用的预设规则列表（名称 + promptInjection） |
| `presets.check_compliance` | 传入章节内容，检查是否违反启用的预设约束，返回违规项列表 |
| `beat.get_current` | 获取当前节拍信息（模板名/当前节拍序号/情绪基调/字数建议/网文提示） |

#### 工具启用层级

| 层级 | 说明 |
|------|------|
| 必须启用（不可关闭） | `jingwei.read_context`、`chapter.read`、`cockpit.get_snapshot` |
| 默认启用（可关闭） | `presets.get_rules`、`presets.check_compliance`、`beat.get_current`、`cockpit.list_open_hooks`、`hooks.manage` |
| 默认关闭（可启用） | `character.check_consistency`、`narrative.read_line`、`rewrite.segment` 等 |

#### 不同 Agent 角色的默认工具集

| Agent | 默认启用（除必须启用外） |
|-------|------------------------|
| 写书 | presets.get_rules, presets.check_compliance, beat.get_current, hooks.manage, rewrite.segment, candidate.create_chapter |
| 审校 | chapter.audit, character.check_consistency, health.read_summary |
| 伏笔 | cockpit.list_open_hooks, hooks.manage, narrative.read_line |
| 大纲与经纬 | outline.suggest_next, beat.get_current, narrative.read_line, narrative.propose_change |
| 章末钩子 | cockpit.list_open_hooks, hooks.manage, beat.get_current |
| 用户自建 | presets.get_rules, beat.get_current, cockpit.list_open_hooks, hooks.manage |

#### 用户配置界面

对话栏顶部一行显示当前工具启用状态：

```
🔧 经纬+章节(锁定) | ☑预设 ☑节拍 ☑伏笔 | ☐角色一致性 ☐叙事线
```

- 锁定工具显示但无开关
- 其余工具为 toggle 开关，点击即切换
- 配置存储在 session 级别，修改后立即生效
- 套路管理中可查看所有全局工具的完整列表和说明

### FR-9：Agent 工具按钮归属对话面板

以下功能作为 Agent 对话面板的快捷按钮，不在驾驶舱画布中：
- 生成下一章
- 续写草稿
- 选段写作（扩写/改写/补写）
- 多版本变体
- 章末钩子生成
- 连续性审校
- AI味检测
- 伏笔建议

这些按钮触发对应 Agent 执行工具调用，Agent 自带文件读写能力。
对话面板右侧为资源管理器，显示章节/经纬/大纲等文件，Agent 修改文件时实时展示变更。

顶部工具栏仅保留低频发布操作：
- 合规检查（CompliancePanel）
- 导出（ExportPanel — TXT/Word/ePub）
- 快照（CheckpointPanel）

其余按钮（生成下一章、续写草稿、扩写/改写、连续性审校、去AI味检测、伏笔建议）移至 Agent 对话面板。

## 删除/降级/废弃的功能

- ❌ 日更进度卡（DailyProgressCard）— 删除，不是人在写
- ❌ "下一步建议：继续写作第 N 章" — 删除，用户通过 Agent 对话写
- ❌ 统计卡片网格（总字数/下一章/已通过审校/风险）— 合并到底部状态条
- ❌ 下一章建议卡（NextChapterSuggestionCard）— 删除，Agent 对话中处理
- ❌ 候选稿摘要 — 保留在资源树中，不在驾驶舱展示
- ❌ AI味检测摘要 — 合并到质量监控面板
- ❌ 写作工具面板 Dialog（WritingToolsPanel）— 废弃，功能已拆散到驾驶舱面板和 Agent 对话按钮中
- ❌ 顶部写作动作按钮（生成下一章/续写/扩写/审校/AI味/伏笔）— 移至 Agent 对话面板

## 非功能需求

- 经纬图谱在 200+ 节点时仍流畅（react-flow 虚拟化）
- 面板展开/收起动画 < 200ms
- 状态条数据实时更新（章节写完后自动刷新）
- 响应式：1280px 以下时分类侧栏可折叠

## 成功标准

1. 用户打开工作台后，第一眼看到的是经纬关系图谱（世界观全景）
2. 底部一行就能知道当前进度、质量、有无问题
3. 需要调预设/看质量时一次点击展开，调完关掉
4. 不再有"全是零"的空洞感——经纬图谱本身就是有内容的工作区
